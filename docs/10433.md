# 获得软件开发人员的第一份工作

> 原文：<https://dev.to/jennifer/getting-your-first-job-as-a-software-developer>

# 前言

这篇文章主要是针对那些正在转型到另一个行业的人，而不是刚毕业的学生。新毕业生可能会从这篇文章中有所收获，但他们不是主要受众。也就是说，这篇文章将主要讨论如何获得一份 web 开发人员的工作，因为这不仅是我的经验，web 开发工作通常是初级开发人员可以获得的。

# 获得面试机会

在某些方面，这可能是最令人生畏的部分。我将打破这一点，并希望帮助你看到你(是的，你！)具备申请所需的条件。

### 准备好你的简历

我总是告诉人们的一件事是，让他们现有的工作尽可能地技术化。例如，当我申请我的第一份软件开发工作时，我让我在家得宝的工作(这是一份 PM 工作)听起来像是我做了更多的开发工作。我做了所有我放在简历上的事情，但我强调了更相关的任务，没有提到我每天大部分时间都在 Excel 中工作。你在工作中优化过流程吗？这显示了你如何解决问题！有没有写脚本帮你分析数据？肯定要加上那个。此外，编写实际代码并不是开发人员需要的唯一技能。Pretty Quick 的首席技术官克里斯·多伊尔(Chris Doyle)很好地总结了这一点(略微转述了推文):

除了代码之外，还有许多有价值的开发技能，其中许多你可能都拥有！从那里开始。此外，你的开发技能可能相对较少，但这并不意味着它们没有用处。[【2】](https://twitter.com/archslide/status/721412741634912256)一位初级开发人员给我发了一封求职信，指出我的网站有一个潜在的 UX 改进，并说“这是我可以帮助你的事情。”[【3】](https://twitter.com/archslide/status/721413114810511361)这就是主动性的具体体现！他们是最受欢迎的人，最终被录用了。[【4】](https://twitter.com/archslide/status/721414041554575360)

对于新开发人员，我希望投资于他们的培训，所以这实际上是关于“他们是要加倍努力还是浪费精力”。[【5】](https://twitter.com/archslide/status/721414744180199428)我确实考虑到他们目前的能力，最低杠杠很低，但是在轨迹上获得自信要重要得多。[【第六期】](https://twitter.com/archslide/status/721415368183562240)

其他有价值的开发技能是什么？Chris 已经创建了一个开发人员能力的完整列表。不要担心，如果你看着这个列表，觉得你少了一些。然而，这个列表可以帮助你决定在简历上写些什么。例如，如果你现在是一个客户支持角色，你可能非常擅长提出错误的可能原因！更多我对简历的想法，请看[我在](http://www.jenniferkonikowski.com/blog/2016/2/24/a-quick-post-about-resumes)这个话题上的帖子。

### 创建您的投资组合

申请工作的这一部分不太重要，因为你有更多的相关经验。例如，访问我的 [github](https://github.com/jmkoni) 或 [gitlab](https://gitlab.com/u/jmkoni) 页面会让人觉得我从来没有编码过！错误的...我所有的代码都是专有的，我在 PyLadies 和 OSS 上花了很多时间。对我有用，因为我以前的雇主和同事会支持我的代码质量。如果你刚开始，你需要表现出来。正如 [Carlos Alonso](https://twitter.com/calonso/status/721454863566159872) 所说，作为一名初级开发人员，“你的公共代码是你简历中最重要的部分。”达斯汀·金建议“写一个小游戏或者其他有趣的演示，然后用代码放到网上。”你的项目不需要很大，但是你应该有一个朋友对它进行质量检查，确保没有明显的错误。我的项目最终成为一个选择你自己的冒险游戏，这是经历 Zed Shaw 的[艰难地学习 Python](http://learnpythonthehardway.org/book/)的一部分。然而，我建议更进一步，在申请之前建立你自己的网络应用程序。参见我的[“开发人员入门”帖子](http://www.jenniferkonikowski.com/blog/2016/3/22/getting-started)的结尾，获得更多关于如何开始的建议。

### 找工作申请

你是怎么找到工作的？有很多很棒的工作板，我最喜欢的是[堆栈溢出工作](https://stackoverflow.com/jobs?t=c&r=home)。但是不要只坚持一种。搜索任何你能找到的招聘信息，正如[克里斯蒂安·斯坦纳特所说](https://twitter.com/chrsteinert/status/720881405337579520)“不要只考虑普通的科技公司。其他行业(比如金融服务业)可能会提供有趣的东西。”在这一点上，大多数公司至少有一些软件开发人员。如果有你喜欢的公司，看看他们的招聘信息。如果他们看起来有一些技术性的工作，那就主动出击吧！对一家公司的使命/产品充满热情通常能让你走得很远。举个例子，如果我住在三藩市，我会追逐[贝塔布兰德](https://www.betabrand.com/referrals/landing/friend/?r=4L)，直到他们雇用我。如果你在某个领域有广泛的经验，看看那些专注的公司，这样你过去的经验会更加有益。例如，如果你是一名正在接受培训的生物学家，并打算转向开发，你可能会发现像 [AddGene](https://www.addgene.org/careers/) 这样的地方非常适合你。我个人也很喜欢和招聘人员一起工作，比如波士顿的 Talener。至少，他们会让你站在一大堆公司面前，让你参加面试。即使这些都不成功，这也是一个很好的练习，当你找到一份你真正感兴趣的工作时，你会有更好的准备。还有，别忘了挤你自己的人脉关系！你认识在招聘开发人员的公司工作的人吗？联系他们，看他们是否能和你一起喝杯咖啡(给他们买杯咖啡)，和他们谈谈在他们公司工作的感觉和招聘流程。

现在你已经有了一长串的招聘信息，你可能开始注意到它们似乎都在说你需要所有这些不同语言的经验，也许其中一种你已经用过了...你怎么能准备好呢？好消息！大多数招聘信息都是愿望清单。是的，即使是写着“必需”的部分也是可以协商的。如果一份工作听起来很有趣，你应该申请。让审查你简历的人来决定你是否合格。唯一能让你停下来的短语是“高级”。如果一份工作招聘的是“高级应用程序开发人员”或类似的职位，招聘经理不太可能会聘用初级人员。即便如此，如果是一家你真正热爱的公司，伸手也无伤大雅。

# 在面试中表现出色

### 以前

准备的一个方法是列出一系列常见的面试问题和一些 T2 练习题。几乎我参加过的每一次面试都有关于 SQL 的问题。鉴于我做过的每一份工作都要求我以某种形式使用 SQL，我建议在申请任何工作之前学习一点。如果你还没有跟上进度，请完成[的练习 12《艰难地学习 SQL》](http://sql.learncodethehardway.org/book/)。看起来大部分的课程可能不会太费时间，而且非常值得你花时间去学！

### 在...期间

去年 9 月，我写了自己的“求职回顾”，其中谈到了我在最近一轮面试中做的一些事情，这些事情让我感到更加自信。作为大三学生，最需要记住的是，说“我不知道”或者“我不知道，但是我最近在看这个，我觉得是[x]”是可以的。在这一点上，你真正想要的雇主不会期望你对发展非常了解。保持冷静，确保你所有的优秀品质都展现出来。最重要的是，[别忘了提问](http://jvns.ca/blog/2013/12/30/questions-im-asking-in-interviews/)！

### 在...之后

如果你不确定你在面试中表现得很好，不要绝望！你还有时间留下好印象。回到 Chris Doyle，他有一个人在一次面试中提交了一份重构版本的编码练习。这是一个很好的主动性的展示，也是你继续考虑和思考你的解决方案，即使你已经“解决”了问题。给面试官发一封邮件表示感谢是很好的。这是一个添加“我一直在思考我们所做的那个问题，我认为解决方案可以通过[x]来改进”的完美地方。

# 开始工作

### 获得报价

山姆·菲彭先生唯一的建议确实触动了我的心弦:

充电。更多。进入这个行业的人总是低估他们合理的工资范围 20%左右。

这让我大吃一惊，因为到目前为止，我的工资少了大约 25%。为了了解其他地方的支付情况，看看像[这样的网站和最近的](http://www.indeed.com/salary/q-Junior-Web-Developer-l-Boston,-MA.html)[栈溢出开发者调查](http://stackoverflow.com/research/developer-survey-2016#work-salary)。如果你在事后发现自己低估了自己，是可以修复的，但是需要一段时间。我是凭经验说话的。

### 做工作

永远不要因为问问题而感到难过。利用和你一起工作的高级开发人员，尽可能多地从他们那里学习。如果你的公司支持结对，尽可能经常地结对程序。那样你会学得更多更快。此外，负责一个项目或一个功能。这并不意味着你必须做所有的工作，只是你有责任确保它通过终点线。沿着这些思路， [@codepaintsleep 说](https://twitter.com/codepaintsleep/status/720755778144100353):

不要因为你资历浅就被单调的工作困住。当有更多有经验的人来帮助时，推动你的知识！此外，不要把繁重的工作当作繁重的工作。什么都学！

为了举例说明你可以从任何事情中学习，本周我将代替一位同事和工作支持。我没有做任何实际的编码。然而，在短短一周内，我对我们的系统如何工作以及我们的用户想要什么的了解是令人难以置信的。学习。从。一切。
