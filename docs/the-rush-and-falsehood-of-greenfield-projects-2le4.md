# 绿地项目的匆忙和虚假

> 原文：<https://dev.to/jskulski/the-rush-and-falsehood-of-greenfield-projects-2le4>

*最初写于 2014 年，为开发者社区重新发布*

我以前很喜欢绿地项目的想法。我羡慕那些白手起家的人。这一次，一切都会好的。

[![](img/3f54d4049dcf17d24483df34bc48b21e.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--J2HZp5v5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2A0BnzC8ndIKeXHaXbc1ypWw.jpeg)

当我在大学里遇到难题时，当我无法让程序做我需要的事情时，*我通常会删除我的整个项目，然后重新开始*。有时这是有效的，但大多数时候我只是把自己编码回到我在的地方。

[![](img/9e98e3864288cdd0845e249f3add91d8.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--q0gQaVYY--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2At4YZAa0V01Mdnvv2RiS4SA.jpeg)

它奏效的次数增强了人们的信念，即项目是走钢丝。早期思想上的失误会导致项目的失败。为了成功地完成一个项目，每个决定都需要*正确、公正和神圣。*

[![](img/d213b8de5a2f895c9b09e4f7434897c9.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--nkUC1bQf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2A992euKF2IXed0P1SvJOCSg.jpeg)

我现在改变了对这一切的看法。我现在有点老了(通常都是这样)。可以说，我在驾驶舱已经工作了很长时间。现在，项目对我来说就像大船。他们有很大的惯性，你必须控制住。朝着一个方向前进可能会让你偏离方向，但是你几乎总是可以慢慢地让船回到正轨。

不可否认，*有时候，*重启的做法很有效。为什么？好吧，有重新开始的冲动。但更重要的因素是因为我不得不重新*思考这个问题。*我又一次处理了这个问题，这次我对*的思考更深入了一点。*

我试着保留那一部分。开始重新思考问题，一无所知。但是这次我保留了我的代码。*初学者的心态，不是初学者的项目。*
