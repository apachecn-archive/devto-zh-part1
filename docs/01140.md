## 通过 if 使用 case

分场合使用 if 的情况不多。
使用 case 比较好的情况很多。

```
user = Repo.get(query, id)
if user != nil do
  ...
else
  ...
end 
```


## 错误返回值形式不同的不通过 with 汇总，而是通过 case 嵌套

```
with {:ok, foo} <- spam(),
     {:ok, bar} <- ham() do
  ...
else
  :error ->
    ...
  {:error, reason} ->
    ...
end 
```

<svg width="20px" height="20px" viewBox="0 0 24 24" class="highlight-action crayons-icon highlight-action--fullscreen-on"><title>Enter fullscreen mode</title></svg> <svg width="20px" height="20px" viewBox="0 0 24 24" class="highlight-action crayons-icon highlight-action--fullscreen-off"><title>Exit fullscreen mode</title></svg>

像这样 spam ( )返回: error，ham ( )返回{:error，reason}的情况下，用
with 总结的话就不知道哪个返回哪个错误形式了，很痛苦。
所以一般用 case 嵌套书写。

```
case spam() do
  {:ok, foo} ->
    case ham() do
      {:ok, bar} ->
        ...
      {:error, reason} ->
        ...
    end
  :error ->
    ...
end 
```

## 如果满足条件则追加到列表中

制作 List 和 Map 时，如果满足条件的话会追加。
比如说这样的。

```
@spec build_spams() :: list()
def build_spams() do
  spams = base_spams()
  spams = if condition1() do
    [additional1() | spams]
  else
    spams
  end
  spams = if condition2() do
    [additional2() | spams]
  else
    spams
  end
  spams
end 
```

将追加要素的部分提取为函数的话，有时会更有前景。
如下所示。

```
@spec build_spams() :: list()
def build_spams() do
  base_spams()
  |> maybe_add_spams1()
  |> maybe_add_spams2()
end

defp maybe_add_spams1() do
  if condition1() do
    [additional1() | spams]
  else
    spams
  end
end

defp maybe_add_spams2() do
  if condition2() do
    [additional2() | spams]
  else
    spams
  end
end 
```

在 CI 上，mix credo 一定会回答。
. credo.exs 是按照以下感觉做的。

```
%{
  configs: [
    %{
      name: "default",
      files: %{
        included: ["lib/"],
        excluded: []
      },
      checks: [
        {Credo.Check.Readability.MaxLineLength, max_length: 120},
        {Credo.Check.Readability.ModuleDoc, false},
        {Credo.Check.Refactor.Nesting, false},
        {Credo.Check.Refactor.PipeChainStart, false},
      ]
    }
  ]
} 
```
