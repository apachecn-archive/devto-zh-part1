# 开源的新贡献者，请多写博客

> 原文：<https://dev.to/shubheksha/new-contributors-to-open-source-please-blog-more>

不久前，我写了这篇文章来帮助新手贡献者认识到，当他们不理解一个巨大的代码库或者不同的模块如何相互作用时，他们不是唯一感到失落&沮丧的人。我们很多人都在同一条船上。去那里也没关系。很多人联系我，告诉我他们被我的帖子激励着开始或继续投稿。一些维护者也被鼓励在阅读之后回去回顾一些拉请求。我不能要求更好的回答了！

经过相当长时间的思考，我得出的结论是，这种反应的主要原因是，对话在很长一段时间里是单方面的。维护者已经发布了很多关于如何让新的贡献者参与到他们的项目中的帖子&他们可以做的所有事情来使他们的项目对初学者友好，从标记问题到设置清晰的贡献指南，等等。Quora 和其他平台上有大量关于如何开始投稿的指南和问题，但没有多少新来者的帖子来说明他们开始投稿后的情况。我自己也读过很多这样的指南，但是它没有一个新手投稿者的第一手经验有帮助，比如他们开始投稿的项目，他们遵循的道路，他们的期望和结果。当只有一方在沟通时，很难取得平衡。

因此，我想鼓励所有新的贡献者记录他们的经验，如果他们希望帮助我们建立这种平衡。我向你保证这值得你花时间和耐心。写作的时候，不能粉饰细节，不能把你认为不需要的东西刷到地毯下面。它迫使你去感知事物的本来面目，而不是你认为的样子，并把它们放在正确的角度。它还能帮助你意识到你从哪里开始&你已经走了多远。写作是为了更好地学习。

我知道很多人会想——我知道的不多，不足以写一个特定的主题。你不必成为某个话题的专家来写这个话题。写你知道的。如果你错了，有人会指出来&你可以填补你知识中的空白。这是双赢的局面。相信我，没有什么比知道你写的东西帮助了一个正在挣扎的人更好的了，无论是技术话题、动机还是灵感。即使它对很多人没有帮助，但最终肯定会对你有帮助。

我害怕我上面提到的所有事情&我已经堆了几个草稿，等着发表，担心它们还不够好——它们可以更好，更精致，更技术性。我一直在网上寻找写得更好的灵感，甚至只是找到我觉得有资格写的话题。就在那时，我偶然发现了一篇文章——[写一篇优秀的编程博客](https://emptysqua.re/blog/write-an-excellent-programming-blog/)，作者是 [A. Jesse Jiryu Davis](https://medium.com/u/4bd421fec9d9) 。如果我觉得自己没有主题，或者觉得自己不够资格写我所知道的东西，我会不断回头重读它。我一直寻找灵感的另一个地方是[Julia Evans](https://medium.com/u/152f65dab15)’[博客](http://jvns.ca/)——她的帖子简短，一点点&读起来很有趣&但她设法让你从几乎每篇帖子中学到一些有趣的东西。我偶尔会上乔尔·斯波尔斯基的软件博客。他们有一些高质量的内容，涉及各种与技术相关的东西，我喜欢偶尔去看看。如果你有一个经常/定期访问的最喜欢的编程博客，请在评论中告诉我。:)

四处寻找灵感，我相信你会找到很多，因为每天都有很多关于各种技术相关的高质量文章发表。如果你没有足够的信心开始写技术主题，没关系，你会慢慢地、稳步地达到目的——从记录你的旅程开始:

*   你是如何参与到你所服务的组织中的
*   他们有什么样的社区&对新贡献者有多欢迎
*   你选择参与的项目&是什么让你做出了同样的决定
*   建立本地开发环境有多困难，您在哪里遇到了困难&您是如何跨越这个障碍的
*   你是如何发现第一个需要解决的问题的
*   如果你被指派了一个导师，你和他/她一起工作的体验如何&你得到了什么样的帮助
*   你所期望的开源是什么样的，它实际上是什么样的
*   一旦你有了足够的经验，你有什么建议想给新手贡献者吗
*   如何表现&在一个组织的 [IRC](https://en.wikipedia.org/wiki/Internet_Relay_Chat) 频道或 [Gitter](https://gitter.im/) 房间里问什么样的问题

这些只是我能想到的一些话题。你不必每天或每周都写，也不必遵循严格的时间表。当你觉得你有值得讲述的事情，可以帮助别人的事情，反过来也可以帮助你，无论是一个小的胜利还是一个大的贡献，就写下来。

慢慢地&稳步地，随着你获得更多的经验，你可以根据你的技能开始写技术主题。它可以是你花了很长时间在做的事情，或者是你学习一门新语言/框架/库的旅程。在这一点上，你可能会想，关于这个话题已经写得太多了，还有什么我可以补充的吗？写了多少并不重要&是谁写的，它从来没有阻止任何人根据他们的理解，从他们自己的观点写另一篇文章。你写的每一件事都有你自己的观点，尽管主题相同，但观点可能与他人大相径庭。一切都在细节中——你如何分解一个主题，你如何接近和理解它。所以，不要因为已经有太多关于某个特定主题的文章，而阻止你写自己的文章。

因此，我希望你写下&广泛地记录你的经验，以便你更好地学习，成为更好的开发人员&可能帮助一个迷失的、受到恐吓的新手贡献者在未来疯狂地搜索互联网，在开放源码软件的广阔世界中找到出路。

附:如果你有任何其他的想法，我想知道它们&也可能添加到这个帖子中。:)
