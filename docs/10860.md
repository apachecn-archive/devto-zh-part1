# 最小可行产品的心理收益

> 原文：<https://dev.to/ben/the-psychological-benefits-of-the-minimum-viable-product>

根据[维基百科](https://en.wikipedia.org/wiki/Minimum_viable_product)，最低可行产品 MVP 的目的:

*   能够用最少的资源测试一个产品假设
*   加速学习
*   减少浪费的工程时间
*   尽快将产品提供给早期客户
*   其他产品的基础
*   建立建造者制作所需产品的能力

这些都很好，但从我的角度来看，建立和推出一个最小可行产品的最重要的好处是别的东西:保持势头。势头总是稍纵即逝，在持续的建设过程中至关重要。失去了动力，团队就失去了能量，时间就浪费了，人就拖后腿了。产品发布和迭代势头是确保热情和能量以及快乐的员工队伍的最佳方式。

我认为人们淡化了动量的重要性，因为它是无形的，过于人性化。在我们的伪决定论和“可测量”领域，人们避免陷入心理动力的虚无缥缈，但它就是一切。我做运动的时候，我们叫它“莫”，它就是一切。

动力可以来自不同的来源，但发射是最好的。无论我们是向 10 个人还是 1000 万人发射，产生进步的感觉和继续努力所需的能量都是令人惊奇的。然而，整件事的关键是平衡动力需求和编写可维护代码的需求，因为我们不能只考虑短期。积累过多的技术债务是现在积聚廉价动能的最简单方式，同时确保它在未来几乎消失。如果我们承认 MVP 的心理益处，就更容易强调对长期心理健康的其他益处，比如分配时间来修整测试套件和有条不紊的重构，这些事情经常因为一时的速度而被搁置。这就是最小可行产品的概念发挥作用的地方。我们削减功能，以便我们能够推出一个东西，同时也为未来的增长奠定基础。“写一个扔掉”可能有一个时间和地点，但是这不是一个像构建 MVP 那样的可持续的实践。
