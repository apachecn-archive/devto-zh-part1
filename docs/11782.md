# 为什么我必须安装 Linux

> 原文：<https://dev.to/briannipper/why-i-had-to-install-linux-43al>

[![](img/5a1244711d1395e643576dc24068db5a.png)](http://upload.wikimedia.org/wikipedia/commons/9/91/Alternate_Ubuntu_logo.png) 最近我决定学习一门新的编程语言。我的理由主要是不想在接下来的 5 年里继续研究遗留代码。我已经开始意识到，在我年轻的编程生涯中，我已经落后于编程世界的潮流。

无论如何，我决定和几个同事合作一个副业项目，并不是说我们有一个项目本身，只是一些一般的想法和制作东西的愿望。我们这个项目的部分目标是学习新技术的各个方面，Pearl、Python、GIT、Tornado 等。

起初，我的印象是学习这些不同的语言/技术不需要我改变我的默认操作系统窗口(目前在我的台式机上分为 windows 7 和笔记本电脑上的 windows XP)。然而，即使对 windows 上的命令行驾轻就熟，我发现当我阅读各种讨论这些新工具的方法和技术的网站时，我感觉自己像一个二等公民。

当我在互联网上搜索与 Windows 相关的各种技术信息时，我会遇到“如果您正在寻找 Linux 支持，请参见…”会有一两个段落，也许是一页，但没有太多深入的信息。与此同时，我在一页又一页的 WIN32 信息中筛选。

现在，当我开始学习这些令人惊奇的新工具时，我面对的是真正不是主要受众的那类用户。对于使用 windows 操作系统和 Linux 工具的用户来说，现有的信息量很少，非常少。

起初我在想“这个社区是怎么回事，为什么他们冷落了这么大一群可以扩大市场份额的用户？”然后我决定“不，我想学这些东西，所以我要坚持下去。”我意识到的是，Windows 主要不是一个编程操作系统，但 Linux 感觉像是“由程序员为程序员制造的”。

是的，你可以在 windows 平台上编程，微软甚至有一些很棒的工具(甚至是免费的)，例如 Visual Web Developer 2010([http://www.microsoft.com/express/downloads/](http://www.microsoft.com/express/downloads/))，它提供了一个非常好的 JavaScript 智能感知/自动完成功能，包括你自己编写的功能，但它仍然是一个面向大众的操作系统。当我开始深入 Linux 世界的时候，我发现即使是最“用户友好”的版本(Ubuntu，这是我目前正在尝试/学习的),它也非常适合编程。

从一开始，我就可以看到我需要的工具有多少是现成的。python——安装完毕，准备就绪。SSH-安装完毕，可以使用了。VIM -已安装并准备就绪。甚至终端似乎也有助于黑客入侵。我通常不是一出门就成为粉丝的人，但我想我可能会成为一个粉丝。

拓展我的视野并找到一个新的理由对我的电脑感到兴奋是令人耳目一新的。
