# 结对编程

> 原文:[https://dev.to/funkysi1701/pair-programming-4ep6](https://dev.to/funkysi1701/pair-programming-4ep6)

因此，我一直在努力应对免费代码营网站上的挑战，这个网站的独特之处之一就是给你一个与他人一起编程的机会。

我是一个内向的人，如果你让我拿起电话给某人打电话，我会在做之前拖延很久(如果我做的话)，我也不擅长与陌生人交谈，但我一直很喜欢我迄今为止所做的结对编程。

到目前为止，我只编程过几次，所以我不是专家。第一次是和一个来自北美的人，刚开始和你的电脑说话并听到美国人的声音回应是很奇怪的，但是你很快就会忘记这一点，因为你专注于你正在处理的问题。

第二次是和一个荷兰人。这一次我们解决了一些更棘手的问题。两次我都更多地退居二线，但我认为贡献了有用的东西，我也确实从其他人那里学到了很多。

我认为这是结对编程的关键，你学到了很多如何从另一个角度看问题，或者学习一个你以前没有听说过的函数。如果你正在读这封信，而我与之配对编程的正是你，那么非常感谢你花时间陪我，我学到了很多。

我今晚试图再次配对程序，但技术问题阻止了我。我发现的一个问题是寻找一个合作伙伴，昨天我希望做一些，但没有其他人可以合作。

我有兴趣了解哪些公司使用结对编程。我想一个头发尖尖的老板会说，当你可以让两个人做不同的事情时，为什么要让他们做一份工作。然而，如果你想建立你的团队，给他们更多的技能，那么结对编程会很棒。我还认为可能有一些问题迫切需要结对编程。
