# 你把它放在哪里很重要

> 原文:[https://dev . to/pablo portugues/it-do-matter-where-you-put-it](https://dev.to/pabloportugues/it-does-matter-where-you-put-it)

*最初发表在我的博客上:[你把它放在哪里很重要](https://bytemybits.gitlab.io/post/2017-06-27-it-does-matter-where-you-put-it/)*

每当你想写一些东西并与开放世界分享时，你把这些内容放在哪里呢？你为什么决定把它放在那里？能接触到多少人？

我建立这个博客是因为我想写作，我把它放在 GitLab 页面上是因为我想[控制内容](https://bytemybits.gitlab.io/post/2017-06-13-it-actually-works/)，所以对我来说有意义的是，人们想与世界分享的任何东西，每个人都应该可以访问。脸书不会这么做。我会试着向你解释这一点的重要性，所以也许下次你想与世界分享一些东西时，你实际上正在做你打算做的事情。

如果你有一个脸书账户，并决定写一些东西给所有人看，你需要记住一些事情，比如帖子的隐私设置。每个有账户的人都可以轻松访问，但没有账户的人就不行了(是的，有些人没有脸书账户，是的，他们是像你一样的普通人)。即使您将其设置为公共，对于未登录的用户来说，这也是他们获得的体验:

[![facebook experience](../Images/168cd1d370ab0959e4c081fb46da394b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--fnM63Axx--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://bytemybits.gitlab.io/post-content/2017-06-27/facebook.gif)

此外，您无法控制任何格式。想添加一些**粗体**或*斜体*文本，一个[自定义链接](https://daringfireball.net/projects/markdown/basics)也许？甚至有些`code`？好吧，你可以忘了它，因为他们知道什么对他们的平台是最好的(是的，脸书是一家私人公司，永远只关心他们自己的利益，而不是你的，别忘了这一点！).

另一件需要考虑的事情是，你写的每一篇文章都不会被互联网档案馆保存，因为[脸书不允许](https://web.archive.org/web/*/https://www.facebook.com/marc.haynes.7583/posts/793204930854561?match=bWFyYyBoYXluZXMscm9nZXIgbW9vcmU%3D)，**也不会被搜索引擎**编入索引，这意味着谷歌或其他搜索引擎不会返回你的文章，除非它的链接位于其他地方，并且其他地方被编入索引(例如博客)。我能访问它的唯一方法是在脸书境内搜索，为此你需要一个账户。

现在，你甚至可能会想:“那么，你为什么不创建一个帐户，这样你就再也不用麻烦了？”对此我的回答是:*有一大堆[的](http://www.nybooks.com/articles/2016/12/22/they-have-right-now-another-you/)的[的](https://www.technologyreview.com/s/604082/we-need-more-alternatives-to-facebook/)的理由[为什么你](https://veekaybee.github.io/facebook-is-collecting-this/)不想拥有一个[的脸书](https://www.washingtonpost.com/news/the-intersect/wp/2016/08/19/98-personal-data-points-that-facebook-uses-to-target-ads-to-you/)账户，只要声明你不想要就够了。*

即便如此，如果你仍然不在乎这些，还有其他的问题。你无法控制脸书，所以你不知道这些内容会在那里存在多久。众所周知，脸书身体健康，可能还会活得更久，但是如果他们决定改变他们的隐私政策，删除你的内容或者封锁你的账户，无论如何你都无法控制。

我说这些废话的主要原因是试图唤起一些意识。网络应该是开放的，脸书恰恰相反。你想与全世界分享你的内容吗？您关心这些内容的长期变化吗？然后把它放在你控制的某个地方，这样每个有网络连接的人都可以公开使用。在那里链接你想要的任何东西(脸书、推特、Pinterest 等)，而不是相反！

那么，你如何实现这一点呢？嗯，你有几种方法做这件事。GitHub pages 用 [Jekyll](https://pages.github.com/) 提供了一个非常简单的方法来完成这个任务。你只需要学习 markdown 来写内容，使用 GitHub 桌面应用程序，你甚至不需要打开终端来更新**你自己的博客**。

最后，如果你真的不想麻烦地使用静态页面博客，你可以选择类似于[实用开发者](https://dev.to/)、[博主](https://www.blogger.com/)、 [WordPress](https://wordpress.org/) 甚至[媒体](https://medium.com)的东西，它们最近已经获得了相当大的吸引力。

通过这样做，你将把你的内容提供给万维网，任何人都可以在任何地方访问它！它将被你选择的搜索引擎搜索到，它将被存储在互联网档案中，你将能够为任何想要使用它的人提供同样的用户体验。
