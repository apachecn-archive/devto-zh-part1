# 感觉卡住了怎么办？

> 原文:[https://dev.to/yelluw/what-to-do-when-you-feel-stuck](https://dev.to/yelluw/what-to-do-when-you-feel-stuck)

每个开发人员都曾在某个时刻感到被卡住了。有时我们觉得被别人抛在后面。我认为这是因为科技行业的变化太快了。我们应该拥抱变化，而不是害怕变化。这些年来，我已经学会了如何将停滞不前的感觉转化为不断增长的机会。以下是方法:

## 在你的舒适区之外工作

我犯过的最大错误就是给自己贴上了`$language`开发者的标签。这是一个错误的原因是它减少了我对其他语言和技术的关注。一个真实的例子是 Javascript 社区中的寒武纪大爆发。我本应该早点少关注 Python，多关注 Javascript，但是我没有。我熟悉 Python，并且知道 Python 的市场份额在增长。随着时间的推移，我开始觉得自己被 Python 卡住了，因为有些事情我想用 Python 做，而我只能用 Javascript 做(就 web 开发而言)。这让我觉得我被困住了。解决方法很简单。我开始发展我的 Javascript 技能。我花了一点时间才意识到我在 Javascript 上取得的进步消除了停滞不前的感觉。

你不一定要学习另一种语言。也许你是一个有 Java EE 经验的 Java 程序员。你可以通过在你的生态系统中学习新的东西来克服被卡住的感觉。也许你会学到一个新的单元测试框架。或者开始听关于 Java 的播客。任何能提供新的不同视角的东西都可以。卡住的感觉来自于一遍又一遍的做同一件事。为了挑战自我和成长，我们需要改变我们所做的事情。停滞不前的感觉就是生长停滞的感觉。

## [](#share-with-others)与他人分享

我从使用社交媒体和网络论坛中获得的最大好处是，我可以与其他开发人员分享，并从他们的工作中学习。一个很好的例子是黑客新闻。我喜欢参与这个社区，因为这里有各种各样的观点和经历。你可能正在和某人谈论某项特定的技术，而开发这项技术的人出现并加入了谈话。能够以这种方式互动是一个强大的学习工具。与更聪明的人交谈是消除卡住感的好方法。确保你的方法是关于倾听和学习的。否则，你不会得到同样多的好处。

## [](#dont-be-afraid-to-fail)不要害怕失败

我过去讨厌分享我代码。我讨厌它。我知道人们会因此评判我。我对自己的编码能力很不自信，因为我希望自己能做得更好。害怕别人会说什么让我躲在一个角落里。事实是每个人都不擅长编码。我们都是。最好的程序员总有一段代码会追捕他们。成功的程序员知道只有失败才能成功。好的编码技能是坏的编码技能随着时间的推移而改进的结果。对失败的恐惧让我们感到停滞不前，因为我们只是封闭了自己的思想，让不安全感占据了我们的头脑。我们应该接受学习是一个充满错误的过程这一事实，而不是让恐惧放慢我们的脚步。只有完成足够的学习，才能取得进步。为了成功，你必须先失败。

## [](#be-patient-with-yourself)对自己有耐心

我很害怕承认，我花了比我想接受的更长的时间来学习这个。你必须对自己有耐心。逼得太紧不好。绝对有可能是自己造成的烧伤。我不想让你经历这些。我怎样才能对自己更有耐心？明白事情需要时间。我们必须分清轻重缓急。我想制造很多机器人，但我也想提高我的 Javascript 技能。与其同时做这两件事，我决定致力于提高我的 Javascript 技能，让机器人稍后再做。设定优先事项意味着我们也必须设定目标。设定目标很棘手。我学到的一件事是，我们的估计总是错误的，事情需要更长的时间才能实现。耐心意味着我们将重新考虑我们的时间表，并做出相应的调整。时间是我们唯一没有的东西，我们需要像管理宝贵的资源一样管理它。

我真的希望这能帮助你前进。感觉被卡住并不好玩，而且会令人沮丧。我相信，如果你愿意尝试我概述的事情，你可以向前迈进，克服这种感觉。它们可能对你不起作用，但你会获得一些重要的东西:接受你感到停滞不前是让你走出困境的第一步。
