# 数据挖掘者 23 可用

> 原文：<https://dev.to/patricktingen/datadigger-23-available-2maa>

就是它，一个新版本。准时作为复活节礼物。感谢 DataDigger 测试团队，这个版本有一些不错的新特性和许多错误修正(谢谢大家！)

## 多重排序

现在，您可以在数据挖掘器中对多个级别进行排序。看一下下面的屏幕。我已经按照客户所在的国家对他们进行了分类:

[![nurc24p](img/f3e92a470c21563b087b3fb60e0e77bb.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--bn2h0WSg--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://datadigger.files.wordpress.com/2017/03/nurc24p.png%3Fw%3D840)

当我按 CTRL 键并单击“State”列时，排序会展开:

[![n765zey](img/99669d12bb42211db8e83733495466c3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ODElsV_T--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://datadigger.files.wordpress.com/2017/03/n765zey.png%3Fw%3D840)

如您所见，记录现在先按国家排序，然后按州排序。通过这种方式，您可以添加多达 9 个级别的排序。

或者，您可以按下过滤器字段左侧的小按钮。有两个上下箭头的那个。它将打开一个对话框，您可以在其中指定或更改排序:

[![6fz35ns](img/6e40444cce304ffd166d5f5e5e934ecd.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s---XCtLPGA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://datadigger.files.wordpress.com/2017/03/6fz35ns.png%3Fw%3D840)

## **题目中的物理名称**

数据挖掘器已经在窗口的标题中显示了数据库和表名。在 settings 屏幕中，您现在可以选择是否要查看逻辑数据库名称、物理数据库名称或不查看。默认情况下，这将是逻辑名称，一如既往。

[![ok8vuwv](img/daa436b78799223be5de9e42a59a85be.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--KSAthBk---/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://datadigger.files.wordpress.com/2017/04/ok8vuwv.png%3Fw%3D840)
