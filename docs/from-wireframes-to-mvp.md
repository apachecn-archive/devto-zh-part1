# 从线框到 MVP

> 原文：<https://dev.to/bravelabs/from-wireframes-to-mvp>

这个话题在我脑海中挥之不去，因为我正试图花更多的时间给我的副业——一个以健身为主题的社交应用——注入活力。具体来说，如何不迷失在尚不重要的细节中，并专注于在实现关键的最小功能集方面取得实质性进展；确保这个特性集充分抓住了预期用户体验的本质，并在一些早期用户(即朋友)手中获得一个版本。

一旦我有了一些用户反馈，我就可以开始进行迭代改进，然后整个事情就像滚雪球一样变成下一个独角兽创业公司，对吗？我很确定事情就是这样的。要么这样，要么我就继续享受发展这个想法的乐趣，同时扩展我的技能。

无论如何，这都是一段值得的时光。

除了定期投入时间的明显努力，我发现我还必须留意那些偷偷摸摸的小生产力杀手，他们试图在我的待办事项清单上把自己伪装成有生产力的项目。比如

*   从头开始实现身份验证解决方案，而不是利用许多受到良好支持的开源 oauth 库
*   一遍又一遍地重新审视用户界面设计，而不是继续做一个目前已经足够好的设计
*   花太多时间评估 PaaS 选项，寻找最能支持我可能永远得不到的用户群的选项
*   当我可以利用我在基于浏览器的技术方面的现有专业知识时，我学会了一种“一次编写代码，就能解决所有问题”的框架来构建多平台移动应用程序

是的，这些都是感觉像生产力的事情，但是我必须记得问我自己- *这真的是帮助 MVP 迈向下一个里程碑的最合适的选择吗？*
