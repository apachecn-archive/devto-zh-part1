# 第一份编程工作需要经验，但是...怎么会？

有抱负的年轻开发人员一直问我的一个问题是:我如何获得我需要的第一份编程工作，以积累大多数编程工作所需的经验？

这似乎是一个先有鸡还是先有蛋的问题。你需要工作经验来获得第一份工作，但没有第一份工作你就无法获得这种经验。这似乎是一个不可能的谜。实际上，这远非不可能。这里有五个不会让你失败的策略。

## 策略 1——在你完成学业之前获取经验

当我还在学校的时候，我做了很多工作，这让我以后的生活变得容易多了。在完成学业之前，我积累了大量的专业编程经验，甚至在我对全职工作感兴趣之前，这些经验就使我成为了一名高级开发人员。

专业编程体验是一种有偿参与，它让你了解如何与雇主互动，并构建你的简历。在学习期间，有几种方法可以获得工作经验。

这里有一些想法:

### [](#be-an-intern)做实习生。

实习是让你沉浸在软件开发组织中的一个很好的方式。作为一名实习生，你将融入一个经验丰富的工程师团队，每天与他们一起工作，为未来的编程工作积累宝贵的经验。

实习可以带薪也可以不带薪。实习最重要的一点是选择一家能让你从事有意义的项目的公司，也就是那些将被运送给客户的东西。

许多实习经历会让你成为同一家公司的全职员工。这是一个很好的入门方式。

### [](#freelance)自由职业。

自由职业是从事真实项目和制作简历的一种极好的方式。有无数的方法可以找到自由职业的机会。一开始，你可以先在 freelancer.com、upworks.com、T2、rent-acoder.com 或 T4 等网站上寻找，然后在那里树立自己的名声。

这是一个充满竞争的世界，但是如果你很优秀，你将开始从你的客户那里获得直接推荐。你的工作量会增长到让你在空闲时间忙个不停的地步。

我 14 岁时找到了第一份真正的自由职业。我被要求建立一个 C 库来管理 MS-DOS 下的文本菜单。客户是一家开发数据库产品的软件公司。他们想在屏幕上打印任何文本，当调用我的库时，需要扫描视频卡的内存，在屏幕的给定区域找到单词，并突出显示其中一个单词，就好像它是一个菜单选项。文本高亮显示需要使用箭头键在屏幕上从一个单词移动到另一个单词。当用户按下 ENTER 时，必须返回突出显示的单词。该库用于构建该公司的旗舰产品。

第二个项目是创建一个具有独特特性的源代码控制管理系统。客户希望用它来管理他们的源代码，但是也希望它能够对编译后的工件进行签名。给定通过编译任何托管项目生成的任何可执行文件，他们希望能够找出每个源文件的哪个版本被编译到工件中。一个我乐于开发的奇怪项目。

当我还在学校的时候，我作为一个自由职业者建立了十几个项目。在我需要寻找我的第一份编程工作之前，这给了我一个坚实的经验。当我申请工作的时候，我已经有了一份很好的简历。

### [](#collaborate-on-open-source-projects)合作开源项目。

GitHub 上有无数的开源项目。如果你在学校，在某些方面合作是非常有意义的。选择一个可见的，大多数人都知道的。几个好的起点:

*   [阿帕奇项目](https://github.com/apache)
*   [轨道](https://github.com/rails/rails)
*   [脸书项目](https://github.com/facebook)
*   [谷歌项目](https://github.com/google)
*   [网飞项目](https://github.com/netflix)
*   [苹果项目](https://github.com/apple)
*   [微软项目](https://github.com/microsoft)

熟悉一个你感兴趣的项目，并开始做出贡献。修复一些 bug，让一些东西更快，增加功能等等。从小做起，熟悉流程，成为摇滚明星。对公共项目的贡献会让你的简历更精彩，你的技能更敏锐。

### [](#commit-personal-projects-on-github)在 GitHub 上提交个人项目。

你不必只做现有的项目。你也可以创建你自己的，并把它们推送到 GitHub。您的 GitHub repo 可以成为展示您的技能和风格的窗口。雇主会看着它，对你能做什么有一个很好的感觉。

当您在 GitHub 上推送项目时，请确保正确地记录它们。包括一个为第一次查看您的存储库的人编写的强大的 README.md。你应该总是假设到达那里的人对你、你的项目、你的目标以及你试图解决的问题一无所知。

### [](#write-a-technical-blog)写技术博客。

当你学习成为一名开发人员时，你可以在日志中记录你的旅程。你的日志可以作为公共博客保存。写博客是一种很好的方式，让你的名字在那里，并建立你的个人品牌。不管你在做什么，你都应该写博客！还不是作为专家，而是作为学习者。关于博客的一些建议:

*   **从免费平台开始。**如果你还没有工作，你没有钱花在托管上。有很多免费的选择，包括使用 [GitHub 作为博客](https://www.smashingmagazine.com/2014/08/build-blog-jekyll-github-pages/)。
*   坚持一个主题。避免写关于你的猫的博客。请记住，你的目标是作为一名开发人员建立个人品牌。
*   把任何抱怨藏在心里。收集抱怨的博客会显示出糟糕的性格和缺乏常识，这与你想要的正好相反。保持积极、充满活力和热情。没有人打算雇佣一个爱抱怨的人。
*   不要和巨魔打架。许多科技界的人通常刻薄且固执己见。你不必回答他们。网上的公开斗争将永远存在，它们不会展示你的品牌。巨魔会打倒你，用经验打败你。
*   **定期发布，不用担心产生流量。**你的目标是学习，建立个人品牌，给雇主留下深刻印象，记录你的学习经历。你不是想靠博客谋生。还没有。
*   保持谦逊。讨论你学到的东西是可以的，但是说教不适合一个初学者(或者其他任何人)。分享，不教。付出，不要奢望。要有主见，而不是一套规则。
*   **培养一种“专业的声音”，**这是你第一次面试编程工作时要用的声音。

### [](#start-your-business)创业。

你还在上学的时候就可以开始创业了。我是在 90 年代做的，那时候要困难得多。记住，像脸书和谷歌这样的产品就是这样开始的，它们成长为科技巨头。有了正确的想法和态度，一切皆有可能。经验是有帮助的，但不是必须的。

制造一个产品，把它放在那里，看看你是否能把它做成生意。它可能不会有任何进展，但是它会建立你的技能和简历。这些技能不仅包括熟练的技术，还包括商业意识，当你去寻找你的第一份编程工作时，这将使你在竞争中领先。

创业完全掌握在你手里，给你一个雇主喜闻乐见的充实体验。互联网创造了一个平台，将一个好主意发展成一项业务，利用它就像想出一个好主意一样困难。很难，但并非不可能，你可以坐在办公桌前完成。

## [](#strategy-2-use-what-you-have-to-your-advantage)策略二——利用你所拥有的优势

如果你是一个寻找第一份编程工作的年轻开发人员，你有一些可以利用的特点。以下是一些可能描述你的例子(如果不全是真的，不要担心):

*   你还年轻。
*   你对编码和技术充满热情。
*   你精力充沛。
*   你渴望并愿意学习。
*   你没有教条。
*   你学得很快。
*   你是一个很好的学员。
*   你有新的想法。
*   你没带很多行李。
*   你已经准备好全力以赴。
*   只要有合适的机会，你就可以去任何地方。
*   你有很多时间。

将你对技术的热情带到桌面上来。你的灵活性和对学习和成长的渴望是你拥有的优势，应该加以利用。

你不能带来经验，但你可以带来新的想法。公司寻求年轻的技术人才，以消除似乎不可动摇的陈旧观念。在一个不断自我改造的技术世界里，教条是危险和令人窒息的。它们像癌症一样生长，使得工程组织发展缓慢。保持谦逊，但质疑一切。

在有好导师的公司里找和你水平相当的工作。如果你找到一个愿意帮助你成长的优秀技术专家，即使最初的报酬不高，你的时间和努力也是值得的。导师们很高兴能给渴望的学生上课。成为一名导游想要的学生。

做任何工作至少两年，除非你非常痛苦。跳槽不是一件好事。你不希望简历上的工作比多年的经验多。

## [](#strategy-3-know-what-to-expect)策略三——知道期待什么

科技行业的面试可能会很难。当你经历第一份编程工作的过程时，你可能会感到不知所措，但是不要烦恼！只要确保你明白科技公司的[面试流程是如何运作的，事情就会变得简单。期待并接受以下一项或多项:](https://www.coderhood.com/6-step-tech-interview-strategy/)

*   **电话筛选。**在安静的地方接听电话。在你面前放一台可以上网的电脑，一张纸和一支笔。
*   **亲自筛选。**通常，这是一次真正的一小时面试，而不仅仅是“聊天”如果你可以在电话筛选和亲自筛选之间选择，选择后者。
*   编程挑战。编程练习是展示自己能力的绝佳方式。在成功筛选之后，有时会给你一个编程问题。你在家做，然后按要求提交。非常认真地对待这个练习，展示你能做什么。
*   **白板编码。**作为一名软件开发人员，你应该期望面试包括在白板上写代码。你必须在 99%的面试中这样做。不用担心；很少有人擅长。要准备，给自己买个大白板，每天练习。当你练习的时候，大声向想象中的面试官解释你的思维过程。展示你的想法是很重要的。好好谈谈。避免只是默默地写代码。
*   **软问题。**期待非技术性问题，如:
    *   你认为我们的产品怎么样？
    *   你为什么要和我们一起面试？
    *   你会如何改进我们的产品？
    *   你的职业目标是什么？
    *   你认为三年后你会在哪里？
    *   我们为什么要雇用你？
    *   描述一个你不成功的项目。

这些问题旨在了解你是谁，你是否在乎你申请的工作，以及你是否适合这个团队。提前准备好可靠的答案，写下来，并随着时间的推移不断完善。让你的故事直截了当非常重要。

## [](#strategy-4-know-how-to-present-yourself)策略四——知道如何展示自己

设计你想如何展示自己，并坚持下去。为你的职业自我建立简单的规则。我不是建议你伪装自己，而是建议你定义自己并保持一致。比如，我建议要谦虚、自信、乐于学习、渴求知识、准备成长。

当你去面试的时候，做好你的功课。确保你对公司和产品了如指掌。展示自己对公司的工作充满热情。清楚地表明你正在面试你的第一份编程工作，并表现出你有多兴奋。

试着弄清楚你想长期做什么，你的目标是什么。你可以随时改变这些。没关系，但是有计划并能够分享计划是很重要的。雇主会对有明确目标的人印象深刻。

## [](#strategy-5-decide-that-you-can-do-this)策略 5——决定你能做到

最重要的是，从中获得乐趣。找工作不应该是一种折磨，它可以是一次学习经历。一些需要记住的事情会对你有所帮助:

*   记住，根据定义，你需要找一份第一次编程的工作，而不是很多。
*   你在面试公司，就像公司在面试你一样。
*   这个世界充满了可能性。如果你愿意搬到任何地方，有成千上万的选择供你选择。
*   开发商的需求量很大。不要让你以前没有工作经验的事实让你认为你不够格。你公开申请一个初级角色，你会找到的。
*   永远不要说谎。这对你没有帮助。
*   仔细听着。即使你比面试你的人更聪明，也要听他们说些什么，并集中注意力。
*   不要相信你必须满足工作描述中的所有要求。准备好解释为什么即使你不符合某些要求，公司也应该雇用你。保持谦逊，但要表现出自信。
*   永远不要过河拆桥。技术社区很大，但没那么大。从长远来看，桥梁对你没有帮助。
*   你能做到的。
