# :悬停应该是死的

> 原文:[https://dev.to/designfrontier/hover-should-be-dead-1am0](https://dev.to/designfrontier/hover-should-be-dead-1am0)

时日无多了。它在这个世界上剩下的时间不多了。

我是认真的。

以下是主要原因:

*   它不存在于触摸设备上

*   它并不容易接近

*   这经常会产生令人沮丧的互动

### 触摸设备

我说触摸设备，而不是移动设备，是有原因的。触摸设备正以越来越快的速度从我们的手机扩展到我们的环境中。他们还扩展到我们的笔记本电脑和台式机。所以说你的“目标市场”是台式机并不意味着你可以使用`hover`。

因为那个花了 650 美元买了一台[戴尔 Inspiron](http://www.dell.com/us/p/inspiron-15-5559-laptop/pd?oc=fncwv2319s&model_id=inspiron-15-5559-laptop) 的家伙，当他用手指触摸你的网站时，也将会与它们发生冲突。他可能会切换回他的触摸板...但他也可能会沮丧地离开，不知道如何打开你的菜单。

### 易接近

键盘用户得不到`hover`。他们得到了`active`，但没有得到`hover`。这是一件大事。屏幕阅读器也不擅长伪装`hover`,如果他们真的这么做的话。不过他们和`active`相处得很好。

### 令人沮丧的互动

想想嵌套的悬停状态菜单。桌面应用程序要求你至少点击第一级菜单是有原因的。因为不小心把鼠标从刚刚打开的菜单上移开应该不会关闭它。

这也可能落在可访问性下。想象一下，用一只颤抖的手浏览你的网站。想想如果我的手发抖，你的菜单对我来说会有多难。

[UX 运动](http://uxmovement.com/navigation/why-hover-menus-do-users-more-harm-than-good/)有一些关于“悬浮隧道”的漂亮图片和讨论，以及为什么它们对用户来说是个大问题。

### 死亡来了`hover`

所以还是去掉`hover`吧。这实际上并没有让事情变得更好，反而让很多事情变得更糟。这是一种“聪明”的互动，而不是一种好的互动。
