# 大卫·海涅梅尔·汉森谈铁路的未来

> 原文：<https://dev.to/ben/dhh-on-the-future-of-rails>

Ruby on Rails 框架自 10 年前创建以来产生了不可思议的影响，成为当今一些最强大的科技公司的选择。框架推广的模式和概念也对应用程序架构的讨论产生了不可磨灭的影响。Rails 是一个开源项目，为了保持其方向和地位，它依赖于核心团队的创新和良好的软件设计。新的范例不断涌现，Rails 在发展方面做得很好，仍然是许多开发人员的首选。

Rails 是一个开源项目，为了保持其方向和地位，它依赖于核心团队的创新和良好的软件设计。Rails 的发展方向在很大程度上取决于固执己见的创始人兼 Basecamp 首席技术官大卫·海涅迈尔·汉森(David Heinemeier Hansson)的观点。我联系了 David，以便收集一些关于 Rails 方向的见解。David 与 Practical Developer 分享了他对未来的展望。

## 自从 Rails 4.0 在 2013 年夏天发布以来，发生了很多事情，尤其是在 Node 社区。与过去相比，Rails 现在处于什么位置？

Ruby on Rails 的目标和以往一样:成为一个全栈框架，让构建像 Basecamp、Github、Shopify、Kickstarter、Airbnb、Zendesk 等应用成为可能，并让小团队感到愉快。当然，编程从来不是一成不变的。我们已经看到 Node、Elixir、Go、Rust 以及其他各种语言和框架的出现，这太棒了。越多越开心！网络是一个令人惊叹的平台，因为它将所有这些方法放在一个公平的竞技场上，当涉及到生成 HTML、JavaScript、CSS 时。在网上，没有人知道你的服务器是运行 Ruby 还是 Go 或者 Node。对我来说，我所做的项目和我上面列出的例子，我还没有看到任何真正让 Rails 物有所值的东西。

## Turbolinks 和 API 模式是开发视图的两种非常不同的方法。你和你在 Basecamp 的团队打算利用这两种设计模式吗？在过去的几年里，你对客户端渲染应用的个人立场有什么变化？

Turbolinks 真的在加速前进。早期的膝跳反应已经消退，许多人正在重新审视它，并真正喜欢他们所看到的。我们在 Turbolinks 5 中进行了彻底的重写，通过关注与原生 iOS 和 Android wrappers 的集成，进一步推进了这一点。这是一个巨大的飞跃，它的技术使我们能够在不建立庞大的本地团队的情况下，为 Basecamp 3 提供出色的本地客户端。我真的很高兴分享所有这些进展，并让更多的团队能够在重要的地方(主要是导航和一些峰值 UI 保真区域)进行本地化，然后依赖他们的 Rails 应用程序作为集成系统。美丽的巨石。

与此同时，API 模式扩大了 Rails 对所有使用客户端 MVC 方式和 100%原生移动应用的人的吸引力。我们没有理由不合作改进 Active Record、Action Pack、Action Mailer、Active Job、Action Cable 以及 Rails 提供的所有其他与 HTML 视图生成无关的框架。绝大多数 Rails 既与全栈应用相关，也与纯 API 应用相关。所以明确我们帐篷的大小是令人兴奋的。

## 你能总结一下你认为 2016 年的“铁路之路”代表什么吗？

目前，我实际上正在写一本名为*Rails Doctrine*的新迷你书，该书详细介绍了这一切。但简而言之，Rails 的任务是装备小团队来处理大项目。我们在这里并不是为了让这个 500 人的公司变得更有效率，这只是一个副作用。对 Rails 的忠诚是让一个、两个或几个程序员有可能创建跨越 web、电子邮件、移动、本机、API 和我们需要在 2016 年覆盖的所有其他平台的惊人应用。为网络提供服务变得越来越复杂。当事情变得困难和条块分割时，有一千个复杂的供应商和小贩从中受益。Rails 在这里是作为一个计数器。保留 web 构建的美观和简单，而不需要一大群专家。

## 您认为您的“重新考虑”文章如何影响了对 Rails 的总体看法？

Rails 从第一天开始就是反文化的。许多基本原则，从配置上的惯例到菜单，仍然存在很大的争议。Rails 代表了一些东西，这意味着它肯定会疏远一些被冒犯的人。[重新考虑](https://signalvnoise.com/posts/3972-reconsider)，我写的那篇痛斥圣弗朗西斯科的经济模式、风险投资和*完全控制市场的春梦*的博客文章只是相信某些东西的自然延续。我怀疑这不会有什么影响，真的。据我所知，目前很多风投资助的科技初创公司已经开始涌向最新的科技热潮。不管是好是坏，Rails 已经不再是最新的技术热点。它现在是主流，至少在影响范围方面是如此(尽管许多原则仍然是反文化的)。我认为这是一件非常好的事情。

## 还有哪些编程趋势引起了你的兴趣？

函数式编程的兴起很奇妙。我让我的编程风格受到关键概念的极大启发，而不需要抛弃 Ruby。Ruby 是一个极好的海绵，可以吸收编程中的最佳想法。它显然不是单范式语言，就像 Rails 不是单范式框架一样。我们可以采用函数式编程的最佳思想并从中受到启发，而不需要放弃面向过程或面向目标编程的最佳思想。愿我们继续受到这样的影响。过去几十年的编程语言的丰富历史仍然有很多东西可以教给我们。
