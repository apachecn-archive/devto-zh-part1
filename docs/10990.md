# 如何创建一个欢迎和包容的开源空间

> 原文:[https://dev . to/captainsafia/how-to-create-a-welcome-and-inclusive-open-source-space-196 a](https://dev.to/captainsafia/how-to-create-a-welcoming-and-inclusive-open-source-space-196a)

*最初发表于[OpenSource.com](https://opensource.com/life/16/3/creating-welcoming-and-inclusive-open-source-space)。*

在 2013 年的一项调查中，11%的自由和开源软件的贡献者是女性。但是也许未来看起来更光明？我们可以通过研究女性参与[谷歌代码之夏](https://developers.google.com/open-source/gsoc/)的情况来回答这个问题，这个项目为专上学校的学生提供津贴，让他们在一个夏天为开源软件做出贡献。从 [2011 年](http://google-opensource.blogspot.com/2012/05/google-summer-of-code-2012-by-numbers.html)到 [2015 年](http://google-opensource.blogspot.cz/2014/06/google-summer-of-code-2014-by-numbers.html)，该计划由大约 7-10%的女性参与者组成。这是一个极低的百分比，对于开源多样性的未来来说不是一个好兆头。

最近，北卡罗莱纳州立大学和圣路易斯奥比斯波的加州理工学院联合发布了一项关于开源性别偏见的研究。尽管该研究已被多家媒体广泛传播和分析，但我还是要对我自己的分析提出警告，声明该研究尚未经过同行评审。此外，研究中的发现仅适用于 GitHub 用户群的 35%,这在任何方面都不能代表整个开源社区。也就是说，这项研究确实得出结论，对于这个特定的用户子集，女性的拉取请求确实比男性更容易被接受。这是什么意思？我们可以得出这样的结论，尽管女性并不占开放源码贡献者的大多数，但她们的贡献与男性的贡献同等重要，甚至更重要。

女性似乎是非常能干的开发人员，但是在开源社区中只占一小部分。看起来，女性对开源的参与看起来不会增加。我们如何改变这种情况？通过实施行为准则，有效地调节关于拉请求和问题的交流，为新贡献者创造一个健康和友好的环境，建立非数字空间，并实践同理心，开源项目可以创建多样化和包容性的社区。

### 行为准则

行为准则是任何开源软件项目的重要组成部分。如果你是一家打算将内部项目上市的公司，在这样做之前，确保你有一套行为准则。如果你是一个单独的开发者，准备推动一个个人项目的第一次公开提交，确保你在这样做之前有一个行为准则。对于行为准则是否应该包含在代码库中，不应该有任何犹豫或疑问。我们有管理我们的州、我们的国家和我们的全球社会的行为准则，那么为什么我们的软件中没有行为准则呢？行为准则巩固了项目的期望和目标。信不信由你，尽管你可能对项目社区中什么样的行为是可接受的、什么样的行为是不可接受的有所期望，但有些人却没有。行为准则有助于为围绕数字世界行为的全球对话设定基线。

有很多行为准则可以作为你项目的一部分。我个人推荐使用[贡献者契约](http://contributor-covenant.org/)。文档的语言简单、简洁、易于理解，并且已经包含了几种翻译。此外，它目前被一些非常著名的开源项目所使用，比如 [Atom](https://atom.io/) 和 [Ruby on Rails](http://rubyonrails.org/) 。

### 关于减贫战略和问题的对话

在开源社区中,“拉”请求和问题可能是最紧张的环境之一，这是理所当然的。在其中，除了他们写得很好的代码和错误报告之外，人们还暴露了他们仓促编写的代码和半生不熟的特性请求。拉请求和问题的暴露性质让新老贡献者对他们在项目中的位置感到紧张。因此，在网上与其他开发者交流时，保持文明、同情和尊重的语气是很重要的。这可能看起来要求不多，但是你会惊讶于情绪心态和一个容易使用的回车键可以造成多大的破坏。

在数字空间中，语气很难辨别，我们经常会将当前的情绪和背景应用到我们在网上阅读的东西上。如果你对某个请求或评论感到很恼火或受到了侮辱，那就换个时间再读一遍。如果你仍然对评论的语气感到矛盾，要求澄清。永远不要——我重复一遍，永远不要——用你个人的心态来影响他人的评论。这种简单的误解引发了许多沟通失误和问题，如果你在交流时假定对方是无辜的，就可以很容易地避免这种误解。

也就是说，分歧确实经常发生。当他们变得激动或情绪化时，项目维护者在每次讨论中坚持以下规则是很重要的:

评论想法，而不是想法者或他/她的能力。在所有非常固执己见的讨论中包括一个公正的主持人。适当执行项目的行为准则。你会注意到，在这一节中，我没有提到任何关于边缘化个人的东西。这是因为拉式请求和问题上的负面交互会对所有贡献者产生负面影响，但来自边缘化背景的贡献者受到的影响更大。当你必须时刻警惕你的语气和想法在空间中的表达方式时，有人以负面的方式收到你无害的评论或不太完美的请求会产生不利的影响。

### 新贡献者的价值

新的贡献者是任何开源项目的生命线。虽然有些人可能会被诱惑去赞美核心开发人员和维护人员，但实际上是新的贡献者保持了项目的精神。

新的贡献者防止开源开发团队成为小团体。新的贡献者将有趣的新想法引入项目。新的贡献者提出了与现状相反的想法。因此，与新贡献者的有效合作应该是任何开源项目的首要任务。

在编写开源软件几年后，你可能已经忘记了第一次创建一个库的感觉，紧张地敲击键盘编写你认为不好的代码的感觉，在拉请求按钮上徘徊几个小时想知道你的贡献将如何被接收的感觉。成为一个新的贡献者是一个在情感和精神上令人疲惫的过程，但是有办法让新的贡献者不那么累。

首先也是最重要的，确保你有[写得很好的投稿指南](http://pandas.pydata.org/pandas-docs/stable/contributing.html)是很重要的。无论是简单的减价文件还是纯文本文件，都要确保您的指南包含了关于开发设置、测试驱动的开发过程、您使用的任何风格指南以及您提交 pull 请求的任何过程的完整信息。不要害怕复制别处存在的信息。提供关于分叉和克隆、创建分支和提交变更的信息是完全合理的。尽管作为指南的作者，它给你带来了很多开销，但是它为新的贡献者提供了一个参与项目的单一的、权威的来源。

除了包含完整的投稿指南之外，还可以考虑包含一个截屏，直观地引导新投稿人完成投稿工作流程。如果截屏太费力，可以考虑制作一个描述工作流程的快速信息图。例如，查看我为[项目 Jupyter](http://jupyter.org/) 制作的[贡献工作流信息图](http://jupyter.readthedocs.org/en/latest/_images/contribution_workflow.png)。不同的人以不同的方式学习，为新的贡献者提供不同的方式来学习如何参与你的项目，让他们知道你意识到并注意到他们独特的视角。

然而，在新的贡献者开始参与项目之前，他们必须知道他们能做什么。用关于完成问题所需的贡献类型的信息来标记项目中的问题(文档、测试用例、特性、错误修复等)。)、特定问题的难度(低、中、高)以及优先级可以帮助新贡献者为他们的第一个拉动式请求找到最合适的问题。标记问题的类型和优先级可能相对容易，但标记难度确实需要更多的细微差别。在标记问题的难度时，我认为考虑以下问题很重要:

*   解决这个问题是否需要跨多个文件进行更改？
*   解决这个问题是否需要特定主题的特殊知识(线程、底层网络通信等)。)?
*   解决这个问题涉及到与代码库中未记录或未测试的部分进行交互吗？

回答这些问题不仅有助于您向新的贡献者提供更多的信息，还有助于您评估自己的代码库的质量并努力改进。

### 非数字空间的重要性

除了数字论坛之外，有机会围绕开源项目建立现实世界的联系也很重要。这些联系不一定发生在围绕项目组织的会议上——它们可以发生在任何地方。当人们在现实世界中相互交往时，一定程度的尊重和友情是可以实现的。社区成员可以在当地的咖啡店或合作空间举办弹出式活动，包括协作编码、随意交谈和知识共享。不管我们喜欢与否，人们有不同的数字和现实世界的人物角色，允许一个社区在互联网之外围绕你的项目发展给了具有强大社交技能的潜在贡献者参与项目的机会。

### 增强共情

在一个联系日益紧密的世界里，在工作环境中，与全国各地甚至全世界的人一起编写软件是理所当然的。在开放源码中，世界范围内的交互更加普遍，任何人只要能访问互联网和使用文本编辑器，就可以了解开放源码项目并为之做出贡献。当我们试图与来自我们甚至可能在地图上无法识别的国家的人进行技术和非技术对话时，这必然会引起很多紧张。我们如何有效地穿越开源软件的多元文化生态系统？它涉及到你可能已经听过一遍又一遍，但可能很难完全理解的东西:工程共情。我相信同理心是你可以增强的肌肉。也就是说，下面的行为对提高你的移情能力大有帮助:

*   阅读来自不同文化的开发人员撰写的技术博客帖子。
*   观看在你居住的国家之外的国家举行的会议的技术报告。
*   如果你能流利地使用另一种语言，试着用那种语言阅读新闻。这将让你了解在开源软件中，非英语母语的人是什么感觉。

这些技术将帮助你发现不同背景的开发人员是如何思考、写作、演讲和分享的。随着时间的推移，你会对不同的人对待软件开发的方式产生一种洞察力和欣赏力。

### 最后的想法

你会注意到，我描述的上述建议并不是专门针对边缘人群的。当你创造了一个欢迎和接纳边缘化个体的空间，你就创造了一个欢迎和接纳所有人的空间。
