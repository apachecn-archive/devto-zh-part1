# 了解你的术语

> 原文:[https://dev.to/stipegrbic/know-your-terminology-5agp](https://dev.to/stipegrbic/know-your-terminology-5agp)

Windows 应用程序术语是怎么回事？我知道它变得一团糟，很容易混淆，特别是当你不在 Windows 世界里的时候，微软在过去几年里做的平台重置肯定没有帮助。但是有相当多的人应该理解它，但是他们没有。他们只是不想花那么少的时间去理解他们在做什么，这就是这篇文章的原因。

### Windows 应用的状态

自从 Windows(手机)应用程序出现以来，我一直在和它们打交道，我可以证明，那些不喜欢这个平台的人有很多困惑。微软再一次重置了这个平台，去掉了移动操作系统名称中的“电话”部分，并且没有为它提供任何官方名称。我认为这是他们的意图，确保人们明白这是同一个操作系统，同一个 Windows 10。虽然它实际上是一个操作系统，一个版本可以在各种设备上运行，但人们现在更加困惑了。客户和项目经理会说:“*你怎么说它不是 Windows Phone？我想要一个在手机上运行的应用程序。*

### 真实故事

我**知道**当 Windows Phone 7 出现时，至少有一个这样的困惑，客户想要一个应用程序。现在，我不知道客户是否不能很好地沟通，或者项目经理是否懒得理解需求，但不知何故，一个新的 Windows Mobile 项目创建了。你可以从这样的例子中看出，为什么决策者真正理解他们正在处理的术语是至关重要的。

### 不只是窗户

不幸的是，这种“*我不在乎，把它做完*”的态度并不局限于 Windows 应用程序开发，或者根本不局限于软件开发。当你在处理一些业务时，你应该确保你至少理解基本的术语，没有任何借口。尤其是当你想做一个项目，想从中赚点钱的时候。
