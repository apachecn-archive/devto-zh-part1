# 为什么做事情很难

你有没有发现你被成千上万的小事淹没，这些小事阻止你取得任何真正的进步？

在软件中，这种事情经常发生。也许比其他行业更常见——我们甚至为这些尴尬的琐事想出了一个方便的名字。“阻碍者”是我们不前进的所有借口。

人们可能很想就此打住，去做别的事情。当我们遇到问题时，我们回避它。我们必须在其他事情上“取得进展”，同时等待问题自行解决。我们的工作最终看起来像这样:

[![](../Images/9ab5c9f6d5a241bf5d3ffbeb4aa16fd6.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rIJBLxf4--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AzG5hUZ4AKhWSDN8kxvnMsQ.png)

所有这些前任代表着我们留下的半成品。如果运气好的话，我们可能会完成其中的一件事，但是看起来我们开始的比我们完成的要多，不是吗？

如果我们继续开始多于完成，我们会发现事情堆积如山。在我们意识到这一点之前，我们会被那些已经完成了一半的工作弄得不知所措，这些工作模糊了我们对什么是重要的事情的看法。

这时候我们开始救火，拼命地为谁喊得最响就给谁挑活干。

这是一种可怕的生活方式。

这种情况是我们都需要努力避免的。解决方案描述起来简单，但做起来很难。在你开始一件事之前先完成一件事。停止在工作流之间跳跃，专注于要完成的事情。

难的是，这可能意味着在你的障碍被解决之前，什么也不做。最好的团队学会识别这些情况，并一起尽快解决问题。通过以这种方式专注于清除路障，你将最终完成并交付*某样东西*。

而且一件交付的东西绝对比十件事情做了一半要好。
