# 我是一名网络开发人员

> 原文：<https://dev.to/kdv24/whoami-i-m-a-web-developer-1466>

就在六个月前，我决定要成为一名程序员。现在，我在一家大公司做网页开发。在如此短的时间内发生了如此大的变化，令我感到惊讶。我开始使用 PHP、ReactJS、少量 NodeJS/npm、Ruby on Rails 和 MySQL。我使用 git 进行版本控制，使用 Foundation 进行 CSS，使用 MAMP 作为我的 PHP 开发服务器。我在控制台工作，最低限度，维姆。在三月之前，我不知道如何做这些…一点也不知道。

问题是，我不是一个了不起的程序员，我在这里的道路也不是那么特别。这是我想写这篇文章的主要原因。我不是火箭科学家(好吧，实际上我在电视上扮演一个，但那是一个单独的帖子)。我只是一个想学习编码的人，我想完整地讲述我的故事，讲述我的不安全感和不足之处。我希望其他人会读到这篇文章，并因为我的分享而感到更加自信。

所以，这就是为什么我*不应该*成为一名程序员(基于刻板印象)，以及我的故事的一个相对简短的版本:

1.  我没有编程背景，
2.  我是一个年龄足够大的女人，我的孩子已经上高中了，
3.  我并不特别想知道我的计算机内部发生了什么，
4.  我不太喜欢幻想、物理、电子游戏(除了马里奥赛车，我有点不擅长)，或者外太空(这让我有点害怕)，还有
5.  我甚至不确定我是否正确地列出了我的介绍段落中的所有内容，或者我是否以这样的方式写了它，它真正做的是突出我没有真正得到 node.js 或 MAMP 或任何我歪曲的东西！

尽管有上述清单，今年 3 月 2 日，我还是参加了在 [Epicodus](http://www.epicodus.com/) 为期 15 周的训练营的第一天。我参加的课程主要是 PHP、JavaScript 和 Drupal，但这与我完全无关。在此之前，我是一名小学和中学教师，没有真正的编程知识或特定的语言选择。所以，我选择了 PHP 课程，仅仅是因为它是在对我和我的家庭影响最小的时候开设的。

在我的 PHP 班上，我们 30 个人每周有 5 天从早上 9 点到下午 5 点聚在一起。我们每天从短暂的起立时间开始，充满了公告和分享；然后，我们分成不同的小组去完成分配给我们的任务。我们的日常工作是基于从以前的作业和家庭作业中学习，但也包含了超越我们所学的机会。我们学习的大部分内容是弄清楚如何向 Google 提出正确的问题，然后将我们收集到的信息转化为代码中可用的内容。除了上课时间，晚上和周末的聚会和黑客马拉松总是比一个人实际参加的多，涵盖了几乎任何可以想象的话题，以及许多关于额外阅读材料、教程等的建议。来消化。

对我来说不容易。有好几次觉得自己脑子很笨，思维像泥巴一样。我觉得自己不够好，因为我把自己和那些比我理解得更快的人进行了比较。当我明显缺乏理解时，我挣扎着。一天结束的时候，我的大脑又满又累，但我努力让自己停止思考，因为我无法停止思考和编程，即使是在睡觉的时候。很难在家庭和学业之间平衡我的时间，我经常觉得我牺牲了两者。至于我的朋友，那几个月我完全忽略了他们。

我很喜欢，现在也是。开发过程中创造性和分析性思维的结合让我上瘾。我喜欢任何任务都有多种方法和不同的解决方案。我喜欢在通往“最终”解决方案的道路上建立小的成功(在这一点上，可能是在多次失败之后)。坦白地说，我喜欢键盘上的咔嗒声和用十六进制代码做出漂亮颜色的能力！我感到如此幸运，有人认为我值得为此付出代价。

不过，这是不安全的部分-

每周，我都不得不把自己从恐惧中抽离出来，我害怕我会被发现，我的老板会意识到我不知道自己在做什么，他会决定他不应该雇用我。实际上，我怀疑这有点像老师没有注意到学生只知道字母表中的六个字母。当然，他知道我还不擅长这个，但是因为我目前的编程知识，他没有留下我。他把我留在身边是因为我的潜力，因为我已经拥有的技能，随着时间的推移，这些技能将使我成为一名伟大的程序员。

他看到了我的好奇心、我提问的意愿、我学习和提高的愿望、我的视野、我的沟通技巧和与他人合作的能力、我向他人学习和教学的愿望。他看到我愿意尝试，我采取主动，当我意识到我正在失去动力时，我会寻求帮助。这些是他付钱让我去的原因。

不要误解我，我并不讨厌我所做的。然而，我是一个初学者，有一个学习曲线。每个人都是如此。不管我觉得自己与另一个人相比如何，或者那个人似乎理解得有多快，任何人知道的东西都远远少于应该知道的东西。而且，我相信真正伟大的领导者、教练、远见者和教师是那些足够谦逊，寻求从每个人身上学到东西的人。将自己与他人比较对任何人都没有帮助，只会让我感到不安全、不舒服或高人一等——这些似乎都不是一个好地方。

所以，除了提醒我们不要拿自己和别人比较之外，我希望人们从这篇文章中学到的，以及当我感到不安全并再次阅读这篇文章时，我想听到的是:

我应该在这里。我擅长我所做的事情，那就是追求如何变得更好。不管我进入这个领域是六个月还是六十年，我总是想做好那些让我今天站在这里的事情。我要好奇，要开放，要坚定。我想继续寻找、倾听和关心。而且，我总是想笑得很快，无论是对自己还是对别人。显然，我想知道更多关于如何成为一名更好的程序员的技术部分，但那将是因为我已经是谁了。这就是为什么我应该站在这里。
