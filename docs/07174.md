### Dan lebrero 5 ' 17 " 3 分钟阅读

这篇文章在评论部分引发了热烈的讨论，许多开发人员和用户都在权衡静态类型的利弊:

[![mesosupe profile image](../Images/121be6e1afda62bfdb8a460c9cb6f80f.png) ](/mesosupe) [ Supe ](/mesosupe) • [<time datetime="2017-06-05T22:00:38Z"> Jun 5 '17 </time>](https://dev.to/mesosupe/comment/9aa) 

我可能遗漏了一些东西，但是我一直认为静态语言的价值在于 API 和框架的可发现性(也就是强大的自动完成)，而不是避免错误。当编译器知道一个变量的类型时，它就能更容易地知道你能用它做什么，并避免去查阅文档。这也是我喜欢不碍事的静态类型系统的原因(多 C#，少 Java)。

此外，社区也发挥了作用。除了一些例外，错误密度更高的语言似乎吸引了更多的初学者。
