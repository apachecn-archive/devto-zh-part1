# 我做出了我的第一个开源贡献…它甚至不是代码！

> 原文:[https://dev . to/chiangs/I-made-my-first-open-source-contribution and-it-not-even-code-11b 8](https://dev.to/chiangs/i-made-my-first-open-source-contributionand-it-wasnt-even-code-11b8)

你没看错……我终于为开源做出了我的第一份贡献，它只是一个 ***TypeScript 框架集合*** 的图标/ logo，一个与 Angular: [link 配合使用的 Java Collections 框架的端口。](https://dev.to/chiangs/passing-data-between-nested-components-with-angular-50cm)

> 对于一个初学者来说，为开源做贡献可能是令人害怕的，并且很难决定从哪里/如何开始以及整个过程是如何发生的。"

尽管难度很小，但我非常自豪终于度过了难关。很久以来一直想参与开源，但是发现过程有点混乱和恐怖。直到发现 ***24 拉请求*** : [链接，我才真正知道从何下手。](https://24pullrequests.com/)

在那里，我找到了几篇关于如何找到好的起点以及如何选择一个起点的文章。它们是为你聚集的相当多的工具，但最终我只是使用 GitHub 的搜索栏并输入`label:"good first issue"`，它返回了所有被标记为这样的项目的列表。

许多文章建议选择你的第一个项目的标准是难度，或者是你经常使用的工具。我选择了这个，因为它帮助我快速完成了从搜索到拉取请求的整个过程。因此，让我们详细介绍一下我的步骤:

#### [](#search-amp-selection)搜索&选择

如上所述，在浏览了各种聚合器和工具后，我最终只选择了 GitHub 的搜索栏和条目:`label: "good first issue"`。

接下来，浏览列表，我过滤出已关闭的问题和关键词，我还检查项目的活动级别，以及他们是否接受拉取请求。当然，我想增加回应的可能性。

一旦我发现了这个问题，我会通读这些信息，首先看看是否有人已经提出了这个问题或者已经接近解决方案，然后发布我自己的信息，宣布我想要尝试解决方案。

#### [](#clone-the-repo)克隆回购

接下来我分叉了主项目，做了一个名为`chiangs`的分支。然后我把它克隆到我的机器上。

我制作了我的图像，然后将它们复制到我的本地回购中，然后再次确保图像符合预期的规格。最后，我提交了更改并将其推送到我的分支。

#### [](#make-a-pull-request)提出拉取请求

接下来，我返回到主回购，并开始拉请求。在这里，我确保引用了解决方案#和描述，并对我的贡献进行了深入描述。

就这样，现在我期待着下一期——当然这次的难度会更高。但是现在，整个过程不再那么令人畏惧，而是相当令人满意。

快速更新:我最近听说了这个很棒的开源笔记应用， [Boostnote](https://boostnote.io/) ，我一直在 GitHub 上关注它，考虑参与进来。我发现了一个快速简单的`main page needs translations`问题，所以我很快提出了一个丹麦语版本的请求。这可能不是我最初声明要寻找的更高难度，但这是一件快速简单的事情，我可以说我现在非常沉迷于为开源做贡献。然而，我想我可能已经找到了我的下一个项目，在我了解了桌面和移动应用程序的全部背景之后，我可以贡献实际的代码。
