# 实习 vs 实习

> 原文：<https://dev.to/brightdevs/internship-vs-internship-1p0>

你好！我是 Radek，我是一名程序员。上周一，我在光明发明公司做全职软件开发人员。

成为程序员有很多方法。但是当你在理工大学学习计算机科学时，有 99%的机会在毕业前很久就能得到一份 IT 实习，而且几个月后就能得到——你的第一份全职工作。好像我可以。我经历过两次实习，都是在看似相似的公司。第一份工作后，我得到了一份全职或兼职工作的机会，只要能更方便地将工作与学习结合起来就行。我说...没有。第二次实习后，当我得到一份工作时，我高兴极了。

这两个有什么不同？

一切。

# 尊敬的 CEO，实习如何操作。

## 1。不要给实习生编造项目。

哇，谁会这么做？说真的，我有种感觉，好像又要重新学习了。我来这份工作不是为了玩沙盒。这是什么原因呢？如果你认为我不适合为客户做专业项目，那你当初为什么还要雇佣我呢？让我学习的安全环境和某种门上挂着“未来开发者农场，请勿敲击玻璃”牌子的怪异孵化器有着显著的区别。这个项目本身在理论上非常有趣，我对此有一些想法和意见，但几周后我问自己...干什么用的？知道你的工作完成后会被扔进垃圾桶，而且没有人会使用它，这是非常令人沮丧的。另一方面，在 Bright，我第一天就被分配到一个真正的项目。当然不只是这样，但这是一个明确的信息，即我的工作是重要的，是需要的，这使我的动机达到了最高水平。真的很重要。

## 2。给他们好的报酬，尊重他们。

我并不是说我在第一份工作中受到了任何形式的虐待。但是让我们澄清一些事情——在走廊上微笑，不时友好地交谈，甚至公开你对这个项目的意见都不能掩盖这样一个事实:如果你得到的是糟糕的钱，你会感到糟糕。这甚至与你在公司的感受无关，因为这影响到生活的方方面面。作为一名实习生，从公司最底层做起，报酬当然不多。但是普通初级 JS 开发人员工资的 15%？你不能指望它会让任何人满意，即使是没有专业经验的学生。甚至不要尝试无薪实习。

## 3。让其他员工意识到对实习生有帮助。

如果你想让我学，我会的。但大部分是来自更好的人。如果你只把我放在一组实习生中，我们如何发展我们的技能，避免编程中的不良做法？在大多数情况下，我们无法从教程中学到这些，也无法自己解决。实习的主要目的是填补为分数和为客户开发项目之间微小但有意义的差距。我们需要代码审查，我们需要反馈，我们从第一天就需要反馈。不仅仅是为了您的最终产品变得更好，也是为了我们，这样我们才能变得更好。这是双赢！如果你不能指定一个人来做评估，就把它当成一项投资。会有回报的，我向你保证。让别人变得乐于助人，至少是几个人。在 Bright，他们知道，嗯，我有这种感觉，我可以向任何人寻求帮助，而不会被忽视，即使在问了第一百个问题之后，根据我在前一家公司的经验，这是非常令人惊讶的。

## 4。如果你不是公司，就不要试图成为公司。

老实说，我不知道是什么让这家公司成为一家公司。但我所知道的是，评估一名员工需要的不仅仅是一份带有花哨图表的报告和开发者坦白代码的官方会议。我看到我的同事在总结会上解释为什么他们的工作不能在项目的前端看到。因为他们在做后端和数据库...总结、规律化、比较让你感觉自己像一个随时会被取代的无关紧要的齿轮。首先，了解项目的性质。其次，和你的实习生谈谈。如果你没有时间，那么依靠那些有时间的人，相信他们的意见。除非你经营的公司超过 100 名员工，否则不要躲在报告和数字后面。他们有时会撒谎。

## 总结

我刚刚结束在光明发明公司的实习。和前一个比较让我明白了工作环境是如何影响你的工作态度和动力的。你可以充满活力地开始一份新工作，但是接下来会发生什么呢...

...不仅仅取决于你。
