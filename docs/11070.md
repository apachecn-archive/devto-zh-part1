# 2016 年的决议

> 原文:[https://dev.to/craigbuchek/resolutions-for-2016-aoh](https://dev.to/craigbuchek/resolutions-for-2016-aoh)

在过去的几年里，我已经写了一份与职业相关的决心清单。我已经相当成功地实现了这些目标，所以我将继续这个传统。

## [](#coding)编码

今年想增加我写的的[开源代码。基本上就是我写的和工作没有直接关系的东西了。](https://github.com/boochtek)

去年年底，我接手了维尔图斯号的维护工作，但一直没有找到合适的时间。这可能更多地是对 GitHub 上的问题和合并拉取请求的响应。但最终，我可能会添加一些功能并做一些重构。

我的目标是今年开发 6 个应用程序和库。这是一个相当大胆的目标，但我想把重点放在学习新东西和构建一些简单但有用的东西上。想想 MVP——专注于不需要太多的努力就能获得巨大的成功。

我想开发一个新的 Rails 应用程序，让自己熟悉 Rails 5。我想体验一些更现代的宝石。我还想看看我是否可以在 Rails 应用程序中使用一些新技术，尤其是可能有助于我们接近六边形架构的东西。我也希望能与一些不同的 web 框架合作——可能是 Lotus、Rodakase、Trailblazer 或 Phoenix。或者可能是 Crystal web 框架之一。

我还想做几个 Elm 应用程序。我成立 STL Elm 小组，作为学习语言的借口。我想做一个 Twitter 客户端，最终可能会变成一个更通用的阅读应用。除了 Twitter，它还可以记录我想看的东西。我想写的另一个 Elm 应用程序是一个游戏，基于我喜欢在 Commodore 64 上玩的旧的太空出租车游戏。

我去年写了一个名为 [Ruby Preserves](https://github.com/boochtek/ruby_preserves) 的微表单。我已经开始从处理原始 SQL 转换到坐在令人敬畏的《T2》续集《T3》的顶端。到目前为止，简单性仍然存在，并且基于 Sequel 比我预期的要容易得多。但是我还没想好怎么做关系(联结)；这将是真正的考验。

## [](#conferences)会议

我对自己 2015 年的会议经历相当满意。我希望我的会议水平保持不变。我真的不想承诺比去年更多的演讲。会议演讲需要花费大量的时间和精力来写作和练习。

我打算去 RailsConf，RubyConf，和 Strange Loop。如果可以的话，我希望能在每一个地方做一个演讲。但即使我不发言，我也可能会出席所有的会议。我还提交了一份关于敏捷 2016 的报告。我不太可能参加任何其他会议，更不太可能做超过 4 次的会议演讲。

## [](#writing)写作

我真的想在今年完成我的有效敏捷书。我会试着让阿莫斯偶尔和我一起合作，推动它向前发展。如果我能完成的话，我还想写一本关于榆树的书。

实际上，今年我想少写点博客。嗯，算是吧。我想把编码和写书放在博客之前。如果这意味着少写博客，我没意见。所以我写博客的目标更像是每隔一周。当我写博客文章时，我更希望它与我正在编写的一些代码相关，一些我可以在我的书中使用的内容，或者一些关于编程语言设计的更深入的想法。

## [](#job-hunting)求职

我最近在 CenturyLink Cloud 开始的合同是无限期的，所以我可能会在那里呆一整年。到目前为止我都很享受；这是一个挑战，但我认为我可以产生重大影响。然而，我真的很有兴趣在 2017 年搬到旧金山。所以我会从几个方面着手准备。

我要复活我的 LinkedIn 账户。我已经忽略了几年了。我会确保一切都是最新的，并建立一些我一直推迟的联系。我还想收回我的 Stack Exchange 和 Hacker News 帐户。

我打算请一位专家帮我修改我的简历。我认为它相当不错，但是它可能需要一些更新。我想让它脱颖而出。我想更好地解释我真正做的事情——加入一个团队，帮助他们改进流程和技术技能。

我还需要清理我所有的网站，这样当人们来看它们的时候，它们看起来会很漂亮。这包括我的个人网站、咨询业务网站、wiki 和博客。我还应该将我的服务器升级到 Debian 的最新版本，并使用 Ansible 和其他一些工具，使用基础设施作为代码的原则来提供它。
