# 调酒如何帮助我成为一名更好的开发人员

本周，我决定离开我的技术博客，把放大镜转向自己，进行一些老式的反思。

我来自一个完全非技术背景的家庭，我目前正在寻找我的第一份工作，即开发人员。希望这能帮助和我处于相同位置的其他人，并帮助他们认识到，他们多年来从以前不相关的工作中获得的一些非技术技能实际上会比他们想象的受益更多！

问题是(或者说我是这样认为的)，在寻找一份开发人员的工作时，拥有非技术工作经历对我不利。但是在与该领域的工作开发人员进行了几次交谈后，这可能比我最初认为的更偏离事实。

有人想知道，我目前在拉斯维加斯大道当酒保。我制作美味的(99%的时间)酒精饮料，供来自世界各地的游客在这个星球上最热门的旅游目的地之一度假时享用。我可以和来自世界各地的人交谈，我帮助那些来这里享受美好时光的客人创造难忘的经历。

但是塞尔吉奥，那段对你一生工作的精彩描述，几乎和编程没有任何关系。语法在哪里？我可以把我的`package.json`放在哪里？负责缩小我的代码以提高生产性能的任务运行人员在哪里？

嗯，你可能是对的。酒吧招待和酒店服务可能与科技没有什么关系。然而，考虑到这一点，我决定有意识地寻找我目前的角色和我梦想中的未来角色之间的相似之处。我很兴奋地得知，我在我的职业中做的一些事情可以完美地转变为一个开发人员的角色。

# [](#communication)沟通

这是我在职业生涯中能够不断磨练的一项技能。当我年轻的时候，我是一个非常胆小的孩子。我总是和我自己、我的朋友或我的电脑保持距离。当然，我总是对人很好，但我绝不会特意去和许多陌生人交谈。当我在长滩水族馆找到第一份收银员的工作后，一切都开始改变了。突然之间，我不得不和每一个来到我的冰淇淋车上寻找美味的人交谈。根据他们的喜好推荐口味，但愿我们没有某种口味，现在我不得不开始思考这些小问题的解决方案。

快进到现在，我开始喜欢和各种各样的人交流。每张脸的背后都是不同的故事，不同的教养，不同的喜欢的鸡尾酒。我相信对话是我们称之为生活的巨大被子的结构的缝线。作为一名调酒师，如果我的谈话出现了差错，我可以立即开始新的谈话，尝试不同的方法。

回到技术领域，我认为在找技术工作时，能够有效地沟通并不断提高自己的语言水平已经被证明是一个巨大的优势。我可以接近一个开发人员/CTO/项目经理，就像他们是我酒吧的客人一样，然后开始谈论任何事情。我不明白一个错误吗？我是不是太赶时间了？这些都是有效沟通可以轻松解决的问题，我很高兴在我的职业生涯中能够不断地解决这些问题。

# [](#urgency)尿急

关于服务行业，提供紧急服务是必须的。当一个超过 10 人的婚礼团队来到酒吧时，你的首要任务是想出一种方法来提供最好、最快、无差错的服务。有时你会有一个团队来分配职责，有时你会独自一人，有多组客人等待服务。

无论如何，这取决于你是否能想出解决问题的办法。我发现最好的方法是把每件事分解成更小的、可管理的步骤。希望每个人都想要一种类似的饮料，这样你就可以将配料分批，然后让每个人各走各的路。有时你会遇到这样的情况，每个人都想要一种完全不同的饮料，每个人都想用单独的信用卡付账。

在技术方面，这可以转化为你在项目中可能遇到的任何问题。有时，您可以编写一个简单的函数来接收相似的数据并返回预期的输出，或者有时数据可能如此多样化，以至于您要么必须编写单独的函数来处理每个边缘情况，要么提出一个简洁的解决方案来正确处理每个情况。最重要的是，所有这些都必须在保持高执行质量的同时进行处理。

# [](#customer-service)客服

作为一名酒保，顾客服务永远是你的第一要务。在你需要做的所有其他“工作”中(服务员/保镖/经理/收银员)，提供最好的客户服务应该永远是你的首要任务。

这帮助我认识到，当我开发一个应用程序时，在另一端有真实的人使用你的应用程序来解决他们自己的问题。当然，你旋转并使其看起来像一个星星的 div 在页面中间看起来很酷，但这是一个房地产应用程序，我认为客户会更喜欢向他们展示更实用的信息，而不是你酷的新设计技能。

提前预测最终用户想要什么有助于消除开发中浪费的大量时间。毕竟你是在开发一个产品给别人用，而不仅仅是自己用。

# [](#teamplaying)团队合作

团队合作让梦想成真，或者他们这么说。在接待方面，你应该能够依靠你的队友来履行他们的职责。只要大家各司其职，事情进展顺利，大家都开心。

能够自己启动、执行和交付质量代码是一项很好的技能。然而，如果你在一个团队中工作，并且有一群真正聪明的人在为同一产品工作，那么了解每个人是如何工作的，并找到一种方法来融合你们所有的工作方式，以便为同一个最终目标而合作，这将是对你最有利的。

你能独自一人赢得对金州勇士队的七场四胜系列赛吗？当你有一个像斯蒂芬·库里这样可笑的远程射手或者像德雷蒙德·格林这样的现代主力球员时，你为什么还要烦恼呢？关键是，当你有一群拥有多种技能的人时，这让你可以专注于你真正擅长并热衷于工作的事情。

# [](#conclusion)结论

这些只是我在职业生涯中能够找到的一些可转移技能，这些技能在各种职业道路上都无处不在。

如果你正在寻找进入技术行业的道路，并且有一段不相关的工作经历，我建议你重新审视一下你的职业道路，并尝试获得比你获得的技术技能更多的东西。

我坚信，如果你心里知道软件开发是你的归属，那么技术技能将伴随着努力、决心和坚持而来。但与此同时，不要低估自己，开始开发你一生中获得的其他技能。你可能会惊喜！
