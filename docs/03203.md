# 学习新的东西，应对挫折(或者至少尝试)。

> 原文:[https://dev . to/vici vancg/learning-something-new-and-treating-to-or-that-less-trying-to-eck](https://dev.to/vicivancg/learning-something-new-and-dealing-with-frustration-or-at-least-trying-to-eck)

有时当我们学习新的东西时，会有一个黑暗的时刻，你会感到失落，因为有些东西非常复杂。当我年轻的时候，我认为它最终会消失，实际上我认为这是学习的一部分，证明我们有很多东西可以享受。

这对我来说是一个很常见的场景；我在家，正在读一本书，这本书是我一个月前买的，我对一项特殊的技术感到非常惊讶，几周前我开始看一些不错的教程，并且一直在读一些文章。是的，我正处于爱上一项新技术的阶段，这太令人兴奋了！

我会说这就像“与合适的女孩约会”，你只是想知道更多，享受这个过程，一切都很好，直到一些可怕的事情发生，我一直在阅读一个令人困惑的章节，当我编写练习时 *taadaaaa* 它不起作用！更糟糕的是:我不知道为什么，尽管作者声称在高级章节中会很清楚，但我很困惑，想把这本书扔掉(笑话)。

这只是一个场景(一个有趣的场景)，但我的意思是挫折是我们作为开发人员(或人类)处理的一件常见的事情。有时这是一件令人头痛的事情，因为新事物看起来庞大而复杂，所以会有一种失败感。

[![Sherlock thinking](img/ceadd5801ccad1e3fe0f6940aaea093f.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--vZkJD_2A--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/x3ih90s3hkq3jgwg5h38.jpg)

老实说，我过去曾多次感受到这种恐惧，我指的是当你认为即使尽了最大努力也无法理解某件事时，你真的在考虑放弃。

我不是专家，我知道有很多变量在起作用，但不久前我就遇到了这种情况，经过思考，我得出了以下几点:

*   我认识的人中没有人能在一秒钟或更短的时间内学会所有的东西:我知道你对新技术感到兴奋，你也很聪明，但请记住那些学习可能需要几周时间，而你无论如何都会喜欢的学年，自我是一件危险的事情。
*   做些笔记，下定决心:我相信有很多方法可以让你继续前进，但我更喜欢 2；在你不理解的地方停下来，专注于它，或者以后当你对该技术或语言如何工作有了更好的认识时再解决它
*   **试着找出问题到底是什么(或接近什么)，并询问同伴或社区:**这是关于知识的伟大之处，实际上有很多人分享他们所知道的只是为了建立更好的东西(dev.to 是一个很好的例子)，记住要尊重和帮助别人，如果你能。
*   **我们热爱学习和编程，我们不需要独自学习:**如果你在某项技术或某个问题上感到有些困惑，试着和朋友一起编写代码，或者你可以教你的伴侣/女朋友/男朋友如何编写代码，听一些你最喜欢的音乐，看一部相关的电影或喝杯啤酒，学习你的方法！
*   伟大的事物，伟大的享受:当你在学习新的东西时，你可能会认为你在创建一个非常简单的东西，有很多伟大的应用程序你只能想象来创建，记住你正在学习，尽可能多地学习并享受它，伟大的应用程序是你喜欢构建的，难道只有我永远不会忘记第一个 Hello World 吗？
*   **不要只看代码里的世界，把代码看做世界的一部分:**享受其他有趣的东西，比如艺术、历史、数学等。这些事情不仅令人愉快，而且很有帮助，它们可以向你展示其他的方式或观点。
*   休息一下，放松一下:这是一个经典的方法，我敢肯定它很好用，只是记得不要花时间去想你不能解决的问题，回忆过去的情况以及你是如何解决的，或者只是放松一下！

这是一些想法，我认为没有神奇的公式，每个人都是不同的。我希望你喜欢我的帖子，我会很高兴知道你的经历。
