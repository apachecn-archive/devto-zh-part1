在 JavaScript 库的海洋中寻找合适的库可能相当困难。原因隐藏在 JavaScript 的流行中，它被认为是当今最常见的编程语言。你知道吗,[的 JavaScript 开发者](https://gbksoft.com/)比 Python 和 PHP 加起来还多？

由于这种倾向，每天都会产生几十个库。这种流动永远不会停止，因为有时创建一个新的库比花时间去寻找一个合适的库更容易。这种过度生长有时真的很可怕。

这里有一个小的统计数据，便于理解这个方案。5 月，GitHub 上发布了 110 万个 JavaScript 项目。7 月，14 万个新项目被添加到存储库中。但是这种增长并没有看起来那么混乱。基本上，新项目与现有的强大技术和框架相连接。然而，所有这些过程肯定需要一些结构。

有了大量使用 JavaScript 的经验，我们收集了一些最有用的轻量级 JS 库。我们将我们的列表放在一个文档中，并在公司内部共享，但现在我们决定是时候与其他开发人员共享这个宝藏了。

当然，你可能以前遇到过这样的收藏。有一些特殊的项目，收集库，但是这些平台包含了成千上万的库。所以我们决定分享我们自己精心挑选的微型 JavaScript 库列表。

## [](#top-104-lightweight-javascript-libraries)前 10+4 名轻量级 JavaScript 库

### [](#qwery)[Qwery](https://github.com/ded/qwery)

大小:0.6 kB
开发人员:达斯汀·迪亚兹、雅各布·桑顿

Query 是一个经典的库，允许你用 CSS1/2/3 查询选择器快速选择元素。这个选择器引擎构建在 querySelectorAll 之上。提示是将 Qwery 与诸如 [Bonzo](https://github.com/ded/bonzo) 和 [Bean](https://github.com/fat/bean) 这样的 DOM 模块一起使用。

### [](#transparencyjs)[Transparency.js](https://github.com/leonidas/transparency)

大小:0.6 kB
开发人:Jarno Keskikangas

Trasparency 是 jQuery 的模板引擎。它的功能是根据 id、类或数据盲属性将 JSON 对象映射到 DOM 元素。它允许用普通的 HTML 编写模板，并且兼容所有的浏览器。

### [](#minivents)[迷你风](https://github.com/allouis/minivents)

大小:0.2 kB
开发人:杨奇煜·奥卡罗尔

这是一个很小的事件库，只有 3 个功能:监听事件，停止监听事件和发出事件。

### [](#bowser)[布瑟](https://github.com/lancedikson/bowser)

大小:1.4 kB
开发人:达斯汀·迪亚兹

这个库用于检查浏览器版本、HTML、CSS 和 JavaScript 特性。在 1.9 版本中 jQuery 浏览器被移除后，我们开始使用它。

布瑟也是一个很好的工具，因为它与移动版本。非常相似的工具，非常流行。

### [](#storejs)[Store.js](https://github.com/marcuswestin/store.js)

大小:1.1 kB
开发人:马库斯·威斯汀

这个库允许你克服网站上 cookies 大小限制的问题。Store.js 是浏览器中值的跨浏览器存储库和本地存储包装器。这是防止数据丢失的完美工具。

### [](#smoothie-charts)[冰沙排行榜](https://github.com/drewnoakes/smoothie)

大小:2.8 kB
由 Joe Walnes 和 Drew Noakes 开发

此工具帮助您构建实时流数据的图表。它可以用于在健身应用程序或交易网站等项目中创建平滑的线图。这些台词非常吸引人，每个人都想把它们放到自己的网站上。

### [](#passmeter)[通计](https://github.com/syntaqx/pass-meter)

大小:0.7 kB
开发人:Chase Hutchins

Pass-meter 是我们见过的最小的图书馆。这个工具是为拒绝弱密码如“12345”或“qwerty”而设计的。当您为满足非常严格的安全规则的大型企业工作时，这可能非常方便。是的，它只有 0.7 kB 重！

顺便说一下，它既可以像独立模块一样使用，也可以像 jQuery 插件一样使用(1.7 及更高版本)。

### [Javascript-状态机](https://github.com/jakesgordon/javascript-state-machine)

大小:1.3 kB
开发人:杰克·戈登

这个图书馆的目的是什么？创建有限状态机；)我们认为这个小小的库的完美用途在于游戏开发，但是你可以在任何类型的项目中使用它。

### [](#swipe)[刷卡](https://github.com/thebird/Swipe)

大小:1.9 kB
开发人:Brad Birdsall

如果你正在开发一个移动应用程序，这个小库将会非常有用。它允许为任何移动设备创建非常精确的移动滑块。该库只包含很少的样式，但一般来说已经足够了。

### [](#hyperapp)[Hyperapp](https://hyperapp.glitch.me)

大小:1.2 kB
开发人:Jorge Bucaran

这个小库用于构建前端应用程序。知道吗？您不必学习新语言来创建可伸缩的基于浏览器的应用程序。

### [T1](#lithtml)[Lit-HTML](https://github.com/PolymerLabs/lit-html)

大小:~2kB
开发单位:PolymerLabs

JavaScript 中的 HTML 模板文字。lit-html 利用了 html 元素和 JavaScript 模板文字的一些独特属性。所以先了解他们是有帮助的。

### [](#innerself)[内在自我](https://github.com/stasm/innerself)

尺寸:350B
开发单位:stasm

React + Redux 的替代。使用 innerHTML 的微型视图+状态管理解决方案。innerHTML 很快。如果你是一家财富 500 强公司，或者即使你的应用程序有很多浏览量，这也不够快。但是如果你关心代码的大小，它可能对你来说已经足够快了。

### [](#wade)[韦德](https://github.com/kbrsh/wade)

大小:1kB
开发人员:kbrsh

超快的 1kb 搜索引擎。搜索背后的算法相当简单。

### [](#pell)[阿贵](https://github.com/jaredreich/pell)

大小:1kB
开发人员:Jared Reich

Pell 是最简单和最小的网页所见即所得文本编辑器，没有依赖性。
