# TDD 辩论:多么错误的二分法！

> 原文：<https://dev.to/ben/the-tdd-debate-what-a-false-dichotomy>

我经常发现在实践测试驱动开发时很难有成效。这是很常见的情况。许多问题很难用远见解决。在我完全知道我在建造什么之前，我经常需要把手伸进黏土里。TDD 感觉就像在解决问题时增加了一层认知官僚主义。但是根据问题的不同，这是我能理解的唯一方法。典型地，当我在重构或者解决一个缺失的边缘案例时，TDD 使之适合我的大脑，而当构建新的特性时，它就不那么适合了，但是这也不是一个固定的规则。这真的是个案处理。

我一直认为随着我变得更有经验，TDD 会在我的过程中变得更有意义，但它仍然没有在很多时候成为我的一项有成效的活动。也许永远不会。
