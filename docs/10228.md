# 完美与足够好

> 原文:[https://dev.to/grahamcox82/perfect-vs-good-enough](https://dev.to/grahamcox82/perfect-vs-good-enough)

有一句话，“完美是好的敌人”，这句话经常被认为是伏尔泰说的。本质上，这意味着如果你努力追求绝对的完美，你将永远不会真正取得任何成就。

这在软件开发中起着很大的作用，在独立项目或副业项目中更是如此。我对此特别内疚。

在某个点上，代码已经达到了 MVP，在这个点上它做了它需要做的事情，并且有可能在将来维护它并解决任何问题。当你到达那里时，你应该停下来。如果你个人投入到代码中，你会发现它很难，但是如果你一直努力直到它变得完美，那么你将永远不会开始下一个工作。反过来，这意味着你永远也完成不了。

不要让完美成为好的敌人。为了足够好，为了 MVP，为了以后更好。这样，你会发现自己进步得更快，也许会完成一些事情。

(而且，作为一个额外的好处，当您使用您认为需要改进的代码时，您会发现它需要改进的概念可能是错误的)

1) MVP -最小可行产品。上市所需的最少工作。
