# 为什么我从 Atom 转到 Visual Studio 代码

> 原文：<https://dev.to/ben/why-i-switched-from-atom-to-visual-studio-code>

我最近买了一台新的 dev 机(经过深思熟虑，我买了一台翻新的 2016 款 MacBook Pro)。这是一个重新考虑我在软件开发中使用的应用程序的好机会。我一直在我的旧电脑上用 Atom 写代码，我对它很满意，不用费心切换...在很大程度上。

我喜欢 Atom 背后的想法，即它可以使用众所周知的 web 技术进行超级定制，以确保它在为其编写的包和新技术的快速采用方面处于前沿。但是它很慢，而且没有任何特别的进步。我乐于接受改变。

我主要在 web 开发领域涉猎各种技术，我不想要一个全功能的 IDE。基本代码编辑器有几个选择，而转向 Vim 这样的东西从来没有真正吸引过我。Atom 和 VS Code 都[在性能上输得很惨](https://blog.xinhong.me/post/sublime-text-vs-vscode-vs-atom-performance-dec-2016/)升华。

## 那么我为什么要用 VS 代码呢？

人们他妈的喜欢它。特性、性能、用例等之间。你可以为任何编辑做一个案例，我不能花两个月的时间沉浸在每一个案例中。我去社区寻求见解。在几个月前[的这个帖子里，“你用哪个编辑器，为什么？”裁决对我来说很清楚。没有人像对 VS Code 那样对任何编辑器赞不绝口。我通读了整个帖子的大部分内容，对这位编辑的正面感受比其他人都要突出。](https://dev.to/ben/which-editoride-do-you-use-and-why/comments)

VS 代码肯定！绝对喜欢！Intellisense 特别适用于 TypeScript，但对 JavaScript 和许多其他语言也有很好的支持！比 Atom 快得多，尤其是在打开非常大的文件时。我还发现 Atom 经常有太多的包错误，这是我在 VS 代码中从未见过的。我也更喜欢它而不是 Sublime Text，原因有几个:它完全免费，我不必为它支付 70 美元。我也发现这是一种更现代的体验。它还每月更新，每次都显著改善体验，并且有一个非常棒的扩展社区。所有这些编辑器都比 Webstorm 好，Webstorm 对我来说太笨重太慢了。

我过去大量使用过 Sublime、Atom 和 Webstorm，所以我的决定很大程度上是基于经验。

人们喜欢不同的编辑有不同的理由，但是人们*喜欢*这个编辑。对我来说这已经是几天前的事了，我明白了为什么人们喜欢它。git 集成非常方便，总体上看起来设计良好。我不是一个真正的技术迷，但在我使用 Atom 的整个过程中，我从来没有真正感觉到我使用的编辑器给了我那么多。

就功能和度量标准而言，我仍然不能为任何一个编辑器提出强有力的论点，但是对这个编辑器的热情脱颖而出，这足以让我买入。为了对选择感觉良好，一个人不仅要对项目方向上的现有产品有信心。我喜欢微软最近所做的事情，我相信他们会支持和学习 VS 代码，我对这种转变感到高兴。最终，编辑器的作用只有这么多，只要它不碍事，让你写软件，但我对自己的现状很满意。
