# 相信自己解决算法

> 原文:[https://dev . to/rachelralston/信任自己求解算法](https://dev.to/rachelralston/trusting-yourself-in-solving-algorithms)

今天的编码课是关于相信你自己。

我注意到自己在编码时有一种暴躁的模式:我遇到了 x 问题。我理解这个问题，并提出了一些解决 x 问题的逻辑。我实现了我的代码，但不可避免地它不起作用。

好的，酷，当我开始修理时，我是一个禅宗大师，从几个不同的角度接近我的想法和项目。我花了 5 个小时有条不紊地研究这段代码，但最终一无所获。到这个时候，我已经沮丧到可以向技术顾问寻求帮助了。

在一个小时的时间里，问题被解决了*，使用的是我最初想出的初始逻辑*。

这周我花了很多时间思考这个反模式。以下是我从这个重复的场景中吸取的教训:

*   相信你自己。你的想法很好，你知道的比你想的多。
*   不要因为代码不会马上运行就认为算法是错误的。

为了改善这一新暴露的无知领域，我将研究如何将思想转化为工作代码。我的流程:

1.  拿一个算法问题( [HackerRank](https://www.hackerrank.com) 是喜欢的)，在纸上解决。
2.  用边缘案例写出问题。
3.  用简单的古英语把算法伪代码写出来。
4.  编写实际的代码。还停留在纸面上。
5.  一行一行地检查并搜寻错误。把你的问题清楚地记在脑子里，确保你的想法行得通。
6.  最后，按原样将它输入到您选择的代码编辑器*中，并记录任何错误，查看它是否/在哪里出错，纠正您的错误。*
7.  在一些评论里，分解问题。写下你的想法和你对代码的看法。

花椰菜和学习都让我感觉像在跳舞。

我把这当成数学作业，这是一件有点痛苦的事情，我每天都在做，把一些知识灌输到我的内心深处。额外收获:这也是面试学习的一部分。
