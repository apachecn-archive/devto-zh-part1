# 学习编码并取代你工作的机器

> 原文:[https://dev . to/Ben/machines-that-learn-to-code-and-take-your job](https://dev.to/ben/machines-that-learn-to-code-and-take-your-job)

程序员倾向于意识到当计算机学会做更多人类做的工作时，世界将面临的经济问题，比如开出租车和翻馅饼。如果社会能够顺利适应，这种结果可能是一件好事；如果不能，这种结果可能是灾难性的。只有时间能证明一切。但是程序员通常认为他们自己的工作不会受到我们新的机器人霸主的威胁。我想说的是，虽然程序员非常适合适应这种转变，但程序员现在每天实际完成的任务自动化的时机已经成熟。

计算机将学习完成程序员目前强调的任务，方法是研究模式以实现结果，并向人类提供实现这些目标的声明性选项，而不是强迫开发人员自己提供一行一行的实现。许多编程依赖于基本模式，训练有素的机器能够非常准确地执行这些模式。像命令性地指导机器执行 CRUD 操作这样的任务，以及像对象关系映射技术这样的关注点，都是将被机器抽象掉的实现细节。就像大多数开发人员不再关心昨天字节级的头痛问题一样，未来的程序员将与经过训练的机器一起工作，处理实际编写代码的任务。

这不会是一个突然的转变，而是一个缓慢的过程。我们使用的应用程序编程接口将会一次一个创新，离金属越来越远，程序员将会相信机器做出的决定。当我们要求我们的应用程序“获得唐纳德·特朗普关于某人愚蠢的所有时间”时，我们的编程基础设施将任意编写必要的代码来完成任务，就像我们告诉我们的同事编写代码一样准确。可能会有沟通错误，但我们会让计算机知道，它会从错误中“学习”。

除了“低级”任务之外，没有理由认为计算机不能同样熟练地处理程序中与人类互动的部分。如果开发人员要求他们的机器“设计一个界面，让人们能够访问他们的医疗保健信息并调整他们的覆盖计划”，那么一个经过任务培训的机器应该能够立即提供一个针对手头任务进行优化的图形用户界面。因为机器在管理数据传递的复杂性方面应该没有问题，所以界面可以根据用户的独特需求无限定制。只有时间才能证明，当你要求机器让设计“流行”时，它是否会对你感到恼火。

目前有一些公司正致力于构建我所描述的一切。

计算机程序的哲学“目的”也许可以被描述为“组织世界的信息”。这是谷歌宣称的使命，但我相信这或多或少是程序普遍重要的原因。移动信息是我们编程的原因。抛开人类动机不谈，程序就是为了这个目标。随着谷歌这个最有可能利用计算机学习能力的组织推进这项任务，它的计算机将被赋予更多的自主权来执行我们原本期望人类程序员执行的监督任务。因为谷歌已经充当了世界信息的事实上的接口，我们可以看到一个未来，谷歌本身是处理几乎所有世界编程需求的单一机器。显然，这是计算的无限潜在方向之一。

人类做什么和机器做什么之间的界限是根据计算机“擅长”什么和人类“擅长”什么来划分的，这条界限一直在变化。据说模式匹配是一项重要的技能，它让人类，尤其是程序员，擅长于他们所做的事情。随着机器提高其模式匹配能力，这正是机器学习引入计算机的“技能”，它们处理更多的编程工作只会有意义。人类将在一段时间内保持重要地位，我希望是永远，但他们的角色会发生变化，那些认为自己是专业计算机程序员的人必须和其他人一样准备好适应这些变化。这种结构性转变的终结可能还有一段路要走，但是大地已经在我们脚下震动了。
