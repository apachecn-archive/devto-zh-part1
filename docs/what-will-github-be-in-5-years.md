# 5 年后 GitHub 会是什么样？

> 原文：<https://dev.to/ben/what-will-github-be-in-5-years>

GitHub 对软件产生了深远的影响，因此也影响了整个世界。每个人都知道这家公司筹集了大量资金，推动了公司的发展。这个事实应该得到更多的关注。作为开源软件的一个关键部分，GitHub 目前的角色存在灰色地带，未来的角色也存在不确定性。

这家独角兽软件公司的估值为 20 亿美元。它的投资者，像 Andreessen Horowitz 和 Sequoia Capital 这样的知名人士，最近给了他们 3.5 亿美元，认为 20 亿美元的估值只是冰山一角。GitHub 需要真正做大，给投资者带来丰厚的回报。

继续为构建开源软件提供一个免费的平台，为个人和创业公司提供一个廉价的平台来托管他们的代码库，这并不是 GitHub 的明显未来。这可能是正确的选择，但这远非必然的结果。对于 GitHub 来说，有许多决策和场景可能会以有趣的方式进行，使用该服务的程序员应该会对它们感兴趣。虽然已经有各种替代方案和竞争公司，但 GitHub 是行业中的巨鲸，随着其功能的发展，供应商锁定问题可能会变得更加突出。因此，单独关注 it 是有意义的，但许多相同的原则也适用于任何当前或未来以相同的典型业务结构运营的竞争对手。关于代码回购巨头的未来，我想提出一些非互斥的想法。

## GitHub 被脸书或谷歌收购

脸书和谷歌在试图开发 GitHub 现在拥有的服务时都有各自的问题。脸书曾几次试图进军“社交”商业领域，但都没有取得多大成功，而谷歌在与 GitHub 更直接的竞争中失败了，在社交领域也全面失败。这些公司足够大，收购 GitHub 是完全合理的支出。

出于各种原因，GitHub 对每家公司来说都是一笔值得收购的交易。开源已经成为每个企业的重要组成部分，这里的 synergy 对任何一家公司都有好处。无论是脸书还是谷歌，都不会羞于进行让很多人反感的收购，或者关闭自己喜欢的项目和平台。

## GitHub 与 Dropbox 和 Google Drive 正面交锋

GitHub 一直在扩展其浏览器功能，包括[直接文件上传](../use-cases-for-githubs-new-direct-upload-feature)功能，我认为这些功能可以让它与 Dropbox 并驾齐驱。GitHub 作为不断扩展的工作流的核心部分有很多优势，集成 Dropbox 式的功能对该公司来说可能非常有吸引力。文件共享的市场很大(而且在负责？).如果 GitHub 在这个方向上成功扩展，这将是对当前产品只关注开发者的重大偏离。

史蒂夫·乔布斯曾嘲笑 Dropbox 是“功能，而不是产品”。当 Dropbox 成长为一家价值数十亿美元的公司，而苹果的云竞争对手却在苦苦挣扎时，他对此提出了质疑，但 Dropbox 尚未脱离困境。其他公司不断威胁该公司集成其最佳功能，而不是作为独立产品提供。在这里，功能与产品的争论是有争议的，我可以看到 GitHub 在其中发挥作用。

## GitHub 全面企业化

当与几个大企业打交道时，公司获得 80/20 的回报并不罕见，这胜过与数百万小人物打交道。我怀疑 GitHub 的大量收入是通过这种方式产生的。如果连免费和便宜的层级都变得无利可图怎么办？GitHub 是一个以盈利为目的的企业，即使他们在开源社区保持不败和受欢迎，这也不意味着维持这项服务符合他们的最大利益。

## 另一个人出现了，彻底击垮了 GitHub

这是有可能的。企业总是会失败，或大或小，GitHub 在当前的模式中有巨大的惯性，永远有创新的空间。对于 GitHub 在功能集上的自鸣得意，绝对有很多抱怨，一个有远见的竞争对手完全可以击败他们。

## 一刀切的方法失败了，取而代之的是更有设计感的方法，行业也分裂了

GitHub 现在确实为所有人做了这一切，这可能不一定是理想的。虽然这种统一的约定有所帮助，但不同的项目可能有不同的需求，随着时间的推移，GitHub 的核心抽象可能会崩溃。情况可能是不同类型的项目、语言、框架、构建工具、容器等。，通过具有专门特性的代码托管平台得到更好的服务。如果这个行业分裂了，如果它能被一些共享的惯例统一起来就好了，这可能会发生，但正如我们在消息传递、数据馈送等方面看到的那样。，这似乎是一厢情愿的想法。

## 结论

公司的内部需求和市场的外部力量意味着 GitHub 的未来还远未确定。该公司肯定有一个他们与股东讨论过的私人愿景，一个让他们从 20 亿美元到 1000 亿美元甚至更多的愿景。当程序员在对连续性的需求和对创新的渴望之间进行平衡时，我们应该意识到其中的利益，GitHub 的利益是否与使用它的程序员的利益完全一致并不清楚。开源的生存能力不是长期确定的，它与单一风险投资支持的企业的联系也是脆弱的。
