# Windows 还是 Linux？

帖子的标题是我每天都在纠结的一件事。一方面你有 Windows，一个我用了一辈子的操作系统系列，从 Windows 3.1 到新的 Windows 10。你可以说我喜欢我的 Windows 操作系统。另一方面，我也是一名 web 应用程序开发人员，基于 Linux 的系统非常适合运行服务器，它们只是安装新程序的一种乐趣。我如何选择一个系统来坚持，或者两个不同的系统之间的折衷是可能的？

### Windows 的优势

Windows 的主要优势之一是它与游戏的兼容性。游戏在基于 Linux 的系统上变得越来越大，但是还没有接近 Windows 的水平。几乎所有的主流游戏都可以在 Windows 上运行，这就是为什么我更喜欢 Windows 而不是 Linux。除了兼容大多数主流游戏外，大多数程序还兼容 Windows 操作系统。这使得普通用户安装程序变得非常简单。

### Windows 的缺点

当然，使用 Windows 也有缺点，尤其是作为一名网络开发人员。缺点之一是 Windows 是基于 DOS 的，而不是像 MAC OS 和 Linux 那样基于 UNIX 的操作系统。在大多数情况下，这不是一个大问题，但在我看来，使用 UNIX 比 DOS 直观得多。Windows 的另一个缺点是它不是一个免费软件。当然，我很乐意为让你的生活更轻松的软件付费，但是每个人都宁愿有免费的东西。

因为许多程序都与 Windows 兼容，而且为 Windows 机器开发很容易，所以很容易感染病毒。在 Windows 机器上，很容易安装来自不可信供应商的软件，因此当您不是有经验的用户时，很容易在您的系统上安装病毒。对大多数人来说，这意味着安装病毒扫描和防火墙是防止这种情况发生的唯一方法。这会带来额外的成本和不必要的软件计划。

### 基于 Linux 系统的优势

对于普通用户来说，在 Windows 机器上安装一个程序非常简单，而在 UNIX 上，这对于有经验的用户来说非常容易。您可以使用命令`apt-get install [package-name]`简单地安装一个程序。同样，对于普通用户来说，这可能非常复杂，并不是安装程序的理想解决方案，但由于我正在使用 Node 和 PHP 的命令行，这种转换是非常自然的。

除了命令行和 UNIX 之外，Linux 还有很多不同的发行版！总有一款能满足您的需求，或者像您希望的那样工作。这些发行版最好的部分是什么？它们都是免费的，你不需要为它们支付任何费用，除非你身边没有一个启动 USB。因为不同的发行版对于不同的软件会有不同的表现，所以这些发行版有自己的(官方的)程序库供你安装。如果这个库中没有您需要的程序，您可以添加一个有它的程序。因为官方的软件仓库只存放可信的软件，所以安装程序时感染病毒的可能性是不存在的。除非某个软件从漏洞中溜了出来，或者你安装了非官方软件库的程序。只有这样，你的系统中才会有病毒。

在一些发行版中，你可以期望预装一些有用的软件，比如 Node 或 Python。通过预装这些程序，您可以立即开始编程或为您正在处理的项目设置服务器。

### 基于 Linux 系统的缺点

但是，总有坏处。其中一个我之前提到过，游戏。尽管对基于 Linux 的系统的支持越来越好，但仍未达到游戏玩家对 Windows 的期望水平。不过话说回来，我还没有遇到多少人使用基于 Linux 的操作系统来玩游戏。所以这种劣势并不适用于所有人。

如前所述，在基于 Linux 的系统上安装程序可以通过`apt-get install [package-name]`来完成。这并不适合所有人，因此对于普通用户来说，学习某些发行版的曲线非常陡峭。像 Ubuntu 这样的发行版都有一个应用商店，但是并不是所有的发行版都是这样，所以当你选择一个适合你的应用时，这是一个值得注意的事情。我遇到的最后一个不利因素是像 iPhone 这样的设备的识别。对我来说，它出现了，但仅此而已。它需要修修补补才能工作，对于一些只想让它工作的用户来说，这可能是一个转折点。如果你是这样的人，也许基于 Linux 的系统不适合你。

### 妥协

那么 Windows 和 Linux 之间有可能达成妥协吗？利用两种系统的所有优点，同时弥补其缺点。是的，有。我想到了两种情况。双启动系统，意味着你的硬盘上安装了一个 windows 系统，以及一个基于 Linux 的系统。启动电脑时，您可以选择要使用的操作系统。当你想要一个真实的体验，没有缓慢的加载时间，而是一个真实的 Windows 或 Linux 环境时，这可能是很棒的。但是，如果您不想处理这个问题，虚拟机是一个很好的选择。这允许您在基于 Windows 的系统的窗口中启动基于 Linux 的系统。这样你就不必为在你的系统上安装一个独立的操作系统而烦恼，并且你可以很容易地在你的 Windows 系统和基于 Linux 的系统之间切换。这对于测试目的、应用程序开发或快速任务来说是理想的。

因此，根据您的需求，您可以选择基于 Windows 的系统、基于 Linux 的系统或两者的组合。我个人有两个不同的系统，一个用于 Ubuntu(一个 Linux 发行版)，另一个用于 Windows 10。这使得开发应用程序变得非常容易，因为我需要一些只能在基于 Linux 的系统上使用的软件。我只需简单地在两个系统之间建立一个连接，它们就能完美地配合工作。但这只是一个孤立的例子，在许多不同的场景中，两个操作系统的组合都是非常理想的设置。但是试着自己看看。尝试不同的发行版，尝试建立连接，整合系统，看看你能用它们做什么。
