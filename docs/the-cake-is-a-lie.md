# 蛋糕是个谎言

> 原文：<https://dev.to/codemouse92/the-cake-is-a-lie>

*最初发布于[indeliblebluepen.com](http://www.indeliblebluepen.com/?p=763)。*

请允许我让你的世界听一会儿——编程不是一门科学。当然，我们称之为计算机科学，但是这个过程本身并不科学。

“我们依靠的是机械装置！你可以坚持。”计算机科学建立在物理学的基础上。一方面，这是真的，就像绘画是基于光和颜色的科学一样。染料和材料是利用科学发展起来的，但是绘画本身就是一门艺术。没有固有的“正确的绘画方法”。莫奈的全景无法与达芬奇的肖像和场景相比，也无法与毕加索的抽象作品或梵高的印象派作品相比。每种风格的支持者和流派都存在，增加了每种风格的数量和受欢迎程度，但对其他风格的影响甚微。

所有的计算机科学都建立在二进制数学的基础上，但当我们探索其他基础时，即使是二进制数学也分崩离析，例如以 3 为基础的量子计算的勇敢的新前沿。

没有什么一成不变的编程法则可供我们依赖。不像牛顿物理学，虽然不完整，但仍然非常正确，编程中的每个假设都可能被打破，或者当探索得太深时，莫名其妙地停止工作。

新的黑客字典描述了神秘的[“schrodinbug”](http://catb.org/jargon/html/S/schroedinbug.html)，一种编程结构，它按照预期工作，直到有人简单地观察到这是不可能的。同样的，[【海森堡】](http://catb.org/jargon/html/H/heisenbug.html)每次被观察到都会改变自己的行为。虽然这些听起来像是摘自刘易斯·卡罗尔的作品，但我可以肯定，根据我在这个领域的多年经验，这些以及其他许多奇怪的事情都是非常真实的。编程提供了一个自己的仙境，看似不可能的事情像柴郡猫一样在每个角落向我们露齿而笑。在黑客中呆上一段时间，你会发现我们都是疯子。

这并不是说编程不合逻辑。就像仙境一样，有一种稳定可靠的逻辑形式:所说的一切都会成真，以一种令人恼火的字面时尚。计算机不会屈从于你的意志，直到你学会从你的思维过程中驱除所有的模糊性，即使你学会了，这些机械怪物也会挑战你所珍视的逻辑假设。

然而，在规划方面仍然存在不稳定性。正如杰拉尔德·温伯格曾经指出的...

> “如果建筑者像程序员写程序那样建造房屋，那么第一只啄木鸟就会毁灭文明。”

这经常被黑客称为“温伯格第二定律”。

这种不稳定性是我们自己造成的，我谦卑地承认，它来自于对编程固有的艺术特征的不完全承认，如果不是彻底否认的话。几十年来，我们建立了自己的数字解决方案，然后愚蠢地给它们打上了与牛顿定律相同的权威印记。我们这样做是因为这是我们做事的方式。

计算机不能很好地处理无限循环，这似乎包括人类程序员的循环逻辑，也许是在一些无法解释的原则上。

因为我们在很早的阶段就宣布我们的想法为法律，所以我们在这些想法上建立的其他一切现在都不稳定。当某些东西向二进制层次分解时，整个结构就会在我们耳边崩溃。

也许这就是为什么我们让所有计算机科学的学生为他们的家庭作业编写小程序。从经验来看，我可以说我们的方法不能扩大规模！当我们在制作小小的乐高模型时，我们可以否认我们正在搭建的桌子是用透明胶带和橡皮筋粘在一起的。一旦我们试图做出任何有意义的事情，那张桌子就会分崩离析。

面向对象编程范式的发明者 Alan Kay 也做了类似的观察。

> “当你把某样东西放大一百倍时，它的能力就会减弱一百倍，事实上，这个狗屋会发生什么事情；它只会坍塌成一堆瓦砾。”

他继续指出了我们领域否认的习惯。

> “最受欢迎的(回应)是说，嗯，这是我们一直在努力做的。在上面放更多的垃圾，用石灰石覆盖，然后说，“是的，我们真的是在尝试建造金字塔，而不是哥特式教堂。”"

然而，我们不断在此基础上建立新的层次。我们一直试图通过一个叫做“抽象”的过程，让人类更容易理解和使用这些混乱的东西。我们增加了层次，使编码过程更接近英语，相反，更远离二进制。乔尔·斯波尔斯基在他的文章中指出:

> 抽象会失败。有时一点点，有时很多。有渗漏。事情出了差错。当你有[他们]的时候，它会在任何地方发生。

所有这些并不是说我们应该抛开一切，从头开始。我们做了很多正确的事情，否则你面前的这台机器将无法正常工作。相反，我们需要学会的是，不要认为任何事情都是理所当然的。我们在编程中唯一的法则是我们自己发明的法则。当我们开始问一个危险的问题时，创新就开始了，“我们为什么要这样做？”，然后要求一个比“因为我们一直是这样做的”更好的答案

与此密切相关的是整个最佳实践原则的问题，这是一个与任何计算机科学法则不相上下的神话。我不想说标准本身就是不好的——我们需要一些东西来让这堆闷烧的瓦砾保持有序，这样我们就可以继续清理它，而不是让它变得更糟。然而，标准和最佳实践之间有着巨大的差异。

一方面，标准从一个团体到下一个团体是变化的，并且有很好的理由。以早期的绘画为例，如果你是现实主义学院的学生，你必须遵循某些技巧来达到你想要的风格。除非每个人都以同样的方式工作，否则没有人能研究、学习或改进你的工作。

然而，当你决定加入印象派的时候，这个标准就变得完全无关紧要了。你采用一套完全不同的标准来适应你的新目标。类似地，如果你正在开创一种新的绘画风格，那么标准现在完全由你自己来定义。

同样值得注意的是，即使在给定的一套标准中，偏差不仅不常见，而且对于定义一个人自己的风格是必要的。比较某一特定学校中的任何一位大师，你会发现相当大的差异。记住一句古老的格言:“一个人必须知道规则，才能知道什么时候打破规则。”

这同样适用于编程，这就是为什么最佳实践是一个如此分裂和危险的概念。它表明有一个统一的标准，所有的程序员都必须遵守，偏离这个标准是一种无法形容的罪行。这是阻止创新的最快方法，因为它断言我们已经知道所有的答案。这种观点的一些支持者甚至会说，我们已经发现了软件中所有有待发现的东西；剩下唯一要做的就是在我们的奖杯陈列柜上多涂几层上光剂，称之为好。事实上，有些人已经这样说了。

然而，即使在我们舒适的基于二进制编程的世界中，我们也在继续寻找这台机器的新用途。如果我们如此大胆地推翻我们在寻找错误之处时所做的每一个假设，那么我们的任务将是以更稳定的方式重建整个计算世界。随着量子计算等新前沿的出现，我们无疑将继续在这个固有的人类构建的数字宇宙中发现新的奇迹、新的可能性和新的刺激物。

这一新的发现之旅始于简单但令人惊讶的挑战性的第一步，即承认我们对编程的所有假设往好里说是不稳定的，往坏里说是完全错误的。这不是一个我们有朝一日能够了解一切的科学领域，尽管这是摆在许多程序员面前的诱饵，让他们毫无疑问地默默遵循我们长期以来的假设。简而言之，正如这个广受欢迎的视频游戏门户网站的粉丝所断言的那样:蛋糕是个谎言。

拿我来说，我不再寻找蛋糕了。还有新的更好的方法有待发现，还有创造性的方法来改进我们现有的方法。这是我们的时间和努力得到最佳利用的地方。
