最近， [VS Code](https://code.visualstudio.com/) 在开发社区中受到了大肆宣传。它已经存在几年了，TBH 是一个非常可爱的编辑。许多很棒的扩展和社区支持。也挺快的(对于一个电子 app 来说😬)而 peeps 和微软似乎也在保持更新，快速迭代。

然而，缺省情况下，你真正需要用 PHP 工作的任何东西都会丢失。我相信这是为了鼓励一个充满活力的扩展社区。幸运的是，像大多数语言一样，有一个活跃的开源 PHP 扩展社区。PHP 有几个很棒的扩展: [PHP IntelliSense](https://marketplace.visualstudio.com/items?itemName=felixfbecker.php-intellisense) 、 [PHP Debug](https://marketplace.visualstudio.com/items?itemName=felixfbecker.php-debug) (都是由[同一个开发者](https://github.com/felixfbecker)开发的)以及 [PHP Intelliphense](https://marketplace.visualstudio.com/items?itemName=bmewburn.vscode-intelephense-client) (#wat？).Jeffrey Way 在 Laracasts 上甚至有一个非常棒的介绍，概述了为 PHP 开发设置 VS 代码需要做的一切。

有些 PHP 软件包运行良好，安装了超过一百万次。！！).然而，有时它们不工作。一点也不。就在今天，PHP 调试扩展彻底崩溃了。幸运的是，开发人员注意到了这一点，并在几个小时内推出了修复程序。对开源产品非常棒的支持！

但是，如果您为编辑器付费，比如说 PHPStorm，您可以在他们的支持频道上提出问题。你可以抱怨产品不工作，这是理所当然的，因为你已经支付了权利！作为“客户”，你对 Jetbrains 这样的供应商有一定的影响力。开源的情况是*而不是*，我觉得我们开发者忘记了这一点。

这就是我反对的地方。我完全支持开源软件。我的职业生涯就建立在这上面。问题是这个插件的开发者必须自己解决这个问题。人们期望他*有*来解决这个问题，并且现在就去做。如果没有立即完成，人们会抓狂，在推特上抱怨，写博客，发飙🐮伙计。

开源就是这样，开放。如果你发现一个插件，编辑器，扩展或者其他什么的问题，看看你能不能修复它！这才是重点。让我们不要举手抱怨，让我们把手弄脏，把这该死的东西修好。

这就是开源的意义所在。让我们记住这一点。
