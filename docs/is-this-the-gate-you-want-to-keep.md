# 这是你想保留的大门吗？

> 原文：<https://dev.to/wiredferret/is-this-the-gate-you-want-to-keep>

对我来说这是很大的回应。比如“Livetweeting @kelseyhightower”的反应水平。所以让我们停下来讨论一下。

老实说，我写这条推文是因为我尊敬的一个团体呼吁程序员帮助索引不足的程序员做一些与编码完全无关的事情，我很沮丧。

在任何房间里，开发人员都不是最聪明的人，我希望我们不要像对待他们那样对待他们。它会导致各种工作场所中毒。是的，这是关于谷歌的事情。优步事件，还有许许多多其他性别歧视和种族歧视的事情，我们都可以一口气说出来。

当然，神化编码是编码人员的一个贡献部分，他们表现得就像小锡神一样。如果和他们一起工作的每个人都表现得好像他们的技能是公司维持运转的全部，他们为什么不会骄傲自大呢？见鬼，有时整个公司被“收购”只是为了得到一个功能性的程序员团队——支持人员和产品被抛弃，好像他们没有价值。

我和几个创业公司合作过，在某个阶段，投资者不再问你的编码员和开发员，并开始制造噪音，比如也许应该有一个经理，或者一个财务官，或者一个 CTO 从战略上而不是战术上思考。这是因为在某种程度上，编码员是执行愿景的人，但最好有人有愿景。

愿景不仅仅是一个美好的销售计划——它来自对行业、问题空间、用户体验以及其他人看待、使用和接触产品的方式的理解。它包括销售、营销、质量保证、技术文档和支持。如果你不积极争取金钱能买到的最好的支持人员，并支付足够的薪水来表示你对他们的尊重，那么你就是个傻瓜。一个好的支持人员可以成就或破坏客户保持率。

我认识很多开发者，我大多喜欢他们。他们很聪明，有时在工作内外，他们都有惊人的兴趣和激情。但它们并不神奇。你实际上不能用它们来倍增生产力，它们也不会拯救你的公司。他们可以执行。真正优秀的人能够执行*和*预测你的策略并朝着它前进。但是开发人员最终没有成为领先的公司，因为他们在 Javascript 方面非常擅长(T2)。他们最终成为了领导者，因为他们没有让他们的编码技能成为他们的全部定义。他们致力于在人际交往、会议、产品甚至更多的人群方面做得更好。

因此，每当你谈到教女孩编程，或者要求程序员帮助其他程序员做非代码的事情，或者开办一所没有探索软件开发其他方面的代码学校，你都在说，在技术领域只有一条正确的道路。你正在设置一个狭窄的门。我不喜欢它。这让我很生气，不仅仅因为我不写代码，还因为它贬低了所有其他创造奇迹的人的贡献。

> 创造一个产品需要的不仅仅是代码。

敬首相，他努力平衡相互冲突的优先事项，参加了几十次会议，以确保事情尽可能好地发展。

对于 QA 测试人员来说，他们设计并运行无数的排列测试，你会发誓没有人会尝试，因为没有人比测试人员更怀疑人类的智力，他们总是对的。

对 UX 的设计师来说，他们经常被斥为“把东西做漂亮”，而实际上他们是在“让你的产品可用”。他们运行数小时的人工测试，观察和迭代，直到用户几乎看不见你的产品。

对于技术和市场两方面的作者来说，他们花了所有的时间试图弄清楚如何使产品真实、相关并对处于困境的人有用。

对于销售和营销团队来说，他们真的为小事操心，他们实际上有点厌倦了豪华的晚餐、旅行和机场座位，但他们坚持下去，因为他们相信这个产品及其解决问题的能力。有时他们也会给我们做贴纸。

敬保安，他们可能是这个房间里最聪明的人，但是他们把这一切都变成了正义的偏执狂，这样我们就不用像他们那样每晚熬夜担心了。

支持，谁花了一整天的时间和那些因为东西坏了而生气的人说话，还拿起电话说“我能帮你什么？”下一个电话。他们得到了从令人眼花缭乱的愚蠢的用户错误到实际的神秘和间歇性的错误，我们希望他们知道如何在我们的整个堆栈中进行故障排除。他们非常了解产品和顾客，如果我们曾经问过的话。

对系统管理员、操作人员、电缆操作员和我们基础设施的无形骨干。我们只在事情出错的时候注意到你，但是事情变好的次数更多，我们从来没有注意到你已经处理过的棘手问题。

财务、会计、应付账款、人力资源和公司运营，谁让灯一直亮着，谁让冰箱里有食物，谁在凌晨 2 点被呼叫，如果清洁工触发了防盗警报。我期待着有一天我能看到一个办公室经理发一张#oncallselfie。他们是令人惊叹的、细心的、有组织的、注重细节的人，我很敬畏他们。

感谢每一个给我们送食物、打扫厕所、在凌晨 2 点用吸尘器打扫，或者让我们像小贵族一样生活的人——我们欠你们太多了，有时我们甚至不好意思说谢谢。

敬管理层。是的，管理。谁参加了这么多会议，你甚至不知道。他们在那里进行权衡和计算，尽他们所能做好他们所拥有的，当你想一想，他们没有能力在产品中做出任何改变。所有的责任，没有任何权力，至少在那个方向上。

* * *

下次当你听到有人谈论技术工作时，就好像他们都在编码一样，停下来想一想。我不想贬低编码——它很酷。但我确实想把所有这些职位都推到科技的聚光灯下，并且说，

> “我们所有人都在制造产品，这太棒了。”
