# 开发者社区和为什么你的公司应该关心

> 原文：<https://dev.to/jerdog/developer-communities-and-why-your-company-should-care-2b79>

很有可能你的公司目前正在以这样或那样的方式营销自己，从社交媒体到电子邮件活动到在线广告，从广播到电视和印刷品的传统广告形式。这些渠道可能会为你的广告费提供一些实质性的回报，但是你可能会失去一个非常重要的媒介——开发者。

如果你是一家为企业或消费者提供服务或产品的公司，你很可能会和开发者打交道。作为一家拥有工具、服务或 API 等产品的公司，你应该采取措施提高开发者对该产品的认识。开发者要么将你的服务/产品应用到他们的产品中，要么创造新的东西来满足你的需求。没有比通过开发者社区与这些开发者合作更好的方式了。

按照韦氏词典的定义，社区是“在一个更大的社会中生活在一起的具有共同特征或兴趣的一群人。”我们在日常生活中会遇到社区，无论我们是否知道，我们都可能是其中的一部分。将开发人员视为一种用户组，由不同个性的人组成一个整体。让我着迷的是，没有一个社区与另一个社区是相同的——正因为如此，确保您了解组成每个社区的开发人员是非常重要的。

我热衷于社区和开发，以及每一个相关的过程，并且我很高兴在公司和他们的技术观众之间发展理解。有人会说，我致力于让精通技术的人成为你们最大的支持者。开发人员和其他任何人一样，都有一个影响范围——他们可以根据自己的建议或支持来影响一群人，也就是个人社交圈中众所周知的“去找个人”——作为一家公司，如果你不寻求利用这一点，你就会失职。

以 XDA 开发商为例。XDA 成立于 2002 年底，是一个志同道合的移动开发者与其他人分享工作成果、讨论所有移动问题的地方。那时的目的和今天一样:*由开发者提出，为开发者服务。* XDA 在其论坛和新闻门户网站上都是与移动相关的独立开发以及爱好者、业余爱好者和新用户讨论的主要来源。在我与 XDA 共事的这些年里，我们一直看到的一件事是，当一个开发者分享一些东西时，你会看到该开发者的用户(在许多方面是粉丝)蜂拥至该线程进行尝试。如果那个开发者开始在一个新的设备上工作，许多用户会出去得到那个新的设备，这样他们就可以使用那个开发者的工作。用户捐赠是为了确保开发人员将他们的工作带到用户拥有的设备上，而开发人员没有，这种情况并不罕见。

同样，如果开发者对某个设备或某个 OEM 感到失望，因为所述 OEM 选择不遵守开源许可限制，或者他们未能提供足够的售后支持，或者更糟；当那个开发者说他们已经和那个 OEM 结束了，那个开发者的狂热粉丝也会离开那个 OEM。然后，社区将跟随开发人员到他们选择任何 OEM，从而创建一个全新的社区。

我们一次又一次地看到这种情况。“但我为什么要在乎？”你可能会问。我回到势力范围。在你的朋友圈里，你知道有多少人，无论是亲密的还是熟人，来找你并询问你对某件事的看法，无论是关于某个特定的产品或技术，甚至是一些体育琐事？就我自己而言，我估计大概有 100 个人在某个时候问过我对某件事的看法——那是我个人的影响范围。对于这 100 个人中的每一个，如果他们中只有一半人按照那个意见行动，并且这个意见影响了他们的决定，我已经影响了很多决定，销售，等等。现在，如果每个人都重复我对购买某样东西的看法，并影响到另一个人，你就明白了。由于缺乏更好的类比，这是最纯粹的多层次营销——希望没有令人毛骨悚然的 t 恤和满满一车库你永远也卖不出去的东西。

某人对你的产品、服务、小工具等的体验。具有巨大的潜力，不仅影响他们留下来的决定，而且影响无数其他人尝试并可能留下来的决定。这就是为什么我觉得对于一个公司来说，不仅要提高开发者的意识，还要与他们建立诚实的关系，让他们参与到你的产品中来，同时建立信任，这是非常重要的。当你这样做的时候，你将会在你的道路上很好地让技术娴熟的人成为你最大的拥护者。
