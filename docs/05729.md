# 言语表达的美

> 原文:[https://dev.to/rapidnerd/the-beauty-of-verbalexpressions](https://dev.to/rapidnerd/the-beauty-of-verbalexpressions)

所以在生活中的某个时刻，我们都不得不遇到正则表达式(Regex)的奇迹。如果你不知道什么是正则表达式，简而言之，它是一种特殊类型的字符串，允许你输入描述来搜索模式。它在开发中非常有用，我经常发现自己经常使用它(有时用得太多了)。它通常看起来像这样

[![alt text](../Images/cf78a5f12a4f183e6191e2bc7d446f25.png "yeah it looks like a mess")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--0QS_6ZCJ--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/vpbyj8gewf54g2un9du6.gif)

所以这看起来有点混乱，虽然正则表达式在很多领域非常强大，但我不认为有人喜欢写它。直到我发现了一种叫做 VerbalExpressions 的小美。VerbalExpressions 采用普通的正则表达式，并对其进行了大量的清空，以使其更高效、更易于使用。举个例子

[![alt text](../Images/20e4ec3cc7bee486884a6ca98ee68a7d.png "omg the beauty!")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--76IODTRb--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/q9yy82m2b6s1tens1ncm.png)

现在与上面的相比，它更容易理解，更有效的工作，你的代码更干净。就我个人而言，我喜欢使用这个工具，很长一段时间以来，我发现在 Regex 中工作总是会给我带来很多问题，因为一个丢失或放错位置的字符将意味着整个事情都将很难修复。

最棒的是。它有 32 种不同的语言版本。所有这些都可以在他们的 github[https://github.com/VerbalExpressions](https://github.com/VerbalExpressions)上找到
