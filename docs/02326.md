# 使用 Github 的好处

> 原文:[https://dev.to/apium_hub/benefits-of-using-github-9ej](https://dev.to/apium_hub/benefits-of-using-github-9ej)

如果你从事软件开发行业，那么你很可能听说过 GitHub，这是一个 Git 的存储库托管服务，也有一个基于网络的图形界面。今天，我想讨论一下，并提及使用 Github 可以从**获得的主要好处。**

GitHub，可以分为 Git，和 Hub。该服务包括访问控制和许多协作功能，如用于基本任务管理和您处理的所有项目的工具。GitHub 以各种不同的编程语言托管您的源代码项目，并跟踪对每个迭代所做的各种更改。所以，“Git”意味着版本控制系统；一个允许开发人员跟踪他们代码的不断修改的工具。“枢纽”是由志同道合的个人组成的社区。这完全是关于社区的协作努力，回顾、改进和从上传的代码中获得新的想法。这绝对是值得研究的东西！

## [](#so-what-are-the-main-benefits-of-using-github)那么使用 GitHub 的主要好处是什么？

**1。它可以很容易地为您的开源项目做出贡献**

说实话，几乎每个开源项目都使用 GitHub 来管理他们的项目。如果您的项目是开源的，并且包含一个 wiki 和问题跟踪器，可以很容易地包含更深入的文档并获得关于您项目的反馈，那么使用 GitHub 是免费的。如果你想做贡献，你只需要派生一个项目，进行你的修改，然后使用 GitHub web 界面向他们发送一个 pull 请求。

**2。文档**

通过使用 GitHub，你更容易获得优秀的文档。他们的帮助部分和指南包含了您能想到的几乎任何与 git 相关的主题的文章。

**3。展示你的作品**

你是一名开发人员，希望吸引招聘人员吗？GitHub 是你可以信赖的最好的工具。如今，在为他们的项目寻找新成员时，大多数公司都会查看 GitHub 的简介。如果你的个人资料是可用的，即使你不是来自一所著名的大学或学院，你也有更高的机会被录用。

**4。Markdown**

Markdown 允许你使用一个简单的文本编辑器来编写格式化的文档。GitHub 通过 Markdown 引导一切，彻底改变了写作:从问题跟踪，用户评论，一切。由于要学习这么多其他编程语言来设置项目，所以以一种格式输入内容而不必学习另一种系统确实是一个很大的好处。

**5。GitHub 是一个资源库**

这个之前已经提到过，但是需要注意的是，GitHub 是一个资源库。

这意味着它允许你的作品出现在公众面前。此外，GitHub 是目前最大的编码社区之一，所以它对您的项目有很大的影响。

**6。跨版本跟踪代码中的变更**

当多人在一个项目上合作时，很难保持跟踪修订——谁更改了什么，什么时候，以及那些文件存储在哪里。GitHub 通过跟踪所有被推送到存储库的变更来解决这个问题。就像使用 Microsoft Word 或 Google Drive 一样，您可以拥有代码的版本历史，这样以前的版本就不会随着每次迭代而丢失。

**7。集成选项**

GitHub 可以与 Amazon 和 Google Cloud 等常见平台、Code Climate 等服务集成，以跟踪您的反馈，并可以突出显示 200 多种不同编程语言的语法。
