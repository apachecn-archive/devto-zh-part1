> 原文:[https://dev . to/kathryngrayson/looking-back-2017-we B- dev-trends-we-should-carry-into-2018-3g0n](https://dev.to/kathryngrayson/looking-back-2017-web-dev-trends-we-should-carry-into-2018-3g0n)

是啊，是啊，我们现在都应该向前看，但是嘿，2017 年并不是一场彻底的失败。我们创造了一些有趣的东西，并在社区中看到了一些有趣的变化。当每个人都在急切地猜测接下来会发生什么时，我想花点时间回顾一下 2017 年的一些事情，我们不应该太快就告别它们。

## 1.易访问性编码

今年，我们作为一个社区谈论可访问性和包容性网络标准的数量发生了巨大的变化。越来越多的人认识到，编写通俗易懂的代码并不是一件“超越”的事情，而是“你工作的基本要求”。这种意识和体贴绝对是至关重要的，因为每个人在一生中的某个时刻都会受益于一个更容易访问的网络。

## 3.反思框架

与 CSS Grid 的高级和新的可用布局功能携手，今年有很多关于远离框架的讨论。在开发社区中有一些关于批判性地看待我们包含在构建中的代码的大讨论:它如何影响加载时间和项目权重，我们包含的所有东西真的有必要吗？通常情况下，嗯...不是的。框架在很多情况下都是有用的，但是对于开发人员来说，独立判断每个项目并根据具体情况做出判断是很重要的——而不是因为习惯或懒惰而让框架成为默认框架。

[为什么你不需要带有 CSS 网格的框架](https://rachelandrew.co.uk/archives/2017/07/01/you-do-not-need-a-css-grid-based-grid-system/)

## 4.技术的多样性

不言而喻，多样性和包容性是今年所有领域的热门话题。但是，多亏了某个“宣言”，今年科技领域的多样性也进入了公众视野。虽然这肯定不是这个话题获得关注的理想方式，但它仍然是公开讨论的一件非常重要的事情——我们需要确保这种讨论也能持续到 2018 年。在任何项目的构建中，你能包含的不同视角越多，它就越好。
