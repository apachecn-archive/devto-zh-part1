# 软件开发人员的“希波克拉底誓言”应该是什么？

> 原文：<https://dev.to/ardennl/what-should-be-in-the-software-developers-hippocratic-oath-2hh>

在我上周的博客中，我说如果软件开发者有一个希波克拉底誓言，大多数(科技)公司将会破产。我是开玩笑说的，但当我读到《卫报》的这篇文章:[“全世界的编码者，团结起来:硅谷工人能遏制大型科技公司的力量吗？”我想知道它是否有价值，你们对此有什么看法。
出自文章:](https://www.theguardian.com/news/2017/oct/31/coders-of-the-world-unite-can-silicon-valley-workers-curb-the-power-of-big-tech)

> 前硅谷劳工组织者、比尔·克林顿的助理劳工部长马丁·曼宁(Martin Manning)认为，组建工程师工会是不可能的。“这并不是说，一群关心隐私、人工智能或任何东西的工程师不应该聚在一起，分享这些担忧，”他告诉我。“但他们应该考虑一个专业组织。”曼宁认为，工程师应该像医生或图书馆员一样，建立行为准则。(20 世纪 80 年代有一个这样的组织，叫做计算机专业人员社会责任组织。)

让您开始的一些问题:

*   “开发者希波克拉底誓言”是什么？
*   你认为“开发者希波克拉底誓言”真的能改变什么吗？
*   “开发者希波克拉底誓言”可行吗？为什么不可行？
