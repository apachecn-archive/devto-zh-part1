# 把自己编码到一个角落

> 原文:[https://dev.to/funkysi1701/coding-myself-into-a-corner-1j6j](https://dev.to/funkysi1701/coding-myself-into-a-corner-1j6j)

我在工作中花费了大量的时间向遗留数据库系统添加新功能和改进。

传统系统是一种旧的方法、技术、计算机系统或应用程序，“属于、关于或是以前的或过时的计算机系统。”通常是一个贬义词，将一个系统称为“**遗产**”通常意味着该系统已经过时或者需要更换。

我的特别传统系统是微软的 Access。我们使用 Microsoft Access 为我们所有的内部数据库提供前端。我们使用的特定技术是仅在 Access 2000、2003 或 2010 中受支持的 ADP 文件。

微软已经停止了对 Access 2000 和 2003 的支持，对 2010 的支持也将在 2020 年停止。

为什么我要把自己关在角落里？对于我在 Access 中创建的每个功能或改进来说，这很简单，我增加了我需要创建或移植到新前端的功能的数量。

这是一个非常令人沮丧的想法，我创造的一切都需要重新创造。我不是在为自己节省工作，而是在增加我需要再次做的工作量。

坏消息是，我的雇主像几乎所有企业一样要求结果，就像一个好员工一样，我一直在提供结果。我被许诺神话般的“当我们安静的时候”你可以重建数据库前端。我知道这可能永远不会发生，所以我有什么选择。

我已经警告过我的经理，这项工作需要完成，他们应该给我所需的资源。这是不可接受的选择。首先，我并不是在预测未来，2020 年很快就会到来。此外，我并没有把自己培养成一名开发人员，因为我获得的唯一经验是使用已经过期或很快就会过期的遗留技术。没有人愿意雇佣只有遗留经验的人。

做点什么我需要不断地交付成果，做要求我做的事情。像任何优秀的工程师一样，我应该将我的估计乘以 4，这意味着我有一些时间可以用于查看更大的图片。这确实是一个双赢的选择。雇主得到的是一个经得起未来考验的解决方案，员工得到的是最新技术的宝贵经验。
