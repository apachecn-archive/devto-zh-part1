# “真正大”的问题

> 原文：<https://dev.to/briannipper/the-really-big-problem-2do0>

我有机会参加了一个会议，会上演示者讨论了 SharePoint 2010 将解决我所在公司所有问题的所有令人惊叹的方法。

我在一家由遍布全球的 1000 多名员工组成的公司工作，当我列举 facet search 和 managed metadata 将如何解决沟通和协作问题时，我开始问自己困扰我工作的公司的根本问题是什么，SharePoint 将如何解决这个问题。

考虑到大多数问题都可以归结为沟通问题，我意识到，虽然我工作的公司(一家中庸的金融机构)无论如何都不是“最大”的公司，但它确实存在“大”问题，也就是说，公司是由很多人组成的。

当你有一个组织时，无论是政府、公司还是家庭，随着组织的成长，它在保持团队中每个人“在同一页上”时面临的问题也会增加。

这其实可以用数学上的 *(n <sup>2</sup> -n)/2* 来表示。这个等式是我从乔尔·斯波尔斯基在*公司杂志*([http://www . Inc . com/Magazine/2010 02 01/a-little-less-conversation . html](http://www.inc.com/magazine/20100201/a-little-less-conversation.html))上发表的文章中发现的，它显示了每当你将另一个人添加到一个群时，让整个群保持一致就变得更加困难。下表(也是从 *Inc .杂志*文章中借用的)很好地说明了这一点。

| 人 | 连接 |
| --- | --- |
| one | Zero |
| Two | one |
| three | three |
| four | six |
| five | Ten |
| six | Fifteen |
| seven | Twenty-one |
| eight | Twenty-eight |
| nine | Thirty-six |
| Ten | Forty-five |

正如它所显示的，一旦你有了 5 个人，让每个人都跟上速度的问题就变得非常复杂。你可能倾向于认为，如果只有一个人做任何决定，这有助于限制问题；然而，你愿意在一个五个人中只有一个人有实际贡献的地方工作吗？我知道我不会在团队中工作的部分原因是你可以从其他人的想法中受益。

那么 SharePoint 2010 计划如何解决这个问题呢？它没有 SharePoint 2010 是一个工具，就像任何其他工具一样，只有使用它的工匠才会有用。

希望随着 SharePoint 2010 的继续推出，这个“真正大”的问题将会在更多人的脑海中出现，对它的关注将会增加，而对用于解决它的工具的重视将会减少。
