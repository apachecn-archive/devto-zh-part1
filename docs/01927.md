# 为什么我喜欢 Skype 会议

> 原文：<https://dev.to/_patrickgod/why-i-love-skype-meetings-df9>

让我搞清楚。我说的是与程序员同事的 Skype 会议，而不是所有与会者都在一个会议室，有人通过 Skype 远程连接的管理或营销会议(尽管我确实喜欢谈论管理或营销话题)。

不管怎样…

我认为大多数现代软件开发团队都使用 SCRUM。因此，他们每天都有自己的单口相声。如果你有远程工作的团队成员，你可以通过 Skype 或 Google Hangout 进行这种单口相声，每个人都可以在打开网络摄像头的情况下舒适地坐在办公桌前享受谈话。你可以享受你的茶或咖啡，同时可以看到你昨天已经完成的任务和你下一步要做的任务。如果有什么事情难以口头解释，任何人都可以**分享他的屏幕并展示提到的问题**。这给我带来了另一大优势。

作为一名后端开发人员，我和我们的前端同事聊了很多。我们讨论接下来的任务，我们头脑风暴最佳的 UI/UX 解决方案(如果 UX 人不在的话…)，当然，我们讨论 HTTP 调用和我们必须交换的 JSON 内容。我们通过 Skype 做所有这些，同时有人分享屏幕。这非常有效。

当然，我们必须这样做，因为我们不在同一个地方工作。但即使当我有机会坐在一个屏幕前讨论技术问题时，我也更喜欢 Skype 的方式。更容易切换共享屏幕。我可以把注意力集中在我同事的屏幕上，就好像它是我的一样，因为事实上它看起来真的像是我的。戴上耳机几乎可以消除任何干扰，因为我可以专注于同事的声音。而且，如果音量设得很低，我甚至可以听背景音乐。如果我们必须与三个或更多的人一起澄清技术问题，没有人必须挤在一张桌子后面，**没有人站在我身后并从我的肩膀上看过去**(我讨厌人们从我的肩膀上看过去——也许这能引起你的共鸣)。尽管如此，即使是配对编程也能很好地与 Skype 配合使用。正如 [Scott Hanselman 所说:](https://dev.to/shanselman/im-scott-hanselman-ask-me-anything-416/comments)

> “有时我会用 Skype 和同事通话，把他们放在另一台显示器上，然后我们就像室友一样安静地一起工作。”

所以远程工作再也不用感到孤独了。有了合适的工具，如果你想的话，感觉就像你和同事在办公室里一样。

最后但并非最不重要的是，另一个很大的优势:当其他人沉浸在与你的任务无关的闲聊中，你也不真正感兴趣时，你可以在他们没有注意到的情况下秘密地工作。但是不要告诉任何人。

您对 Skype 或 Google Hangout 有什么体验？你认为这些工具比以往任何时候都更重要吗，或者你仍然相信面对面的交谈吗？
