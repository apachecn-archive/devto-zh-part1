# 应该学习哪种编程语言？

> 原文：<https://dev.to/pinotattari/which-programming-language-should-you-learn-9gb>

这个问题不时出现在论坛、脸书等等。因为我厌倦了总是写同样的东西，我决定在这里写一次我的建议。

像通常一样，对于这种问题，一切取决于你的出发点(你是初学者吗？业余爱好者？)和你的目标(你想做 web 开发吗？为开源项目做贡献？像 STM32 这样的小器件编程？).通常问这个问题的人是一个初学者，并没有说任何关于具体目标的事情，因此通常我认为目标是在这个领域获得一些“文化”。下面的答案是根据这个一般目标写的。

### 第一步:学会编程(简单是关键)

如果你完全没有编程经验，我会建议你从简单的开始，以便在*真正学习语言*之前，学会*如何编程*。

是的，因为“学习编程”不同于“学习编程语言”学习编程意味着学习如何将一个问题分解成更小的问题，并指定一个过程来解决它们。知道如何做到这一点是一种独立于特定语言的技能，并且在某种程度上比知道编程语言更高。你甚至不用电脑也可以写程序，只需要纸和笔。

> [阿达·洛芙莱斯](https://en.wikipedia.org/wiki/Ada_Lovelace)，拜伦的女儿(是的，**那个**拜伦！)，被认为是历史上第一个程序员，因为她在由[查尔斯·巴贝奇设计(但从未建造)的](https://en.wikipedia.org/wiki/Charles_Babbage)[分析引擎](https://en.wikipedia.org/wiki/Analytical_Engine)上对[文章](http://www.fourmilab.ch/babbage/sketch.html)的注释，是一种 800 年的计算机，但寄存器由齿轮制成，由蒸汽驱动……(我建议阅读[阿达·洛芙莱斯文章](http://www.fourmilab.ch/babbage/sketch.html)，这很有趣)

当然，即使是学习编程语言的技能也是必要的，但我会坚持一些简单的东西。

在我看来，适合初学者的语言是基本的。它在 80 年代相当流行，当时每台家用电脑都内置了某种 BASIC 解释程序。虽然简单，但它拥有所有必要的特性，人们可以用它编写非常复杂的东西，尽管最终的代码不太容易维护。BASIC 的一个优点是它是解释型的:您可以编写一个命令并立即执行，而不需要 IDE。

由于现在 BASIC 已经不那么流行了，我认为 C 语言是另一个不错的起点，因为它相当简单。c 也是相当低级的，让你可以看到一些机器级别的东西，比如指针和堆栈，这很有用，因为它让你对“幕后”发生的事情有一种感觉，甚至对其他语言也是如此。(我发现当我学习一门新的语言时，这种直觉对我帮助很大)。一定要学习 C 语言的所有微小、晦涩的细节，比如指针、结构、联合、带有可变参数列表的函数(比如 printf)。

此外，使用 C，您将了解宏扩展，这是一种在许多地方使用的方法，它需要的思维方式与 C 等命令式语言中使用的思维方式略有不同。老实说，我不喜欢宏扩展，因为它很容易出错，并且会使代码可读性更差，但在某些情况下它是有用的。

### 第二步:最佳实践&用 Ada 进行 OOP

由于面向对象编程(OOP)广泛传播和流行，作为第二语言，许多人会建议使用 C++或 Java。但是，我要异端，建议 [Ada](https://en.wikipedia.org/wiki/Ada_(programming_language)) 。除了支持 OOP，它还是一种现代语言，非常强大，非常强调正确性和可维护性。它还可以向您介绍其他语言中不常见的东西，如契约、类型不变量和形式检查。学习它，你会学到一些编程的“好方法”,这些方法甚至对其他语言也有用。

> 由于 Ada 没有被广泛使用，在网络中查找信息并不容易。几个有用的链接:你可以在这里得到一个开源的、基于 gcc 的编译器:[http://libre.adacore.com/](http://l.facebook.com/l.php?u=http%3A%2F%2Flibre.adacore.com%2F&h=zAQFgNuwR)。关于这种语言的介绍，请参见 [Adacore 大学](http://u.adacore.com/)，关于简单的参考，请参见 [Wikibook](https://en.wikibooks.org/wiki/Ada_Programming) ，关于混合资源，请参见 [Ada 信息交换所](http://www.adaic.org/)(在那里您可以找到参考手册)和 [Adacore](https://www.adacore.com/) 的网站。最后，你可以在 usenet group comp.lang.ada 和一些 LinkedIn 群组上认识其他 Ada-ist。

#### 现在你有了最重要的基础知识

一旦你对 C 和 Ada 有了很好的体验，学习其他语言只是学习一种新的语法，也许还有一两个新概念。例如，我仅仅通过一个教程就在几个小时内学会了 Ruby。最后，大多数当前的语言彼此非常相似。

### 打磨它:Ruby/Python 和汇编

最后，总结一下，作为蛋糕糖衣，我建议使用一种脚本语言，比如 Ruby 或 Python，也许还有一些汇编语言。的确，现在汇编已经不怎么用了(虽然这取决于你的上下文)，但是汇编的一些经验会让你对“引擎盖下”发生的事情有一些很好的理解此外，离处理器如此之近，以至于你几乎可以感觉到芯片，这是一件非常有趣的事情...:-)

### 奇怪的东西:函数式语言，PROLOG，…

如果你真的想要更广阔的视野，你也可以尝试一些奇特的东西，比如函数式语言(Eiffel，Haskell，…甚至 LISP)和逻辑编程，比如 PROLOG。那些语言有一个不同于通常的“过程”模型的“模型”。我没有看到它们被用得太多，我想知道它们是否适合非常大规模和非常长寿命的软件，但是知道它们的存在和它们如何工作是有用的。

### 结论和结束语

我希望你觉得这很有用，或者至少给了你一些编程的想法。最后一个建议:我的经验是，学习一门新语言的最好方法是使用它。在你熟悉了语法之后，选择一个中等难度的项目(即使是一个愚蠢的项目，它也不一定有用)去做(并且，可能的话，从中获得乐趣……)
