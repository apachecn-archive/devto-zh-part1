# 我还是初级开发者吗？

> 原文:[https://dev . to/click clickonsal/am-I-still-a-junior-developer](https://dev.to/clickclickonsal/am-i-still-a-junior-developer)

我在 2014 年 8 月的大会上开始学习如何编码。在完成了为期 3 个月的 Web 开发沉浸式课程后，我可以自称为“初级全栈开发人员”。2015 年 2 月，我在 Skyline Technology Solutions 找到了第一份工作，担任初级全栈开发人员。几周前，我开始问自己“我还是初级开发人员吗？”，因为我做编码已经 2 年半了，专业编码也 2 年了。我怎么知道我什么时候升级了？我要永远做初级开发员吗？？？

[![alt text](img/eaafcce25ba7a9b009f04d39274aa37b.png "Scared Kermit")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--BoirsAjE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AT-VmyFjh4tqGpCcRidA9ng.png)

所以我开始回想我开始的时间和我今天的位置，我记得我没有完全理解 JavaScript 中的回调、this 和原型。不知道为什么我需要使用 JavaScript 的。调用()或。apply()方法。或者像 requireJS、webpack、grunt 或 gulp 这样的工具实际上为我做了什么？我不记得什么时候，但我记得一切都开始点击，我终于明白了这一点(双关语)！

[![alt text](img/635d9f180aee054760c2917d255d697d.png "Alan from the Hangover Thinking, hard, difficult, math")T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--oHKksqwI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AmHtTVyi5EIB_y9IlF9mjrw.jpeg)

在我回答这篇文章标题中的问题之前，我会列出一些要点来说明我认为初级和中级开发人员之间的区别。

## 晚辈

*   理解基本的编程概念
*   有时会在不考虑性能或可维护性的情况下生成代码
*   可以使用库和框架，但并不总是理解工具实际上为他们做了什么和/或它是如何做的。
*   可能会从网上复制粘贴代码(如堆栈溢出)而不理解它，如果它只是工作
*   需要偶尔的指导
*   有时遇到困难时不会寻求帮助，因为害怕认为自己不够好(你很好！求助ðÿ˜š就可以了)

## 中级

*   更深入地理解一种或多种语言
*   将产生考虑性能和可维护性的代码
*   理解工具为他们做了什么，经常钻研源代码以理解它是如何工作的&也发现工具的未记录的用法。
*   甚至在考虑复制和粘贴代码之前，就能理解在网上找到的代码。
*   知道自己卡住了会求助(因为求助是可以的！)

以上要点只是我在互联网的帮助下收集的高层次要点&与我的同事交谈。所以我很自豪和自信地说，我现在是一个中级开发人员ðÿ！

> *等等！我真的可以自己回答这个问题，自己改标题吗？你实际上可以，因为你不需要一张纸就可以在这个领域获得任何头衔。在我看来，在改变你的头衔之前，你应该做的是和你一起工作的人，甚至是你在技术社区交往的人讨论这个问题。*

当我感觉我已经达到高级水平时，我一定会写一篇博客！:-)
