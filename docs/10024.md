# 我在数据工程师中寻找什么？

> 原文:[https://dev . to/jesset Anderson/what-do-I-look-in-data-engineers](https://dev.to/jessetanderson/what-do-i-look-for-in-data-engineers)

[最初发表于此](http://www.jesse-anderson.com/2017/02/what-do-i-look-for-in-data-engineers/)

我想和你们分享一些我发现的优秀数据工程师的特质。这些特质可能不是每个数据工程师都具备的，但是你会发现几个。

对于一个数据工程师来说，拥有强大的编程背景是多么重要，我怎么强调都不为过。数据工程师通常在职业生涯中处于中高级职位。那些刚从学校毕业的学生通常拥有计算机科学的硕士或以上学位，重点是分布式系统或数据。我看到一些特别聪明的初级工程师为团队做出了巨大贡献。

这听起来很奇怪，因为我谈了很多编程的重要性，但是最好的数据工程师对编程感到厌倦。这意味着他们已经掌握或几乎掌握了编程这门学科。写另一个企业系统或者小数据项目没有太大的兴趣。

因此，他们开始交叉训练进入其他领域。这些可能与数据科学等编程相关，也可能与营销或分析等无关。

数据工程师厌倦了创建小型数据系统。它们没有那么复杂。他们想创造更大更复杂的系统。其主要驱动力是他们希望创建每个人都可以使用的数据产品。

这种创造数据产品的愿望来自于对数据的共同热爱。你可能见过一个软件工程师热爱编码，或者甚至热爱一门语言。他们编码的时候最开心。数据工程师热爱编码和数据。如果没有爱，至少有对数据的兴趣。我发现这是伟大的数据工程师与优秀的数据工程师的区别。

他们使用这些数据是因为他们天生对正在发生的事情和原因感到好奇。他们会用他们的数据来证明或否定这个假设。

我不关注一个数据工程师懂什么技术。我关注他们对系统和分布式系统的理解。他们显然需要了解一些大数据技术和 API。然而，一旦了解了大数据系统的基本架构和设计模式，学习 API 或其他技术就会容易得多。一名数据工程师已经证明他们可以学习一些大数据技术，他很可能有能力学习其他技术。

当我培训一个已经在使用大数据技术的团队时，我经常看到这种情况。他们更快地理解这些概念，因为它们与其他大数据技术有相似之处。团队从训练中学到了更多，因为他们不是从零开始。
