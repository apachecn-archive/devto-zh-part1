# 你在脱自行车吗？大概吧！

你不知道我在说什么，对吧？这是因为没有多少人使用这个术语。这不是一个人们经常随意使用的术语。

当你为一些无意义的事情争论，并倾向于回避手头更重要的问题时，就叫做“脱单车”。

例如《城市词典》告诉我们:

> 针对次要、边缘问题进行技术争议，而忽略了更严重的问题。隐含的形象是，当房子还没有完工时，人们在争论给自行车棚刷什么颜色。

我们就以上图为例。你看到棍子工试图把沙堡里的水倒掉，但却回避了一个事实。这座城堡是用沙子和泥土建成的。他们的城堡下面有一条河淹没了城堡。

在某些方面，这就像拖延症。你浪费时间去做/谈论那些原本不应该是你主要关注的事情。

这在技术社区是一个越来越大的问题。我参与过一些项目，其中更重要的问题，比如可伸缩性，被忽略了，因为我们不能决定一个查询应该包含多少项。

所以，我建议，就像克里斯·科伊尔在他的文章中所说的那样，下次你在开会的时候看到两个人在争论一些没有意义的事情，大喊**骑车离开！**

直到下一次，代码长，繁荣！
