### 对 Python 和图形界面未来的思考

盯着我的同事，已经知道这种情况不可避免，我的眼睛转动着，争论又开始了:

“我告诉过你，我可以用一半的代码行写两倍的代码，而且它们会比你的代码更干净、更易读！Python 太牛了！”——一个人说。

“不管你说什么，你永远也做不出有这个一半好的 UI。它看起来不漂亮，没有人会想用它！”——另一个回答说——“可能连在 Windows 上运行都不行”——他一边走开一边喃喃自语。

几年前，这是同事之间的定期交流，虽然他们大多是在胡闹，但仍有一定的真实性。不管它的能力如何，Python 通常被认为是一种“脚本”语言——而不是真正的图形界面——世界仍然在他们的 GUI 应用程序中寻找本机操作系统的感觉，如果不做大量的工作，这是无法从 Python 中获得的。
