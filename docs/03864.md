# 19 类开发商解释

> 原文:[https://dev . to/lpas qualis/19-开发人员类型-解释-1lj](https://dev.to/lpasqualis/19-types-of-developers-explained-1lj)

这篇文章最初发表在 [CoderHood](https://coderhood.com) 上，被称为 [19 类开发者解释](https://dev.toPOST-URL)。CoderHood 是一个致力于软件工程人类层面的博客。

[![](img/30bb40fbe1766863811bda9c36cd786d.png)T2】](https://www.coderhood.com/19-types-of-developers-explained/)

软件开发的前景不断变化。每隔几个月就会出现新的专业领域、技术和方法，这些都是由软件行业的不懈创新打造的。有了它，描述特殊类型开发人员的术语成为非官方行业术语的一部分，并出现在招聘广告和公司名称中。

在互联网出现之前，这些专业化并不存在。万维网已经改变了我们生活的大部分方面，包括彻底改变了软件工程师的职业道路。

没有官方的行业术语表。理解每种类型的开发人员需要具备的技能对于新手来说是令人困惑的，对于非技术人员来说可能是令人畏惧的。

在这篇文章中，我定义了 19 种最常见的开发人员，并简单描述了他们使用的技术和必须具备的技能。请注意，这些术语的定义反映了我的专业理解，但它可能因公司、地区或行业而异。

## 1 -前端开发者(又名客户端开发者)

这是一个专门从事可视化用户界面编程的开发人员，包括其美学和布局。前端开发者代码在网站用户的计算机上的网络浏览器上运行。

这是非常高级的工作，通常远离硬件。它比计算机科学理论更需要对人机交互和设计原理的理解。前端开发人员的大部分时间都花在处理跨浏览器兼容性问题和调整 UI 的视觉呈现细节上。

前端开发技能包括用户界面(UI)和用户体验(UX)的设计、CSS、JavaScript、HTML 以及越来越多的 UI 框架集合。

## 2 -后端开发者(又名服务器端开发者)

这是一个专门从事设计、实现、功能核心逻辑、运行在远离最终用户的机器上的软件或系统的性能和可伸缩性的开发人员。

后端系统可能会变得非常复杂，但是用户通常看不到它们的复杂性。例如，考虑谷歌搜索引擎。前端部分是一个非常简单的 UI，有一个标题、一个文本框和两三个按钮。后端是一个非常复杂的系统，能够抓取网页，索引网页，并通过越来越多的复杂机制找到你想要的东西。

后端开发人员使用 Java、C、C++、Ruby、Perl、Python、Scala、Go 等编程语言。后端开发人员通常需要集成大量的服务，如数据库、数据存储系统、缓存系统、日志系统、电子邮件系统等。

## 3 -全栈开发者

这是一个既做前端又做后端的开发者。他或她拥有创建一个功能齐全的 web 应用程序所需的技能。

## 4 -中层开发人员

这是一个开发人员，他编写在浏览器中运行的非 UI 代码，并经常与在服务器上运行的非核心代码对话。一般来说，中间层是系统的“管道”。

术语中间层开发人员用于描述不擅长前端或后端，但两者都能做一点的人，而不是全栈开发人员。工程师很少用这个作为头衔，因为它更多的是对技能的描述，而不是职业道路。

## 5 -网络开发者

Web 开发人员是专门创建网站的软件工程师。他们要么是前端开发者，要么是后端开发者，要么是中间层开发者，要么是全栈开发者。

在 90 年代末和 21 世纪初，网络开发成为进入软件工程世界的一种非常常见的方式。它的起点很低，只需要基本的 HTML 和 CSS 知识。只需几个月的经验，一个入门级的 web 开发人员就可以开始编写可以交付给生产系统的代码。对于没有 CS 基础，想投身编程界的人来说，是一个特别有吸引力的选择。

## 6 -桌面开发者

这是一名开发人员，他开发的软件应用程序可以在桌面操作系统(比如 Mac OS、Windows 和 Linux)上运行。

早在 80 年代，这是最常见的工程师类型之一，因 Turbo Pascal、Turbo C、Visual Basic、Quick C、Visual Studio 和 Delphi 等廉价开发环境而流行。

桌面开发人员经常使用 GUI 工具包，如 Cocoa、XAML、WinForms、Gtk 等。

## 7 -移动开发者

这是一个为在智能手机和平板电脑等消费者移动设备上运行的应用程序编写代码的开发人员。在本世纪初和智能手机市场爆炸之前，移动开发几乎闻所未闻。在此之前，移动开发被认为是嵌入式开发的一个子集。

移动开发人员了解 iOS 和 Android 等移动操作系统的复杂性，以及用于在这些操作系统上编写软件的开发环境和框架。这包括 Java、Swift 和 Objective-C。

## 8 -图形开发者

这是一类专门编写渲染、照明、阴影、着色、剔除和场景管理软件的开发人员。这些开发人员通常负责整合游戏和视频制作行业的技术。

图形开发曾经是低级开发的一种形式，需要高级数学和计算机科学培训。随着商业和开放源码框架和系统的引入，它变得越来越容易获得。例如，现在很少有人需要从头开始编写着色器。

框架包括 DirectX，OpenGL，Unity 3D，WebGL。对于更高级的图形开发人员，低级开发需要 C、C++和汇编。

## 9 -游戏开发者

这是一个通用术语，用来标识专门编写游戏的开发者。游戏开发人员可以归入其他类别的开发人员，但他们通常在设计和实现引人入胜的交互式游戏体验方面拥有特定的知识和技能。

游戏开发者使用的框架包括 DirectX、OpenGL、Unity 3D、WebGL 以及 C、C++、Java 等语言。Adobe Flash 曾经是网页游戏的标准游戏平台。[自从 Flash 被抛弃后](https://coderhood.com/adobe-flash-end-life-timeline-infographic/)，JavaScript 和 HTML5 成为了新的标准。在移动设备上，Swift 和 Java 现在是 iOS 和 Android 游戏的首选技术。

## 10 -数据科学家

这种类型的开发人员编写软件程序来分析数据集。他们通常负责统计分析、机器学习、数据可视化和预测建模。

数据科学家使用的语言通常包括 SQL、R 和 Python。

## 11 -大数据开发者

这种类型的开发人员编写软件程序来存储和检索数据仓库、ETL(提取转换加载)系统、关系数据库、数据湖管理系统等系统中的大量数据。

大数据开发人员通常熟悉用于分布式存储和处理大量数据的框架和系统，如 MapReduce、Hadoop 和 Spark。大数据开发者使用的语言包括 SQL、Java、Python、r。

## 12 - DevOps 开发者

这类开发人员熟悉开发系统所需的技术，能够构建、部署、集成和管理后端软件和分布式系统。

DevOps 工程师使用的技术包括 Kubernetes、Docker、Apache Mesos、HashiCorp stack (Terraform、vagger、Packer、Vault、Consul、Nomad)、Jenkins 等。

## 13 -开发商 [CRM](http://searchcrm.techtarget.com/definition/CRM)

这种类型的开发人员专门从事收集用户和消费者数据的系统领域。这些开发人员的任务是通过改进客户支持代表、客户经理和销售代表使用的工具来提高客户满意度和销售额。

这些开发人员使用的技术包括 SAP、Salesforce、Sharepoint 和 ERP(企业资源规划)。

## 14 -测试中的软件开发工程师(SDET)

这种类型的开发人员负责编写软件来验证软件系统的质量。他们创建自动化测试、工具和系统，以确保产品和流程按预期运行。

SDETs 使用的技术包括 Python、Ruby 和 Selenium。

## 15 -嵌入式开发者

这些开发人员使用的硬件通常不被归类为计算机。比如微控制器、实时系统、电子接口、机顶盒、消费类设备、物联网设备、硬件驱动、串行数据传输等都属于这一类。

嵌入式开发人员经常使用 C、C++、汇编、Java 或专有技术、框架和工具包等语言。

## 16 -高级开发者

这是对用 PHP、Perl、Python 和 Ruby 等高级脚本语言编写远离硬件的代码的开发人员的统称。Web 开发人员通常是高级开发人员，但并不总是如此。

## 17 -低级开发者

这是一个通用术语，指用汇编和 c 等低级语言编写非常接近硬件的代码的开发人员。嵌入式开发人员通常是低级开发人员，但并不总是如此。

## 18 - WordPress 开发者

我把 WordPress 开发者包括在这个列表中，因为他们是一个庞大的专业 web 开发者群体。他们为 WordPress 创建和定制主题和插件，并管理 WordPress 站点。

这类开发人员使用 WordPress 系统、PHP、JavaScript 和 HTML。

## 19 -安全开发者

这种类型的开发人员专门创建系统、方法和程序来测试软件系统的安全性并修复安全性缺陷。这种类型的开发人员通常充当“白帽”道德黑客，试图渗透系统以发现漏洞。

安全开发人员通常使用 Python 和 Ruby 等脚本语言编写工具，并详细了解用于攻击软件系统的许多模式。更高级的安全开发人员需要阅读和理解用 C 和 C++编写的操作系统源代码。他们还可能对库和商业软件系统进行逆向工程，以发现和利用漏洞。

## 结论

软件开发人员可以通过许多途径进入职业生涯并取得进步。无论你是从正规的计算机科学教育开始，还是偶然进入个人项目的 web 开发，或者试图通过开发 iPhone 游戏赚一百万美元，可能性都是无穷无尽的。

一旦你选择了一条道路，你可以随着你的技能和知识的提高而改变。即使你选择在整个职业生涯中坚持走一条路，你也永远不会学不完东西。技术发展如此之快，以至于落后要比厌倦容易得多。
