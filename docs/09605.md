# 本创造一个成功的兼职项目的五个关键

> 原文:[https://dev . to/Ben/bens-five-key-to-creating-a-successful-side-project](https://dev.to/ben/bens-five-keys-to-creating-a-successful-side-project)

自从我进入软件行业(大约 12 岁时的 HTML)以来，我一直需要从事完全属于我自己的工作。所以如果我有自己的个人项目，那会吸引我所有的注意力。如果我在做学校的作业，或者职业发展的任务，我总是发现自己被那些次要的项目所吸引。

我从我的项目中学到了很多，总是做得比上一次好一点。随着时间的推移，我真正明白了我最成功的副业项目，这个网站、 [dev.to](https://dev.to/) 和 [@thepracticaldev](https://twitter.com/thepracticaldev) 。在许多其他有希望的项目，但最终因为这样或那样的原因失败后，我(非正式地)为下一个项目制定了一些标准。无论用什么标准来衡量，这都是一个成功的方案，我愿意与你们分享。

成功可以用很多方式来衡量。我认为这是一个有用的建议，涵盖了很多不同的“成功”概念

## 最初的顿悟

在回忆我在大学时做的一个项目时，我意识到如果我坚持下去的话，它会是一个巨大的成功。就像，绝对的。这个项目最终失败的原因有很多，但主要是缺乏耐心。我在卧室里开始的一个项目变得很重要，但由于管理不善，我最终让它失败了。当时我有这样的想法，我必须*成长成长成长*，如果我不扩大这个企业，它将会失败。这是有缺陷的想法。如果我能够保持项目的新鲜感和趣味性，即使是最适度的增长也会随着时间的推移而增加。

有了一些空闲时间，以及编写稳定的软件和建立持久的东西或其他东西的愿望，我列出了一些标准，以建立在过去工作的基础上。然而，这一次，它不会失败。

## 🔑让它成为你自己的

那一定是你。如果你不能自豪地谈论你正在建造的东西，并且喜欢把自己倾注进去，那最好是你能很快“完成”的东西。如果这是一个必须持续的项目，它最好能反映你的兴趣和激情。如果你坚持下去，对自己诚实，时间会把你的项目变成成功，但是如果你不能保持对你正在围绕的主题的根本兴趣，它就会失败。

## 🔑给自己约束

当事情不太顺利时，你总是想要改变路线，或者扩展你的想法以获得成功或让事情变得更容易，但请战胜这种冲动，给自己一些规则来遵守。对我来说，随着 [@thepracticaldev](https://dev.to/thepracticaldev) 的成功，我不得不不断地与做需要更多人参与的事情的冲动作斗争。通过给自己一些事情必须由我来做的约束，我变得更善于修剪想法和建立自己的管理功能来扩大我的努力。我的核心标准是，我必须打造一台运转良好的机器，而不是试图雇佣帮手来维持一个繁忙的流程。

这一限制至关重要。这从根本上使这个东西变得可扩展。约束是美好的事物，它们有各种形状和大小。但是不管你的过程或技术是否存在限制，找到它们并热爱它们。随着时间的推移，你可以克服它们，但要和它们一起生活，直到它伤害你，永远不要轻易打破束缚。

## 🔑消除时间限制

我确实认为，对于这种成功的附带项目，必须从一开始就消除的一个时间限制就是时间限制。你必须把这看作是会永远存在的东西。你不能着急。 *[原著](https://www.amazon.com/Originals-How-Non-Conformists-Move-World/dp/0525429565)* 这本书真的给了我不操之过急的权限。作为一个旁注，我肯定会推荐这本书的部分内容，但其中一些内容完全没有意思。

你需要建立一个工作流程，这样你的项目才不会烂掉。让一个项目失去前进的动力，它可能会失败。宽松的时间约束是建立一个健康的异步工作流的关键，但是您需要保持纪律，在没有截止日期或匆忙的情况下保持它。当你需要赶时间的时候赶时间，通过记录你的见解来接近想法，即使你不能立即实现它们。

## 🔑该项目需要随着时间的推移而发展

从一些基本的和(可能)对世界其他地方不感兴趣的事情开始。

即使你的最终目标是创造性的和令人兴奋的，也不要去建立它。这可能是一个艰难的障碍，因为你的第一个版本可能并不那么不同或有趣，但你会通过获得一些快速胜利和启动*任何东西*学到很多东西。这将是令人沮丧的，因为当你解释你在做什么的时候，听起来会很无趣。但这需要成为过程的一部分。我认为你需要理解没有人会关心你的项目，你只需要坚持下去直到它开花结果。不要急于求成，即使你希望人们能看到你的未来愿景。

从技术角度来说，一开始创新少一些，但给自己一个可以启动的平台是关键。如果你遵循既定的模式，你的代码会好得多，你会发现一些地方蚕食规范，直到最终你构建出独特的、真正令人敬畏的东西。

## 🔑自己动手，最终寻求帮助

我并不固执于“自己动手”这一部分，但那绝对对我有用。我是一个独立工作者，但那可能不适合你。不过，我会鼓励从副业开始。这是一个很好的约束，以确保项目适合你的优势，并且在动机、时间承诺、成功指标等方面没有问题。即使多一个人也能让这一切变得一团糟。考虑到这一点。某些社会关系，比如配偶关系，可能更容易克服期望不平衡。

最终，会有一个*寻求帮助*的拐点，你必须自己去发现。不要过早跳到这一部分。这是我之前几个副业最大的失误。我尽可能长时间地忍受一个人的限制，并最终在[李佳薇](https://dev.to/jess)找到了这个项目的一个很好的补充，这真的让整个事情成功了。她是我的朋友，我们相处融洽，但我们有完全不同的技能。当我把她带进来的时候，这个项目已经开始了，但这是确保这个项目成功的第一个重要转折点。

## 闭合思维

我希望这是一个有益的阅读。这是一个令人满意的项目，我迫不及待地想看到它成长。最近又出现了一个转折点，杰西和我在这个项目上得到了更多的帮助。这是一项正在进行的工作，但它非常令人兴奋。这个所谓的副业前途光明。我希望你能找到一个你愿意倾注多年生命的项目，并从中找到满足感。

请不要迫于压力将其商业化。有这么多令人满意的开源项目可以从我在这里概述的想法中受益。创建一个旨在最终赚钱的企业固然很好，但一旦这成为一种压力，消除时间限制的能力很快就会崩溃。只是要意识到这种动态。

附言:如果我今天开始另一个副业项目，我会强烈考虑让它成为一个专注于围绕某事构建公共 API 的项目。人们通常会直接构建带有用户界面的完整堆栈，但是如果您对某个领域感兴趣，我强烈推荐构建和维护一个纯 API 的服务。找到世界上没有很好的软件模型的东西，花时间来构建和维护一个可吸收的 API。此外，让你的项目开源宜早不宜迟。这个项目最终将是开源的，但我后悔没有一开始就这么做。
