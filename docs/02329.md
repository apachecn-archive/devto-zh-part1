# 像 noob 一样处理错误

> 原文：<https://dev.to/helloanavee/handling-errors-like-a-noob-74h>

[![frustrated dev](img/7b694fc1d2efa4cc25d1a23270f422c5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--RH0sD2of--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/8yshapm3gnx108l5f78z.jpg)

我们刚刚完成了一个大项目，比截止日期晚了两天。我说的“我们”指的是我(摸索着去理解这个项目的进展)和我的老板(他指导了我大部分时间)。

客户在程序中想要的:当一个人批准向他提出的请求时，csv 格式的请求将自动下载到不同服务器的文件夹中。

我不明白的地方(反正现在还不明白):如何将 csv 文件下载到特定的文件夹。

经过一系列的尝试和错误，以及 StackOverflow 的提示，我得到了 cURL。有了 cURL，我可以随时通过链接下载一个 csv 文件。该 csv 文件保存到 cURL 文件所在的位置。从现在开始，我将把这个带卷的文件叫做 listen.php。

耶！至少我知道 listen.php 真的下载了一些东西。

下一个问题:我需要将动态数据传递给 csv。事实上，只有当我直接转到 listen.php 所在的页面时，它才是可访问的，csv 只是输入 302 错误。

所以我需要从我的控制器发送数据到 listen.php 文件。该 listen.php 文件将转到该链接，并希望下载正确的 csv 文件。我的问题在发送部分:我如何在没有表单、提交按钮或 js 的情况下发送$_POST 数据？

老板的建议:研究狂饮。

而我说:(一字不差的用他加禄语:“Tama ba pagkakaintindi ko 先生？Magpopost ako 从我的控制器监听. php？”)我理解的对吗？我会把数据从控制器上传到 listen.php？

他回答:对，没错。“是的”。(塔玛道。)

在阅读了 Guzzle docs 的请求部分，复制并修改了它的示例代码后，我能够将数据传递给 listen.php。耶！并且下载该文件。我知道我传递的数据是正确的，因为 csv 文件的名称是正确的和动态的。我还在 POST 请求中传递了 csv 的标题。

但是在 csv 里面有一堆不同的错误。

下一个问题:所以我可以下载一个 csv 文件，但是这个文件只包含:
a)空白页
b) 302 重定向错误
c) csv 文件但是包含带有错误堆栈的 html 代码

我尝试了一些不同的事情，我忘了具体是哪一个，这导致了我的 3 种错误。

我有逃避错误的倾向。每当我不能理解一个特定的集合，或者我在一个任务上卡住了，我就去列表下面的另一个任务。我不认为这是有效率的，但当我完成一项任务时，我会感觉更好，即使是一件简单或次要的事情。

我想这是我在项目上花费更长时间的主要原因。每当最后期限临近时，我往往会有一堆难题要解决。

这造成了更多的恐慌。

然后我不能正常思考和集中注意力，因为我担心最后期限的临近，我不能解决它。

致命的循环还在继续。

当我得到带有 stacktrace 错误的 csv 文件时，我的直觉是尝试我在 StackOverflow 中找到的另一个“hack ”,而不是在 stacktrace 中找到错误。

我是希望弹出错误，还是希望有一个没有错误的空白页面(但仍然不工作)？我会随时选择错误页面。但是他们仍然让我害怕。

我的老板说，如果错误被放大，我们就能看到错误，那就好了。至少我们有线索去哪里找。

我觉得自己像侦探柯南。但我发现的不是尸体，而是页面不起作用的原因。

你对程序中的错误有什么看法？
