# JavaScript 赢不了

> 原文：<https://dev.to/plainprogrammer/javascript-can-not-win-25ed>

我在[Jonny Asmar](https://medium.com/u/7e0f0c4bba2b)—[*hacker noon*](https://hackernoon.com/javascript-has-already-won-235b29ed126b)上看过一个故事，Javascript 已经赢了 *。*前提是 Node.js 和 JavaScript 已经变得如此深远，以至于它已经战胜了其他编程语言。

[![](img/7bb54514344a51bfd2b3de355dea7e2c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--0pJFpzfA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://james.thomps.onl/wp-content/uploads/2019/07/f080f-1dmfnkrwdue3lm3ki7qvvqg.jpeg)

不幸的是，这整个思路都是无稽之谈。不是因为 JavaScript 做不到宣称的所有事情，而是因为没什么好赢的。JavaScript 赢不了，因为没有竞赛。JavaScript 赢不了，因为没什么好赢的。

> JavaScript 赢不了，因为没什么好赢的。

* * *

JavaScript 自 1995 年就已经出现了。90 年代其他著名的编程语言包括 Haskell (1990 年)，Python 和 Visual Basic (1991 年)，Ruby (1993 年)，以及 Java 和 PHP (1995 年)。根据 TIOBE 指数，Java 和 Python 都超过了 JavaScript。而且，即使是早在 1972 年发布的古老的 C 语言，其发展速度也超过了 JavaScript。

但是，那又怎么样呢？所以，没什么。这才是重点。如果我们继续关注 TIOBE 索引，像 COBOL (#24)、Fortran (#33)和 Prolog (#41)这样的语言仍然在排名中——而且，前两种语言来自 20 世纪 50 年代后期。那么，JavaScript 到底赢了什么？没什么。因为没什么好赢的。

对于编程语言来说，没有普遍性这个奖项，因为不存在普遍性。任何能够达到一定流行程度的编程语言都不太可能永远*消亡。COBOL 自 1959 年就出现了，据说[70%到 80%的商业交易都在上面运行。因此，如果有人赢得了什么，那就是 COBOL，而不是 JavaScript。](https://blog.hackerrank.com/the-inevitable-return-of-cobol/)*

 ** * *

[![](img/7334bec9984f5893226112f2b1b0a471.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--JhDh-r6h--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://james.thomps.onl/wp-content/uploads/2019/07/02227-1liiglwrpjzse8don8_roag.jpeg)

无论一种语言变得多么受欢迎，都没有奖励可言。不仅仅是因为人气来来去去，还因为竞争比虚荣更糟糕:那是想象出来的。如果用 JavaScript 重写 COBOL 的每一行，没有什么能阻止 JavaScript 出现同样的混乱。这是因为没有完美的编程语言、范式或生态系统。这是太多软件开发人员的巨大虚荣心。

我从 2003 年开始开发网络软件。我知道，与我出生前在这个行业的经历相比，我这些年的经历微不足道。我还没有老到可以看到任何技术起飞，然后消失在默默无闻中，但我知道它已经发生了。我没有不幸地与一个欣欣向荣的技术一起工作，并看到它停滞不前，萎缩到一无所有。但是，我听说过那些经历过这些的人的故事。而且，即使对那些人来说，他们的损失也只是暂时的。他们丢了一个工具，找到了另一个。最终，他们什么也没失去，因为他们没有什么可失去的。

编程语言是工具。合适的，灵活的，才能流行起来。即使是不讨人喜欢的也能找到自己的受欢迎程度。但是，曾经说过一种编程语言赢得了*对我来说似乎是胡说八道。也就是说，除非你说的是 COBOL。因为，如果统计数据是真实的，那么无论编程语言的竞争是什么，COBOL 都是赢家。**
