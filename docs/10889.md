# 也许重新发明轮子

> 原文:[https://dev.to/ben/maybe-reinvent-the-wheel](https://dev.to/ben/maybe-reinvent-the-wheel)

不要重新发明轮子——除非轮子是用 PHP 编写的，固执己见，你永远反对的不是轮子，而是一堆热气腾腾的代码。这个网站不是 Wordpress 网站，我很高兴我“重新发明了轮子”我选择了一个我喜欢使用的堆栈——Rails，它在项目的约束和自由之间给了我恰到好处的平衡。Rails 并不比 Wordpress“更好”。它们是风马牛不相及的，但是我自己铺设混凝土给了我真正做出我想做的东西的能力。此外，在我的日常工作中，我没有得到构建一个基本的 CRUD 应用程序的简单乐趣，不要从我的反对者那里夺走这一点！

很多时候，是的，你应该达到现有的实现，但在你这样做之前，你应该考虑后果，并以某种方式达到某种证明，即使你有不完整的信息，你将有不完整的信息。

你很少能想出完整的答案，但有时这是关于旅程而不是目的地。仔细考虑是自己编写还是使用库/框架/服务的后果是一个非常有用的练习，有助于了解未来的技术需求。

写软件是一个永无止境的跳跃过程，希望你能落在柔软的东西上。你永远不知道未来的需求是什么，但是要避免像“不要重新发明轮子”这样的笼统的规则不要害怕“浪费时间”,出去走走，思考一下将要做决定的所有事情，以及未来会发生什么。你现在节省了时间，但以后却增加了负担吗？会有回报的。

快乐编码。
