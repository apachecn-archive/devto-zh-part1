# 我主持了我的第一个技术播客

> 原文：<https://dev.to/ben/i-hosted-my-first-technical-podcast>

不久前，我被邀请去主持《软件工程日报》的一期节目。我从来没有做过播客的嘉宾或主持过播客，主持技术播客的想法让我的冒名顶替综合症变得疯狂，但为了尽快克服我的不安全感，我只是答应了。在另一个世界，我第一次尝试主持播客会被十几个朋友和家人听到，但这次 fire 的尝试是在一个有数千人收听的成熟技术播客上进行的。

《软件工程》的每日听众对节目中的讨论有很高的期望，我感到满足他们的压力很大。如果我有一些巨大的技术盲点会在面试中尴尬的暴露出来怎么办？更不用说当我把所有东西都拼在一起的时候，还得听到我自己的声音。

尽管我很害怕，录制播客还是很棒，在录制的一个小时里我学到了很多。我的客人是胡安·布里蒂卡，T2 游乐城的工程副总裁，整个讨论非常棒。有很多事情我可以做得更好。我可能已经偏离一些技术问题太远了，因为我对 Juan 每天所做的一些管理选择非常感兴趣，但是软件工程的大部分是关于人和交互的，所以我个人认为这个方向提供了很多价值。

在采访中，我真正学到了很多关于 Juan 如何在 Ride 组织他的工程团队，以及他们为什么做出一些技术决策。Juan 的团队分布在美国和拉丁美洲，因为 Juan 是哥伦比亚人，他能够利用自己的关系网组建一个非常有才华和多元化的分布式团队。这一集今天播出了，现在是第一集。

如果你想主持一个技术播客，以下是我的建议:

1.  做吧，会比你想象的容易。
2.  不要花太多时间开玩笑或闲聊。吃肉吧。
3.  问问题是为了真正学到东西。其他人也可能从中受益。
