# (初级)开发人员必备读物

> 原文:[https://dev . to/yordiverkroost/初级开发人员必备读物](https://dev.to/yordiverkroost/essential-reads-for-junior-developers)

您可能不止一次听说过，为了跟上快速变化的软件开发世界，开发人员必须是终身学习者。这是真的:框架和编程语言比人类诞生得更频繁(不，它们不是，但你知道我的意思)。如果没有多任务处理的能力，要跟上他们是不可能的，但是除非你是一台秘密的电脑，那也是不可能的。

[![Multitasking](../Images/4f195d2a6f704edad43d860fcd6318aa.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--NDipvZ6f--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AbBKLqHoxKZd9OukvoVbkvQ.jpeg)

作为一名开发人员，我试图通过听播客来跟上(后端和前端)开发中的大多数新技术。NET Rocks，Hanselminutes 和 Developer On Fire。像这样的播客给你(至少)介绍了各种新的框架和其他与开发相关的东西。但我更喜欢的是学习那些有着非常普遍用途的东西。你可以学习一种特定的编程语言，但是你也可以学习如何编程。您可以了解 Angular 的所有细节，但是您也可以了解 Javascript 框架是做什么的，以及它的主旨是什么。很明显，我喜欢后一种选择。有鉴于此，我想推荐两本我目前正在读的书(关于不能同时处理多项任务，我就说这么多)。第一本是著名的书《设计模式》,由 Erich Gamma、Richard Helm、Ralph Johnson 和 John Vlissides 组成的所谓“四人组”撰写。顾名思义，它包含了对设计模式的解释，这些设计模式可以应用于所有的(面向对象的)编程语言。能够在你写的代码中使用这些设计模式肯定会帮助你写出更好的程序。作为备忘单，我喜欢使用。NET 设计模式网站提供一些快速刷新和解释。肯定会有其他网站有其他编程语言的同类例子。

[![Design Patterns](../Images/017dab1a3d71093a425646928cc04481.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--oRSD6HQM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AFpEusOlKSUkIX5hccJuDKw.jpeg)

另一本书不一定与发展有关，但让你领略了“亿万富翁、偶像和世界级表演者的战术、套路和习惯”，正如书的封面上所写的那样。这本书名叫《泰坦的工具》,作者是《纽约时报》畅销书作家蒂姆·菲利斯，他因与这些亿万富翁、偶像和世界级表演者的著名播客而闻名。事实上，这本书和书中讨论的人之前已经出现在费里斯的一个播客中了。你在这本书里找不到很多实用的开发技巧，但是却有很多你可以在自己的工作生活中使用的技巧和诀窍。有太多的建议，不可能全部都用上，但是你可以尝试其中的一些，看看它们是否对你有用。就我个人而言，我正在通读这本书，并在我想在稍后阶段重温和尝试的引用处留下标记，或者甚至听听菲利斯与引用者的播客。我刚刚读了 20 多岁的人的简介，已经留下了足够的标记，让我在接下来的几个小时里忙于听这些人必须要看的东西。

所以，对于接下来的时间，足够的阅读材料，我退出继续设计模式和工具的泰坦之旅，我已经开始了！如果您对一般开发主题有其他好的(书籍)提示，不要忘记将它们添加到讨论中！
