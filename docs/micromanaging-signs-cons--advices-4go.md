# 微观管理:迹象、缺点和建议

> 原文：<https://dev.to/apium_hub/micromanaging-signs-cons--advices-4go>

如今，微观管理是人们对管理者抱怨最多的事情之一。如果我们考虑到根据丹尼尔·平克的观点，21 世纪的持久动力是由自主性、掌控力和目标推动的，那么我们绝对应该讨论这个问题。因此，在今天的文章中，我们将看看“微观管理”是什么意思，这样做的原因是什么，微观管理的主要缺点以及如何提高团队效率的宝贵建议。

## 微观管理？这是什么意思？

微观管理是一种管理风格，管理者密切观察和控制他或她的员工的工作，过分关注微小的细节。

在大多数情况下，微观管理通常被认为有负面影响，但它仍然是世界范围内普遍接受的管理风格，尤其是在管理新来者时。有时间做这个！在大多数工作场所，经理负责培训新员工，这是微观管理的理想时机。由于新员工仍然不真正了解公司的文化、价值观、流程、关键的成熟流程，微观管理将是一个很好的选择，实际上可以教会他们公司是如何做事的。所以，微观管理者在工作过程中指导他们，激励他们，在“微观”层面上教他们。因此，在这种情况下，你可以与你的新团队成员密切合作一段时间，可能需要密切监督，直到一个单位成立并表现良好，通常只有在试用期才有必要。

但是，如果你开始对有经验的员工进行微观管理，并开始问“你现在在做什么”这样的问题，你实际上激励了他们，并对他们做什么和如何做表现出缺乏信任。如果你给人们一份工作，那么他们有能力完成任务，不要扼杀他们的自主性、创造性和主动性。

尽管微观管理通常很容易被员工认可，但微观管理者很少这样看待自己。他们通常对自己的管理风格给出不同的描述，例如:“结构化的”、“有组织的”或“完美主义的”。

## 事无巨细的迹象&这么做的主要原因

微观管理最常见的动机是注重细节、情绪不安全感和对员工能力的怀疑。还有，也可能和管理者的性格有关。因此，微观管理者希望确保每件事都按照它应该的方式进行，事情总是按照他们期望的方式进行。此外，尤其是当我们谈论软件行业时，这些微观管理者希望能够准确地估计交付项目所需的时间，以便进行有效的成本价值分析，并给客户一个准确的交付预测。问题就在这里！这些微观管理者中的大多数只是不使用敏捷方法和 scrum 方法，在这两种方法中，一切都被很好地规划并分成冲刺阶段(通常为 2 周)，有清晰的路线图和用户故事。有了敏捷，你就有了透明度，你就有了每日会议和冲刺回顾。事实上微观管理者想要的一切。也没有必要每小时问一次“你在做什么”的问题，因为你有像吉拉这样的工具，它会向你展示进度图和“进行中”的活动。

## 识别微观管理者的关键标志

*   抵制委派工作
*   让自己沉浸在分配给别人的工作中
*   看细节而不是大局
*   阻止他人做决定
*   期待定期的详细报告
*   有一个士气低落的团队
*   很少对可交付成果感到满意。
*   想知道所有团队成员在哪里，他们在做什么。
*   要求经常更新情况
*   更喜欢电子邮件抄送给我。
*   准确地告诉团队他或她希望事情如何完成，不给团队任何采取主动的空间。
*   承担项目经理的角色，即使已经有了项目经理。
*   团队人员流动率过高。
*   感觉如果你想把事情做好，你必须自己去做。
*   当下属未经咨询就做出决定时，通常会感到恼怒，即使这些决定属于下属的权限范围。
*   提出无法实现的最后期限。
*   引入“优先”任务。
*   例行公事地要求人们停止他们正在做的任何事情，去处理紧急的事情。
*   要求事情为“现在”。

## 微观管理的主要缺点

微观管理是一种领导形式，可能在短期内产生效果，但随着时间的推移会伤害员工和公司的士气。通常，微观管理对团队有负面影响，因为员工可能会觉得微观管理者对他们居高临下，因为他们对员工的能力缺乏信心。此外，实施这种管理风格的经理会创造一种环境，在这种环境中，他的团队对工作缺乏安全感和信心。有时候，微观管理会完全消除信任，扼杀学习和发展人际交往能力的机会，甚至会引发反社会行为。微观管理者痴迷于控制，觉得被驱使去推动他们周围的每个人成功，这样做他们冒着削弱同事权力的风险。这摧毁了团队的信心，损害了他们的表现，让他们沮丧到放弃的地步。

此外，微观管理者可能会觉得，通过充当看门狗，他们可以确保每次都得到他们想要的结果。然而，微观管理的后果可能会造成比经理或企业主意识到的更大的损害。当所有的决定都必须通过经理，员工的每一项职能都受到严格审查时，这就扼杀了创造力，导致了停滞。员工变得不愿意提出新的想法来改进流程、产品或服务，从而提高生产率。员工甚至可能故意保留能够带来创新和积极变化的想法。由于微观管理者不允许团队中其他人的主动性和投入，员工们学会了将所有的决策权交给管理者，变得完全依赖于他或她。微观管理不仅对员工不利，还会对微观管理者的身心健康产生可怕的影响。这里重要的是花时间退一步，喘口气，并意识到团队可以在没有微观管理者不断控制的情况下处理任务。而当员工被微观管理时，他们往往会做一件事……辞职。它影响了公司的士气，粉碎了团队的精神。

## 如何改善微观管理问题？

“与你能找到的最优秀的人在一起，下放权力，只要你决定的政策正在执行，就不要干涉。”罗纳德·里根

*   基本上，micromanager 可以为任何给定的项目设置几个定义成功的度量标准，并简单地忽略其他没有定义的细节。
*   更多地关注“需要做什么”而忽略“如何做”
*   为团队成员提供一个门户开放的政策，以便在他们需要的时候提供指导或进一步的指导。
*   为指定项目的每个阶段设定一个截止日期，之后应召开一个有合理时间限制的会议，以了解工作的最新情况。
*   让员工参与目标设定和评估。所有人都是不同的，不要做你的估计，让团队去做。此外，当员工参与目标设定过程时，他们可以看到自己工作的目的，当他们自己做评估时，他们不会在后来说最后期限不现实，因为他们做了。
*   促进双向对话。微观管理的最大问题之一是它是单方面的。记住，做事的方法不止一种。员工甚至可能知道最好的方法。不要发号施令，问问员工的想法和意见。讨论他们如何工作得最好，如何工作得最有效率。
*   注重结果反馈。良好的反馈能让员工学习和成长。如果一个经理简单地说，“你需要用这种方式来做事情”，员工会觉得被控制了。他们没有被引导；他们被微观管理。更好的选择是关注事实和结果，而不是过程。
