# 为初学者建立一个社区

> 原文:[https://dev.to/jennifer/building-a-community-for-beginners](https://dev.to/jennifer/building-a-community-for-beginners)

*最初发表于 2017 年 2 月 18 日在[加勒比海](http://www.pycaribbean.com)的演讲。为网络稍作修改。x-被派往[jmkoni.com](https://www.jenniferkonikowski.com/blog/2017/2/10/building-a-community-for-beginners)。*

我六年前开始用 Python 编程，过去五年一直在做 Ruby 开发。当我作为一名新开发人员去用户组时，那是非常令人生畏的。人们似乎互相认识，我不知道该向谁寻求帮助。最后，我不再去了，只是创建了自己的群。但是许多人非常沮丧，他们认为编程不适合他们。

这就是我认为为初学者建立一个社区如此重要的原因。我来解释一下原因。我们希望确保我们的社区对初学者开放，因为我们需要扩展和多样化。我们的社区越多样化，我们的团队就越多样化。根据《哈佛商业评论》的报道，“和与你不同的人一起工作可能会挑战你的大脑，让它克服陈旧的思维方式，提高它的表现。”我还认为每个人都应该能够学习编程。编程不应该仅限于那些在小学就有足够特权学习编程的人。不管他们的年龄、性别或背景如何，如果有人想加入我们的社区，我们应该乐于帮助他们学习。让社区来帮助学习不应该只对愿意为代码学校支付数千美元的人开放。大约四年前，我创办了 PyLadies Boston，目的是让更多的女性加入 Python 社区。我还参与了波士顿 Ruby Women 的活动，主持每周的学习会议，回答人们向我提出的任何问题。随着我们的继续，会有更多关于这些和其他一些的信息…

你如何尽最大努力确保你的小组对初学者开放？首先说一下新群体。正如我提到的，现有的团队仍然可以从这些想法中受益，所以如果你已经有了一个团队，不要完全脱离。

1.  确保你描述你的团队和活动的方式包含初学者

    《出埃及记》“没有大小问题”对所有层次的人都有好处

2.  明确需要哪些知识和技能

    明确需要什么层次的知识。人们常常会低估自己，所以在描述什么是需要的时候要记住这一点。《出埃及记》“需要基本的 Python 知识，应该已经完成了类似“艰难地学 Python”这样的教程”

3.  找出你当地社区的需求

    这很重要。每个社区需要不同的东西。有了 PyLadies，我们有了一个由学者和科学家组成的大型社区，因此对教程和代码评审有着巨大的需求。对于波士顿 Ruby Women，我们有很多最近的新兵训练营毕业生，所以我们花了很多时间谈论面试和找工作。

4.  一直寻求反馈。

    每年，我都会举办一个周年聚会，并询问每个参加聚会的人对过去一年的反馈(你喜欢什么，不喜欢什么)以及对未来的建议。对皮拉第斯的定期评估让我们有了我自己从来没有想到过的新类型的活动，并摆脱了那些我认为会成功但不会成功的活动。

5.  为整个团队尝试不同类型的活动。根据你所在的社区，有些可能比其他的更适合你。以下是我成功应对的一些事件类型:

    1.  演讲之夜是标准的，但通常有一种想法，你必须是一个专家才能做一个演讲。当要求做演示时，要清楚地表明你对初学者项目的演示持开放态度。
    2.  闪电谈话是让人们进行第一次公开谈话的好方法。我鼓励人们的方法之一是说，虽然它应该是相关的，但如果你有一个爱好想与大家分享，我们很乐意听到关于它的闪电谈话。皮拉迪斯的一个成员最后做了她的第一次观鸟报告，并且获得了巨大的成功！
    3.  *教程*总是成功的。它们给有经验的人一个有意义地分享知识的机会，给初学者一个学习新技能或工具的机会。然而，对于教程来说，在开始时，或者在活动开始前，留出额外的时间来做准备是很重要的。相信我，即使你给出了明确的指示并要求人们事先做好准备，你仍然可能需要额外的时间。
    4.  乌合之众编程(Mob Programming)是指整个团队都在关注同一个问题，并试图一起解决它。我们开始运行 mob 编程和代码审查相结合的活动，它们非常成功。每个人都可以参与:即使只有有限的编程知识，你也可以知道其他人面临什么样的问题。
    5.  举办单独的以初学者为中心的活动。这些活动将吸引那些仍然过于拘谨而不愿参加主要会议的人。我们经常有女性出现在我们的初学者活动中，这些活动几乎从不去主要小组，因为她们觉得自己还没有准备好，尽管我鼓励她们。
        1.  学习小组可以帮助人们互相学习。在皮拉迪斯，我们尝试每周都有学习小组，每次都有导师。然而，我们也鼓励我们的成员在他们的社区里成立学习小组，并且取得了巨大的成功。我用这些来瞄准那些刚刚开始学习编码的人。如果你试图在学习小组中提供导师，这可能是一个挑战。向更有经验的会员推销的一个方法是，这是分享他们的知识和提高他们对基本面的理解的一种方式。我有一个每周都来的女人，她总是挑战我，让我比以前更深入地学习这门语言。
        2.  导师会议类似于学习小组，但更侧重于职业发展。这些目标人群是那些知道如何编码并希望进入这个行业的人。大三找工作通常非常令人沮丧，定期与告诉你你能做到的人会面会有所帮助。此外，通过结识更多的初级开发人员，你会更容易找到那些还没有机会的优秀开发人员。通过这些团队，我已经让 Akamai 和之前的公司雇佣了两个人。通常，在一个更大的群体环境中更难认识人。像导师会议这样的小组会议可以帮助更有经验的成员了解那些刚刚起步的人。

这是开始一个以初学者为中心的小组的一些基础，但是如果你现在正在管理一个小组呢？以下是一些成功地将更多初学者引入波士顿 Python 和波士顿 RB 的建议。我将从最简单的开始:

1.  给每个加入的人发一封电子邮件，强调每个人都是受欢迎的，无论他们的编程经验水平如何，并让他们知道在你的活动中会发生什么。使用 Meetup，您可以一次编写一条消息，然后自动发送给每个新成员。
2.  如果可以的话，让别人在他们走进来的时候打招呼。理想情况下，它将是组织者之一，定期在那里的人。这个个体应该尽自己最大的努力去了解刚加入的人。它会给所有新来的人一张友好的脸，每次他们回来的时候，还有一个熟悉他们水平的人。
3.  对于演讲之夜，确保有适合初学者的常规讲座。这些讲座不一定是关于“如何编写 for 循环”，而是更多地“这里有一个问题，这是我如何解决的”，而不是强调纯编程。与我交谈过的组织者说，当他们有来自各种领域的多名演讲者时，他们在晚上收到了大量新注册者的涌入。
4.  对于项目之夜，有一些建议:
    1.  准备几张初学者的桌子，如果可以的话，请几个有经验的程序员给他们配备人员，帮助新人解决问题。
    2.  一开始就做介绍。让每个人介绍自己并提及他们正在做的事情。如果有经验的人愿意整晚提供帮助，你可以请他们举手。这可能很耗时，但它将有助于建立社区并为人们合作创造机会。
    3.  向人们保证，如果他们没有在做项目，那也没关系。如果人们想在一个项目上合作，你可以让他们在一开始就举手。

好的，最后但同样重要的是:举办研讨会，找到教人们编程的最佳方式。

1.  就我个人而言，管理研讨会是组织者面临的最大挑战之一。这里有一些问题的概要和一些关于如何处理它们的建议。
    1.  空间:鉴于研讨会至少持续 6 个小时，你不能在工作日晚上举办。所以，大部分商家是不会想托管的。然而，你应该尝试联系当地的大学和社区学院——如果你的团队中有人在那里工作就更好了。
    2.  志愿者在任何时候都是一个挑战，但是让人们放弃他们的周六(也许加上他们的周五晚上)完全是另一个问题。预计至少有几个人会在最后一刻退出，所以要有后备计划。确保比你认为需要的多几个志愿者，并做好准备，如果应该出席的人没来，你就可以出席。
    3.  如果你正在做 Django 或 Rails 的研讨会，内容可能是最简单的，因为已经有针对周末时间框架的完整教程。如果你想为这两者中的任何一个开一个工作室，去看看 DjangoGirls，RailsGirls 和 RailsBridge。如果你想为一个不同的语言或框架开一个研讨会，考虑看看那些例子，看看你应该包括什么，并把它用于你想涵盖的框架。如果考虑一个关于语言的研讨会，回顾一下波士顿 Python 研讨会的内容。虽然这些材料是 Python 语言，但是您可以修改它们以适应其他语言。
    4.  食物——当有人被困在一个房间里一整天时，至少提供午餐是很重要的。你可以联系使用你所教授的语言或框架的当地公司，并找人提供食物。通常，他们也会想派一名志愿者参加研讨会，这样他们就可以有人代表他们的公司了
    5.  持续参与可能是最大的挑战。当人们来到研讨会时，确保他们知道接下来的步骤是什么。当 RailsBridge 波士顿研讨会举行时，他们总是确保在下一周晚上有一个波士顿 RB 项目，这样人们就可以继续学习。你也可以在不久之后进行闪电谈话，鼓励人们谈论他们想要解决的问题或者他们想要构建的应用程序。
2.  没有教人们如何编码的“最佳方法”。然而，我用一些方法比用其他方法更成功。
    1.  不起作用:
        *   如果人们付费的话，一周又一周建立在自身基础上的班级风格设置可能会起作用。然而，如果你像我一样，只是想为你的社区提供免费服务，请不要选择这个选项。我刚开始做 PyLadies 的时候就这么做了，因为有初级班的需求。我每隔一个周末只上两个小时的课。第一周我的出席人数非常多——30 个人来了，而且非常投入。第二周还是不错的——20 个人。然后开始急剧下降。到了第五周，只剩下我和我的合作组织者了。
        *   只是给一个文本教程(像学习代码的艰难方式)，没有支持。由于没有支持团体或寻求帮助的地方，当人们陷入困境时，他们可以认为自己不适合编程，然后退出。仍然有一种耻辱感，认为你必须擅长数学才能成为一名程序员，一些非技术人员认为只有天才才能编程(已经被告知我必须超级聪明，因为我是一名开发人员)。通常情况下，这只是看到一个概念有意义的正确例子的问题。仅仅因为某人学习使用一种资源有困难并不意味着他们不能学习使用另一种资源。
    2.  确实有效:
        *   关于基本编程概念的简短的一次性教程，它们互不依赖。你不必做太多，因为大多数编程都需要了解其他概念。但是你可以在不涉及大量代码的情况下教授面向对象编程的思想。也有其他的语言你可以在两个小时的时间内学会基础——SQL 是我最喜欢的，但是 HTML 也是可能的。目标是分享知识，所以要有创意！
        *   举办以初学者为中心的活动，人们可以从他们选择的任何教程中提出问题。正如我上面提到的，这是皮拉迪斯波士顿的重要组成部分。我总是推荐我最喜欢的两个教程，但如果有人通过其他方式(比如 MOOC 或视频)学得更好，那么他们可以使用这些教程，我仍然会在那里回答问题。我也试图在所有的交流中清楚地表明，我总是可以通过电子邮件联系到的。除非你有一个超过 5000 人的团队，否则这没有你想象的那么重要。我通过自己管理的团队让大约 1500 人可以联系到我，通过我的网站让更多的人可以联系到我，但我可能一周最多收到一封电子邮件。它会在人们需要的时候给他们一根救命稻草，但不会占用你太多的时间。

以上是我对如何建立新手社区的建议。无论你如何参与，成为一个欢迎每个人学习的空间的一部分是一个有价值和有益的经历，它真的可以给刚起步的人带来变化。
