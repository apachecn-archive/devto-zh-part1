# 为 12/3 事件制作 Diff 的“工作马拉松”总结

> 原文:[https://dev . to/saronyitbarek/make-a-diffs-work athon-recap-for-123-event](https://dev.to/saronyitbarek/make-a-diffs-workathon-recap-for-123-event)

这里是我们第一次 Make Diff work athon 的回顾，这是一个全天的活动，程序员、设计师和技术人员在三个特色的社会公益项目中工作。

### 主动项目回购

**回购** [市政科技项目](https://github.com/make-a-diff/civic-tech-projects)

**Stats** 27 次提交，2 个分支(1 个用于托管 gh-pages)，6 个问题已处理

**完成**在工作结束时，他们有了一个完整的项目工作流程设置(为规模化而构建)、项目提交/评审的初始流程、带有虚拟文本的项目卡和托管的 gh-page

**参与进来**后端工作流程仍然需要正规化。应该使用标准化的 civic.json 格式从各种来源(API & static)提取数据。更多信息，请参考我们在 Github 上的更详细的问题。

### 代号军团

**回购** [代码军团](https://github.com/code-corps)

**Stats** 我们从 4 个贡献者的工作中打开了 8 个 PRs。

**完成**每个人都专注于应用程序中未被记录的一个领域:认证。现在，我们有机会围绕这一点改进开发流程。因此，该团队进行了大量的入职培训，并针对两次回购中的一些突出问题公开了 pr。

**参与其中**加入懈怠！对于任何有兴趣参与其中的人来说，这是一个完美的切入点:【http://codecorps.slack.com。这里先睹为快我们的路线图:条纹管理账户，用户界面重新设计，组织入职流程，当然还有代码重构。我们的近期目标是在假期后将该服务的其余部分上线，这样我们就可以开始使用该平台来管理其余的构建。如果你想对一个项目产生影响，这个项目将很快为很多人提供重要的实用工具，现在是参与的好时机。

如果你已经开始了 Code Corps 项目，我们想让它尽可能容易地开始，但是我们需要从你那里知道什么是有效的。现在，我们正在尝试不同的东西，看看什么能坚持下去。请花两分钟时间给我们您的反馈[这里](https://codecorps.typeform.com/to/HPe83D)

### 假新闻检测器

**回购** [事实核查网站](https://github.com/EricSchles/fact_checker_website)

**完成**完成了网站的设计，开发并整理了一些卡片，建立了一个 AWS 实例，将一个 repos 归档，定义了内容策略，定义了工具的长期策略，并开始构建网站的后端和前端。

**参与其中**有一大堆事情要做，所以你可以从检查问题开始，着手做你感兴趣的事情。加入我们在[制作差异松弛](http://make-a-diff.herokuapp.com/)的松弛频道！
