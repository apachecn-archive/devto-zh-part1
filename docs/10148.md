# Bloc 编码训练营:前 4 个月

> 原文：<https://dev.to/kaseycodes/bloc-coding-bootcamp-the-first-4-months>

今天正好是我在 Bloc 开始兼职 web 开发人员的四个月，那么前四个月是什么样的呢？

我在开始日期前一周左右注册了，我利用这段时间做了一些免费的编码教程，以确保我没有发疯，这是我真正想做的。

训练营开始后，我选择了一位导师，并熟悉了他们的设置。第一周我接触了 HTML、CSS、命令行、Git 和 Github。我在较短的 27 周轨道上，但放慢到 57 周轨道，这样我就有时间吸收信息并保持理智，因为我也在全职工作。

从那以后，就像坐过山车一样，被卡住了，不知道怎么回事，感觉就像“我搞砸了，我到底在想什么”到那个特殊的时刻，点击刷新，看到我的工作完成了，然后感觉就像“我是宇宙的主人！！！！".这个过程令人惊讶地上瘾。

随着我获得了更多的经验，我变得更有耐心，接受我停滞不前的地方，花时间后退一步以不同的方式看待它，或者完全离开一段时间，以便我可以带着一个新的想法回来。

对我来说，最难的部分之一是让我的头脑接受在互联网上查找答案是正确的方式的想法，很难放弃这是“作弊”的想法。我不需要记住每一点代码，并从记忆中回忆所有代码。我可以利用现有的信息，并相信随着经验的积累，我会回忆起更多的信息，但是事物更新和变化的速度是如此之快，我总是要查阅资料。

另一个困扰是我还不知道当我完成训练营的时候我到底想做什么。作为 web 开发的新手，我不确定我是想关注前端还是后端。训练营的设计方式是，你会接触到这两者，然后在课程的最后三分之一时间里专攻这两者，所以我知道我会一路摸索出来。

迄今为止，训练营最棒的部分之一就是和导师一起工作。我无法想象在没有导师的情况下经历这个过程，当我花了几个小时试图弄清楚一些事情或者质疑这个过程最终是否真的让我做好了工作准备时，有人可以求助。他在编码方面的经验，尤其是在指导其他学生参加训练营方面的经验，是一个非常宝贵的资源。

到目前为止，在最初的四个月里，我已经了解了命令行、Git、GitHub、HTML、CSS、JavaScript、jQuery 以及最近的 AngularJS。我对命令行、Git、GitHub、JavaScript 感觉很舒服，我喜欢 AngularJS。我正在进入一个没有指导的有角度的项目，比我迄今为止所做的有指导的模块更能自我维持。之后，我将开始模块 3:后端基础。
