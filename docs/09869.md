# 你不是一个真正的开发者

> 原文:[https://dev.to/arvesolland/you-are-not-a-real-developer](https://dev.to/arvesolland/you-are-not-a-real-developer)

因此，它再次浮出水面，地下暴民的丑陋的头，觉得他们应该标签谁是开发商，谁不是。

这些说法不时出现，来自不同的个人和团体，因为他们试图以某种方式维护他们的“社区权威”。一些人试图宣称自己是纯粹主义者、思想领袖或领导者，他们本质上是说，如果其他人不做他们做的事情，或者不听从他们的建议，他们就不是真正的开发人员。

真的……那是你必须去的深度吗，为了得到一些关注？真的很难过。

我想更糟糕的是，这些人实际上认为这是正确的。他们基于什么是开发人员的通用定义，来决定谁是谁不是。

在我看来，那些抨击他人工作方法、试图否定他人的人不应该被视为思想领袖，也不应该被听从任何建议。

当然，每个人都可以自由表达自己的观点，分享自己的方法并为之辩护——这是我们的技术社区变得伟大的原因——但不要仅仅因为你的专业领域中的其他人使用的工具或方法与你不同，就否定他们。

前端开发人员、后端开发人员、Web 开发人员、移动开发人员或者仅仅是开发人员。重点是…你用代码做东西。你用你自己的方式创造它们——这很好——这就是创造新想法和多样性的方式。

永远不要因为别人不喜欢某个工具或框架，就觉得有义务去跟随和接近对你来说没有意义的东西，或者不去使用某个工具或框架来简化你的任务。务实一点，使用任何能让你的任务变得简单的方法或工具。

做你自己，做一个开发者，这很好，很棒——你会喜欢的:)
