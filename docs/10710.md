# 单页面应用的好处

实时信息，部分页面加载，快速导航到页面。Javascript 有很多优点，并且越来越受欢迎。网站不再只是简单的 Javascript 和 jQuery。越来越多的 Javascript 框架和库被开发出来，并迅速取代了传统 web 开发技术的角色。LAMP (Linux、Apache、MySQL 和 PHP)栈正在慢慢被更快、更灵活的开发方式所取代，比如 MEAN (MongoDB、ExpressJS、AngularJS 和 NodeJS)栈。Javascript 允许更快地浏览网站和应用程序，甚至允许开发者开发手机应用程序。

### 更流畅的用户交互

速度和灵活性固然很好，但这如何应用于现实世界的解决方案呢？首先，单页面应用程序(SPA)邀请用户进行更多的交互体验。因为 SPA 在初始加载过程中加载所有数据，所以在页面间导航时加载时间会更短。这种行为非常类似于加载本地移动应用程序的过程。对用户来说，这个应用程序看起来更流畅，不像一个典型的网站，你必须等到下一个页面被加载。典型的网站感觉不是动态的，感觉是一堆静态的页面，通过这些页面你可以点击。原生应用感觉更像是层中的层的堆叠。这些层可以改变并响应用户输入。这是一个典型的网站永远无法顺利完成的事情。然而，SPA 正试图在 web 环境中复制本地应用程序的这种动态感觉。通过异步调用和响应 Javascript，页面加载速度更快，能够更好地响应用户输入，从而改善整个应用程序的用户体验。

### 降低服务器负载

其次，单页面应用程序通常占用较少的带宽和服务器的计算能力。这是因为一个非常简单的原因，服务器不需要不断地为整个网页服务。相反，它提供部分页面并异步加载数据，减轻了 I/O 或 CPU 的压力。典型的网站同步工作，意味着一个任务在另一个开始处理之前完成。Javascript 允许异步调用，这意味着服务器可以对任务进行排队。然后，它将为每个 CPU 线程完成队列中的一个又一个任务。这意味着它将能够同时执行多项任务，使单页应用程序在执行相同任务时胜过任何典型的 web 应用程序。

### 可转换为混合应用

最后，本文强调的第三个好处是可兑换性。越来越多的公司推出 iPhone、Android 等应用程序。这些天。通常，这些应用程序都是从零开始，由 iOS 和 Android 开发人员构建的。这是一个非常昂贵的过程，单个应用程序通常要花费 50.000 多英镑。如果有一种方法可以将您现有的网站转换成移动应用程序，而无需进行大量额外的开发，会怎么样？对于用 Javascript 编写的单页面应用程序，这是可能的。有无数的程序可以帮助你将一个简单的网站转换成一个混合应用程序，例如 PhoneGap。这个程序本质上是围绕你的网站构建一个外壳，允许它像手机上的移动应用程序一样运行。基于 Javascript 构建的单页面应用程序可以很容易地转换成这样的应用程序，只要应用程序可以访问底层 API 端点。当然，这永远不会像实际的原生移动应用程序那样流畅，但它提供了一种测试移动应用程序的快速而简单的方法。

这些只是单页应用程序的众多优点中的一部分。但是请记住，利益总是伴随着交易。构建单页面应用程序也有缺点，这些将在下一篇博文中重点介绍。本文中的重点是我在去年构建几个单页面应用程序时遇到的一些主要部分。我相信更多的优点和缺点会出现，但只有时间会证明。
