# 关于技术面试

> 原文：<https://dev.to/mohamed3on/on-technical-interviews>

### 背景

我是一名即将毕业的 CS 学生。我花了一个月的时间试图在软件行业找到我的第一份工作。我面试了几家公司，但我(你猜对了)被拒绝了。我是来说面试流程的。

### 流程

大多数公司都有类似的面试程序。你首先要通过一个在线测试(在 Codility、HackerRank 或类似的平台上)。如果你通过了，你将会有另一次技术面试，你可能需要在白板上写代码。

### 为什么我觉得它有缺陷

你被问到的问题通常围绕着实现数据结构、数学和算法。问题是，在工作中，您几乎从来不需要从零开始实现数据结构，而且您也不应该这样做。

您通常使用语言或标准库中包含的数据结构。您使用它们以及其他工具、包和框架来实现您的业务逻辑。作为一名软件开发人员，你大部分时间都在将不同库和包中的代码粘在一起，以构建你的应用程序。你永远不需要完全靠自己从零开始构建一些抽象的东西，比如堆或者二叉树。没有 IDE 或者谷歌的帮助，你永远也不需要编写代码。但不管怎样，这就是你被评估的依据。

低级基础的理论知识是伟大的，但是没有它你也能成为伟大的开发者。

你在申请的工作中所做的(尤其是初级工作)和你在面试中被要求解决的抽象问题之间有着巨大的差距。如果你遇到复杂的事情，通常你会在谷歌上查找，或者咨询你的团队。在没有互联网连接、没有外界帮助、没有 ide 的情况下，你不必独自面对它。

那么，为什么你要经历所有这些步骤来证明你只是一个优秀的入门级开发人员呢？

### 方案

我知道很难找到好的候选人。我只是说目前的系统明显有缺陷。一种解决方案是根据你的代码库的一部分给候选人一个任务，在这个任务中，他们试图理解你的代码，以及如何在此基础上构建并编写代码来解决问题，同时不破坏现有的代码，这在实际工作中经常发生。这样你会对候选人以及他们如何融入团队有更多实际的了解。
