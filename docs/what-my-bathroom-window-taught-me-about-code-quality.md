# 关于代码质量，我的浴室窗户教会了我什么

> 原文：<https://dev.to/atyborska93/what-my-bathroom-window-taught-me-about-code-quality>

*这篇文章最初发表在[我的博客](http://angelika.me/2017/08/06/what-my-bathroom-window-taught-me-about-code-quality/)上。*

当我的男朋友搬进我们现在的公寓时，他意识到浴室里有一个小缺陷——只有窗户的底部是由毛玻璃制成的。上半部分非常透明，可以将浴室里发生的一切暴露给楼上的邻居。

他需要遮住窗户来保持一些隐私。

遮盖窗户的正确方法是窗帘、百叶窗、卷帘或磨砂窗膜。前三种的优点是有两种状态，窗户可以被盖住或打开，而不需要拆卸东西，但它们也阻挡了阳光。最后一个一直遮住窗户，但让阳光进来。

所有这些都需要计划和资金，不可能在一个晚上完成，尤其是当商店已经关门的时候。

那天晚上，我男朋友需要在那个浴室洗澡。

他的解决办法是在窗户上贴一块纸板。

将近一年后，那扇窗户仍然被同样的纸板覆盖着。为什么？因为这是性价比最高的解决方案。

*   它又快又便宜。所需的材料，胶带和纸板，已经在公寓里了。
*   它是有效的。它完全阻止了邻居看到他们不该看到的东西。
*   纸板阻挡阳光不是问题。浴室已经从底部的窗户得到了充足的阳光。
*   如果不撕掉纸板，窗户就不容易被盖住和打开，这不是问题。窗户太高，没有梯子够不着。
*   纸板丑不是问题。我们真的不在乎。
*   没有年轻易受影响的室内设计师来到我们的公寓，看着那个窗户，认为粘在窗户上的纸板是一个好的设计图案。

## 如此...有什么意义？

这只是一个愚蠢的例子，但它让我思考我如何对待自己编写的代码的质量以及我对他人的期望。

我患有完美主义。我的同事抱怨我对他们(也包括我自己)要求太高。有时候，我发现自己在代码审查期间真的很固执(真诚地向所有受到影响的人道歉)。如果发生这种情况，我会试着停下来问自己——这是什么代码？

代码的价值是它给用户带来的可用性，而不是它的干净或美观。

代码的代价是开发它所花费的时间和精力。

通常，糟糕的代码是昂贵的。糟糕的代码很难理解。糟糕的代码很难调试。糟糕的代码很难扩展。由于所有这些原因，糟糕的代码也降低了将来可以交付的价值。

但是有时候，只是有时候，你知道你写的代码永远不会被再次阅读，也永远不会被修改。你知道它会被使用，也许只是一次或两次，然后它会被扔掉。这是坏代码便宜的时候。

我现在明白了，一个好的软件开发人员并不总是交付好的、干净的代码，而是能够评估情况并交付价值与(**长期**)成本比率最高的解决方案的人。即使这意味着偶尔写一个快速肮脏的黑客。

哦，但是要确保没有年轻易受影响的初级开发人员会来到你的代码库，看着那些糟糕的代码，认为这是一个好的设计模式，因为像你这样受人尊敬的开发人员写了它。或许在评论里留下一句警告？
