我想写一篇关于 Dev.to 的文章。我的兴趣广泛，经验丰富，但是我从来都不是一个“真正的”IT 人员。做一点 Python，一点 Rails。但是，我做了几件事，希望与大家分享，并希望得到反馈，主要是围绕研究实验室的中等规模存储系统的开发和管理。可以想象，管理每月数十 TB 的数据流量并不容易，但我认为我们已经掌握了核心原则。

然而，我不确定自己能否写下这段旅程。所以我想找一个合著者或者编辑，至少是一个“橡皮鸭”来激发我最初的想法。或者可能有人会问我好问题，关注我，也许会给我一个很好的建议。

所以这个帖子给像我这样的人提供了一个在写作开发上寻找合作的机会。对你的提议进行评论(“我想写关于 X 的文章”或“我想帮助写关于 X 的文章”)，也许会有知识渊博的人认同你的想法。

我认为这可能有用的几个原因是:

*   你会写得更好
*   你会写得更快
*   你会学到新的东西
*   也许写这样一篇文章将是一个辅导/教学练习
*   也许你会找到朋友
