## **简介**

“重要的不是你做什么，而是你怎么做。”这种对生活的简单观察肯定可以应用于试图改变现状并涉及来自企业不同部分的许多利益相关者的 IT 计划和项目。以业务为中心的 IT 转型计划就是一个很好的例子。这些旨在随着业务的变化和整个生命周期规划和监控 it 支持的绩效，确保业务战略和需求得到理解、优先考虑和执行，并确保在决策过程中了解和考虑 IT 支持的成本、质量和风险。他们是负责 IT 转型的规划师、战略家、架构师、投资组合经理和 CIO 办公室团队的推动者，使他们能够将业务愿景转化为成功的项目和可靠的运营。

支持业务转型的 IT 转型计划是一个突破性的计划，它将引入新的 IT 流程，改变或淘汰现有流程，定义新的角色和职责，并涉及广泛而深远的利益相关方。在 IT 转型计划的某个成熟阶段，有必要使用软件来管理项目的复杂性。在“购买还是构建”问题中，许多人会决定寻找一个标准软件解决方案，在企业环境中，“通常”会提供以下优势:

*   更快获得预期结果
*   降低开发和拥有的总成本
*   更好的软件质量
*   具有类似项目经验的顾问
*   可持续部署

然而，如果标准软件不能被配置来满足公司的需求，它就可能成为一种束缚。在 IT 转型中，只有高度可配置的解决方案会或应该受到质疑，原因有几个:

**以业务为中心的 IT 转型是一项涉及许多利益相关方的协作活动**。果断而敏捷的 IT 规划和管理需要关于广泛问题的最新准确信息。这需要类型多样、数量众多的许多利益攸关方做出贡献。仅靠软件无法自动收集数据，如果只让 it 规划人员收集，信息将很快过时。业务分析师、应用程序所有者、项目经理、解决方案架构师、企业架构师、IT 人员以及涉及 IT 风险和 IT 财务的人员都拥有一些对 IT 规划和管理决策非常重要的信息。用于收集这些信息的软件必须在外观和感觉上吸引每一个利益相关者，信息与手头任务的相关性，并为执行任务提供清晰的程序。

**以业务为中心的 IT 转型需要高度的数据完整性**。由于上述原因，数据质量很重要。实施 IT 转型解决方案的一个重要部分是能够控制谁可以访问哪些软件功能以及谁可以对数据进行更改。与此相关的是安全问题。因为面向业务的 IT 转型涉及记录业务策略、目标和能力，以及管理项目组合，所以系统中的许多信息可以被认为是敏感信息，不是每个人都应该看到。用户配置文件、角色和权限的配置控制着这种访问。不仅大型企业中的 IT 转型涉及大量用户，而且特别是职责的不断变化和信息环境的不断变化，都需要在 IT 转型解决方案中灵活定义角色。大型企业中的大量用户也导致了对信息用途和维护方式的大量解释。数据完整性规则和约定是 IT 转型解决方案的重要组成部分，也需要进行配置，以适应各个企业的组织策略和定义。

**以业务为中心的 IT 转型管理着 IT 规划和投资流程**。尤其是在一个不断变化和不确定的环境中，决策过程既不能含糊也不能缓慢。这需要标准流程来保证一致性和效率。创建自动化工作流以将一致且高效的流程锚定到企业中的能力非常重要。自动化工作流确保与策略和时间表保持一致，以维护治理并使 IT 跟上需求和产品组合流程。工作流的可配置性对于能够反映单个企业的过程和组织模型也很重要。

审计人员对以业务为中心的 IT 转型很感兴趣。因为它涉及关键的决策制定流程，并且汇集了如此多的信息以进行有效的决策制定，所以审计人员可以在 it 转型解决方案中使用这些信息来回答许多问题。例如:哪些应用程序符合 SOX 标准？哪些数据与 GDPR 相关？是否建立了项目风险管理流程并得到遵守？为审计人员配置特殊报告的能力使他们更容易理解 it 转型信息。扩展元模型以包含与审计员相关的数据的能力可以确保他们获得所需的特定信息。
