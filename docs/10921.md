所以你是一名程序员，想接触一些人工智能(AI)和机器学习(ML)？我接受了 TechWorld 的采访，就开发人员的机器学习建议进行了采访，但最终的书面报告没有包括大部分材料，所以下面是我的完整答案:

### [](#1-how-should-developers-get-started-on-this-topic-what-do-you-recommend-as-a-first-step)1。开发者应该如何开始这个话题——你建议第一步做什么？

学习。当我开始学习人工智能和人工智能时，我读了彼得·诺维格的《人工智能:一种现代方法？，而且我现在大概已经看了三遍了。重要的是从根本上理解你将使用的算法，否则每一个 ML 方法都是一个黑盒，你能做的最好的事情就是盲目地把它们扔向问题。然后尝试做 ML 项目，比如 Kaggle 竞赛，在那里你对你要学的东西有一个清晰的定义——比如一个特定的算法或工具包。你经常会失败，但这就是学习 ML 技术的最佳方式。

### [](#2-can-you-talk-me-through-the-process-of-setting-up-a-machine-learning-framework)2。你能告诉我建立一个机器学习框架的过程吗？

第一步，也是最重要的一步，是**理解你试图解决的问题**。什么是输入和期望的输出？数据是什么？在实验算法之前，彻底理解数据是至关重要的。否则，你对结果的质量或可靠性没有实际的信心。您很可能还需要对数据进行预处理和转换，并可能获得更多的数据。ML 过程的下一步是考虑你所拥有的算法和方法。对于适合你的问题和数据的算法，有很多标准需要考虑，比如可靠性、效率、可扩展性等等。请记住，任何给定的方法(例如深度学习)都不是一把锤子可以钉上每一颗钉子。我经常看到工程师对他们的结果不满意，因为他们在没有首先了解真正问题的情况下，就立即向他们的数据扔深度学习模型。有了 ML 算法，下一步就是运行、研究和改进。使用 k-fold 交叉验证和统计显著性测试等技术评估算法的结果。现在，您可以清除那些善于发现和使用数据中的结构的算法，以及那些正在解决问题的算法。使用这些算法进行更集中的实验将有助于您微调和改进结果。现在，有了性能最好的算法，你可以尝试集成方法或更多的特征工程来获得最佳结果。
那是很多，所以总的来说:

1.  定义问题
2.  分析和准备数据
3.  选择算法(从简单开始)
4.  运行并评估算法
5.  通过集中实验改善结果
6.  通过微调最终确定结果我应该提到，如果您打算共享或部署这个 ML 框架，**记录过程中的每一步**。这对于你和其他工程师来说会很有用。### 3.你能告诉我建立一个机器学习工具包的过程吗

NuPIC 是运行我们的技术——分级临时内存(HTM)的开源工具包。NuPIC 为开发人员提供了一个独特的机会，让他们可以亲自动手操作最先进的机器智能。对我们来说，有两个主要举措:建立和支持一个强大的社区，并纳入我们的尖端机器智能研究。我们的开源旗手马特·泰勒在前者方面做得非常出色，我可以对后者说更多。在研究中，我们在神经科学理论、数学推导和算法实验方面做了大量工作。大多数算法从未超越白板，因为它们不符合我们关于大脑皮层如何工作的框架。为了开发使用的算法，有大量的原型和测试。在 NuPIC 中发布一个修改的或新的算法之前，我们包含了大量的文档。我们还与 NuPIC 社区公开讨论算法，并评价他们对理论和代码的贡献。

### [](#4-which-are-the-best-machine-learning-tools-or-platforms-for-devs)4。对于开发者来说，最好的机器学习工具或平台有哪些？

这实际上取决于你试图解决什么问题，特别是你正在处理的数据。也就是说，开源至高无上。有许多高质量的开源机器学习工具包可供开发人员利用，社区也很活跃，很有帮助。在 Python 世界中，我非常喜欢 Scikit-learn，因为它提供了广泛的机器学习和数据分析工具。出于同样的原因，我喜欢面向 C++开发人员的 mlpack。NLTK 是自然语言处理(NLP)方法和数据的 goto，尽管深度学习包实现了高质量的 NLP 算法。如果你想做流分析，NuPIC 是最好的；它是最先进的人工智能，拥有令人难以置信的知识库和社区，并且用 Python、C++、Java 和 Flink 实现。特别是对于深度学习，我们看到了很多好的、快速的框架:看看 Theano、Neon 或 Google 的 TensorFlow。Java 方面，我强烈推荐 deeplearning4j 它是深度学习加上更多的机器学习工具，由一些非常聪明的工程师编写。

### [](#5-are-there-any-languages-that-are-best-for-them-to-focus-on-for-ml)5。有没有最适合他们专注于 ML 的语言？

[Python](http://www.kdnuggets.com/images/cartoon-recommendation-python-machine-learning.jpg) ，毫无疑问。大多数机器学习和数据分析工具包都是用 Python 编写的，它有一个巨大的、有用的社区。像 HTM 和深度学习这样的高级方法通常是用 Python 实现的。

### [](#6-are-there-any-devsexamples-of-work-that-youd-recommend-looking-at-for-inspiration-in-this-area)6。在这方面，你有没有推荐一些开发/工作实例来寻找灵感？

对于最需要在 ML 中进行更多开发的流分析，我建议查看 [Numenta 异常基准(NAB)](https://github.com/numenta/NAB) 。我们展示了真实世界的数据流，测试了几种最先进的异常检测算法，以及传统的数据分析技术。这一切都是开源的，ML 工程师和开发人员很容易参与进来。对于其他的人工智能项目，我跟踪该领域的许多研究人员，例如在[的谷歌大脑](http://research.google.com/pubs/MachineIntelligence.html)和[的脸书人工智能](https://research.facebook.com/ai/)的团队；他们已经在 NLP 中做了一些伟大的工作。总的来说，我喜欢浏览 ShowHackerNews 网站，寻找其他开发者分享的新项目。

### [](#7-whats-it-like-doing-machine-learning-research)7。做机器学习研究是什么感觉？

大多数 ML 研究都是调整现有的算法，或者开发 ML 管道来大规模部署。**机器智能(MI)是一种不同的动物**。MI 呼吁进行基础研究，因为智能背后的确切属性和算法仍然未知。为了构建 MI，我们正在努力发现智能的基本属性，将它们抽象到一个计算框架中，并在机器学习应用程序的软件中实现这些算法。

### [](#8-hows-machine-learning-different-from-machine-intelligence)8。机器学习和机器智能有什么不同？

ML 正在构建软件系统，旨在随着经验的增加而改进。ML 算法执行特定的、狭窄的任务，比如玩特定的游戏。深度学习或聚类算法就是一个例子。MI 是智能的软件体现:不是执行特定任务的能力，而是通过感觉-运动互动发现世界结构的能力，然后使用这些知识来实现目标。也许 MI 的更清晰的定义是通过它的要求:机器必须从未标记的数据流中连续学习，同时进行预测，检测异常，并进行分类；即人脑的功能。HTM 是机器智能。

### [](#9-whats-in-store-for-numenta-whats-in-the-future)9。Numenta 的未来是什么？未来会怎样？

尽管我们取得了很大进步，但仍有工作要做。我们正在构建核心理论，并在实验中测试算法。给大脑皮层的算法编程不是一件容易的事情。我们不能确定它的确切发布日期，但我们相信 HTM 将对神经科学和人工智能领域做出重大贡献。
