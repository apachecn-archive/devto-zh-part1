# 写代码的 100 天

> 原文:[https://dev.to/ben/100-days-of-writing-about-code](https://dev.to/ben/100-days-of-writing-about-code)

这是接受个人挑战的季节。阅读 [#100DaysOfCode](https://medium.freecodecamp.com/start-2017-with-the-100daysofcode-improved-and-updated-18ce604b237b#.ulzr6kc9r) 挑战赛让我感受到了这种精神。这挑战了一个简单的个人目标:坐下来连续编码 100 天。对于新程序员来说，这是一项美妙的事业，他们需要汲取经验，可能会感觉到生活中的其他事情在拖他们的后腿。

对于稍微有经验的程序员来说，也许已经为工作编写了很多代码，这可能不是我们所需要的。就我个人而言，吃饭、睡觉、编码、重复的心态对我的心理健康没有太大的好处。然而，本着 100 天代码挑战的精神，我将参加**关于代码挑战的 100 天写作**。

[![2017](../Images/c8f7a114ac82a7cce27463f8ed26e3d0.png)T2】](https://i.giphy.com/media/3o6UB9IxAjNdyIgUKY/giphy.gif)

许多人[提到](https://dev.to/ben/what-are-your-programming-goals-for-2017)写博客和更多地参与社区是他们 2017 年的目标，我也有类似的抱负。我想达到一种节奏，在这种节奏下，我能够更有规律地发表我对代码的想法。我经常陷入困惑，不知道一个想法是否值得分享。

所以我要去做。(几乎)从 1 月 3 日开始的每一天，我都要写*一些关于代码的*东西，即使它很短而且毫无用处。我有很多事情要做，也有很多事情要贡献。分享我的想法让我感觉很棒，每次分享都从反馈中学到很多。我真的很喜欢写关于 web 性能的文章，并且我认为在这方面我可以提供很多东西。我还会写一些我尝试的新语言和工具，以及全新的尝试，比如游戏开发，我计划今年兼职涉足。

我将把它全部发布到我的 dev.to/ben 的博客上，并把它变成一种练习。我会在我的个人推特上分享这一切，[@ bendhalpen](https://twitter.com/bendhalpern)。一些不错的作品可能质量足以出现在 [dev.to](https://dev.to) 主页上，甚至更少会出现在 [@ThePracticalDev](https://twitter.com/thepracticaldev) 上。

我将在这个网站和 Twitter 上采用 [#100daysofcode](https://dev.to/t/100daysofcode) 标签，但我个人的挑战是关于写作部分。如果你正在阅读这篇文章，并且和我有同样的感受，我鼓励你参与进来！非常欢迎你和我一起发表到[发展到](https://dev.to)。这将使我更容易跟上你的努力并为你加油，但是你也可以发表到你自己的博客上(或者两者都发表！)如果你让我知道，我会努力跟上。

2017 年见！✊
