# 首先，解决问题，然后编写代码

> 原文:[https://dev . to/shubham 9411/first-solve-problem-then-write-code-4g6l](https://dev.to/shubham9411/first-solve-problem-then-write-code-4g6l)

在我开发旅程的开始，我曾经编写 HTML 和 CSS 代码，并创建一些看起来很棒的静态页面(如果你不与其他在线网站相比的话)。我过去常常在没有应用任何风格的情况下首先在划分和架构上进行实验，所以我只是简单地向它添加类和 id，这就像创建一个没有任何风格的框架，只是一个普通的框架。

这样我就可以通过在我分配给它的类中改变或添加颜色来创建尽可能多的设计。这样我就可以通过改变 CSS 文件来改变页面。在我大学的“Kaivalya”科技节上，我完成了我的第一个 web 项目。任务只是为大学计算机科学俱乐部做一个网站的前端。做好前端后，你必须为它做一个后端。

当我第一次看到这个任务时，我在想我该如何开始建立网站。所以我搜索了很多网站，选择了一个设计非常简单的网站。那时我不知道任何软件原理和设计方面。我会浏览网站，如果我喜欢某个功能，我会把它添加到我的网站上。

没有适当的规划和设计，我创建了这个网站，因为它是一个小项目，如果有任何问题发生，我也可以从头开始重新创建它，但如果我要从事更大的项目，并使用这种类型的方法，我用来创建我的网站，然后会发生什么。然后，网站是不可管理的一个例子，当我第一次在 WordPress 主题创建工作时，我在侧边栏中遇到了问题，每次我改变一点主要内容的 CSS，侧边栏就会爆炸。

这是因为我没有确定主内容和侧边栏内容的划分宽度，但我首先创建了一个图表，并定义了屏幕的多少部分将被不同的容器覆盖。这个主题还有很多其他的缺陷，但这仅仅是我的 WordPress 之旅的开始。

设计是计划的创造，是问题的解决方案，就像一个人如何从他的角度看待问题，每个人都有不同的解决方案。我们只需要全方位思考。我正在读一本名为 [Code Complete](http://www.amazon.in/Code-Complete-Steve-Mcconnell/dp/9350041243) 的书，书中有一个例子，题目是“设计是一个棘手的问题”，是关于一座名为“[塔科马海峡大桥](https://en.wikipedia.org/wiki/Tacoma_Narrows_Bridge_(1940))”的桥梁的设计。

在建造这座桥的时候，工程师们主要关注的是赋予它足够的强度来支撑荷载。但是由于风的[气动弹性颤动](https://en.wikipedia.org/wiki/Aeroelasticity#Flutter)和桥梁没有准备好处理速度 68 公里/小时的空气，所以它倒塌了。当这一切发生时，附近的一名摄影师记录下了它的镜头。这段视频仍然被展示给工程、建筑和物理专业的学生作为警示。

但是这个故事的寓意是什么呢？在我看来，寓意是从各个方向看问题，并产生自己的测试案例，就好像他们知道空气动力学一样，他们将做出一个设计，使桥梁能够承受高速空气。这就像试错法，他们创造了一座桥，仅仅在 4 个月内就倒塌了，所以他们发现了桥的空气动力学，并为下一座桥创造了新的设计，这样它就不会在未来发生。

我们必须从以前的错误中吸取教训，必须尝试新事物。原因设计不是一个算法过程，它本质上是启发式的。我们三个人[尼廷](https://www.linkedin.com/in/nitinkshatriya)、[迪拉杰](https://www.linkedin.com/in/dhirajkathayat)和[我](https://www.linkedin.com/in/shubham9411)在为 ColoredCow 的招聘系统开发插件时遇到了这样的问题。问题是，当我们被告知要创建一个显示可用职位空缺的插件时，我们认为它会像一个网站的小部件一样，以一种轮播的方式显示职位空缺。

因此，我们创建了一个小部件，但交付后，我们发现他们想要的不是这个小部件，他们只是想要一个短代码方法，无论他们想把传送带放在哪里，他们都必须添加短代码。这没什么大不了的，但这就像是一个例子，没有以不同的方式看待问题，或者可能没有以客户使用软件的方式看待问题。现在我们也增加了这个功能。

对于我们的大学网页开发小组，我们创建了一个任务，就是检查学生的设计技能。这更像是一个实验，测试他们如何对任务做出反应并得出答案。

任务是为一家公司创建一个具有基本功能的网站，如主页，关于网页等..每个人都想出了不同的设计，他们中的一些人添加了我们提到过的确切的页面，一些人利用他们的创造力，为网站添加了其他功能和页面。这让我想到了在“代码完成”中管理复杂性的话题，我们提到的网页是一个网站的基本要求和必要条件。但其他偶然的特征也是任何公司所要求的，他没有提到，就像上面的例子，客户可能认为我们添加了事件，并注意到了我们自己或任何东西。

这取决于人与人之间的设计和时间因素，也可能是其他因素。首先解决问题，然后开始设计阶段，这在编码阶段比没有计划或设计就直接开始编码更有帮助。正如约翰·约翰逊所说

> *首先，解决问题。然后，编写代码。*
