# 我们决定和更有经验的人一起去

> 原文:[https://dev . to/georgeoffley/we ve-decide-to-与更有经验的人同行](https://dev.to/georgeoffley/weve-decided-to-go-with-someone-with-more-experience)

我相信任何找工作的人都经历过这种情况。你完成了一天的工作，然后收到了你最近面试的公司发来的邮件。你的心跳停止了，取决于你有多想要这份工作，打开邮件的那一刻似乎会持续几天。当恐慌开始出现时，你打开电子邮件，你真正读到的唯一一行阐明了“谢谢你的兴趣，但是我们已经决定和更有经验的人一起去。”你屏住呼吸，重读邮件，完全掌握了信息的内容。然而不幸的是，你的士气已经受到了伤害。

现在这一切都没有上面描述的那么戏剧化了，但是不管你经历了多少次，胃部的小刺痛仍然存在。来自自学背景，这可能是你经常听到的。即使来自传统的四年大学背景，你也会遇到这种情况。这常常让人想起《蝙蝠侠前传》中的场景，由克里斯蒂安·贝尔饰演的布鲁斯·韦恩的豪宅被烧毁。孤独地躺在那里，精神崩溃；由伟大的克尔·凯恩扮演的阿尔弗雷德重复了托马斯·韦恩经常告诉布鲁斯的一些事情。“我们为什么会坠落？所以我们可以学会重新振作起来。”我的生活一点也不有趣或英勇，被拒绝工作的想法也是如此。然而，这个信息也是相关的，尤其是当它涉及到试图进入软件开发的时候。

不幸的是，人们通常对软件大师的看法会让人联想到天才的形象，他们从四岁起就开始编写代码，有着与愤怒的瓦肯人相似的礼仪，并且是半机械人。这些人是存在的，我并不嫉妒他们与“天才”这个绰号相关的辛勤工作。当他们达到工作年龄时，他们通常有大量的工作可以展示给潜在的雇主。对于我们其他人来说，当我们在生活中的某个时刻转向并决定进入技术领域时，我们会有不同的体验。

当你没有专业经验时，找工作时经常会被忽略。有很多方法可以让我们在下次面试中脱颖而出。在谷歌上快速搜索“如何获得软件开发经验”将会找到大量解释如何克服这一障碍的线索。作者完全赞同从搜索中收集到的建议方法。这是我已经做了一段时间的事情。最近我的个人项目一直在增加，我一直在空闲时间帮助创建网络工具。结果，我的编码技能提高了，我开始发现自己越来越适应使用新工具和新语言。

我喜欢这样做，我喜欢和有相似想法的人一起工作，一起写代码。这就是你继续前进的方式。你专注于你的项目和产品。你是你最伟大的产品，投资它正是你在个人项目中应该做的。拓展业务，尝试新的语言，尝试新的方法，建立你喜欢使用的东西，或者写作。

求职须知。我见过的关于求职的最好建议来自计算机专家伊莱。你应该申请你想要的每一份工作。这就是我一直在做的。大约每两周，我会浏览一次精选的招聘清单，寻找我可能有点资格申请的工作。对我来说，我的目标是那些我读过评论或亲自使用过他们产品的公司，并以这种方式申请工作。我还将搜索范围扩大到我所居住的国家不同地区的不同城市。简单来说，有时候你必须去有工作的地方。我不会建议任何人去尝试和申请所有的工作。当你在十个你真的不想工作的地方结束面试，并击退顽固的招聘者时。

这篇文章没什么新意。你努力工作，有时这种努力会有回报，你会进入一个你以前不属于的行业。有时候不会。那次经历是继续前进的战斗口号。它甚至不应该被称为一次失败，因为很多时候没有足够的事情让你被选中做那份工作。你不断前进，你不断学习；坚持是失败的敌人。
