# 如何制作一个节目——回答 12 岁的我的问题

> 原文:[https://dev.to/monknomo/how-to-make-a-program-回答-12 岁-mes-question-4oi4](https://dev.to/monknomo/how-to-make-a-program---answering-12-year-old-mes-question-4oi4)

当我小的时候，我喜欢电脑。我认为他们非常酷，我真的想让他们做一些事情。我一直沉迷于赛博朋克和电子游戏，所以我有了这样的想法:通过摆弄键盘和编程，你可以做任何事情。我所没有的是任何让我前进的知识或资源。

我想写一个程序，但是我不知道谁会编程，我能得到的编程书籍要么是古老而无用的(想想 COBOL 手册)，要么是崭新而无用的(想想 90 年代末的 C++门栓)。

我对如何编程有所了解。你按照特定的顺序写下特殊的单词，计算机就会做一些事情。比如 excel 宏之类的。我不知道的是如何把我的文字从文本文件转换成计算机可以运行的东西。就像字面上的机械编程过程。我不知道该怎么做。

那是 90 年代，我在阿拉斯加。不是阿拉斯加的乡村，但绝对是乡下。我有互联网，或者一些合理的传真，如果没人需要打电话的话。我阅读 [Slashdot](https://slashdot.org/) 和论坛。我可以读到很多关于编程的东西——人们是如何做的，有哪些语言，到处都有一些教程。

我读过的所有东西都缺少如何编程的基本步骤。当然，他们会告诉你如何说“你好，世界”，但在没有支持网络的情况下把自己提升到那么高是不可能的。也许是我反应慢了，或者是我想多了。看起来计算机很复杂，那么为什么创建一个程序不会很复杂呢？

让我切入正题，回答 12 岁的我的燃眉之急。以防有人也有同样的问题。

## 如何用 4 个简单的步骤制作一个程序

1.  打开文本编辑器。文本编辑器类似于 Windows 中的记事本或 OSX 中的文本编辑器。
2.  写一个程序。这一步只需从教程中复制粘贴一个 hello world 程序
3.  将此文件另存为纯文本文档
4.  这是我小时候困惑的复杂部分。根据您使用的语言类型，有两种选择
    1.  用编译器编译刚刚保存的文件。运行生成的可执行文件
    2.  用解释器运行你刚刚保存的文件

就是这样！你所要做的就是写一个文本文件，把它输入编译器或解释器，你就有了一个程序。说真的，不要想太多了

### [](#compilers-and-interpreters)编译器和解释器

我在第四步中使用了一些专业词汇。12 岁的我读了它们，但不理解它们，所以我最好解释一下。

编译器是把你写的文本文件翻译成你的计算机能理解的机器代码的东西。编译语言是像 C，Java，Go，Rust，[等等。](https://en.wikipedia.org/wiki/Compiled_language)

解释器直接从文件(有时是命令提示符)中读取程序，并动态地“解释”指令。解释语言是像 Python、Javascript、Ruby、Lisp、Perl、[等等。](https://en.wikipedia.org/wiki/Interpreted_language)

有些语言既有编译器又有解释器。
