# 想要加强你的 Android 学习游戏吗？你需要先看看这个

> 原文：<https://dev.to/nisrulz/want-to-step-up-your-android-learning-game-you-need-to-read-this-first>

> 声明:如果你认为自己是 Android 领域的初学者、中级、专家或忍者，这个秘密酱仍然适用于你。

##### TL；大卫:建造一些东西。

真的没有！我不是在开玩笑。听我说完。

[![header](img/8f07c552670fabaf581b0f018a150511.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--R4mtAJwA--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ktvo1fkx6xjy9ee8ulps.gif)

变得更擅长 Android 开发的秘密是，每年都有新的闪亮的东西发布，让你经历错过的恐惧，甚至在阅读博客帖子或文档之前，只是构建一些功能性的东西。

很容易说你阅读了文档，浏览了解释复杂概念的博客文章/由 Google 或一些第三方开发者/公司引入的新库。我并不是说这是不必要的或者是浪费时间，因为理解概念，然后继续使用新的和闪亮的库或者你刚刚知道的新概念是非常重要的。您应该始终理解底层概念，以避免误用 API/框架/架构方法。

但是在这里，我想暂停一下，提一下一种不同的方法，这种方法对我和我向其介绍这一概念的人一直有效。这个概念并不新鲜。这是接近 Android 领域新库/架构的一个简单的额外步骤，即…

##### ..构建一个简单的、孤立的、基本的 android 应用程序示例，看看你正在尝试学习的工作示例。

仅此而已。我把这种 ***例子驱动开发***【approachðÿ】。

这样做很好的原因是，这些简化的 Android 应用程序没有你自己的应用程序中的任何复杂性。如果你像我一样，喜欢在我正在开发的复杂应用程序中实现它之前，先看看事情是如何工作的，那么这种方法会让你非常感兴趣(当然，除非你已经意识到并使用了它，在这种情况下，跳到这个故事的底部，ðÿ˜…)

这些简化的 android 应用程序示例还为您提供了试验您正在尝试实现的概念/库的机会，以及在您直接在复杂的 Android 应用程序中实现之前发现可能的错误或问题的机会。

有些人可能会认为这是一个耗时的过程，最终会浪费时间和精力，因为在一天结束时，示例应用程序会被扔出窗外，而概念/库必须被实现到您正在开发的 android 应用程序中。

我强烈反对这一点，我有自己的理由来证明这一点(基于我的经验)。这可能不是对每个人都适用，但我发现这种方法对我推荐的大多数人都有效。我相信这种简化的 android 应用程序就像

*   ***单元测试*** ，简单且专注于理解概念/库的功能和工作方式。
*   ***代码*** 中的注释，它们是为了让你回来，了解概念/库如何在最基础的层面上与平台一起工作。

如果我不能展示我是如何为自己所用的，那么我提倡一种方法就没有意义，对吗？

所以我碰巧遵循这种方法已经有一段时间了，而且碰巧社区里的一些人找到我，与他们分享同样的事情。因此 [Android 示例](https://github.com/nisrulz/android-examples)成为了今天的样子…

[![android_examples](img/78176e5661368263d2b414706893d6bb.png)T2】](https://github.com/nisrulz/android-examples)

…我去年创建了这个 repo，在那里我开始放入我在工作中进行 android 开发时或只是为自己学习一些新东西时在某个功能/库/概念上构建的所有示例应用程序。事实证明，它现在已经发展得相当大，拥有 **100 多个 android 示例**(包括来自我自己构建的 [android 库的](https://github.com/nisrulz/nisrulz.github.io#open-source-contributions)[示例应用](https://github.com/nisrulz/android-examples#example-apps-from-my-android-libraries))

一定要去看看，体会一下我到目前为止一直喋喋不休的话题。 ***直接跳进去，构建一个演示你一直想理解的概念/库实现的应用程序，亲自动手编写代码！*T3】**

更好的是，用更多(缺失的)简化的 android 示例或改进 Github repo 中存在的示例来为 repo 做出贡献。任何规模的贡献总是受欢迎的。

如果你仍然认为这种方法是有缺陷的，我很乐意听听其他对你有效的方法。

如果你有建议或者想让我给这里的内容添加一些东西，请告诉我。
