# 所以你想开源一个项目，是吗？

> 原文：<https://dev.to/captainsafia/so-you-wanna-open-source-a-project-eh-5779>

由于我已经在开源生态系统中呆了一段时间，我收到了很多来自个人和公司的关于如何加入开源世界的问题。有些人对为开源项目做贡献感兴趣。关于这一点，我已经写了很多，也讲了很多，所以我将讨论参与开源的另一种形式。许多个人和公司都对将以前封闭的源代码公开化感兴趣。

我想讨论你需要采取的步骤，以便成功地建立你的新的开源项目。这篇文章并没有解决在谈论让封闭的项目开源时所解决的典型问题。我不打算谈论许可证或贡献者合同或文档——我要谈论的是人。

这些观点特别关注那些寻求开源新项目的公司。然而，这些经验可以很容易地应用于个人或小型团队，他们希望在开放的环境中构建一些东西。

**员工同理心培训**

让项目开源是测试团队健康和灵活性的好方法。一个开源项目将为你的团队在沟通、同理心和项目管理方面带来新的挑战。你需要确保你在这种环境下为他们的成功做好准备。我建议你的员工参加同理心和联盟培训，更新沟通技巧，参加开发者福音速成班。这些培训将帮助您的工程师应对额外的挑战，与来自不同背景的人一起工作，在开源的公共论坛上有效地交流他们的想法，并自豪而平易近人地与世界分享他们正在做的工作。

**为员工提供适当的心理健康资源**

对于很多人来说，从事开源工作或帮助管理开源项目是有压力的。由于上面提到的新挑战，你的员工可能会努力管理他们所处的新环境，他们的心理健康可能会因此受到影响。在你的公司将某段代码开源之前，一定要问自己“我们准备好为我们的员工提供他们在开源中取得成功所需的精神支持了吗？”如果答案是否定的，那就开始在你的公司发展精神健康基础设施。确保你的健康保险计划涵盖心理治疗，确保你为员工提供充足的休假时间，确保你的经理了解并准备好应对可能出现的新的管理挑战。

**做好准备，把社区看得比自己的员工更重要**

作为一家公司，关心和重视你为生产产品和改变行业而雇佣的员工是很重要的。随着你的项目越来越受欢迎，参与度越来越高，你可能会发现你的一个员工在项目中是一个敌对的存在。他们可能会贬低提出问题的个人，他们可能不会对没有将他们的变更包含在项目中做出很好的反应，他们可能会采取精英主义的行为，贬低经验较少的开发人员或不是您公司员工的开发人员。如果这些事情发生了，重要的是你要把这个人从项目(和你的生意)中移除。不管他们给你的公司带来多少价值，你都需要认识到与这个人相关的负面外部性，并减少他们可能对你的业务和项目产生的有害影响。

在最好的情况下(这是最糟糕的情况)，这个人会阻碍你公司的营销、销售和招聘工作。在最坏的情况下，这个人可能会让你陷入一些严重的法律麻烦。

**建立适当的管理结构**

管理开源项目时，您可以访问公司内部可用的技术和非技术资源，以及为您的项目做出贡献的每个人。当您开始开源项目时，确保您的管理过程考虑到项目周围社区的贡献和能力是很重要的。不要害怕让那些不受雇于你的公司的贡献者参与到项目的关键方面——特别是如果他们是经常的贡献者。开源是开放的，你的管理团队将需要围绕资源分配和问题委托培养一种开放的思想。

我能给你的就这些了。总之，随着您的公司转向更加关注开源，您将需要确保您的团队准备好接受开源的精神——并管理可能随之而来的包袱。
