# 如何很好地离开一家公司

> 原文:[https://dev.to/seankilleen/how-to-leave-a-company-well-2lok](https://dev.to/seankilleen/how-to-leave-a-company-well-2lok)

最近，我和一家公司的老板共进午餐，我已经离开了这家公司。除了作为一个一般的赶上，它作为一种检查/反思点建设性的批评，从我们在一起的时间。

在我离开之前(这是由于其他情况)，我(我认为)与老板的关系很好，但他在午餐期间说的一件事让我非常高兴:

“你像绅士一样离开了。”

这是我 100%的目标——我认为这应该是每个人离开公司时的目标(假设这段关系没有破裂)。这让我反思我离开公司时做了什么，希望它能成为我和其他人前进的样板。

## [](#prior-to-transitioning)过渡前

*   了解是否有还价可以让你留下来。更多的钱会让你留下来吗？头衔提升？即使你不需要这些东西，也要为它们的出现做好准备，提前了解它们是否会影响你的决定。本文的其余部分假设您已经决定在这次谈话的结果出来后进行转换。
*   **创建过渡区域/文件**。下面会有更多的介绍，但是如果你已经向管理层表明你致力于平稳过渡，这将会建立很多良好的信誉。
*   提前两周通知:只要有可能，作为一种尊重的表示，我喜欢在离开一家公司时提前两周通知。你的新雇主的需求当然是第一位的，但是提供更多的过渡时间有助于你作为一名专业人士有一个坚实的基础。

## [](#the-transition)过渡

问题的核心。

### [](#the-initial-conversation-is-important)最初的谈话很重要。

这可以决定你离开的整个基调。

*   **保持积极**。人们经常渴望让别人知道他们离开公司的原因，但这是完全错误的。你会倾向于提出不太有建设性的批评，这为谈话转移到争论中创造了可能性，这会让你看起来心胸狭窄和/或有报复心。
*   如果非要问你离职的原因，保持一般和积极的态度。如果某人看起来真的很感兴趣，而且没有什么能改变你离开的想法，那就在你离开后安排一次会面来讨论这件事。如果他们认真对待改进，这可以让他们坚持到底，如果他们不认真，这可以节省每个人的时间和挫败感。
*   弄清楚什么时候可以告诉别人，应该告诉谁。我建议提供建议，但要让经理知道你会尊重他们选择的任何顺序/时间。
*   **确保他们知道你致力于平稳过渡**。尽力让他们明白，即使这需要额外的努力，你也不会让他们无所事事。

### [](#create-a-transition-area-and-transition-document)创建过渡区域和过渡文档

找一个任何需要的人都可以合理到达的地方，并创建一个过渡区。这可能是一个 SharePoint 网站，一个谷歌文档文件夹，一个你硬盘上的共享文件夹，等等。

### [](#transition-document-suggestions)过渡文档建议

额外收获:我为你创建了一个过渡文档模板！你可以在 Github 的我的办公模板库中找到[。](https://github.com/SeanKilleen/OfficeTemplates)

我的过渡文档通常包含:

*   **我参与的项目**:我的当前角色、当前状态、未解决的主要问题、建议的新联系人以及文档链接。
*   路线图/“某一天”项目:你想在那里实现或者希望公司在你在那里实现的事情的列表。有时，这些可以催生真正的变化。取决于你的公司有多开放，这可能是一个微妙而积极地推动他们解决你的一些问题的好地方。
*   内部/外部联络关系:你是任何小组的领导或任何委员会的核心人物吗？你是任何供应商的主要联系人吗？列出这些，这样公司可以平稳过渡。
*   公司电子邮件账户:这很重要，尤其是如果公司付钱让你在某个地方开一个账户(比如 MSDN 订阅)。
*   **软件许可**:同样的，这些也可以在你离开的时候释放出来。
*   **您拥有的密码/安全访问权限**:记下您知道密码的任何特权访问权限或帐户，因为这些都应该以某种方式进行更改或处理。
*   我使用的软件:这可能是有帮助的，特别是如果这个软件是你实现任何“神奇”结果的原因。
*   我使用过的网站:类似地，一系列有用的基于网络的工具可以帮助你轻松过渡。
*   **推荐书籍**:如果我发现有适合开发团队或公司阅读的书籍，我可能会把它们放在这里，特别是如果我觉得它们会帮助我们更好地协调以避免我离开的话。
*   **基于评估的下一步行动。**这可能是最重要的部分。根据您在上面写的所有内容，您和其他人需要做哪些事情来确保成功过渡？我甚至会将这些任务添加到一个[特雷罗董事会](http://Trello.com)中，并邀请适当的利益相关者，以便他们可以随时关注事情的进展。

### [](#meeting-with-stakeholders)与利益相关者会面

尽快会见尽可能多的利益相关者。在你离开之前，你需要考虑所有接下来的行动，了解每个人对你的期望是什么，这样你就可以根据整体优先级找出事情在队列中的位置。

在与这些人会面后，将过渡计划带回给你的经理，这样他们可以确认优先级是正确的。这将有助于确保你们在同一页上，以防有些事情你不能得到。

### [](#connect-with-and-praise-colleagues)与同事交流并表扬他们

一旦消息传出，如果你还没有在职业/社交网络上添加同事，并且你想在未来与他们保持联系，那么就添加他们。

我喜欢花时间发送一些个人感谢电子邮件，详细说明人们如何帮助你成长或在你需要时帮助你。我把这些发给尽可能多的人。这种事大有可为，但只要你是真心的就去做(别人会知道)。

此外，如果 LinkedIn 上的某人真的对你的时间产生了影响，或者如果这种联系对你的未来特别重要，那么在他们身上添加推荐也是一种很好的姿态。

### 最后，一个漂亮的退出姿态

你希望你的同事如何记住你的离开？我喜欢通过为办公室做些事情来表达我的感激之情。在之前的一份工作中，我从城里一家高档面包店带了羊角面包，放在休息室，并附上一张感谢卡。这些非常受欢迎。大多数企业永远不会想到这一点，这就是为什么它最终成为一件好事。

## [](#what-do-you-think)你怎么看？

我很高兴与我以前的雇主保持了大体上积极的关系。
