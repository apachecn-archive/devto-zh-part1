# PHP 在 2017 年会如何发展？

> 原文:[https://dev.to/cloudways/how-php-will-fare-in-2017](https://dev.to/cloudways/how-php-will-fare-in-2017)

对于 PHP 社区来说，2016 年最好的消息就是 7.1 版本的发布。虽然许多人最初认为 PHP 7.1 是一个修复了错误的小版本，但事实证明它是一个完全不同的怪物。该版本提供了重要的改进，一个新的返回类型(void)和多重捕获异常处理。

今年，我去了 PHP 社区，询问了对 2017 年主导 PHP 世界的趋势的普遍感受和期望。本质上，我在 Reddit 帖子里问了三个问题:

*   2016 年 PHP 发生的最好的事情是什么？
*   社区如何看待 PHP 7 和 7.1？
*   最好的 PHP 框架是什么？

另外，我问了一下关于官方终止对 PHP 5.6 支持的问题，社区的反应。

除了 Reddit，我还通过电子邮件向几个社区影响者提出了这些问题。我收到了以下回复:

Zend Technologies 的技术经理、PHP 社区的教父 Cal Evans 回应道:

> PHP 的 7 和 7.1 版本并不是我们在后来的 5.x 版本中得到的革命性变化。然而，它们证明 PHP 已经稳定、成熟，并且有一条可预测的前进道路。

在回答关于框架的问题时，Evans 非常明确。

> 框架的存在只是为了让开发者生活得更好。一些开发人员不想为细节操心，他们只想把事情做好。这没有错，像 Laravel 这样的框架将会一直存在以满足这种需求。在我们的生态系统中，Laravel 凭借围绕它构建的工具和生态系统帮助开发人员完成工作，鹤立鸡群。

Yii 的主要贡献者 Alex Makarov 对 PHP 版的问题提出了个人见解。

> 虽然 7.0 是优秀的革命性版本，但返回类型和返回 null 的设计问题阻止了我使用它。7.1 增加了可空类型，现在我很高兴。

在谈到最喜欢的 PHP 框架时，马卡罗夫补充道:

> 取决于“最好”的定义是什么。如果我们谈论的是“在美国最受欢迎的人”，那么应该是 Laravel。如果它“在前苏联和亚洲最受欢迎”，那它就是 Yii。如果我们谈论的是企业级支持，那毫无疑问是 Symfony。如果是关于特性和性能，Yii 有更多现成的。

PHPBenelux 的联合创始人 Stefan Koopmanschap 热爱 PHP 7+。

> 我喜欢它。我认为这是语言发展的一大进步，PHP 现在是一门非常成熟的语言，可以轻松地与其他语言竞争。它正被用于高可伸缩性和业务关键型应用程序。

他对最喜欢的框架问题的回答呼应了“独立于框架的开发人员”的观点。

> 没有最好的框架。每个用例都有最适合的。Symfony，Laravel，Zend Framework，Yii，Expressive，Silex，Slim，不胜枚举，每个框架都有一席之地。哪个框架适合您的用例取决于应用程序的需求、您团队中的开发人员、它将运行的基础设施以及许多其他因素。

我想从 Reddit 用户 leeharris100 的评论开始讨论 Reddit 的民意调查，他非常简洁地总结了 2017 年的 PHP 趋势:

> 这就是为什么我认为 PHP 在未来很长一段时间内会继续流行的原因。创建任何规模和范围的应用程序都很容易。你能想到的每一个问题都有文档和支持。它终于达到了性能基本可以接受的程度。

我在 Reddit 上做了一个[投票，从世界各地的 PHP 专家那里得到了一些精彩的答案。我现在将对所有四个问题的答案进行总结，以便读者能够形成自己的观点。](https://www.reddit.com/r/webdev/comments/5gsrwp/what_do_you_think_about_php_trends_in_2017/)

## 2016 年 PHP 发生的最好的事情是什么？

大多数受访者认为速度是 PHP 在 2016 年大幅提升的一个方面。用 Reddit 评论员的简洁话说:

> PHP 和 h*ll 一样快！

速度的提高是 PHP 7 的主要优势(2015 年底推出)。

对于一些用户来说，2016 年最好的事情是 PHP 库和框架的日益成熟。我认为 Reddit 用户 ajr901 在评论中说得最好:

> 速度，开发的容易程度(某种程度上？)，易于部署，可以说是 PHP 可用的更好的框架和工具。

我相信这两种发展(速度的提高和库的成熟)都是引入 PHP 7 和 7.1 的直接结果。这让我想到了下一个问题

## 社区对 PHP 7 和 7.1 怎么看？

社区喜欢新的 PHP 7。该语言几乎所有领域的改进都集中在一个非常快速的包中，消除了对 PHP 的一个主要抱怨。

用 Reddit 用户 hedsht 的话说:

> 这是朝着正确方向迈出的重要一步，打字暗示有望让一些反对者闭嘴。

社区一致认为 PHP 7+是该语言迄今为止发生的最好的事情。几乎 Reddit 和脸书上的每个回答者都同意，随着 PHP 7 的推出，PHP 已经进入了一个全新的时代。PHP 7.1 通过引入影响每个框架和库的速度相关的改进，进一步增加了甜头。

许多回答者认为 PHP 现在已经为企业级项目做好了准备，因为 PHP 7 的速度很快。Reddit 评论员 the_goose_says 写道:

> 2016 年 PHP 最重要的是速度。数百万人一直在使用 PHP，但最大的公司都避免使用它，因为速度在那个级别非常重要。有了 PHP 7+，我们可能会有更多家喻户晓的名字使用 PHP。

## 最好的 PHP 框架是什么？

讨论的真正内容(对于每一个 PHP 相关的讨论都是如此！)是关于用户更喜欢的框架。每个人都有自己喜欢的框架，并在项目中使用。在回答关于最佳 PHP 框架的问题时，每个人都选择了自己最喜欢的。

我让社区从 Symfony、Yii2 和 Laravel 中选出他们最喜欢的。有趣的是，一些 Reddit 评论员也将 Codeigniter 和他们自己的定制 PHP 框架带入了这场辩论。

令人惊讶的是，评论在三人中倾向于拉勒维尔，Symfony 排在第二位。我认为糖尿病人总结了这场辩论:

> Laravel 是我用过的最流畅的编码。Yii 每一步都在和系统抗争。

最后，关于终止对 PHP 5.6 支持的问题，社区并没有很大的分歧。一些评论者说，如果 PHP 5.6 项目已经使用最佳实践实现了，就没有什么可担心的了。用 Reddit 评论员 scootstah 的话说:

> 任何兼容 5.6 的软件都可以毫不费力地迁移到 7。如果你已经落后时代 5 年，你才会有问题。

## 2017 年 PHP 的(可能)趋势是什么？

我认为 2017 年有三大趋势将主导 PHP 世界。

第一个也是最有可能的趋势是 PHP 7.x 在 PHP 开发的所有方面完全占据主导地位。在某种程度上，这将是停止支持 PHP5.6 的结果。然而，这一趋势最重要的方面是，PHP 7.x 为网站和 web 应用程序的语言和 PHP 开发引入了几个重要的改进。

2017 年第二个重要的相关 PHP 趋势将是所有重要框架的主要版本的发布。现在，所有主要框架(Symfony、Laravel 和 Yii)的主体都是建立在 PHP 的旧版本之上的。2017 年，发布的版本将完全包含 PHP 7，并将提供重要的功能，作为框架工具和脚手架的一部分。

最后，各种风格的主要(和次要)虚拟主机提供商将升级到 PHP 7.x。虽然这是显而易见的，但鉴于 PHP7.x 的流行，任何使用过虚拟主机的人都知道让虚拟主机公司升级任何东西是多么困难！我认为，2017 年将会是 PHP 7.x 作为默认 PHP 版本随处可用的一年。

总之，PHP 社区对 PHP 7+以及新版本为各种描述和规模的项目提供的所有可能性非常乐观。说到框架，Laravel 和基于 Laravel 的 CMS 目前已经吸引了社区的注意，没有迹象表明其他框架会带来游戏改变者！
