# 不要害怕 CFP

> 原文:[https://dev.to/mattstratton/dont-fear-the-cfp-3mb1](https://dev.to/mattstratton/dont-fear-the-cfp-3mb1)

*这篇文章基于我在 2015 年底特律发展周*[](https://legacy.devopsdays.org/events/2015-detroit/proposals/how-to-train-yourself/)**上的一次演讲。**

 *决定在活动中提交一个演讲供考虑可能是一件麻烦的事情——如果我的想法不够好怎么办？我是否有足够大的想法来与“思想领袖”分享一个舞台？为什么会有人关心我必须做的事？

在这篇文章中，我将与你分享我的秘密，来确定一个谈话的想法，决定在哪里提交，并帮助给出克服可怕的[骗子综合症](https://en.wikipedia.org/wiki/Impostor_syndrome)的建议，以帮助你克服“不够好”的担忧，并突破与喜欢听的观众分享你的个人专业知识和见解！

### 提交一份报告真的很吓人

[![](../Images/0928650fb97158e1e59ecb849022476c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--7Ck3qHOt--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2A9syNCFqYS4PTHKaLPo52Lw.png)

因为我们花了很多时间听播客，看演讲，在 Twitter 上关注大思想家，我们认为任何人都想听到的唯一信息是超级有见地的想法。

事实是，不管你的经验水平如何，你都有一个独特的故事要讲。你的故事有价值。你不需要分享一些全新的容器管理方法来让别人听你的——他们可能会很高兴听到你在生产中运行 Kubernetes 的历史。即使你认为“每个人都知道这个”，你对这个故事的独特见解也是特别的，因此是有价值的。没有人确切地知道你所知道的，就像你所知道的那样。

### [](#you-dont-have-to-write-the-whole-talk-to-submit%C2%A0it)你没必要写完整篇演讲来提交它

[![](../Images/7803218c9b8ad1fe2a484e3557e30dad.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--1XSVZIuL--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2Axd9362JyrNVS3s_jSClaRw.jpeg)

一个想法足以创造一个摘要。你只需要一两段文字来阐述你的理论。在你被录取之前，不要花时间写下整个谈话。

你甚至不需要知道这个想法是否可行！例如，当我想出 DevOps talk 的[五种爱情语言时，我只是简单地有了一个理论，爱情语言的隐喻将适用。这是我在提交之前所知道的(你可以在这里](https://www.youtube.com/watch?v=SGCfwvjHKGI)[看到这个演讲的摘要](https://github.com/mattstratton/speaking/blob/master/proposals/five_love_languages_of_devops.md))。如果你最初的假设不完全符合你的期望，你可以在写演讲稿的时候调整它。很可能没事。

### [](#you-can-use-a-talk-to-learn-something)你可以用一次谈话来学习一些东西

[![](../Images/901f66bd34d88a8e450c11e4961fe317.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--dSJKsyWE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2ApCU6g7anwbtwYl1gKL-yGQ.jpeg)

想学点新东西？提议谈谈吧！这将是真正学习它的一个很好的动力(截止日期可以非常鼓舞人心)。

例如:

*   我有一个关于如何建立一个管道来使用 InSpec 符合性测试的理论，以确保我所有的食谱能够很好地一起工作
*   我不需要确切地知道如何写摘要
*   我可以只是提出这个想法，我会在写报告的时候学习如何去做！

记住——你的摘要不一定要被“证实”才能提交。

### 我怎样才能想出一个演讲的点子？

[![](../Images/51a87e63b7a8bcd3ef9d52b260e48a4b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WlJQ0BlC--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AXiaH4cZCtkHmyLCGzRO7Tg.jpeg)

#### [](#something-you-have%C2%A0done)你做过的事情

即使很多其他人谈论过一个话题，你的经历也和其他人不同。你的故事很特别。真实世界的故事比理论更有价值。

#### [](#something-that-you-have-learned%C2%A0recently)你最近学到的东西

你刚刚学了一个很酷的新方法来测试你的 Go 代码吗？做一个关于它的演讲将会有两个很好的结果。

1.  你会非常兴奋地谈论它，这会让你成为一个迷人的演讲者。
2.  它会巩固你大脑中的知识。

我非常相信**学>做>教**的模式。教某人如何做某事比学习它使用你大脑的不同部分，你会更好地记住知识。

#### [](#something-that-you-find-super-interesting)你觉得超级有趣的事情

这个整体是关键。你必须对你的话题感到兴奋。它会在你的演讲中体现出来。也许你对 AWS 如何执行公共事后检查很感兴趣。即使你不在 AWS 工作，甚至不认识那里的任何人，你也可以做研究并展示你学到的东西。

#### [](#something-that-you-want-to-learn%C2%A0about)你想了解的东西

如上所述，你可以把谈话作为学习新事物的一种方式。朱莉娅·埃文斯应该是你的模特。她是把教学作为一种学习方式的典型代表。而**学>做>教**的模式在这里也适用！

### [](#where-to-submit%C2%A0talks)向何处提交会谈

[![](../Images/dabde61144166812e4ff64fa9891f35b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ymkpfhys--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AvmpkkzqzfcpBt5fLcbeH0A.jpeg)

#### [](#forget-about-places-like%C2%A0velocity)忘记速度之类的地方

通常的回音室可能相当吓人。你正在与这里的大牌竞争，如果这是你第一次来，你将面临一个巨大的挑战。

也就是说，也许你不应该避免参加这些会议。但是不要把所有的希望都寄托在大的上面。

#### [](#places-that-arent-familiar-with-your-particular-area)不熟悉你所在地区的地方

对我来说，因为我的主题领域是 DevOps，所以我喜欢考虑对它不太熟悉但会觉得有趣的会议和团体。我喜欢考虑提交 DevOps 类型讲座的一些地点示例:

*   当地开发者聚会
*   敏捷会议
*   软件测试组织

您可以从[280 多个技术、IT 和 DevOps 会议的完整列表](https://docs.google.com/spreadsheets/d/17i6PGTMHd9JHG1CKw0mKckUQehTXFPvabVuC8vfTr5I/pubhtml)中获得灵感。

#### [](#devopsdays-ignites)假期点燃

这些都是尝试(简短)谈话的好地方，但也有一些挑战:

*   Ignite 格式可能很难(但是真的很有趣！)
*   DevOpsDays 对重复的内容很挑剔

好的一面是，大多数 DevOpsDays 通常缺少 Ignites 的提交，所以你可能有更好的机会被接受。

### 你不必刻意打扮它

<figure>[![](../Images/852d7d92648ab13399e1e37b5d9ef1e0.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--onHFh79n--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/600/1%2AN-vXO0yG5pQlJ6kBqgx2gw.jpeg) 

<figcaption>悲保尔悲</figcaption>

</figure>

这是对 J. Paul Reed 的引用，他倾向于希望在他演讲的每个会议上都有一个新的演讲。这是*精疲力尽*。保罗可以做到，但你不是保罗。

秘密？尽管他的所有演讲都不同，但它们往往是一个主题的变体。但是如果你刚刚开始，不要给自己增加不必要的困难。

### [](#dont-overkill)不要矫枉过正

想出一两个摘要，到处投稿。它们应该以一种有意义的方式彼此不同。当你被接受并做这些演讲时，每一次你都会做得更好，一年下来，你会不断完善你的演讲。

这是专业人士(即 Jez Humble)的做法。为每个会议创建新的内容令人疲惫不堪，而且实际上并不能提供所需的价值。

### [](#try-out-new-material-on-the%C2%A0road)试用新材料“在路上”

<figure>[![](../Images/75db929f43753d2b00de135f15e7ad28.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--XZXmrH2p--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/720/1%2AnY7_F2cnVFtV3zZA_yXWRQ.jpeg) 

<figcaption>“集装箱到底是怎么回事？”</figcaption>

</figure>

在一个更大的活动中发表演讲之前，自愿先在当地的集会上发表你的演讲，让它变得更加精彩。这也给了你获得直接反馈的能力。大多数聚会都很乐意让人们自愿介绍内容，并且很乐意和你一起工作。

如果你不属于某个特定的团体或社区，出席一个城外聚会会很有帮助(也不那么令人生畏)——你不必担心你之前的名声(或被认为缺乏名声)。

### [](#use-the-buddy%C2%A0system)使用好友系统

[![](../Images/47101029486c5fc80540134c255f8960.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gkz7Lzgh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2A9tICMZ-n-tdv0wvScQUUQw.jpeg)

当你在做演示的时候，同行评审是一个很好的工具。您可以向以下人士寻求帮助:

*   同事
*   社区里的朋友
*   你演讲的活动组织者
*   问问推特吧(如果你觉得大胆的话)

这并不一定是对你的期末报告进行回顾(尽管这不是一个坏主意)，而是对你的摘要、主题和演讲结构进行输入。如果你把这些想法放进一个谷歌文档，你可以和你寻求帮助的人分享。

### [](#qampa-is-your%C2%A0friend)问& A 是你的朋友

<figure>[![](../Images/6129c9da83b4f642e282f9d304f4bb03.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--v5IVsLiX--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2AdDdaPTmC6JNCQizaWf8KKg.jpeg) 

<figcaption>最好的科技讲座都在 3d</figcaption>

</figure>

如果你担心你没有足够的“内容”来做一个完整的演讲，记住问答真的很有帮助。它使你的谈话更具互动性，并有助于延长时间。

但是，我不建议只问“有什么问题吗？”像这样的开放式声明可能会引起观众的不满。你当然可以从那开始。如果没有，准备好向听众提出“行动号召”的问题——如果你的演讲是关于验尸的，你可以问听众是否有人在他们的组织中有一个好的过程，并与你分享一些关于这个过程的东西。

### [](#protip-1)PROTIP #1

#### 将你所有的演讲材料储存在 GitHub repo 中

[![](../Images/18ad7cc26cecfdfb4e2a6efaa66a27b0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--uAsjV6ll--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/1024/1%2A38kJDOMcKR485gaW3AhD1w.png)

这应该包括你所有的摘要/提议，陈述本身，以及你用来演讲的任何支持项目。

你可以让每个演示都成为一个子模块(如果你不怕子模块的话，这是很合理的恐惧)。这样，你就可以单独提交/处理它们，也许还可以和其他人合作，而不用查看你的整个演讲报告。

你还应该用你的简历在回购中创建一个降价文件；这给了你一个简单的复制/粘贴的地方，或者给人们指明方向。你的头像也要存放在这个回购里。

### [](#protip-2)PROTIP #2

#### 在你的博客/网站上创建一个“演讲者页面”

[![](../Images/3fa631f96b94d3746bb2d510eedde314.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--rOIHPe_c--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://cdn-images-1.medium.com/max/800/1%2AsbH01bM6mymzCxgzuVyrKw.png)

这应该包括你过去演讲的链接，以及一个可以下载你演讲人头像的简历页面。

保持这一页的更新可能是一个挑战，但这是值得的。它不仅可以帮助你记录你在哪里演讲过，而且当你提交 CFP 时，你可以在笔记中添加一个链接，以帮助组织者了解你的经历。
