# 做出正确的决定

> 原文：<https://dev.to/developerscode/make-good-decisions>

我的母校西北大学有一个不幸的记录，它是大型体育赛事中唯一一所从未进入 NCAA 男子篮球锦标赛的学校。情况变得如此糟糕，以至于我们的球迷为球队想出了一个简单的口号。上面简单地写着:

**拍摄。**

虽然听起来我们只是在取笑自己的团队，但我一直觉得这个口号更尖锐，而不是有趣。而且，在内心深处，我想我们粉丝群中的其他人也有同样的感受。

你可以想怎么策划就怎么策划。学习电影。去健身房。每天练习一千次跳高。尽情准备吧。但是，听起来很残酷，人们只会根据你在一场比赛中的投篮和不投篮来评判你。你不会得到一张免费通行证，因为你为一次失误付出了所有的努力。

因此，如果我必须将我所有的编程经验总结成一条建议，它将是类似的:

做出正确的决定。

我共事过的最好的程序员总是坚持做这件事。这并不是说他们知道某种语言或特定的模式，或者是最快地想出一个复杂问题的聪明解决方案。那就是，当面临挑战时，他们总能完美地执行他们的解决方案。他们做出了正确的决定。

做出好决定的棘手之处在于，在看到结果之前，你不知道自己已经做出了决定。就像你不知道你射门了，直到球入网。但是，这就是重复经验发挥作用的地方。你尝试的次数越多，你对自己的评估就越好。

然而，经验并不是唯一的指标。做出好决策的程序员似乎也掌握了这些特质:

*   他们对待小问题和大问题一样尊重。
*   他们走正当的捷径，然后适当地清理债务。
*   他们承担值得冒的风险，并确保尽可能不冒风险。
*   他们认为任何决定都是一种交换。
*   他们添加更多的代码，同时保持干净的代码。
*   他们知道对顾客、客户和代码库来说什么是最重要的，并在两者之间找到恰当的平衡。
*   他们永久性地堵塞代码漏洞，而不是重复地擦拭潮湿的地板。
*   他们以*相对*的准确度来估计交货时间。
*   他们清楚地传达他们的意图(用代码和人类)。
*   和他们一起工作很愉快，也很可靠，尤其是在战斗最激烈的时候。
*   出现问题时，他们不会简单地“承担责任”。他们有这个问题。
*   如果有小路障挡路，它们能够调整航向。

即使拥有所有这些特征，仍然不能保证程序员每次都能做出好的决定。就像最好的篮球运动员仍然会偶尔投篮不中。任何决定都有一点点运气的成分。但是，正如罗马哲学家塞内卡曾经说过的，“运气是当准备遇到机会时发生的事情。”
