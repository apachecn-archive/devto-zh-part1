# 用你的橡皮鸭

> 原文：<https://dev.to/funkysi1701/using-your-rubber-duck-3jj2>

橡皮鸭是一个发展中的过程，在这个过程中，你可以和一个无生命的物体(比如橡皮鸭)讨论一个问题。它的名字来源于《实用程序员》这本书，它第一次被使用。

我是我工作的地方唯一的程序员，我经常发现自己有一个找不到答案的问题。然而，我确实有几个朋友在这类事情上比我更有见识。我经常通过电子邮件向他们解释我的问题。在写这封邮件的过程中，我必须解释我的问题，这有时会暴露问题，我甚至不需要发送这封邮件。在其他时候，当他们回复我的时候，我的大脑已经充分地处理了这个问题，如果我还没有解决的话，我已经接近解决方案了。

其他人使用不同的物品作为他们的“橡皮鸭”，在 [code 新手论坛](http://discourse.codenewbie.org/t/whats-your-rubber-duck/61)上的人使用的物品有一只猫，填充章鱼，一个乐高玩具，或者只是简单的自言自语。

成为我办公室里唯一的程序员很难。一个简单的你怎么看这个可以在办公室问的问题必须在我的脑海中完成，我可能会认为一个不必要的复杂解决方案比一个简单的解决方案更好，因为我还没有想到更简单的版本。作为唯一的程序员当然有好处，我可以做我想做的事情，学习我喜欢的任何技术，但我开始认为在一个团队中编程的好处超过了单独编程。
