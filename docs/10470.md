# 吉格斯普劳特的兴衰

> 原文:[https://dev.to/mariordev/the-rise-and-fall-of-gigsprout-elm](https://dev.to/mariordev/the-rise-and-fall-of-gigsprout-elm)

2011 年夏天，我正在积极寻找一份网页开发员的新工作。我做了这些天我们大多数人都在做的事情，上网并开始在招聘信息栏中寻找一个很好的机会。至少可以说，这次经历令人沮丧。我遇到的大部分求职板，不管是知名的还是鲜为人知的，都有非常烦人的功能，比如弹出窗口、广告和无用的“职业建议”如果我不想要广告，我必须有一个付费账户。有些要求我创建一个账户来搜索或申请工作。其他人甚至把广告伪装成工作列表。这些招聘信息板中的大多数都没有用户友好的界面，更不用说它们是手机兼容的了。

## 我会创造我自己的

我看到了一个机会，在网上找工作时，可以创造一个增强的、集中的、没有干扰的用户体验。作为一名 web 开发人员，我喜欢构建东西，那么为什么不构建更好的东西呢？我自己的产品？我考虑了几天。在我的职业生涯中，我参与过许多项目，但我几乎总是作为开发团队的一员。这将是我第一次独自从头到尾开发一个成熟的商业应用程序(技术方面)。我把这个想法告诉了我的长期女友，现在也是商业伙伴的英格丽德。她真的很喜欢这个想法，鼓励我建立它，并向我提供她非常需要的帮助。团队 [TeclaWorks](http://www.teclaworks.com) 诞生了。

## [](#development)发展

经过几个星期的绞尽脑汁想出一个我们的工作公告板的名字和一个还没有被使用的网址，我们最终登陆了 Gigsprout，意思是“工作机会萌芽的地方”我们进行了大量的研究，我专注于技术方面，Ingrid 专注于商业方面。

我们想按书上说的去做每一件事，有太多的东西需要学习和理解。组建公司的公司章程、税务和会计问题、技术问题、开发框架、托管解决方案、云托管与传统托管、营销、社交媒体、客户服务、最低可行产品等等。

建造 Gigsprout 相当冒险，花费了无数个小时的开发时间，在我们全职工作的同时慢慢地建造它。做出了很多牺牲，我在这里就不多说了。我不想听起来像是在发牢骚。我尝试了三个开发框架，包括我自己的，我在 2012 年放弃了它，转而使用一个更成熟的框架，一个当时相对较新的框架，叫做 Laravel。通过这一切，我们学到了很多东西，凭借毅力和奉献精神，我们克服了许多挑战，使 Gigsprout 成为现实。

## [](#the-tech-stack)科技栈

我设计了 Gigsprout 在云上运行。它完全托管在 AWS(亚马逊网络服务)上。后端是用 PHP 和令人敬畏的 Laravel 框架编写的(第一个可行的版本是用 CodeIgniter 编写的)。数据库是 MySQL。弹性搜索引擎使全文搜索成为可能。源代码版本控制的 Git。在前端，一个高度修改版本的引导 CSS 框架和 jQuery。

## [](#launch-and-shutdown)启动和关闭

Gigsprout 是科技行业的招聘平台。它被设计成对求职者免费，没有广告和弹出窗口。求职者不需要创建一个帐户来搜索或申请工作。雇主可以享受简单、负担得起的按岗位付费的价格。至少，计划是这样的。Gigsprout 的功能有限，但我们的想法是在我们推出后，根据我们的路线图和用户的反馈增加更多功能。

但是发射提出了“先有鸡还是先有蛋”的经典问题。我们无法吸引求职者，因为我们的网站上没有可供搜索的工作。我们无法吸引雇主来发布工作，因为没有求职者来我们的网站。那么我们如何启动？我们决定通过向雇主免费提供所有工作岗位来解决这个问题。这是一个限时优惠，激励雇主在 Gigsprout 上发布他们的工作。

2015 年秋天，我们终于推出了。我在 Google AdWords 上注册了一个账户，开始每天支付可观的广告费。广告公布了 Gigsprout 和提供给雇主的免费工作岗位。我们开始获得一些流量，但雇主们暂时没有签约。最后，几天后，我们开始得到一些注册。然而，我们很快意识到，他们发布的大多数注册和工作都是有问题的，并且/或者违反了我们的使用条款。一些工作岗位明显具有“金字塔计划”的性质，一些网站伪装成工作列表，提供“每天只工作几个小时就能在家赚很多钱”的服务。我们收到的帖子中只有少数是合法的。我们行使权利删除违反我们使用条款的帖子，并关闭发布这些帖子的账户。

这样过了几个星期后，我们决定做些改变。一旦免费提供过期，我们决定对所有帖子提供 50%的折扣。这个想法是，需要通过信用卡付款，只有严肃、合法的雇主才会签约。在 Google AdWords 和 Twitter 上多做一点广告带来了一些流量，但没有注册。广告费用越来越高，我们被迫停止。在为 Gigsprout 的运营买单一年后，但没有产生任何收入，我们被迫决定关闭它。

## [](#lessons-learned)吸取教训

我从这次经历中学到了一些经验，我想在这里分享它们，希望这样做可以帮助其他可能打算推出自己产品的人。

### [](#1-dont-incorporate-too-early)1。不要过早合并

如果你正处于第一个产品的开发阶段，不要太早合并(例如在美国，成立公司或有限责任公司)。一边做产品，一边做全职工作，还要兼顾家人和朋友，这可能需要很长时间。一旦你成立了公司，你就必须开始纳税。如果开发需要很长时间，你将开始支付税款和其他费用，甚至没有机会产生任何收入。等到你即将推出你的产品，然后纳入。

### [](#2-start-small)2。从小处着手

有些想法是从需要“挠自己的痒”开始的。其他人从人开始，需要为他们解决问题。不管是哪种情况，在你开始着手你的想法之前，先用它的一个小版本来测试一下。检查你的想法是否引起你的朋友和家人和/或你的产品将受益的人或社区的兴趣。

### [](#3-set-deadlines)3。设定截止日期

具体的截止日期将帮助你设定具体的目标。每一个截止日期都代表一个里程碑，让你专注于一个目标并朝着它前进。当你设定最后期限的时候，要现实一点，并尽最大努力遵守。

### [](#4-share-the-process)4。分享过程

与他人分享你正在做的事情，并发布你正在取得的进展的更新。这将有希望产生更多的兴趣，你的产品。一些人可能会对你的产品更感兴趣，如果他们是创造产品过程的一部分。这可能有助于你理解我的下一个观点。

### [](#5-build-a-following)5。培养一批追随者

不要发射到蟋蟀身上。通过分享构建产品的过程，你有机会建立一个追随者。也分享你的知识，博客你的项目，分享你学到的东西。随着时间的推移，你会建立一个追随者。当你提前建立了一些追随者时，推出一个产品应该比在蟋蟀的叫声中推出更容易。

### [](#6-maximize-time)6。最大化时间

我不知道你怎么想，但对我来说，两三个小时专注的工作很有帮助，远远超过八个小时不间断的工作。找到一个你可以在工作时集中注意力的空间，每天关闭社交媒体和其他分心的事物两三个小时，你就会取得进步。

### [](#7-get-enough-sleep)7。获得充足的睡眠

我知道当你被连线到一个角色编码时，很难停下来，通宵工作。但是夜晚是用来睡觉的，你的身体需要它。我知道每个人的需求不同，这可能不适合你。但是我发现当我有足够的睡眠时，我会更加警觉，能够更好地集中注意力。这样我更有效率。

### [](#8-outsource-and-get-help)8。外包并获得帮助

你可能听过这句话“你必须花钱才能赚钱。”在某些情况下，确实如此。很可能你不是开发产品的每个方面的专家。你可能是一个出色的开发人员，但是你可能需要市场营销方面的帮助。当我在 Gigsprout 工作的时候，我什么都想做，把花费保持在最低。我首先是个编码员。尽管我对 Photoshop 和 Illustrator 有足够的了解，但我不是专业的平面设计师。我创造了 Gigsprout 标志，并节省了一些钱。但是在更深入地学习工具来做这件事和创建许多原型之间，我花了很长时间才得到我满意的东西。如果你没有资格完成某件事，那就雇用有资格的人。他们会在你需要的时间的一小部分内完成，你会为自己节省一些时间和挫折。这将有助于你更快地推出你的产品，这是我的下一点。

### [](#9-launch-early)9。提早发射

你的产品在发布时不一定要完美。它必须有用。你想想，反正你的产品永远做不完。尽快发布出来，这样你就可以开始收到用户的反馈。它们是改进产品的最佳指导来源。你越早发布，你就能越早开始根据正在使用和将要使用它的人的反馈来改进它。如果你没有得到任何用户或反馈，那么你很早就会知道有些东西需要调整。弄清楚你的 MVP 是什么，最小可行产品。你的 MVP 应该有足够有用的特性，但也要足够小，这样你就可以尽早发布它。越快越好。

### [](#10-free-isnt-always-good)10。免费并不总是好的

根据你要推出的产品/服务的类型，免费服务可能不是最好的。它可能会为你的产品吸引错误的观众。因此，做一些研究，找出任何替代方案，以创造一种激励来吸引付费客户。

如果你打算推出自己的产品/服务，我希望我从这个项目中学到的经验对你有所帮助。永远努力争取成功，但是记住有时候失败也没什么。这东西不容易。如果是的话，任何人都会这么做。如果我们从失败中吸取教训，失败是有价值的。我们所学到的，将帮助我们在未来取得成功。如果你跌倒了，站起来，掸掉身上的灰尘，运用你学到的经验教训再试一次。最美好的祝愿。
