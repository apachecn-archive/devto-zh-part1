这是一个我经常重温的话题，但是这个列表往往会随着网站和时事通讯的变化而定期改变，或者它们以某种方式改变，使它们变得或多或少有用(对我来说)。今天，我将特别关注两种类型的开发人员资源:

*   [链接聚合器](#aggregators)
*   [时事通讯](#newsletters)

我已经将这些资源组织成我个人经常查看/阅读的资源(即每天查看网站或每期简讯)、经常查看的资源(即每周或每隔一期)、偶尔查看的资源(即网站每周不到一次或简讯每月不到一次)和很少查看的资源(即我仍然会查看或订阅，但只是偶尔阅读)。

### [](#dont-get-insulted)不要受到侮辱

在有人不高兴之前，我应该注意一些事情。

第一，这是*我个人的喜好*。这很大程度上取决于我靠什么谋生。举例来说，我不做很多设计或 CSS，所以即使我仍然订阅或查看一个网站，一个专注于设计的网站在我的列表中的有用性往往会较低。

第二(这一点尤其与时事通讯相关)，如果我真的发现一个网站或时事通讯一点用都没有——我甚至懒得去检查它——永远不会。这意味着它不会上榜。如果我发现时事通讯没用，我会退订。我所包括的网站和时事通讯(甚至在“很少”部分)在那里，因为你可能会发现它们对你的工作或兴趣很有用。

## [](#aggregators)聚合器

就我个人而言，我的大部分阅读仍然依赖于 RSS，但即使这样，聚合网站也是一个有用的工具，可以跟上我可能不知道的来源。

### [](#always)永远

[EchoJS](http://www.echojs.com/)
EchoJS，顾名思义，专注于 JavaScript 和其他相关文章。它允许社区提交，投票和评论。这是非常开放的，这意味着一些垃圾有时会通过，但这些通常是显而易见的，很快就会被否决。评论通常相当空洞或轻松(尽管偶尔会变得激烈)。

[前端前端](https://frontendfront.com/)
前端前端专注于前端开发，因此包含了一些与 EchoJS 类似的主题，但也倾向于有更多通用的 web 设计和开发文章，而 EchoJS 则更专注于具体的 JavaScript 编程。它还允许社区提交、投票和评论，尽管我很少看到任何评论，而且它似乎能让首页远离垃圾帖子

与上述网站不同，Collective 是一个每周两次由人工策划的帖子。它确实包括一些赞助内容，但这些内容通常都是切题的，并且有明确的指示。这些链接更倾向于网页设计，但涵盖了广泛的主题，包括编程、设计、工作文化等等。

### [](#frequently)勤

JavaScript Live 是出版 JavaScript 周刊的库珀出版社的一部分。如果你想让 JavaScript 周刊考虑你的帖子，这里是发帖的地方。任何人都可以张贴链接或评论，但没有任何投票功能，所以它可能会不时地出现垃圾邮件。

[龙虾](https://lobste.rs/)
龙虾如果具体到编程，就像黑客新闻。龙虾往往会出现非常深刻的技术文章。你必须被邀请加入这个社区，以我的经验来看，这有时会有点苛刻和不受欢迎。

[黑客新闻](http://news.ycombinator.com/)
我可能甚至不需要向你解释黑客新闻。你知道它是什么，就内容的节奏、广泛的主题和评论线索而言，它可能有点令人不知所措，这是当之无愧的恶名。

### [](#occasionally)偶尔

这个网站的功能与大多数其他聚合器类似，但来自社区的投票和评论似乎很少。

WebDesignerNews
通常的功能(发布推荐，评论和投票)，虽然职位不是自动的，但手动策划。顾名思义，它更侧重于网页设计，而不是网页开发——这也是我不常访问的原因。

任何人都可以注册提交一个被监控的 feed 或发布一个 URL，尽管没有评论或投票，活动看起来也不活跃。

### [](#rarely)很少

是的，DZone 允许链接，但他们似乎更专注于自己的内容，而不是作为一个有用的聚合器(这很好，只是不是我想要的——另外，他们往往广告很多)。

就我个人而言，我发现 Reddit 社区并不友好，甚至毫无帮助。抱歉，Reddit 的粉丝们。

## [](#newsletters)时事通讯

希望你的内容被推送给你，而不是去寻找它？好消息是有大量有用的时事通讯。坏消息是现在有这么多，你可能会淹没你的收件箱。

### [](#always)永远

在我看来，这里列出的最受欢迎的时事通讯仍然是最好的。库珀出版社的工作人员知道如何策划一份时事通讯(我的意思是，他们有大量的时事通讯)，并一直保持这份时事通讯的相关性。

[Frontend Focus](https://frontendfoc.us/)
另一个库珀出版社的时事通讯，但是有更多的 web 标准、HTML、CSS 和相关技术(以前是 HTML5 Weekly)。

与我的好朋友霍莉·辛斯基和库珀出版社的工作人员一起，由你忠实的策划的移动开发周刊
。我们专注于任何移动开发(移动 web、混合、JavaScript 原生解决方案和原生开发)。我们过去纯粹专注于移动网络，但现在已经稍微拓宽了话题。

我认为了解历史能让我们洞察当下。如果你想成为一名 web 开发人员，看看 web 的(短暂)历史会很有用。另外，这份时事通讯除了有用之外，读起来总是很有趣。

网页设计周刊
顾名思义，这更侧重于设计，但倾向于包括许多前端开发主题，并且策划得非常好。

### [](#frequently)勤

[ES。下一条新闻](http://esnextnews.com/)
这是一份由阿克塞尔·劳施迈尔博士策划的相对较新的时事通讯。它通常非常整洁(我的意思是，非常专注于它的主题，并且包括一组有限的链接)。如果你对 JavaScript 语言特别感兴趣，推荐。

虽然现在 RWD 和普通的网页设计已经没有什么区别了，但是这份时事通讯仍然提供了一些可靠的链接。这是一个移动倾斜和一些设计的网页。

CSS 周刊
另一个非常好的时事通讯，如果我关注 CSS 和设计的话，我会多读一些。Zoran 保留的链接不超过 10 个，这样可以快速方便地找到与你相关的链接。

### [](#occasionally)偶尔

这是由我的朋友 Raymond Camden 和 Cooper Press 共同策划的。即使它不是我个人的主要资源，但随着我深入无服务器领域，我越来越依赖它。

WDRL
安瑟姆·汉内曼的网页设计阅读清单很好，因为它的链接数量有限，但也试图为它们提供上下文。它涵盖了广泛的主题，所以并不总是有与我相关的信息，但仍然值得一查。

如果你更专注于设计，HeyDesigner 的每周时事通讯是值得订阅的，尽管它每周都包括一些更一般的与 web 开发相关的话题。

O'Reilly 网络简讯
O'Reilly 的网络简讯很好，因为它提供了有限链接列表的更长形式的摘要。我认为它对我个人来说可能更有用，如果我还没有订阅上面的内容，因为当它到达时，我经常会看到它包含的我关心的文章。

### [](#rarely)很少

请注意，所有这些实际上都是很好的时事通讯，否则我会退订。正如我在下面提到的，只是它们通常涵盖的主题并不总是与我特别相关。

奥莱利编程新闻
这是一份不错的时事通讯，只是涉及的话题范围很广，与我个人并不总是相关。

这是由尼古拉斯·贝瓦夸策划的。这些链接通常是非常深入的技术文章，有点超出我的理解或技能水平(或者两者都有)。如果你比我聪明，这本书值得一读。

这是一篇做得很好的时事通讯，有一个非常独特的焦点。我不倾向于订阅特定框架的时事通讯，但是如果你非常关注 Vue，这可能值得一读。

CSS 动画周刊
我其实很喜欢读这篇文章，因为我希望我有时间、精力和足够的艺术能力来做动画。如果有，我绝对推荐。
