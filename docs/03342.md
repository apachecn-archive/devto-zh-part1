# 新程序员终端介绍

图形用户界面无处不在。它们已经接管了我们与手机和电脑互动的方式。

但并不是每台电脑都有显示器进行交互。事实上，地球上大多数电脑都没有显示器。

相反，它们是通过一种叫做“终端”的东西来访问的。通过这种方式，数百万台计算机被操纵并被告知该做什么，而不需要一个物理监视器来连接它们。

## 什么是终端？

终端是一种基于文本的方式来与你的计算机上的文件和程序进行交互。

通过基于文本的输入，终端允许你完全进入你的计算机的所有内部工作。

从终端内部使用一个名为`ssh`的程序，我们可以把我们正在工作的计算机变成世界上任何地方的远程计算机。然后，终端会执行相同的命令，就像我们在使用本地计算机一样。

现在，几乎每一个主要的操作系统上都安装了终端，尽管今天我们将把重点放在安装在 MacOS / Linux 的操作系统上。

## 我何苦呢？

有效使用终端的能力为成熟的开发人员增加了一项关键技能。它让你能够批量处理文件，远程编写脚本和控制计算机。

如果您对服务器或 web 应用程序开发感兴趣，对命令行有一个相当好的理解将使您能够远程部署和管理您的服务器。这也让您在调试系统停止工作的原因时占了上风。

## 访问终端

在 MacOS 上，同时按下`CMD+SPACE`，然后打出`terminal`，就可以打开终端。MacOS 启动器将打开一个终端，给你一个空白的窗口，准备接收文本。

从这里，您可以开始输入命令，并让它们在一个纯粹基于文本的环境中执行。

## 在终端导航

您想从终端输入的第一个命令是`ls`。那是小写的字母 L，后面跟着也是小写的 S。

按下`ENTER`执行`ls`命令。

这将列出当前目录中的文件。

在 MacOS 中，你可以在终端中输入`open`,你会得到一个 finder，准确显示你当前目录中的文件。

如果我们使用命令`cd`，我们可以改变到其他目录。我们输入`cd`，然后输入我们想要切换到的目录的名字。如果我们按下`TAB`键，终端将尝试完成我们已经开始输入的目录名。

终端中有几个特殊的目录名。其中一个是`.`，表示当前目录。这是一种指明你目前所处位置的方式。

另一个特殊的目录名是`..`(两个句号)。意思是我上面的目录。

目录名可以用`/`链接在一起。所以，如果我们想转到两个目录，我们可以输入../../'.键入`cd ../../`将从我们当前所在的位置带我们到两个目录。

这样，您可以看到终端的基于文本的环境允许您浏览主机的文件和目录。

## 创建目录

可以用容易记住的`mkdir`命令创建目录。

要使用它，只需键入`mkdir directoryname`。这将在当前位置创建一个新目录。

就像之前一样，我们可以使用`/`和`..`分隔符来指定创建目录的不同位置。例如，要在当前目录的上面创建一个新目录，我们可以输入`mkdir ../testname`。

## 创建文件

可以用几种不同的方法创建文件。我们执行的程序可以为我们创建文件，或者我们可以使用一些内置的工具。
jw
其中一个叫做`touch`。触摸可以让我们在电脑上创建新文件，这些文件完全是空的。

我们使用它的方式只是通过传递一个文件名。它的工作方式就像`cd`和`mkdir`，你可以在当前目录或另一个目录中传递一个文件路径。

您可以为使用`touch`创建的文件键入自己的文件扩展名。因此，要在当前目录下创建一个空的 Python 文件，你只需要输入`touch myProgram.py`，那么`myProgram.py`就会在当前目录下创建。

## 编辑文本文件

使用`nano`命令编辑文本文件非常简单。

当`nano`运行时，它接管终端的屏幕。然后，它允许您直接在您创建的纯文本文件中键入内容。从这里，您还可以打开其他文本文件，并保存它们。

要打开我们之前创建的`myProgram.py`文件，我们需要输入`nano myProgram.py`。我们可以只输入`myP`，然后按下`TAB`，终端将为我们完成输入文件名称的其余部分。

打开编辑器后，尝试输入以下 Python 3 程序:

```
print("hello!") 
```

## 执行文件

执行文件非常容易。

我们只需键入想要运行的程序的名称，然后按回车键。

如果我们想知道我们正在运行的程序在哪里，我们使用命令`which`。

它返回程序运行的目录。按`which python3`找到你电脑的 Python3 解释器在哪里。

我的情况是`/usr/bin/python3`。如果我打开`myProgram.py`，并将其更改为以下内容:

```
#!/usr/bin/python3 print("hello!") 
```

我现在有了一个终端可以自己运行的程序。当我给它发送这个文件名时，它将执行`/usr/bin/python3 myProgram.py`。

但是为了让终端知道它可以把它作为一个程序运行，我还需要做一件事，告诉它这个文件现在是一个可执行文件。

我们用`chmod`命令来实现。我们输入`chmod +x myProgram.py`，`chmod`程序将它转换成系统上的可执行文件。

从这里，我们可以用一个`./myProgram.py`来运行我们的程序。

## 管道输出

用我们的新命令，我们正在向终端发送文本。

但是终端的一个主要特点是能够将程序的输出互相传递。

让我们试着将程序的输出传递给`less`程序，看看输出传递是如何工作的:

```
$ ./myProgram.py | less 
```
