# 给初学者的一些建议

> 原文:[https://dev.to/t4rzsan/some-advice-for-beginners-17i4](https://dev.to/t4rzsan/some-advice-for-beginners-17i4)

如果你是编程新手，这里有一些明智的建议给你。

这个列表是根据我自己的经验建立的，因为我们在某些方面都是新手，对吗？

这个清单可能会更长，但我已经挑选了 7 条重要的建议，这些建议最近与我在辅导过程中有些关联。

### [](#1-have-fun)1。玩得开心！

是的，没错。这是第一条也是最重要的建议。编码应该是有趣的。如果你不开心，也许编码不适合你 [![🙂](../Images/0368ccb0684a7320e0200f90e014461b.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--opz_1A18--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://s.w.org/images/core/emoji/2.3/72x72/1f642.png)

### [](#2-use-existing-guide-lines)2。使用现有的指导线

你可能是一个初学者，你可能会不知所措，但是你越早开始使用现有的指导方针，如命名标准和最佳实践，你的编码技能就会越快起飞。您的公司可能有现有的编码标准(他们应该有),它们的存在是有原因的。遵循指导方针和标准将有助于你和你的导师更好地理解代码。

此外，如果不遵循现有的标准，您的代码很可能无法通过代码审查，而且以后再去修复它也是一件非常痛苦的事情。

### [](#3-your-code-is-not-a-special-case)3。您的代码不是特例

很抱歉我不得不告诉你，但是你的代码一点也不具有突破性，也不需要特殊处理。这意味着你不能自由地使用 gotos，不能像以前那样使用继承，也不能把你的程序的 3000 行放在同一个文件中。

其他人都可以没有奇怪的构造，你也可以。

### [](#4-read-the-error-messages)4。阅读错误信息

如果编译器拒绝编译您的代码，那么请务必阅读错误消息。它们可能很神秘，但你可能会习惯它们，因为这不会是你最后一次见到它们。

运行时错误可能很难弄清楚，但是这个列表中的下一条可能会帮到你。

### [](#5-debug-everything)5。调试一切

调试真的非常非常重要。有人曾经给过我一个建议，总是调试你的代码。所以这条建议适用于所有人，不仅仅是新手。

调试将帮助你理解你的代码是如何工作的。每当我写一段代码时，我都会用调试器一步一步地调试它，即使它看起来在工作。做这件事的一个好方法是调试你的单元测试。

所以，要学会如何在刚开始学习的编程环境中调试。我说过这很重要吗？

### [](#6-the-error-is-most-likely-in-your-code)6。错误很可能出现在您的代码中

如果你的代码失败了(这是必然的),那么可以 99.99%确定是你引入了一个 bug。问题不在于编程语言，也不在于平台。这是你的代码！

所以如果你认为。NET 不能保存超过 50 行的文件，或者 C#在简单的数学运算上很慢，那么你就错了。

问题出在你的代码里！

### [](#7-stop-excusing)7。停止辩解

似乎每个人都在为自己的代码找借口。没有理由。你的代码有缺陷，可以设计得更好，但是我还没有遇到一个程序员写出完美的代码。如果你的导师嘲笑你的代码，你需要找一个更好的导师(是的，我们都为嘲笑某个可怜家伙的代码而内疚)。

就是这样。我希望它能让你的编码之旅变得简单一点。
