# 我的第一个 ASP.NET 应用程序/母版页

> 原文:[https://dev . to/krit ner/my-first-aspnet-application master page-5fc 9](https://dev.to/kritner/my-first-aspnet-applicationmasterpage-5fc9)

当我试图让 ASP.NET 应用程序在我的 web 服务器上工作时，我遇到了一些问题——我对 ASP.NET 知之甚少，我以为它会类似于我习惯的 ColdFusion 事实并非如此。

我不明白的是 AppPools，这个条目的存在是为了提醒我(可能还有其他人)。IIS 似乎默认将你的 web 根分配给一个 app pool——默认(至少对我来说)是。NET 框架 2.0。这就像使用 4.0 框架将默认网站更改为新的 appPool 一样简单，对吗？几乎...

将默认的 appPool 更改为 4.0 框架后，我现在成功地编译了标准的“Hello World”。但是在试图将一个母版页集成到我的站点时出现运行时错误。母版页是一种“模板”,可用于在多个网站上创建类似的布局/功能。运行时错误解释说在~/MasterPage.master 中找不到我的 master page . master——但是它在那里！

幸运的是，我的妻子克里斯汀(她认识 ASP.net)让我知道，如果我的应用程序在一个目录中，但应用程序是在一个单独的目录中建立的，那么“~/MasterPage.master”引用是相对于应用程序的，而不一定是您正在工作的目录。

简单修复？在我的代码实际所在的目录下创建一个新的应用程序池。耶！现在一切正常，没有运行时错误！
