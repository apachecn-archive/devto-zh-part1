# 推荐的 Eclipse 插件

> 原文:[https://dev.to/committedsw/recommended-eclipse-plugins-35go](https://dev.to/committedsw/recommended-eclipse-plugins-35go)

我们喜欢 Eclipse，并且每天(和每晚)都把它作为我们的 Java IDE。(好吧，这些天我们用 Android Studio 做 Android 的东西)。

[![micah williams 348739](../Images/ed19be5d1957ac37f373567af19dbfd0.png)T2】](///static/micah-williams-348739-5d1a23f3685d19a8900a98080631bada-7a1db.jpg)

这里是我们为 Java 开发者挑选的插件。

作为起点，我们使用 Spring 工具套件而不是通用 Eclipse，因为我们大量使用 Spring Boot 进行开发。当开发 Eclipse 本身的插件时，我们强烈建议从 [Eclipse PDE](http://www.eclipse.org/pde/) 开始，以节省您的时间和精力。

从 Eclipse marketplace(帮助> marketplace):

*   EclEmma :我们最喜欢的代码覆盖插件，主要是因为它有这么好的 UI 集成，可以让你找到薄弱的地方。
*   **Findbugs** :代码质量
*   PMD:更高的代码质量
*   更高的代码质量
*   更高的代码质量
*   jAutodoc :从代码文档中去除(一些)单调
*   m2e-apt :单独使用没什么用，但是可以和[不变的](https://immutables.github.io/)结合起来生成神奇的代码。
*   **Gonsole**:Eclipse 中的 Git 控制台，非常适合与 Eclipse Git UI 混合使用(尽管我们也使用 [GitKraken](https://www.gitkraken.com/)
*   Infinitest**:每次保存时自动运行测试
*   PITclipse:Eclipse 的突变测试
*   **Eclipse Class Decompiler** :当你想快速检查出代码而你没有源代码的时候。
*   TestNG :我们自己并不怎么使用 TestNG，但是在开发开源项目时有一个插件是很好的。

当然，你必须安装我们的[拼写插件](https://dev.to/opensource/spelling/spelling/)，它可以帮助你确保你的代码是可读的(真正的英语单词)。

可选，但是我们建议您在 IDE 中安装 [Lombok](https://projectlombok.org/) 并减少 Java 的样板文件。
