# 错误的记录

> 原文：<https://dev.to/n_develop/a-log-of-errors-4cn>

我先坦白一件事。我真的很喜欢 dev.to 社区。没有多少开发者社区如此友好。

但是为什么要向社区致敬呢？因为是一个社区成员告诉我这个想法的，我会写下来。评论文章[的是](https://dev.to/maxwell_dev/takes-notes-on-everything-3io)[的](https://dev.to/maxwell_dev)约瑟夫·裘德。Joseph 评论道，他记录了开发过程中遇到的每个错误(以及相应的解决方案)。多棒的主意啊！

遗憾的是，我不得不经常谷歌几个月前我谷歌过的同样的错误信息或错误。“错误日志”的想法太棒了。

## 格式

我使用降价。为什么？因为我喜欢减价。它是一种简单而强大的标记语言。很容易插入代码片段、图像、链接等等。

如果你使用一个好的编辑器，比如[markdownmost](https://markdownmonster.west-wind.com/)或者 [VS Code](https://code.visualstudio.com/) ，那就更有趣了。

### 降价模板

我为自己记录的每个错误创建了一个小模板。它包括

*   一个标题
*   一些标签(使它更容易搜索)
*   问题的描述
*   错误原因的描述
*   当然还有它的解决方案。

看起来是这样的:

```
## Title (for example an error message)

**Tags**:

> these, are, my, tags, for, the, search

### Problem

A short description of the problem/bug/error. 
In most cases it contains the situation 
in which the error occurs 
and the error including the stacktrace.

### Reason

Why did the error happen?

### Solution

What's the solution for this problem? 
```

## 放在哪里

我希望几乎在任何地方都可以访问这个错误日志。这就是我用 GitHub 的原因。

GitHub 还提供了非常好的搜索功能。那很有帮助。

当然，你可以在你的磁盘上使用一个简单的目录结构，然后把它同步到你的 Dropbox，或者类似的东西。但对我来说，GitHub 是一个极好的选择。
