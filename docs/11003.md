# 胖客户端和瘦客户端的辩论

> 原文：<https://dev.to/ben/the-fat-client---thin-client-debate>

关于面向“胖客户机”的体系结构与面向“瘦客户机”的体系结构孰优孰劣的争论一直是最典型的编程争论之一。脂肪(或厚、重等)。)客户端被理解为一种架构模式，其通常独立于程序的中央服务器提供丰富的功能。瘦客户机和胖服务器是一种服务器处理大部分密集计算的架构风格。

每个单独的架构决策都应该考虑到具体的问题，而不是基于更大的趋势，但是作为一个行业，趋势总是会出现的。在历史上的任何时候，都可以感觉到“编程的未来”取决于这种或那种风格。许多人都在争论未来会是什么样子，结果却被不可预见的技术创新所掩盖。新的社会环境和硬件演变意味着两种风格之间的拉锯战，其中一种风格在一段时间内占据主导地位，但趋势却朝着另一个方向倒退。前面提到的硬件变化以及消费者期望、业务需求、网络发展和一系列其他因素是这种基础架构起起落落的原因。

尽管引用的技术发生了变化，但是架构争论的原则仍然是相关的。尽管这场争论完全发生在我们今天为了服务于计算机程序而使用的大多数技术和界面之前，但这些思想仍然存在。曾经关于在许多计算机终端上显示文本的最佳方式的争论，已经演变为关于下载网站和应用程序并将其传送到桌面和移动设备的不同方式的类似辩论，并将随着更多联网设备的出现、虚拟和增强现实成为现实以及计算机智能允许更复杂的基于消息的应用而继续发生变化，更不用说由于尚未出现的发明而将发生的变化了。

## 胖客户端/瘦服务器的优势

*   较低的服务器要求。胖客户机不需要瘦客户机那样高的性能水平，因为单独的客户机设备处理大部分工作。
*   离线工作。只有在网络条件允许的情况下，胖客户端才可以存储和连接服务器，因此不需要持续连接。
*   更好的多媒体性能。胖客户端在富媒体应用程序中具有优势，如果完全下载，这些应用程序会占用大量带宽。例如，胖客户端非常适合玩游戏。
*   更加灵活。在某些操作系统上，软件产品是为拥有自己的本地资源的个人计算机设计的。在瘦客户机环境中运行该软件可能很困难。
*   利用现有的基础设施。由于现在许多人拥有非常快的桌面和移动计算设备，运行胖客户端不需要额外的计算资源。
*   更高的服务器容量。客户端执行的工作越多，服务器需要做的工作就越少，从而增加了每台服务器可以支持的用户数量。这对一些组织的扩展工作非常重要。

## 瘦客户机/胖服务器的优势

*   长期稳定性和可靠性。在浏览器技术和特定于设备的技术中，环境的变化比服务器环境大得多，而服务器环境是按照制造商的时间表发展的，因此不太需要跟上浏览器技术和特定于设备的技术。
*   所需的前端计算不那么密集，这意味着更多市场的更多设备可以更轻松地运行该程序。
*   更容易跟踪和修复错误，节省支持此类问题的成本。
*   少写代码。随着越来越多的功能集中在一个中心位置，编写和更改代码库所需的工作量将会减少。
*   降低安全风险。有了更少的表面暴露和更稳定可靠的基础设施，暴露安全缺陷的场景就更少了。

## 我们现在在哪里？

我认为，总的来说，编程行业最近已经进入了胖客户端时代。更强大的设备以及本地和客户端 web 技术的进步使得丰富的体验成为常态。扩展客户端架构的好处使其对计算扩展工作具有吸引力。拥有更强大设备的大型组织和用户开始期待丰富的体验。一些组织正在利用支持富客户端的生态系统来构建出色的体验。其他人为了提供简单的信息，不必要地强迫用户下载大量的代码。在我看来，新闻机构是伤害用户体验的不必要的大客户类别中最糟糕的犯罪者。我会指出[Mic.com](http://mic.com/)是一个罪犯，尽管它提供简单的文本信息，却强迫我下载不必要的大量 JavaScript 和 CSS，滚动时经常出错，并且有一个主页在没有打开 JavaScript 的情况下甚至不能完全呈现。

## 我们要去哪里？

有许多趋势表明瘦客户机将会回归。“物联网”和消息传递作为平台的重新出现将推动将更多计算转移到服务器的流行。随着最大的硅谷公司努力争取在印度和非洲部分地区等新兴互联网市场获得市场份额，对各种规格和计算能力差异极大的设备的支持将推动这些公司将更多精力放在构建占用大部分计算能力的技术上。然而，在技术更先进的市场中，更聪明、更强大的客户将推动技术人员创建用户完全期待的更丰富的界面以及游戏和媒体应用程序。几家硅谷巨头都有自己的解决新闻网站膨胀问题的方案。脸书的[即时文章](https://instantarticles.fb.com/)和谷歌领先的开放[加速移动网页](http://thepracticaldev.com/@ben/accelerated-mobile-pages)协议是最引人注目的。这些不同方法的优点是有争议的。

在我看来，计算的历史是相当同质的。从网络计算的早期学术和军事用途到个人电脑寡头垄断的兴起，主要参与者努力模仿彼此最重要的功能。但是，随着越来越多的计算设备的出现，以及伴随着不同技术优势的全球市场而来的分裂的计算机需求，我预见这场辩论将远离统一的浪潮，而更多地朝着根据需求变化很大的分裂的最佳实践的方向发展。由于人类的群体性和一定程度的惯例的好处，总会有一定的趋势，但由于这种不均衡的全球化，计算的近期未来将与过去不同，我认为这种论点随着时间的推移将变得不那么二分法。
