# 我在软件开发的头六个月

> 原文:[https://dev . to/ajsmithsw/my-first-six-months-in-software-development-540](https://dev.to/ajsmithsw/my-first-six-months-in-software-development-540)

最初发布在我的[博客](https://mycoding.blog/2017/11/28/my-first-six-months-in-software-development/)

当我开始为这个博客写作时，我刚刚结束了为期 3 个月的自愿失业期，在此期间，我努力建立了一个工作组合，最终导致我获得了第一份技术工作。当时，我写下了那段旅程，目的是支持和鼓励其他人考虑类似的道路。

自从我踏上科技职业阶梯的第一步以来，六个月已经过去了，虽然时间不长，但我已经学到了大量在我进入这个兔子洞之前不可能知道的东西。事后看来，我收集了一些观察和建议，希望对任何想走类似道路的人有益。

## 寻求帮助

你刚刚说服一家公司雇用你，现在你已经开始了，你可能会觉得不知道自己在做什么是一个非常糟糕的开始。在我获得第一份工作之前，我自学编写(并出版了几个)移动应用程序，我认为我将带着广泛的有用技能进入这份工作，这将使我立即投入工作。然而仅仅过了一天，我就感到完全不知所措了。几乎立刻，我就沉迷于高级 Git 命令、Kibana 查询、设置构建服务器、理解云架构、管理 CI、Calabash UI 测试等等。

幸运的是，很多同事和导师都非常乐意帮助我重新站起来。从那以后，我和其他公司有类似经历的许多初级开发人员交谈过，所以谢天谢地，这似乎是这个行业的通病，不仅仅是我的公司。默默承受没有意义。你的团队希望你成功，并且明白你不会知道所有的事情，所以他们看到了回答你的任何问题的好处。一位同事对我问愚蠢问题的道歉回应道:“这里没有愚蠢的问题，只有愚蠢的答案”，这引起了我的共鸣，因为我继续尽可能多地从我的团队中吸取知识。

所以问吧，一直问下去。这是对你的期望。

## 学习曲线

正如我在上一节提到的，要为你的学习曲线中的一个非常尖锐的峰值做好准备。不仅在第一天，而且随着你和你的团队研究使用新技术或更好地利用你已经在使用的技术，将会有大量的东西需要学习。

你可以给自己一点先机。彻底学习版本控制，因为你将与其他人在相同的代码仓库上协作，所以你需要熟悉分支和发出拉请求。确保你已经学习了单元测试，因为你写的测试程序的代码比你写的构建你的功能的代码更重要。根据您选择的技术，了解 mocking 框架并尝试不同的单元测试框架。学习阅读和理解堆栈跟踪，这样您就能够更容易地跟踪 bug。

但即使你掌握了以上所有，你还什么都没看到。你需要为每天必须学习新东西的前景做好准备，并为此感到兴奋。

## 冒名顶替综合症

回想起我工作的头几天，我记得我感觉我已经成功地说服了我的新雇主，让他相信我可以做一份我完全不胜任的工作。我以为被发现只是时间问题，礼貌地护送到出口。

事实证明，很多人都有这种感觉，很多时候，我仍然有这种感觉。这是我已经意识到我应该习惯的事情。随着技术的进步和发展比以往任何时候都快，事实是你可能永远也不会成为你领域的主人。总有太多的东西需要学习和理解。新的框架，新的思维方式，甚至新的平台。你团队中的每个人，从高层到中层到你，都在经历同样的事情。这只是工作的一部分，所以不要担心感觉自己什么都不知道。因为你永远不会。没关系！

## 性别差距

在我进入科技行业之前，我一直在媒体行业工作。在我工作过的所有地方，女性都得到了很好的代表——在我之前的公司，整个员工队伍中有 55%是女性，我的大部分直属经理都是女性，这一点也反映到了高管团队中(我这里只说就业数字——我不知道薪水)。这是非常积极的，我坚信它在公司令人印象深刻的成功中发挥了巨大的作用。

在我进入科技行业之前，我就听说过女性在开发/工程领域的代表性不足。但当我第一次参加工程全体会议时，我仍然感到震惊，在一个挤满人的房间里，我只用手指就能数出房间里的女性人数。我的团队中有非常有才华的女性，她们约占总人数的 30%，但她们的角色主要涵盖产品管理、UX 设计和质量保证——只有一小部分编码工程师是女性。在我办公室的地板上漫步也描绘了一幅类似的画面。

到目前为止，这种观察可能是我遇到的软件工程中唯一消极的部分。仔细阅读这个主题，我认为这是西方特有的问题，这是一种文化偏见，认为程序员主要是男性。在英国，只有 16%的计算机科学本科生是女性，这是一个令人震惊的统计数字，而在印度，这一比例更接近 50:50。我认为我们的学术体系需要改革，更多的年轻人应该从小接触并鼓励他们对工程和基于逻辑的活动感兴趣。

## 一生的事业

当我开始申请工作时，我犯了一个错误。在我简历的顶部，电子邮件地址的下面，我添加了我的电话号码。当时我认为这只是每个人都会做的事情，毕竟如果你能直接与潜在雇主交谈，回答他们的任何问题都会容易得多。不幸的是，我低估了对开发人员的需求。在我把公开简历发布到网上后的几个小时内，数十家招聘公司抢走了我的简历，开始时，它对我有利——到第二天结束时，我已经获得了 3 次面对面的面试和几次电话面试，到第二周结束时，我可以选择工作机会。但是，尽管我已经把简历从所有发布过的地方删除了，6 个月过去了，我仍然每周至少会接到一个电话，每天都会收到几封直接的电子邮件，询问我是否对新的机会感兴趣。

有点讨厌，但我不那么介意。与一些招聘人员交谈会非常有见地和有趣，他们很乐意花时间与你交谈，即使他们知道你不会很快跳槽。这给了我很大的信心，当我离开目前工作的时候(我暂时不打算这么做)，最多只需要几个电话和几封电子邮件。我处在一个幸运的位置，雇主(一般来说)竞争候选人，而不是相反。这也为我提供了从职业生涯中休息一段时间的可能性，也许去旅行或学习一些新的东西，然后一旦我准备好了，就能找到一份工作。很安慰。
