# 6 个参考网站，帮助你学习 2018 年及以后的 web 开发…🚀

> 原文:[https://dev . to/code the web/6-reference-sites-to-help-you-learn-web-development-in-2018-beyond-edn](https://dev.to/codetheweb/6-reference-sites-to-help-you-learn-web-development-in-2018-and-beyond--edn)

[![6 Reference sites to help you learn web development in 2018 and beyond… 🚀](../Images/3aae10ec4a7d31dd89438f350ccb147a.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--_N6iiM1k--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://codetheweb.blog/assets/img/posts/web-development-reference-sites/cover.jpg) 
作为一个学习 web 开发的人，你应该经常查阅你不知道的东西。因为，让我们面对现实吧——在今天的世界里，知道*一种编码语言的每一个*特性是很愚蠢的。相反，作为开发人员，我们要去查资料！今天，我将向你展示六个最好的网络开发参考网站，当你不太记得具体是如何工作的时候。

## [](#what-is-a-reference-site)什么是参考站点？

如果你需要了解某件特定的事情，你可以去参考网站——比如说 CSS 中的`margin`属性取什么值，或者在 JavaScript 中如何求一个数的平方根。这是获得一个非常具体的问题的答案的简单方法。

## [](#how-is-a-reference-site-different-to-a-tutorial-site)参考网站与辅导网站有何不同？

像这样的教程网站旨在向你解释一些事情。因此，例如，在我的 [CSS 背景图片](https://codetheweb.blog/2017/11/29/advanced-background-images/)教程中，我用一种简单易懂的方式教你所有不同的背景图片属性是如何工作的。如果它是一个参考站点，它会显示如下内容:

> 背景位置属性采用一个或多个逗号分隔的<position>值。如果已经指定了多个背景图像，每个<position>被应用于相应的背景图像(第一个<position>用于第一个图像，第二个<position>用于第二个图像，等等)。</position></position></position></position>
> 
> (摘自 Codrops CSS 参考)

正如你所看到的，这非常简洁，如果你需要快速参考或提醒，这很好，但是如果你是第一次学习，需要帮助、例子和指导，这就不太好了。

## [](#1-w3schools)1。w3 学校

[![W3Schools](../Images/f0cb48c638b575ef30a7a2a9bda0e5e0.png)](https://www.w3schools.com/)
[W3Schools](https://www.w3schools.com/)是一个教程网站，但它们更简短，更具参考性。他们也有大多数 web 开发语言的专用参考页面。他们的格式很好。对于每种语言参考，他们都有一个侧边栏，上面有页面列表，所以很容易找到你要找的东西——我甚至不需要使用搜索栏！参考文献也非常简洁扼要，这在你需要快速提醒某件事情的时候很有用。我会说 W3Schools 有目前为止最好的解释。

### [](#reference-links)参考链接:

*   [HTML 引用](https://www.w3schools.com/tags/)
*   [CSS 参考](https://www.w3schools.com/cssref/)
*   [JavaScript 引用](https://www.w3schools.com/jsref/)

他们也有很多其他的引用，比如 [jQuery](https://www.w3schools.com/jquery/) 、 [PHP](https://www.w3schools.com/php/) 和 [SQL](https://www.w3schools.com/sql/) 。

## [](#2-devdocs)2\. DevDocs

[![DevDocs](../Images/db288094292b05063be30aa0be91a630.png) ](http://devdocs.io/) [DevDocs](http://devdocs.io/) 与 freeCodeCamp 非常相似，因为它有许多不同的语言(不仅仅是 web 开发)，从非常流行的到不太知名的。我真的很喜欢这个布局——它类似于 W3Schools，尤其是 freeCodeCamp 的侧边栏和下拉框。该参考资料非常棒，非常深入，而且很容易浏览并找到您需要的内容。

### [](#reference-links)参考链接:

*   [HTML 引用](https://guide.freecodecamp.org/html)
*   [CSS 参考](https://guide.freecodecamp.org/css)
*   [JavaScript 引用](https://guide.freecodecamp.org/javascript)
*   基本上其他的一切😂

## [](#3-freecodecamp-guides)3。免费代码营指南

[![freeCodeCamp Guides](../Images/a2bdc7b81b5025dcde138405b7db166d.png) ](https://guide.freecodecamp.org/) [ freeCodeCamp 指南](https://guide.freecodecamp.org/)是由社区制作，为社区服务。他们有一个 [GitHub 库](https://github.com/freeCodeCamp/guides/tree/master/src/pages)，你可以在那里实际贡献！这是我做过的一件事，很有收获。无论如何，有这么多优秀的人日以继夜地工作，这是一个很好的参考。布局也很好——类似于 W3Schools，它有一个侧边栏，显示所有不同的页面。然而，*每一种语言*都在侧边栏中，这意味着导航非常简单。

要访问特定语言的页面，只需在侧边栏中点击它旁边的箭头，就会出现一个包含所有页面的下拉列表。freeCodeCamp 指南的另一个很棒的地方是，它不仅仅适用于 HTML、CSS 和 JavaScript。他们有一堆其他语言，包括 C、Ruby、PHP 等等。他们还有额外的指导，比如在技术部门工作的[和在设计部门工作的](https://guide.freecodecamp.org/working-in-tech)[和](https://guide.freecodecamp.org/visual-design)，这些都非常有帮助。

### [](#reference-links)参考链接:

*   [HTML 引用](https://guide.freecodecamp.org/html)
*   [CSS 参考](https://guide.freecodecamp.org/css)
*   [JavaScript 引用](https://guide.freecodecamp.org/javascript)
*   一堆其他的东西(见侧边栏)

请注意，这些链接会转到主语言页面，要查看该语言的所有子页面，您需要在侧边栏中展开它。

## [](#4-mdn-web-docs-mozilla-developer-network)4。MDN 网络文档(Mozilla 开发者网络)

[![MDN Web Docs (Mozilla Developer Network)](../Images/fd93831a02ca6d126aa8f33e65a0e02f.png)](https://developer.mozilla.org/docs/Web/Reference)[MDN Web Docs](https://developer.mozilla.org/docs/Web/Reference)是由火狐浏览器的开发者 Mozilla 制作的。所以你可以想象，他们也是 web 开发的伟大权威！就我个人而言，我不喜欢 MDN 的布局，但是，它比 W3Schools 详细得多，并提供了更多的示例。

我想这取决于你在找什么——如果你在找一个你忘记的东西的快速参考，去 w3schools。如果您正在寻找更深入的文档，这些文档概括了您正在寻找的东西的所有内容，请选择 MDN。最终，这取决于你。两个都看一看，看看你更喜欢哪一个。就我个人而言，我总体上并不是 MDN 的粉丝，但许多人会不同意我的观点。

### [](#reference-links)参考链接:

*   [HTML 引用](https://developer.mozilla.org/docs/Web/HTML)
*   [CSS 参考](https://developer.mozilla.org/docs/Web/CSS)
*   [JavaScript 引用](https://developer.mozilla.org/docs/Web/JavaScript)

他们也有一些[其他更具体的引用](https://developer.mozilla.org/docs/Web)比如 [SVG](https://developer.mozilla.org/docs/Web/SVG) 和[画布](https://developer.mozilla.org/docs/Web/API/Canvas_API)的引用。

## [](#5-codrops-css-reference)5。Codrops CSS 参考

[![Codrops CSS Reference](../Images/0adbcdb8b115c97b214dc730f7e8aaa7.png)](https://tympanus.net/codrops/css_reference/) 最后但绝对不是最不重要的是， [Codrops CSS Reference](https://tympanus.net/codrops/css_reference/) 的布局非常棒，易于导航和使用。它们拥有*每一个* CSS 属性、函数、数据类型、规则等等。当你点击其中一个，它会带你进入一个页面，上面有官方语法、数值、例子等等。它甚至可以显示哪些浏览器支持该功能！

很像 MDN，很深入。然而，我更喜欢它而不是 MDN，因为即使有很多内容，它也以更好的方式组织起来，并且更容易导航和找到您正在寻找的内容。不幸的是，Codrops 只有一个 CSS 引用，而没有 HTML 或 JavaScript 引用:(如果有，那就太棒了，但目前我很高兴 CSS 引用的存在。

### [](#reference-links)参考链接:

*   [CSS 参考](https://tympanus.net/codrops/css_reference/)

## [](#6-stack-overflow)6。堆栈溢出

[![Stack Overflow](../Images/a57fbb62baaf18a31f3ee4e3d4dc5ad9.png)](https://stackoverflow/) 虽然[栈溢出](https://stackoverflow/)严格来说并不是一个参考站点，但它是一个获得编程问题答案的无价之宝。大多数时候，当你有一个问题时，你会发现它已经在网站上得到了回答——不用再看了！如果没有，你可以发布一个问题，通常会有人在发布问题后的一个小时(通常更短)内回复你(除非它真的很难懂或很棘手)。Stack Overflow 社区是有益的、友好的、慷慨的和聪明的，将帮助您解决任何问题。

有一个很棒的旅游页面解释了它是如何工作的。
