“你是怎么打入产品管理的？”

这是一个产品经理很快就习惯被问到的问题。随着角色从营销职能演变为交付职能，吸引力也在不断增强。

如果人们对亲临现场的兴趣在上升，那么数字世界正在燃烧。有各种各样的[休闲频道](http://slack.mindtheproduct.com/)、[脸书小组](https://www.facebook.com/groups/productmanagementgrowth/)、 [LinkedIn 小组](https://www.linkedin.com/groups/42629)和培训课程可供选择、参与和学习。产品是🔥。更好地管理它。

鉴于这种趋势，令人惊讶的是，很少有人利用可能是利用率最低的途径来获得实际的产品管理经验。

开源。

### 从开源开始你的产品生涯

我在其他地方写过关于开源的力量，它是进入像技术写作这样的职业的一种方式。[文档经常被描述为现代开发者文化中最大的未解决的问题——这也是我们在 [Corilla](http://www.corilla.com) 的使命。

但是产品管理呢？怎么样？

#### 开源多于代码

我知道你在想什么。开源不就是给开发者看的吗？不就是 GitHub 上的一种代码吗？跟企鹅和长胡子的愤怒的人有关吗？

不再

最初是一场鼓励开放共享源代码的运动，现在已经发展成为一个强大的产品模型，拥有一个非常多样化的社区。以下是有利于产品团队的变化的备忘单:

1.  开源赢了，现在意识到它可能需要赚钱，而不是乞求。
2.  像 Red Hat 这样的公司证明了开源项目可以成为可行的开源产品。
3.  消费技术的惊人进步已经成为[企业设计创新](https://thenextweb.com/contributors/2017/07/10/state-enterprise-ux-design-2017/)的[强大动力](https://uxmag.com/articles/the-future-of-enterprise-design-is-consumer-grade-ux)。
4.  开源社区提炼了一系列分布式协作模型，这些模型被证明对所有 T2 事物都有用。

#### 这对产品经理意味着什么？

初创公司和企业已经意识到[开源方式](https://opensource.com/open-source-way)给了他们市场优势。甚至像 [Twitter 发布 Bootstrap](https://blog.twitter.com/developer/en_us/a/2011/bootstrap-twitter.html) 这样简单的事情也展示了一个项目对生态系统的影响。

回到产品方面，我们看到 [Red Hat 的收入突破了 20 亿美元大关](http://fortune.com/2016/03/22/red-hat-revenue-2-billion-open-source/)，这是由超越“那个 Linux 公司”起源的对云服务的关注推动的。但是对于每一个红帽，其他伟大的技术团队如 RethinkDB 都有[崩溃烧毁](http://www.defmacro.org/2017/01/18/why-rethinkdb-failed.html)。这个部门是有效的——但只对那些有有效的[产品市场契合度](https://en.wikipedia.org/wiki/Product/market_fit)的企业有效。

现在在开源走出地下室拥抱产品文化或者死亡的同时——哦不——普通的消费设备突然变成了神奇的万能机器。而且看起来很神奇！该死的。谁能帮助弥合一个以开发者为中心的软件时代和被苹果设计至上美学的巨大成功宠坏的客户期望之间的差距？

这……这听起来很像是产品领导的机会，不是吗？但是这对你有什么好处呢？

#### 开源为契机

开源的直接好处是能够通过主动做某事来获得经验。通常在团队的鼓励和支持下，团队本身会从你所能做出的任何有意义的贡献中受益。

开源的长尾效应可能是最显著的。作为一名产品经理，你为开源项目所做的每一件事都会成为你发展的有力的公开记录。

证明你与团队合作的能力、主动的意愿以及在角色中的全面发展，更能让你获得突破性的角色。远远超过对以前公司工作或培训课程的模糊提及(也就是“你的简历没有发货的产品重要”)。

#### 开源为社区

同样值得重申的是，开源不仅是技术，也是社区。其中大多数总是需要所有可能功能的额外支持。不仅仅是软件开发将一个热情的项目变成实际的产品，还有像营销、文档、设计、UX 和…产品管理这样的角色。

### 两条开源路径进入产品管理

对于那些刚开始从事产品管理的人来说，有很多途径可以加入开源社区。这里有两个位于生态系统的两边。

#### 企业路线

在天平的一端是更大的和企业支持的项目和产品。这里的优势是社区结构相对较好，并且遵循某种形式的过程。它们由有才华的贡献者和员工提供资源，并包含某种形式的入职流程。

参与开源企业项目或产品的好处是，您将有机会学习一个定义好的过程。最好有知名公司有经验的产品经理的指导。请记住，许多开源公司倾向于直接从聚集在其生态系统周围的社区中招聘员工。

与此相反，你还会观察到规模的低效率以及上下游产品整合的挑战。像任何形式的领导一样，管理者的价值不仅仅来自于能够遵循既定的流程，还来自于能够获得💩完成时间💩倒下了。

#### 启动路线

如果你认为一个开源企业项目需要你，那就等到你遇到一个开源创业公司。你好👋。

参与这一级别的优势在于，你将更直接地参与到通常非常有才华的团队中，并获得更快的学习周期。

像任何初创公司一样，你会面临普遍缺乏流程的挑战。对于一个雄心勃勃的产品经理来说，这是展示他们对任何给定计划的成功的直接影响的完美时机。这些直接的挑战将会给你现实世界的应用(或灵感)来应用你正在进行的任何其他学习途径。创业公司是即时学习的发动机室。

如果企业是你不断学习和建立关系网的地方，那么创业就是你证明自己有能力以有意义的方式前进的地方。如果你和我一样，你可能已经在想…

### 你已经走了这么远(所以继续走)

如果有任何一点需要重复的话，那就是开源需要这种动机和产品焦点来生存和发展。如果你选择了这条鲜为人知的道路，你就进入了这个行业的转折点。这既是推进您的产品管理职业生涯的一个极好的方式，也是您将自己的想法和观点带入生态系统的一个机会，让您意识到[它不能只是不断乞求支持](https://changelog.com/posts/how-to-fund-your-open-source-projects) —它需要通过定义市场的产品来赢得支持。

我自己的旅程离这一经历不远— [科里亚的故事如今被很好地记录了下来](https://www.youtube.com/watch?v=9mNgs_zvsLI)但却是建立在这条精确的道路之上。一群技术作家在内部构建他们自己的内容产品。然后这一次旋转出正确地建造它。什么会出错，对吗？

如果有什么我能做的来帮助你在类似的情况下找到自己的机会，请告诉我。无论你被什么吸引，请开始吧。如果这是你正在执行的任务，那就让它实现吧。你越早这样做，你就能越早为这个世界做出一些令人惊叹的事情。
