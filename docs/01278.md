# 新开发年度决议

> 原文：<https://dev.to/de/new-dev-year-resolutions-1j2a>

[![](img/8998565daa35c70ebec3c2c1806082fe.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--ZwhUO-zg--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/l589cevt6wosnuv5uq3y.jpg)

1.  冒名顶替综合症被取消了。*每个人都有。*虽然这可能是我们行业中最热门的流行语，但我不想再用它了。因为至少对我来说，坐在那里感觉我写软件很烂实际上是令人愤慨的，因为我花了-不夸张-至少 75%的醒着的时间做这件事。不只是这样做，而是一丝不苟地确保我一直遵循最佳实践。今年我要给自己一些信任。作为一名开发人员，我将更加认真地对待我的成就、我的成长和我自己。我会记得我努力工作才获得今天的成就，这个星球上没有人会一直付钱让我做我不擅长的事情，因为我没那么幸运。当有人称赞我的专业技能或者任何相关的东西时，我不会尴尬的笑或者开一些自嘲的玩笑，我只会说谢谢。

2.  整理我的社交生活 -这本来不在我的决心清单上，直到我算了一下，打出“我醒着的时候至少有 75%的时间在写代码。”所以，我就把这个放在这里。

3.  **阅读**——这个真的没什么好说的。大学毕业后，我就没怎么读书了(书库满了，文件多了，我的星座没算进去)，这个问题应该解决了。我可能永远不会成为那个能在一个周末读完一整部《哈利·波特》的书呆子，但是一个月读一本书对我有好处。一年 12 本书。我想我能处理好。

4.  去度假——我在工作中从不休假，这是出了名的。今年我会带很多。或者至少，我能分到多少就有多少。

5.  **Contribute** -就在几周前，我参加了一个研讨会，主持人在演讲时遇到了一个错误。在他试图调试的整个时间里，我让程序在本地运行得非常好，因为我在编写代码时发现了错误。所以我就在那里，看着主持人和与会者一个接一个地研究可能的解决方案，我只是坐在那里想说点什么，一想到要大声说出来就头昏脑胀。在某个时候，我悄悄地把我的发现告诉了任何人，谢天谢地，我旁边的人听到了，大声喊了出来。我告诉你，那天下午我没有帮任何人一个忙。尤其是我自己。我发现我可能有点不理性地害怕更大的开发者社区，这让我可以说是一个壁花。很明显，今年我朝着成为最好的开发人员这一最终目标“升级”的最大努力将包括克服自己(见解决方案 1)并对贡献更加满意。无论是开源项目，还是通过合作，畅所欲言而不必担心被误解或被忽视，甚至只是加入到 [dev.to](https://dev.to) 和其他论坛的技术对话中来。今年我必须彻底摆脱困境，保持我的个人轨迹。
