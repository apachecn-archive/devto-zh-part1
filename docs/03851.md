# 为什么我在开发者大会上发言

> 原文:[https://dev . to/wired ferret/why-I-speak-at-developer-conferences-9cm](https://dev.to/wiredferret/why-i-speak-at-developer-conferences-9cm)

我不以写代码为生，也从来没有过。开发人员从来不是我职位的一部分，我的 Github 历史不会给任何人留下深刻印象。我想这就是为什么人们对我在开发者大会上发言感到惊讶——下个月我将去 RubyConf、PyconCA 和 next 月。

[![](../Images/bdfc0d9391c685891729e396437357a5.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--3KMoKURu--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.heidiwaterhouse.com/wp-content/uploads/2017/10/4d49a60f-0f35-4827-a38f-581c437bda83.jpg)

当我开始在会议上发言时，我认为我只是“被允许或”有资格在技术写作和通才会议上发言。随着我对我的信息越来越有信心，我意识到思想的交叉传播有很大的价值。当我与更多的开发人员交谈时，我意识到他们发现最棘手的谈话不是关于如何做某事，而是关于做什么是可能的。

想想你在会议结束后记得的演讲。他们是如何执行棘手或新的东西的 bravura 现场编码的例子吗？或者是关于你能做什么，你如何用不同的方式思考问题，未来可能会发生什么？展示当前的事物是重要的，但讨论我们想在这个世界上处于什么位置和成为什么人也是重要的。未来。

大多数会议委员会试图根据经验、代表性、预期观众水平、技术深度以及对与会者、赞助商和雇主的吸引力来平衡演讲和演讲者。我们需要进行深入的技术对话，我们需要就精神健康、可访问性和可用性进行对话。不是非此即彼，也是——而且。

所以我在开发者大会上发言来平衡原力。我这样做也是因为我想在这个世界上展示自己，成为技术专家和红发人。我想和那些倾注精力学习不同事物的人分享我几十年的经验。我认为我带来了价值，显然会议组织者也同意这一点。

[![](../Images/14d4b0c971083abdb5c0c222ffc737aa.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--mO40WbYf--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.heidiwaterhouse.com/wp-content/uploads/2017/10/img_0876.jpg)

你有没有想过与众不同能给会议带来什么？如果你觉得你不能参加比赛，因为你对通常涉及的话题没有任何新的东西可说，那么考虑覆盖一个你在会议上没有见过的话题。如果有很多代码演示，可以考虑做一个特性概述。如果你有与会议主题相关的专业知识，有时会有助于人们以不同的方式理解你在说什么。我有一个关于编织和文档以及我们如何教授代码的演讲。

[![](../Images/a2e002e2c10a43bb6713391aeac4cfef.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--hI2t9sDM--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://www.heidiwaterhouse.com/wp-content/uploads/2017/10/img_0875-1.jpg)

如果你是一个“非技术性的技术人员，不要让这阻止你向会议提出建议——你仍然有宝贵和有意义的经验可以分享。如果你想集思广益，请给我留言。
