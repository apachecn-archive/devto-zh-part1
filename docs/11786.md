# 我的“大白开页”问题

> 原文:[https://dev . to/Brian nipper/my-great-white-open-page-problem-ijl](https://dev.to/briannipper/my-great-white-open-page-problem-ijl)

当我第三次重新开始写作时(对自己叹了一口气)...我已经意识到我的大脑是如何工作的。

当面对一个特定的任务时，我的大脑会迅速形成一个解决方案。当我遇到问题时，我似乎无法停止想出解决问题的方法。

当我在工作中遇到一个问题时，问题的范围通常很小，工作中问题的范围通常是在一个区域接受信息/数据，处理/修改所述数据，然后将数据保存/发送到另一个区域。现在，当遇到一个可以用如此狭窄的术语定义的问题时，我很容易想到解决该问题的几种不同的方法，然后当经历这些不同的选项时，每个解决方案通常都有一系列的优点和局限性，这就变成了一个简单的问题，与相关方讨论这些优点和缺点，并提炼解决方案的要求，以隔离哪个选项最适合。

所以循环通常是:

*   描述一个问题，
*   “研究”问题，
*   定义解决方案，
*   基于“范围”过滤解决方案
*   执行剩余的解决方案。

我将根据不同的复杂程度每隔几个月完成几次这个循环，从影响几十个用户的孤立问题到可能影响几千个用户的事情；然而，有可能影响一万名用户的事件(这在我工作的公司永远不会发生)看起来并不那么势不可挡。

鉴于这种类型的日常微笑，人们可能会认为解决我自己的问题会更容易，但这就是问题所在，即“白色大开页”问题。

我知道我需要定期写作，以成为一个更好的沟通者，并提高我的工作技能(作为一名程序员，我认为最重要的技能之一是沟通)。然而，当我开始写作时，我经常发现我只是盯着一张白纸，什么也没有想到。

所以当我在电脑屏幕前来回走动时，我希望在接下来的几周甚至几个月里形成某种类型的结构来缩小我的问题范围，这样我至少可以尝试解决“大白页”问题。
