# 行高...行高...行高

> 原文:[https://dev . to/design frontier/line-height-line height-line-height-k0c](https://dev.to/designfrontier/line-height-lineheight-line-height-k0c)

这是一份公共服务协议:互联网上有一个漏洞。这是一个微妙的错误，但到目前为止，我检查过的每个浏览器都有这个错误的一些变体。

那么这个 bug 是什么呢？它会吃掉我的网站吗？它携带疾病吗？是病毒吗？！会有丧尸吗？

好了，冷静下来。让我以相反的顺序回答这些问题(因为这样更有趣)。

这个 bug 的结果是不会有僵尸了。我保证。我们对此非常确定。

它不是病毒。互联网上有许多病毒，但这不是其中之一。

它似乎没有携带疾病的能力。

一般来说，它不会吃掉你的网站，尽管它可能在你不知道的情况下就存在于你的网站上。这通常不是问题，但是如果你的网站出现奇怪的行距、斑点、模糊的视觉、宏伟的错觉、丑陋的布局、糟糕的设计，或者开始表现奇怪，我们建议你咨询专业人士。此外，如果你的网站开始与你交谈，请寻求适当的医疗或专业关注。

这个错误存在于各种浏览器处理数学，特别是行高的方式中。例如，在你能找到的尽可能多的浏览器中查看这个链接:[http://jsfiddle.net/95cqM/1/](http://jsfiddle.net/95cqM/1/)

行高值的变化取决于它是仅被指定为一个比率，还是使用 em 度量单位(一个比率)。除了 Firefox 之外的所有浏览器都演示了这部分错误，为理论上等价的两者返回不同的值。

错误的第二个方面是，真的不应该有一种方法来指定分数像素。分数像素的概念实际上并不存在。像素大小是一个整数，不允许有小数，因为计算机不知道如何绘制半个像素。至少据我所知…

实际上，这更是一个有趣的发现，而不是一个问题。除非你在 javascript 中基于行高做一些计算，否则你不会看到任何奇怪的行为。如果你是…好吧，你需要在你的计算中考虑这种怪异。欢迎来到互联网！
