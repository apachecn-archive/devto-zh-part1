# 我对 Microsoft Access 的爱恨情仇

> 原文：<https://dev.to/funkysi1701/my-lovehate-relationship-with-microsoft-access-1ok8>

由于历史原因，我使用 Microsoft Access 作为我所有业务线数据库的前端。

Access 允许您创建可以连接到 SQL Server 数据库的 ADP 文件。然而，office 已经放弃了对此的支持，所以我可以使用的最新版本的 Office 是 Office 2010。

在我开始讨厌 Access 之前，这个平台有一些好的方面。知识很少的用户可以相当快地创建一个数据库。开发时间也相当短，因为创建一个新表单并添加几个打开不同报告的按钮并不需要很长时间。

在过去 5 年左右的时间里，我在 Access 中构建了一系列不同的表单，(即使我自己这么说)，我已经非常擅长于创建用户可以更高效地使用的表单。Access 有一个子窗体的概念，它是一个嵌入在窗体中的窗体，每个窗体都可以连接到一个不同的查询，并且可以相互连接，因此创建一个非常强大的东西是相对容易的。

Access 报表是一种创建专业外观报表的简单方法，这些报表可以发送给客户或由管理层使用，它们还有子报表的概念，(与子窗体相同，但嵌入在报表中)，这意味着您不必编写一个查询来取回报表中显示的每一条数据。

VBA 或 Visual Basic for Access 允许您以编程方式轻松地完成一些事情，例如，如果选择了某项内容或单击了某个按钮，则运行某些内容。这不是最容易使用的编程语言，但是一旦你使用了一段时间，你就会习惯它的怪癖。我最喜欢的函数是 IIF，语法相当简单 IIF(expr，truepart，falsepart)，如果 expr 的值为 true，truepart 就被执行(执行的过程可以简单到在特定条件下显示特定文本)，如果为 false，falsepart 就被执行，你也可以把它们链接起来，我已经这样做了，以产生一些看起来非常复杂的东西。

现在说说事情的消极面。Access 创建二进制 ADP 文件，在 git 或 subversion 中，二进制文件不能进行版本控制。这是一个大问题，因为我想跟踪我所做的每一个改变。这也意味着在大项目上分支或与他人合作也很困难。

我发现解决这个问题的一个办法是[访问 SVN](https://accesssvn.codeplex.com/) 。这个程序采用 ADP 并创建每个表单和报告的文本文件。它也可以从这些文件中创建 ADP，但是我在这个特性上取得了不同程度的成功，所以如果可能的话，我更喜欢只创建文本文件。在对 ADP 做了更改之后，我要做的是运行 AccessSVN，创建文本文件，并将它们提交给源代码控制。然后，您可以看到哪些控件已被更改或添加。它也有助于解决 bug，因为我发现有时在测试时 access 会对表单应用一个过滤器，这将破坏部署，这非常烦人。

Access 只能做 access 设计要做的事情。有一些高级功能，我们想添加到我们的数据库。比如发送电子邮件或向网站上传报告。这不是访问错误，因为它从未被设计为能够做这样的事情，但我经常被问及什么是可能的。

对 Office 2010 的支持即将结束，我计划停止使用 Access，但在此之前，我需要继续支持我们现有的前端。
