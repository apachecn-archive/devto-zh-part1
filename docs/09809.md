# 为什么结对编程？

> 原文:[https://dev.to/geisonfgfg/why-pair-programming](https://dev.to/geisonfgfg/why-pair-programming)

最后，我听到了很多关于结对编程的讨论，通常是争论它的重要性，什么时候它是必要的，是否可以被代码审查所取代。因此，在这篇文章中，我会给你我对此的想法和经验。

通常结对编程的结果与代码质量有关，但是从我的经验来看，我可以肯定地说这不是它的主要好处，还有 3 个更重要和不可替代的结果:

1.  **让团队的新成员在工作的第一天就能有所作为:**即使在技术上有能力做出贡献，团队的新成员也没有必要的业务领域背景来修复 bug 或开发产品的新功能，因此与团队开发人员合作可以让新成员在第一天就做出贡献，并加快他的入职流程。
2.  **避免知识孤岛:**当一个团队有至少两个开发人员在代码或业务上下文 it 的确定部分工作时，允许开发人员休息几天而不妨碍团队，因为总是会有其他人在必要的上下文中继续工作。
3.  **解决开发人员遇到的问题:**作为开发人员，有时我们需要解决复杂的问题，当问题发生时，让其他人帮助找到解决方案是非常有用的。只要知道有时候你并不孤单，就足以给你挑战问题的勇气。

你是否意识到所展示的 3 个主要好处与分享知识有关，而与质量代码无关？

因此，如果你问我:

结对编程的重要性是什么？

**答案:**分享知识！

结对编程可以被代码审查取代吗？

**回答:**不，因为代码评审不能让团队的新成员在工作的第一天就富有成效，不能避免知识的孤岛，也不能解除开发人员对某些问题的困扰。

什么时候应该使用结对编程？

**答:**每次分享背景或知识都有助于解决问题。
