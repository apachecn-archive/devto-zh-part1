# 我真正喜欢的是“太好了，他们不能忽视你”和其他漫谈

> 原文：<https://dev.to/warnero/what-i-really-liked-about-so-good-they-can-t-ignore-you-and-other-ramblings-gl3>

所以，自从我拿起这本书，在大约一周的时间里从头到尾读了一遍(可以这么说，我拥有了我如饥似渴阅读的电子书)，已经过去了大约两年。

首先，我想解释一下我在职业生涯和生活中的位置。我不开心。不是生活的一部分，而是我的目标。在我的职业生涯中，我一直认为，为了让自己快乐，我需要“追随我的激情”。我终于全力以赴，做到了这一点。我白天工作，下班后和几个朋友一起创业。

在这样做了 2 年多，经历了更多的挑战和至少一个我们正在建设的支点后，我累了。我的朋友不再帮助我了。我做了所有的重活，为了什么？我们哪儿也不去。我们的用户基数在两位数以下。我累了。我拔掉了插头。临近假期时，我无精打采。我不确定我应该做什么，甚至不确定我想做什么。我不记得我是如何听说“太好了，他们不能忽视你”的，是一个读过它的朋友还是我在某个地方看到的一篇评论，但我立即被激起了兴趣。

据我所知，它有像“不要追随你的激情”，“变得优秀，我的意思是，真的，真的擅长某事，你会找到你的激情”。

在我坐下来度假之前，我对自己充满了疑问。我在做什么？我在做“我喜欢”的事情吗？我在做我自己的事吗？为什么我在职业生涯中从未快乐过？少了什么？

是的，创业很难。尤其是试图从头开始。这绝对是我坚持了两年多，试图去工作的事情。有时候我在建造它的时候很开心，但是仍然缺少一些东西。

所有这些想法，问题和想法在我的脑海里，我坐下来读这本书。

我立即被作者卡尔·纽波特采访的不同人的许多陈述和描述的真实性所震惊，这些人在追求梦想的过程中有成功的也有失败的。让我印象深刻的一部分是这个——“如果年轻的史蒂夫·乔布斯接受了自己的建议，决定只从事他热爱的工作，我们今天可能会发现他是洛斯阿尔托斯禅宗中心最受欢迎的老师之一。但他没有遵循这个简单的建议。”这是在介绍部分，有点让我吃惊。我喜欢史蒂夫·乔布斯，他对创造一些最具标志性和改变世界的设备的热情一直是我的灵感来源。在这之后，我完全被迷住了。

这本书大致分为三个部分:

*   忘了“追随你的激情”吧——那是胡说八道
*   做个好人，让他们无法忽视你——培养你的技能
*   拒绝升职——掌控你想去的地方
*   从小处着眼，大处着手——肩负使命

我特别关注的一个领域是技能部分。他提出了一些有趣的建议，我选择在一定程度上实施了这些建议，并且我仍然在结构方面进行尝试。在这一节中，他谈了很多关于工匠心态的问题，这是我在职业生涯后期才开始重视的。不要在新的闪亮的技术上工作，而是真正学习磨练你的手艺，变得尽可能好，然后走得更远。

我通过项目做到这一点。我有几本书，根据我想学的内容，我把它们放在一边。每当我从事一个项目时，我都有一系列的事情要努力去完成，无论是研究一项新技术，尝试一项新技术，尝试一种不同的语言，或者只是试图在某方面做得更好(测试)。

起初，我试图塞进太多的内容，我试图遵循他的方法，大致包括:

*   深思熟虑的练习——选择你的弱项并努力提高
*   延伸——强迫自己去解决对你来说太大而不能强迫自己去学习的事情
*   小赌注——在许多小领域进行研究，寻找大的目标

刻意练习。对于程序员来说，这可以采取编码形式，这种练习迫使你思考一个问题的解决方案，而不是一个，而是几个。你可以找到几乎任何语言的代码表，这是学习一门新语言的好方法。你也可以拿一个你有完美解决方案的问题，自己编码几次，强迫自己想出一个全新的解决方案。

拉伸。对于程序员来说，这意味着深入研究一个你不熟悉的计算机科学领域，或者一个你有些熟悉但想了解更多的领域。对于 web 开发者来说，这是一个很大的领域。对我来说，一个领域是机器学习。这个地区让我着迷。我只是触及了这个问题的表面，希望在不久的将来能够深入研究。

小赌注。这里的想法是，你有一个整体的任务，你要去，但你不太确定如何到达那里。所以，你想出一堆小点子，你可以在一个月左右的时间内解决，看看你是否取得了一些真正的进展。如果你没有，那就把它放在桌子上，试试你的其他想法。

我从哪里得到这些想法的？我现在和他们在一起做什么？

我试着把它们融合在一起，在一个晚上编码，然后是项目，小赌注等等。但是我没有时间全部做完。相反，我决定在我想做得更好的特定领域，并真正投身于这些领域。我挑选了一些对我日常工作有帮助的东西，那些一旦我学会了就会让我的日常工作变得更容易的东西。我在一定程度上仍然遵循这一哲学。但我也从事个人项目，从长远来看，这些项目将帮助我变得更好，并帮助我随着时间的推移实现我想要的目标。

不过，总的来说，这种新的心态把我带到了哪里？

*   我意识到我真正想做的是从头开始帮助建立一个团队和一家公司
*   我在一个我没有真正热情的领域找到了这样一家公司，但我确实有热情与这些人一起建立一个解决方案和一家公司，它充满了有趣的挑战，有助于拓展我作为开发人员和个人的能力
*   我继续专注于学习更多的东西，挑战自己，并找出我想解决的下一件事是什么
*   意识到我离我想成为的工匠还差得很远，但要继续朝着那个目标努力，也要意识到在任何时候，对我来说，还有其他更重要的事情需要关注

如果你对采用这种思维方式感兴趣，这里有一些想法:

*   做一些[编码动作](http://codekata.com/)
*   读一本书[《好到他们不能忽视你》](http://calnewport.com/books/so-good/)
*   弄清楚你人生中的使命是什么，并努力去实现它

归根结底，这很大程度上是关于找出你擅长什么，你在哪里有资本，并尽你所能做到最好。只有通过这样做，你才会发现你真正的激情是什么。这里只是我回答这个问题的一个小样本——我擅长什么？我在哪些方面积累了资本，可以继续发展？

*   我是一个很好的开发者，我正在变得更好:)
*   我也是一个相当不错的作家

经过很长一段时间的中断，我再次将这个博客复活，并将继续分享我成为软件工匠的旅程。
