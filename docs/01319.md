# 亲爱的互联网，停止通过 HTTP 获取我的信息

> 原文:[https://dev . to/Kyle Galbraith/dear-internet-stop-take-my-information-over-http-3c3k](https://dev.to/kylegalbraith/dear-internet-stop-taking-my-information-over-http-3c3k)

对于开发者来说，通过 HTTPS 配置网站访问从未如此简单。像 AWS Certificate Manager 和 Let's Encrypt 这样的服务让它变得既便宜又简单。

当然，从来没有借口通过 HTTP 获取用户信息。为您的网站配置 SSL 曾经有些困难。如今，这已经变得非常简单，而且这样做的必要性从未像现在这样重要。

SSL 并不能保证 100%的安全，比如有人在中间攻击。但是，它确实降低了潜在的风险。

### [](#anatomy-of-a-person-in-the-middle-attack)解剖一个中途发作的人

中人攻击的一般前提是客户端和服务器之间坐着一个邪恶的人。永远不要相信邪恶的人。这个邪恶的人总是在那里，你永远不应该假设他们不在。

我们的反面角色正在监控从客户端到服务器的请求。他们想要获得关于用户的信息。对他们来说，密码、电子邮件甚至电话号码都是制胜的标准。在坏人看来，通过 HTTP 从客户机传递到服务器的请求是明文。

这是一个人在中招(PITM)时的一般解剖。不应该通过 HTTP 传递密码、电子邮件和其他敏感信息。攻击者很容易窃取信息并利用它来对付你。

### [](#add-ssl-to-your-website)将 SSL 添加到您的网站

如今在你的网站上添加 SSL 证书非常便宜。我最熟悉在 AWS 证书管理器中创建它们。假设您已经有了一个 Amazon Web Services 帐户，您可以按照以下步骤创建一个 SSL 证书。

1.  导航到 AWS 控制台中的证书管理器。
2.  单击申请证书。
3.  在域名输入中输入你的网站域名。
4.  单击向该证书添加另一个名称
5.  在域名输入中输入 my-awesome-site.com
6.  单击下一步。
7.  选择电子邮件验证。
8.  单击查看。
9.  单击确认并请求。

对于输入的每个域名，您必须通过 AWS 发送的电子邮件确认您是该域名的所有者。这将到达你注册为域名所有者的电子邮件地址。批准邮件来自主题为`“Certificate approval for your-site.com”`的`no-reply@certificates.amazon.com`。

单击电子邮件中的批准链接以批准证书请求。

在 AWS 中获得证书后，您可以将它附加到 EC2 web 服务器前面的负载平衡器。如果你有一个静态网站，你可以把它附加到你的 S3 网站前面的 CloudFront 发行版上。

### [](#conclusion)结论

互联网越来越倾向于 HTTPS。像谷歌 Chrome 这样的主流浏览器开始在用户通过 HTTP 浏览网站时向用户显示警告。曾经有点麻烦的事情现在变得如此简单，没有理由不去做。简单明了，如果你在你的网站**上获取用户信息，不要通过 HTTP** 进行。
