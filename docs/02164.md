# 延迟的成本:同步特征开发

> 原文:[https://dev . to/bosepchuk/cost-of-delay-simultaneous-feature-development-9 F4](https://dev.to/bosepchuk/cost-of-delay-simultaneous-feature-development-9f4)

同步特性开发是一种常见的，但通常在经济上不是最佳的软件开发实践。在这篇文章中，我将着眼于同步特性开发的财务影响，并向你展示你可以做些什么来获得更好的结果。

### [](#what-is-simultaneous-feature-development)什么是同步特征开发？

同步特性开发是一种常见的软件开发实践，一个团队可以同时开发多个特性。例如，如果你在做 scrum，你会把一堆故事拉进你的 sprint。如果你在冲刺阶段结束时完成了所有的故事，每个人或多或少都会感到高兴。你的团队对正在进行的工作没有限制。所以你在任何方便的地方工作。

团队参与同步的特性开发，让团队中的每个人都一直很忙，这是不正确的优化变量。

### [](#why-should-you-care-about-simultaneous-feature-development)为什么要关心同步特性开发？

同步特性开发的问题在于，通过同时开发多个特性，你会延迟每个特性的交付。如果这些功能增加了收入或降低了成本，你就是在消耗你公司的利润。因此，您的团队可能会达到所有的速度目标、质量目标，并将最重要的工作放在您的 sprints 中，但仍然会错过大量的盈利机会。

当然，你的团队可能有很好的理由并行开发特性。你可能有一个很大的团队，让 50 个人专门开发一个 200 行的特性是没有意义的。或者，您可能有一个交付一篮子功能的截止日期，而满足该截止日期的唯一方法是并行工作。

### [](#how-you-can-improve-your-outcomes-by-using-cost-of-delay)如何利用延迟成本来改善结果

你不能武断地应用“没有同步特性开发”的规则。你需要计算你的延迟成本，选择最经济的途径。

#### [](#an-example)举个例子

假设你正在开始一个新的四周冲刺，你有 4 个特征要完成。每个功能将需要一周的时间来开发。而你每延迟一个功能的推出，你就损失 5000 美元——这就是延迟的代价。

如果您同时开发所有的特性，并在 sprint 结束时部署它们，这是财务影响。

如果您按顺序开发您的特性，并在开发完成后立即部署它们，这将带来财务影响。

[![sequential feature development](../Images/bfb3aacedc336f6e316f1a4521da74b0.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--IEcz_d5Z--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/gxt4lt267k0h9736r4k4.PNG)

只要对你的工作计划做一个简单的改变，你就能对你的项目的盈利能力产生巨大的影响。在这种情况下，您可以**将延迟成本降低 37.5%！** ((50K - 80K)/80K = 37.5%)

你看到这有多大了吗？你不需要更加努力，雇佣更多的人，或者学习函数式编程。你所要做的就是安排好你的功能，使延迟成本最小化。

关键见解:**等待是有代价的**。如果你的特性需要开发人员花费 50 个小时的时间来完成，但是在最终交付给你的客户之前，它要花费几个月的时间在各种队列中等待，那么这个特性的真实成本可能是程序员 50 个小时工资的许多倍。**大多数企业都专注于 50 小时的程序员时间**，并确保他们所有的人都有很高的利用率。**但是他们对潜在的数万美元损失视而不见，因为这个功能花了几个月才发布，而它本可以在几天内发布。**

### [](#wrapping-up)包装完毕

在我了解延迟的成本之前，我对这个增加盈利的巨大机会完全视而不见。如果我们在冲刺阶段致力于高优先级的任务，并达到我们的速度目标，我们会很高兴。但我们肯定在桌上留了很多钱。如果故事没有通过多次代码审查，我们会经常允许它们在多次冲刺中逗留。这些故事可能会连续几周屏蔽整个功能。但是我们从来没有想过我们在流血。

计算延误的成本并利用它来安排我们的工作需要我们做一些调整，但这是非常值得的努力。我将在后续文章中详细阐述这些想法，并向您展示更复杂的示例。敬请关注。
