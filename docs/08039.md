# 计划您的提交

> 原文:[https://dev.to/rpalo/plan-your-commits](https://dev.to/rpalo/plan-your-commits)

我不知道这是否常见，但我真的很难记住提交我的代码。我知道提交应该是有意义的、合乎逻辑的工作块，从整体上来看，它们讲述了一个故事。可是...我仍然发现自己一直在突突前进，直到我准备好从编码中醒来，却没有做出任何事情！我的提交是一个巨大的 blob，带有类似“完成了大部分特性”的消息希望有人和我一样。希望这篇文章能告诉那些过度关注一个问题而忘记承诺的人(或者移动，或者吃东西，或者出去，或者和其他人说话)。因为我发现了一个解决办法。

计划。你的。提交。

就是这样！神魂颠倒了吧。谁能想到这样的天才？！肯定没有人考虑过在你现在做之前预见你将要做的事情！闭嘴，这对我来说是个新主意。以下是我的思考过程:

我读过的很多包含 Git 的教程似乎都假设你已经是一个专业的提交者了。他们会说这样的话，“好了，这看起来是一个提交的好地方，”或者“你现在想提交你的工作了。”这让我相信，我可以只是跟着代码走，当我应该提交时，这对我来说是非常明显的。

[![Nope.](img/d2a52fdf2df685ba06a34516de602a46.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--ERp56aS3--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://assertnotmagic.com/img/nope.jpg)

然而，我发现如果我预先选择我的检查点，我会忘记得更少。说类似“我完成这个函数后会再次提交”或“我连接好 url 路由后”这样的话效果会好得多。大部分是。我的想法是，一旦我到了那里，我可以承诺，出来透透气，看看我在哪里，看看我需要去哪里。这也有助于我不需要把所有事情都记在脑子里，如果我离开这个项目几天，想回来的话，这也有一定的帮助。写下来的东西越多，在我脑子里蹦蹦跳跳的就越少。

所以，如果你的提交看起来像一堆`[rhymes with commit ðŸ’©]`，试着在你写代码之前写下你的提交信息(在你的头脑中)。如果你有更好的方法，或者，如果你有其他能帮助我长期保持头脑清醒的诀窍，请分享！我很想听听。
