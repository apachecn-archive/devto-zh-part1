# 花点时间解释

> 原文:[https://dev.to/pbeekums/take-time-to-explain](https://dev.to/pbeekums/take-time-to-explain)

有一篇很棒的文章讲述了 Kotaku 如何发布了一篇关于游戏开发者使用的巧妙技巧以及一些开发者如何批评它的文章。为什么要发一个解释如此基本的东西的帖子呢？谁在乎呢。 [![](../Images/3439999889f463012a5d0e49c532d5e7.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--JjkxNqUP--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://blog.professorbeekums.com/img/2017/4/conversation.png) 显然很多人都这样做了。一个人认为基本的东西对另一个人来说实际上是相当聪明和有趣的。为什么会有人想要嘲笑一个人的好奇心？每个人一开始都知道得不多，获得这些知识通常需要有人愿意解释。

更重要的是，指望人们理解某些事情有多难，同时批评任何向他们解释这些事情的尝试，这是荒谬的。

这也发生在游戏开发之外。

我见过很多持类似态度的软件开发人员。他们更希望每个人都盲目相信自己的判断。开发者终究是最聪明的！解释技术决策背后的推理只是占用了他们太多无限宝贵的时间。

还有一些人认为开发人员只不过是实现他们宏伟愿景的普通人。“我告诉你怎么建就怎么建！我不在乎细节。”

这两种态度都会导致团队工作效率低下。

我以前对别人有那种有害的开发者态度。很容易陷入那种部落主义的感觉。“我们对他们”的心态会让人上瘾。在很多方面，抱怨“他们”甚至很有趣。

最终我意识到我的行为给我的同事带来了困扰。奇怪的是，我认为他们是工作时间以外的朋友。由于内疚，我决定在工作时间开始像朋友一样对待他们。如果他们有问题或需要解释，我会花时间尽我所能提供最好的答案。当我有工作要做的时候，即使花去我一个小时的时间也没关系。为了我的朋友，这是值得的。他们也花时间来回报。

一开始是想帮忙，结果变成了极其有效的团队动力。通过真正团队应该有的沟通方式，我们最终更加了解彼此。我们不得不做出的决定变得更快，因为我们有彼此来帮助消除这些决定的不确定性。如果 X 被做了，其他人会有什么反应，内部没有争论。与其他人交谈更容易。

这些决定不仅做出得更快，而且更好。我们提出了一些想法，如果我们都停留在自己的专业和部落泡沫中，我们谁都不会想到这些想法。

解释事情所花的时间不是浪费时间。这是为了以后能够更高效地工作而进行的前期投资。

我建立 professorbeekums.com 的原因之一就是帮助人们进行投资。所提供的课程将帮助每个与开发人员一起工作的人了解软件开发是如何工作的。这些知识是与开发人员有效沟通和工作的坚实基础。
