# 代码正在遭受痛苦

> 原文：<https://dev.to/ben/code-is-suffering>

不，你将无法轻松地扩展你的 CSS。您试图通过编译、内联、严格的约定等来规避它的缺点。会有所帮助，但你并没有从疼痛中解脱出来。你只是在一条路上。你试图在客户机上增加丰富的交互，这将使像引用图像路径这样的基本操作变得更加困难。当你度过这个难关时，情况可能会好转，但另一个难关即将到来。你想提高服务器性能的愿望将充满了为眼前利益而调整解决方案的机会。你将在权衡中挣扎，做你希望最好的事情。对于你解决的每一个复杂问题，宇宙都会以创新的形式给你提供两个新的问题。你使用的抽象是有漏洞的。这将导致处理边缘案例时的痛苦。更接近金属的解决方案会不断增加开发负担。

你的实践，像测试驱动的开发，将会减轻痛苦，但是它们不会使你镇静。你会渴望能减轻你的烦恼的解决方案，但是当你走在代码的道路上时，这些修复只是一个暂时的问题。你将理所当然地寻找和发现更适合你的解决方案。毕竟，你不是受虐狂。也许你的 Go、React、Rails 等等教会会让你麻木。这可能正是你所需要的，但是痛苦仍然存在。不会有救赎。不是来自 Rust，不是来自 Ember，甚至不是 PostCSS。

你会经常爱上它的。有时你会觉得自己像个魔术师。会有苦难困扰你的日子。会有一切都好的时候。当你对你的代码或者其他东西满意的时候，安全、兼容性、复杂性、官僚主义、可伸缩性等等残酷的现实。不会消失。很多时候，这些会是迷人的谜题，会给你带来很多快乐。你与代码的关系中的 ebs 和 flows 只是代码瀑布中短暂的水滴。

代码正在遭受痛苦。
