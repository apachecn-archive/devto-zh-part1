# 作为开发人员，如何跟上时代的步伐

> 原文:[https://dev . to/mporam/how-to-stay-up-as-a-developer-44dh](https://dev.to/mporam/how-to-stay-up-to-date-as-a-developer-44dh)

编程新手经常问开发人员的一个问题是[我们如何设法跟上所有的新技术](https://www.quora.com/How-do-programmers-keep-up-with-whats-going-on-in-the-industry-without-falling-far-behind-considering-it-moves-so-fast)。编程是发展最快的行业之一。有如此多的 JavaScript 框架不断涌现，以至于像[、](https://dayssincelastjavascriptframework.com/)、 [FapFap.js](http://fapfapjs.io/) 这样的网站存在只是为了取笑这种情况。

随着新的库、框架甚至语言的不断出现，人们怎么可能希望跟上，更不用说开始学习它们了？！我不得不承认，我很高兴我在十多年前就开始编程了，那时事情还没有发展得这么快。我敢肯定，这似乎是一个不可能完成的任务。放心吧，不是的！

新进入这个行业的人应该明白的主要事情是，虽然每天都有新的东西出现，但大多数都是一样的。不管使用何种语言，库和框架都遵循相同的一般原则。学习那些使用一种语言的，你会发现学会其他语言相对容易。毕竟，每一个库或框架都必须遵循与构建它的语言相同的规则。如果你理解了这些规则，你就做得很好。这就是为什么在 Mayden Academy，我们不向我们的学生介绍任何语言的框架，直到他们学习了至少一个模块，包括用该语言构建的完整项目。

下一件要记住的真正重要的事情是，你不必一直跟上所有的事情。所有的程序员都知道并理解总是有太多新东西需要学习的痛苦。大多数开发人员选择一些他们感兴趣的技术和语言，并专注于与这些技术和语言保持同步。然而仅仅这样还不够。重要的是:我们互相分享信息。

程序员在自由分享信息方面是真正的典范。我从来没有经历过一个社区不仅仅是愿意，而是像开发社区那样积极地尝试共享知识。TechNation 在 2016 年进行的研究显示，超过[5000 人定期参加巴斯和布里斯托尔地区](https://techspark.co/bristol-and-bath-tech-talent-most-productive-in-uk/)的聚会。如果你不喜欢聚会，像 Mayden、Edo 和 Seccl(我们的招聘伙伴)这样的公司会举办各种各样的内部活动来帮助他们的开发者保持最新状态，从[黑客日](https://twitter.com/MaydenTweets/status/885880968325861377)到[开发者午餐](https://twitter.com/MaydenAcademy/status/883301100320894976)到周一早上[代码大战](https://www.codewars.com/)。雇主们越来越意识到，他们需要将开发人员的一些时间用于培训和更新。

我认识到大多数阅读这篇博客的人还没有从事过开发工作。那么你从哪里开始呢？下面我列出了一些博客、聚会和聊天频道，我强烈建议你花些时间去看看。毫无疑问，他们会像帮助我一样帮助你。
