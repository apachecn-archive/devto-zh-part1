# Sitefinity CMS

> 原文：<https://dev.to/robinvanderknaap/sitefinity-cms-28oa>

为了使命令者能够管理自己的网站，网络盗版者目前正在设置 joom 或内部开发的内容管理系统。这些系统都是在 PHP 中开发的。为了扩大我们的服务范围，我在过去的一段时间里一直在寻找一个基于 ASP.NET 的 CMS

CMS 必须符合一些要求。CMS 首先必须易于使用，并具备广泛的功能。此外，CMS 也必须由开发商轻易扩充及调整，提供客制化的服务，同时维持低廉的成本。

在尝试了几个系统后，我从电视上出来了【T0 站点定义】T1。此 CMS 非常易于使用，标准安装包括各种丰富的功能，并可方便开发人员在不学习新技术的情况下进行扩展。

CMS 包括所有常用的 CMS 组件，包括博客、新闻、文章、花粉和论坛，并提供创建 wiki 页面、发送新闻稿和管理用户组的功能。大多数功能都易于使用和设置。简单的培训或说明就足以帮助您快速启动站点定义。[网站上的](http://www.sitefinity.com/)站点定义提供了系统所有功能的全面概述。

在本文中，我想说明站点定义与其他内容管理系统的一些不同之处。首先，站点定义支持多种语言。用户可以用不同的语言存储文本，而无需为每种语言创建新页面。

siterecognition 内置了一个全面的版本控制系统。存储和查看多个版本的内容。最大的好处是，只需单击一次即可恢复所做的调整，您可以清楚地看到是谁进行了这些调整。

除了版本控制之外，站点定义还包含一个系统，所有新的和自定义的内容都必须经过授权人员的批准才能发布。尤其是在由多个用户提供网站内容的企业中，这是一个很受欢迎的选择。

telerik 表示站点定义很容易被开发人员扩展。为了测试这一点，我写了一个小的 Twitter 模块，它显示了网站上用户的最新 Twitter 信息。事实上，在几分钟之内，我就有了第一个可行的版本。桴萸隅砱妏蚚藩跺 ASP.NET 羲楷埜都熟悉腔撮扲ㄛ瞰翋珜睿蚚誧讽璃﹝。因此，不需要学习新的技巧即可向站点定义中添加新功能。

网站的在线状态直到最近才作为免费下载的社区版提供。很抱歉，此专案已停止，目前只提供专业版。CMS 的成本是 900 美元--每一个领域，这是相对便宜的其他商业套件。

到目前为止，最大的缺点是安装过程。我大概花了半天才让网站的定义说出来。最重要的一课是不使用安装程序，而是手动执行安装。
