# 在编程中，人性就是法律

> 原文:[https://dev.to/ben/in-programming-humanity-is-law](https://dev.to/ben/in-programming-humanity-is-law)

当我昨天去 Heroku 构建和部署我的代码时，我输入了`git push heroku master`，并且没有理由相信推送会被拒绝。令我惊讶的是，构建失败了，出现了这个错误。

```
npm ERR! No compatible version found: left-pad@0.0.3
```

困惑之余，我在谷歌上搜索了这个错误信息，并没有找到一个随机的 Github 问题，我意识到这个问题正在网上被讨论，像 React Native 社区中的[this](https://github.com/facebook/react-native/issues/6595)这样的讨论。一个依赖项被神秘地从 NPM 移除，这破坏了每个人都使用的一些模块。问题很快就被解决了，但是原因仍然很明显。一位程序员公开了他的模块，因为他理所当然地对 NPM 处理法律威胁的方式感到愤怒。他的一个模块，11 行代码，引起了整个矩阵的小故障。

值得注意的是，我们所依赖的系统运行得相当好，尽管它基本上是建立在纸牌屋的基础上。一个愤怒的单干程序员能够为数百万开发人员破解构建。

程序员习惯于将他们的工作视为确定性的、技术性的、逻辑性的，而不是人类的情感和不确定行为。社区已经看到这种思维过程以令人讨厌的拒绝承认道德责任的形式表现出来，就像在 [Opal](http://opalrb.org/) 社区中关于一个跨性别贡献者的讨论。

> 更详细地说，蛋白石是一种技术，技术是没有道德的，如果一个跨性别的贡献者出现在这里，@elia 的观点甚至不会出现在这里，因为为什么会有人关心，带来贡献，他们会被敞开怀抱接受，带来道德和政治，你会看到隐喻的大门。
> [来源](https://github.com/opal/opal/issues/941)

事实是，正如我相信最近的事件所反映的那样，编程结果一点也不道德，一点也不确定。NPM 的生态系统被一个开发者对待他的方式的个人问题所震撼。编程世界变得越来越像一个全球有机体，一个可以被人类以最好的方式、最坏的方式以及介于两者之间的任何方式影响和感染的有机体。NPM 方案可以被公布和删除的自由性质创造了巨大的权力和对巨大的集体责任的需要。

开源编程是给世界的一个巨大礼物，但是固有的人性和个人、公司、政府等之间有争议的相互作用比开发者认为的要复杂得多。今天的程序是建立在世界集体人性的基础上的，如果你想做出准确的技术预测，并最终做出好的选择和构建好的软件，就要承认人性。
