# 开发和运营有什么区别？

> 原文：<https://dev.to/funkysi1701/what-is-the-difference-between-development-and-operations-ii8>

我想学习开发，并最终进入开发角色。但是发展和我已经做的有什么区别呢？

简单地说，开发或编程就是创建新的程序、网站或数据库。

IT 运营或系统管理是对现有服务器、网站或数据库的管理。

自从 2006 年开始工作以来，我一直从事系统管理工作。作为这一角色的一部分，您所做的事情可以非常简单，比如设置新用户或计算机，也可以非常复杂，比如升级公司用于电子邮件的 Exchange 版本。

作为我工作的一部分，我做了很多可以称之为开发的工作。我创建了数据库，构建了新的数据库结构，如表、视图和存储过程。我还协助开发部门进行测试，建立 visual studio，从代码中构建网站和数据库。

我可以把自己描述成一个开发者吗？我会说是的，我做了大量的开发工作。

我能找到一份开发人员的工作吗？有可能，但这取决于角色。我有许多开发人员使用的技能，但我在许多领域的知识有限。我需要做的是将我的精力集中在我所做的开发工作上，并尽可能多地委派管理工作，这样我的知识就能增长。

目前 IT 界的一个热门词是 DevOps。我已经记不清有多少次了。NetRocks 或者 RunAsRadio 都提到过。

DevOps 是 IT 开发和运营的集成，它强调两个部门需要紧密合作以实现共同的目标。在大公司里，如果你不小心的话，这两个部门可能会相互对抗，但在我的情况下，因为我同时担任这两个角色，我自己的开发部门很难将问题归咎于我自己的运营部门。

我认为这让我处于一个很好的位置，我只需要学习更多的开发，我将为某人的开发团队做出有价值的贡献。
