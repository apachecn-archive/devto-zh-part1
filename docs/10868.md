# 亚马逊网络服务第二部分

> 原文:[https://dev.to/funkysi1701/amazon-web-services-pt-2-do4](https://dev.to/funkysi1701/amazon-web-services-pt-2-do4)

上次我开始关注亚马逊网络服务以及它与 Azure 的不同之处。我将继续观察它能做些什么。

## [](#virtual-machines)虚拟机

让我们看看你可以用虚拟机做什么。我选择创建一个新的虚拟机(或者 AWS 称之为 EC2 实例)

首先你为你的虚拟机选择一个名字，然后是运行在其上的操作系统。有 5 个主要操作系统可供选择，包括 Windows Server、Amazon Linux 和一系列最常见的 Linux 版本。

然后，您可以下载证书来保护您的虚拟机。

像 Azure 一样，AWS 需要一些时间来创建你的 VM。在我等待的时候，我看到 AWS 已经配置了防火墙，所以只有我当前的 IP 可以连接到它。

一旦虚拟机准备好了，你可以下载一个 RDP 文件。但是，要获得登录详细信息，您需要使用创建虚拟机时下载的证书来解密密码。

比较 Azure 和 AWS 在安全性上的差异很有意思。Azure 允许直接从控制台重置虚拟机的密码，但是我怀疑在 AWS 中，如果你丢失了你的证书(AWS 声明他们没有保存这个的副本),你将不得不重新创建你的虚拟机。

与网站一样，虚拟机的默认名称比你从 Azure 获得的名称更不友好。然而，我怀疑有其他选项，我还没有发现，可以定制这些。

## [](#azure-portal-vs-aws-console)Azure 门户 vs AWS 控制台

我真的很喜欢 Azure 门户。感觉就像是已经设计好的东西，所以你可以轻松地访问特定 Azure 功能的所有选项。

AWS 控制台可能有所有与 Azure 相同的选项，但是我不认为它看起来有一半好，并且会花我一些时间浏览菜单来找到等效的选项。部分原因是我对 AWS 不熟悉，所以随着时间的推移会越来越容易。
