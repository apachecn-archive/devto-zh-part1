# 如何了解数据库系统

> 原文:[https://dev.to/ozanonay/how-to-learn-about-database-systems](https://dev.to/ozanonay/how-to-learn-about-database-systems)

*这篇文章最初发表在[布莱菲尔德计算机科学学院](https://bradfieldcs.com)的博客*上

不幸的是，大多数自学成才的软件工程师对数据库系统的基础知识掌握得很差。虽然越来越多的工程师了解 SQL 或能够使用 PostgreSQL，但很少有人对底层系统有足够的了解，能够避免停机和调试复杂的查询，或者为某项工作选择并正确配置合适的数据库管理系统。

在大学环境之外学习这些基础知识是完全可能的，但有两个障碍:(一)必备知识；(二)缺乏良好的资源。我希望这篇短文能帮助你克服这两个问题。

#### 先决条件

如果不了解以下情况，就很难理解数据库系统:

1.  C 和 C++
2.  本科生计算机体系结构
3.  本科水平的操作系统

对于 1 和 2，我已经写了简短的指南[如何学习 C 语言](https://blog.bradfieldcs.com/how-to-learn-c-59222a627a4c#.cd43b566n)和[学习计算机如何工作](https://blog.bradfieldcs.com/learn-how-computers-work-e7d33dba0238#.g6gldlo8v)。我还没有对操作系统做类似的处理；在那之前，我建议将[操作系统:三个简单的部分](http://pages.cs.wisc.edu/~remzi/OSTEP/)作为一个容易获得的免费资源。

熟悉编译器对[也有价值，尽管这不是一个很强的先决条件。](https://blog.bradfieldcs.com/in-2017-learn-every-language-59b11f68eee#.jhcyr4swj)

C/C++很重要，因为大多数开源数据库系统广泛使用其中一个或两个，所以无法读取源代码可能是一个障碍。同样，当你开始编写自己的数据库系统时，你会发现自己正在接触类似的低级语言。

计算机体系结构和操作系统非常重要，因为您需要对内存层次结构、I/O 模型、内存虚拟化、进程和线程等概念有扎实的理解。

无论如何，学习所有这些领域都很重要，所以不要担心它们比数据库更重要。

#### 资源

遗憾的是，基本上没有好的、现代的涵盖数据库系统的教材。这可能是因为那些最有资格编写教科书的人正忙于创办数据库公司或在行业中担任有利可图的角色。对于我们的学生，我们建议将[数据库管理系统](https://smile.amazon.com/Database-Management-Systems-Raghu-Ramakrishnan/dp/0072465638/)作为最不糟糕的选择。

谢天谢地，多亏了乔·赫勒斯坦，伯克利的 CS186 课程有一套很棒的[视频讲座。对于那些在本科阶段没有多少接触就自学这些内容的人来说，我强烈建议以这些视频为起点，使用维基百科或教科书来填补空白。](https://www.youtube.com/watch?v=dY48_UZhvhw&list=PL-XXv-cvA_iBVK2QzAV-R7NMA1ZkaiR2y)

一旦你理解了基础知识，下一个很好的资源就是红皮书，这是由彼得·贝利斯、乔·赫勒斯坦和迈克尔·斯通布雷克编辑的论文集。这些论文是该领域的开创性著作，虽然它们不总是像针对本科生的资源那样容易消化，但你的耐心将会得到更深层次的洞察力。

当然还有更多资源，但是在观看 Joe Hellerstein 的讲座和研究红皮书阅读材料之间，您应该会发现自己正在成为数据库用户中的前几个百分点。
