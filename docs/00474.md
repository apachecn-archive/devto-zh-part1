## 前端和后端

理解网站如何工作的两个关键词你可能以前听说过，是*前端*和*后端*。这两者经常被分开讨论(和教授),并被视为两个非常不同的东西，但在大多数 web 开发项目中，它们都是必不可少且紧密相连的。

简单来说，**前端是运行在浏览器**上的任何代码，**后端是运行在服务器**上的任何代码。

## 服务器

从表面上看，服务器后端代码的作用非常简单:

1.  倾听请求。
2.  响应请求。

在一个常见的例子中，请求可能是某人试图访问网站的主页。服务器接收该请求，并发回 HTML、CSS、JavaScript、图像和任何其他与该页面相关的文件。

然而，这些请求和响应可能比这个简单的示例复杂得多，后端代码通常还处理检索和存储数据库中的数据、动态页面呈现、安全性和身份验证检查、密集图像处理以及许多其他处理大量任务以响应传入的请求。

生成的响应也可以采取多种形式。它们可能很简单，就像老式的 HTML 和 CSS 文件，或者像图像和音频文件这样的资产，但是它们也可以采用数据文件的形式，比如 JSON。

后端代码用多种语言编写——Ruby、PHP、Python、Java 和 JavaScript 是常见的选择。后端网站开发工作除了编写网站运行的实际代码之外，通常还包括所谓的“系统管理员”任务。这些包括安装服务器端工具、配置数据库等。

学习后端语言为你在网站上构建什么样的功能提供了许多可能性。它允许你选择一个静态页面，并添加大量新的方式与网站互动。开始后端开发的最佳地方可能是上述语言之一的教程，然后探索您感兴趣的语言的后端框架或内容管理系统，如 Ruby on Rails (Ruby)、Express (JavaScript)、Wordpress (PHP)、Laravel (PHP)、Django (Python)或许多其他语言。

## 浏览器

虽然服务器的响应可以有多种形式，但让我们回到常见的 web 页面请求的例子。处理完请求后，服务器用 HTML、CSS 和 JavaScript 文件的集合以及请求者可能需要的任何附加资产来响应。

然后，浏览器接收并处理这些文件。这些文件中的代码是*前端*代码，因为它运行在浏览器端。

前端的主要作用是:

1.  请求信息(数据、文件等。)从后端。
2.  显示或呈现该信息。
3.  倾听用户输入(点击、点击、滚动、悬停、打字、语音命令等)。).
4.  响应用户输入。

对于第一个角色，当点击一个链接或输入一个新的 URL 时，许多请求由浏览器本身处理。在页面已经加载了浏览器端的 JavaScript 代码之后，也可以请求新的信息。

数据的显示由浏览器处理的 HTML 和 CSS 定义。HTML 告诉浏览器页面的结构和内容，CSS 告诉浏览器如何在屏幕上显示。

当用户与显示器交互时，浏览器和页面的 JavaScript 代码都会侦听交互并做出适当的响应——或者使用内置的浏览器功能，或者使用 JavaScript 中定义的自定义响应。

为浏览器编写 HTML、CSS 和 JavaScript 代码是前端编程。编写这段代码通常依赖于后端的处理，因此这两者虽然是分开定义的，但却紧密相关。

前端有大量的库、框架和其他工具，从普通的 HTML、CSS 和 JavaScript 到 jQuery、React 和 Angular 等工具，但是无论使用什么工具，前端的角色都是一样的。这些工具只是解决类似问题的不同方式。
