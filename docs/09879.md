# 可转移的知识

> 原文：<https://dev.to/mavris/software-developmenttransferable-knowledge>

这是一个关于我作为 iOS 开发人员的知识以及我如何将这些知识转移到不同的语言和 SDK 的故事。

#### **成为 iOS 开发者的路径**

回到 2008 年，当史蒂夫·乔布斯宣布 iOS SDK 的时候(当时我还在学习计算机科学)，我做出了迄今为止我人生中最重要的选择。“我想我会成为一名 iOS 开发者。

由于没有软件开发经验，成为一名专业的 iOS 开发人员并不容易。长话短说，两年前，我获得了第一份 iOS 开发人员的工作。

#### 我的担忧

有一句颇有争议的话:“万事通，无所不能”，我完全同意。这意味着，通过尝试学习很多东西，你不会掌握其中的任何一个。我认为自己是“万金油”，但这不是我的选择。我之前做了五年的系统工程师，在转到 iOS Developer 之后，当没有 iOS 项目的时候，我在做 web 项目。因此，我可以自己构建一个完整的产品，但我仍然不掌握这些技术。所以我想专注于 iOS，并精通这个领域。

我最担心的是:*如果 iOS 设备减少到 1%，没有人想要 iOS 应用程序，会发生什么？*我知道我可以轻松地将我的知识转移到 macOS 应用程序中，但是在我的国家没有人要求 macOS 开发人员。所以，我到目前为止学到的东西将是无用的，对吗？

#### 看情况！

这取决于你决定专注于特定语言和 SDK 的时间。例如，如果你没有多线程的经验，使用 GCD、NSOperations、NSThread 等(iOS 和 macOS 中的多线程)会帮助你理解这个概念。如果你更深入，你将了解竞争条件，死锁，原子性等。另一方面，如果你已经在 GCD 或 NSOperations 上深入体验了多线程，那么对其他语言和 SDK 就没什么帮助了。

#### 从 iOS 开发者到移动开发者

所以，我得到了 Xamarin Developer 这个我无法拒绝的提议。我认为 C#是非常好的语言，我也将有机会熟悉 Android 操作系统，为什么不呢？

#### 可转移的知识

之前说过，Objective-C 和 iOS 是我最先深入的语言和 SDK。所以，我没有传授任何知识，一切都是从零开始。

#### 软件设计模式

来自大学的 C 风格编程，我不知道什么是软件设计模式(在真正的软件中)。在 Objective-C 中创建了几个项目帮助我理解了这个概念。

苹果在他们的例子中使用了一个很差的 MVC 实现。在构建了一些大项目后，我意识到苹果构建应用程序的方式(至少是他们构建例子的方式)并不适合我，于是我寻找替代的软件设计模式。然后熟悉了 [MVP](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93presenter) 和 [MVVM](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93viewmodel) 。我最终选择了 MVVM。

这种知识完全可以转移到其他语言和 SDK 上。例如，要为 iOS 和 Android 开发一个具有共享业务逻辑的应用，MVVM 是一个不错的选择。

#### **面向对象编程**

同样，从过程 C 开始，我在 Objective-C 中使用类作为结构。例如，如果我有一个类电影，我需要操作数据，我可能会在 ViewController 中这样做。但是，随着项目的发展，我也需要在另一个 ViewController 上操作数据，我意识到这个“逻辑应该由类来处理”。

面向对象的编程也是完全可移植的。

#### 多线程

每种语言的多线程方法可能不同。有一些低级和高级 API。我没有深入研究 iOS 中的多线程，但是我熟悉竞争条件、死锁、原子性等等。这些知识也可以应用于其他语言和 SDK。

#### 移动思维方式

构建移动应用程序的方法与构建网站的方法非常不同。

首先，你需要小心向用户呈现什么样的数据。因为屏幕很小，所以没有必要在列表视图中显示不必要的数据。

您还需要考虑移动设备的资源。例如，用户可能正在使用蜂窝数据，因此每次启动应用程序时下载 10 MB 的数据并不是一个好主意。

其他一些技巧，比如在下载新数据时向用户显示缓存的数据，会让用户忙起来，而不用看进度条。

在这种情况下，知识是可转移的，因为我正在从移动设备转移到移动 SDK。

#### 版本控制

当我开始使用 Git 时，我只使用提交、推送和拉取。然后我学习了分支、标签、冲突、拉请求等，并开发了自己的流程。这种知识也是可以转移的。

#### 结论

就我而言，我能够将大约 80%的知识转移到一种新的语言和 SDK 上。因为我在开始做 iOS 开发者之前完全没有经验，所以我能够转移很多基础知识。我从 mobile 迁移到 mobile SDK 也很有帮助。

我还在学习 C#的新东西(比如控制反转)，将来我可以应用到其他语言中。

因此，您可以传授哪些知识，取决于您的经验以及您是否正在迁移到类似的 SDK。
