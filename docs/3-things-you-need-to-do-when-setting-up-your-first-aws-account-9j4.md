# 设置第一个 AWS 帐户时需要做的 3 件事

> 原文：<https://dev.to/kylegalbraith/3-things-you-need-to-do-when-setting-up-your-first-aws-account-9j4>

“如果你当时知道你现在所知道的，你会做得更好，不同，等等。”

学习亚马逊网络服务很有趣。开始学习新的服务并将其应用到您正在构建的解决方案中是令人兴奋的。如果你专注于你试图解决的问题，你可以学得很快。

这篇文章是你学习轮中众所周知的“活动扳手”。

停一下。确保您不仅为更多的学习做好了准备，也为未来的扩展和安全性做好了准备。以下是围绕您的新 AWS 帐户的基于社区的最佳实践的集合。你应该在写完所有令人兴奋的特性后休息一下，看看你是否已经完成了这些。

### 1。为您的 Root 帐户打开多因素身份验证(MFA)

当您创建您的亚马逊网络服务帐户时，您正在创建一个“根”帐户。这个 root 身份拥有您帐户中的所有权力。它可以访问所有 AWS 服务和资源。

因为该帐户的权限太大，所以最好将其锁定。在这方面，一个好的步骤是为此启用多种纤维安排。AWS 提供虚拟、物理和 SMS 文本消息 MFA 交付机制。根据你的需求，哪一个都是合适的答案。

你可以通过手机上的应用程序选择虚拟 MFA。缺点是，现在一个受损的手机可以访问您的 MFA 令牌。您可以选择物理设备。缺点是你必须把它放在安全的地方，不能丢失。

任何选择都有缺点，但是对你的根帐户的 MFA 将是一个巨大的安全胜利。加分，是 AWS 认证考试上的常见问题。

### 2。太好了！现在停止使用你的根帐户

好了，您已经为您的 root 帐户配置了 MFA。现在停止使用它。在 IAM 中创建一个拥有`BillingPolicy`和`AdministratorAccess`策略的新用户。为该帐户生成唯一的密码，并在新帐户上启用 MFA。为此用户生成您的 CLI 密钥。删除您的 root 帐户拥有的任何 CLI 密钥。

一旦你这样做了，你现在可以登录为 IAM 用户，并退出访问你的根帐户。

### 3。为你的预期花费设置账单提醒

您保护了您的 root 帐户，现在您有了一个可以登录您的帐户的用户。看看你已经在进步了。

我见过刚接触 AWS 的人最容易忽视的是他们的计费设置。在你学习期间，大多数服务都不贵。如果您的帐户是全新的，您还有一整年的“免费层”资格。但是，如果你迷上了 AWS，你将通过学习阶段，烧穿你的免费层。

如果你认为这可能是你，避免神圣的法律法案。配置您的账单提醒，当您超出自己的支付能力时，它会向您发送电子邮件。打开它的 AWS 文档是这里的。

### 结论

这不是一个全面的列表。这三件事是*所有*账户一开始就应该做的。如果您希望通过该客户推出产品或服务，也要考虑这些因素。

*   创建包含私有和公有子网的非默认 VPC。
*   创建可以在实例中添加/删除的细粒度安全组。
*   考虑构建一个允许通过 SSH/RDP 访问的堡垒主机，您可以使用它跳转到另一个实例。
*   在您的 root 帐户上设置 CloudTrail 日志，以检测是否有您不期望的访问。
*   考虑为您将要使用的任何地区打开 CloudTrail 日志。即使你现在没有搜索它们的计划，在你需要的时候拥有它们也是很好的。
*   定期(但不可预测)旋转 IAM 键。
*   尽你所能使用云编队。从 CloudFormation 模板开始比以后做要容易得多。
