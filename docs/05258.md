# [](#developer-tools-for-a-backend-programmer)后端程序员的开发工具

任何一个开发者都会有一套他们信誓旦旦的开发者工具。一套工具，做我们需要它做的一切。人们可以说工具是可以互换的，在某种程度上的确如此。然而，开发人员使用的工具集通常决定了工作流。也就是说，我想转到告诉您我每天使用哪些工具的部分。

## [](#my-developer-tools)我的开发者工具

我的一个主要编程工具是一个名为 PHPStorm 的 IDE，我想几乎所有程序员都至少听说过它。编辑器带有内置的终端，我发现这是一个非常有用的特性。我通常同时使用 3 或 4 个终端，编辑器使管理它们变得非常容易。我经常使用的另一个有用的功能是搜索功能。您可以使用几个关键字来搜索整个项目中的字符串，这使得开发变得更加容易，不再那么繁琐。

## [](#updating-and-managing-code)更新和管理代码

如果出于任何原因，我需要更改任何实时代码，我会使用命令行编辑器 Nano 或 Atom 结合 Filezilla。幸运的是，我不经常走这条路，因为任何错误都会立即反映在生产中。通常我在本地修改我需要的所有东西，并通过 Git 和 Github 将其投入生产，这是我日常使用的另外两个工具。当然，除了 Git，还有 NPM 和 Composer 来获得我的项目中所有需要的包。如果你在 2017 年没有为你的项目使用包管理器，你应该检查一下。它使您的应用程序保持最新变得轻而易举。这也意味着您可以利用他人构建的数千个开源包。

## [](#testing)测试

测试是构建过程中非常重要的一部分。幸运的是，我在大多数项目中使用的 PHP 框架 Laravel 内置了 PHPUnit 支持。这意味着编写测试非常容易。只需几行简单的代码，您就可以知道您编写的方法是否如您所愿。在准备发布代码之前，这是一个非常好的运行过程，只是为了确保你写的东西确实有效。

## [](#browsers)浏览器

有时候你真的需要一个浏览器来测试你的应用程序。例如，在创建 SPA 时，我几乎 100%的时间都在使用 Chrome 开发者工具。我测试了两种浏览器，看看是否一切都按计划进行。第一个当然是谷歌 Chrome，第二个是安装了 Firebug 插件的 Firefox。它带有一个控制台和一个网络选项卡，用于查看是否有任何记录的错误，并查看您的浏览器实际上从服务器加载或接收了哪些数据。这对于调试和确保浏览器接收到您需要的数据非常有用。

因此，我每天都会使用一些工具来确保开发过程按照计划进行，并且您想要发布的代码以有序的方式发布。因为归根结底，您希望本地代码在生产环境中运行。通往那里的路不止一条，每个人都有自己的路。
