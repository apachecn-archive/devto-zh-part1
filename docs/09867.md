# 用户测试会让你成为更好的开发者

> 原文:[https://dev . to/April wensel/user-tests-will-make-you-a-better-developer](https://dev.to/aprilwensel/user-tests-will-make-you-a-better-developer)

我的第一次用户测试让我大吃一惊。我坐在一个房间里，看着一位女士将敏感数据输入到我开发的一个应用程序中，以便发现她的孩子是否有障碍。

突然间，关于我们是使用本地应用还是 web 应用，我们选择哪个表单库来实现，以及我使用哪个文本编辑器来编写代码的担忧消失了。

重要的是这位女士对这种形式的侵犯性问题的焦虑。

重要的是当按钮不像她预期的那样工作时她的沮丧。

重要的是当我们的应用程序告诉她她的孩子可能有障碍时，她的恐惧。

从本质上来说，那一刻最重要的是她的*感受*、，没有什么比在那个房间里亲眼看到她的反应更能清楚地向我展示这些感受了。

感觉并不是软件开发人员之间最常见的话题。我们喜欢谈论效率、代码质量和框架。当然，这些都是重要的话题，但是没有一个涉及到我们为什么要写软件。

我们写软件是为了帮助人类，为了写出更好的软件，我们需要开始关心那些人类以及他们混乱的、不合逻辑的、令人困惑的感觉*。*

 *这就是为什么我建议每个开发人员至少偶尔参与他们产品的用户测试。对于开发人员、用户和团队的其他成员来说，都有很多好处。

### [](#1-you-will-develop-more-compassion-for-the-user)1。你会对用户产生更多的同情心

观察用户测试有助于你培养对用户的同情心。移情正在成为设计词汇中一个更常见的术语，但很少有人谈论同情心。

理解同情的一种方式是将其视为一种更积极的移情形式。同理心意味着理解他人的想法和感受；同情意味着理解某人的痛苦，并积极地想为此做些什么。

当一个开发人员看到用户亲自努力时，她天生的解决问题的能力就会发挥出来。例如，在一次用户测试会议之后，她可能会发现修复一个看起来很小的错误更有意义，因为这对用户来说比整个网站的重新设计有更大的影响，对她来说，重新设计在技术上更有意思。

通过这种方式，记住使用我们软件的人可以集中我们的注意力，帮助我们区分工作的优先级，最终产生更适合我们用户的产品。

### [](#2-you-will-communicate-more-effectively-with-designers-and-product-owners)2。你将更有效地与设计师和产品负责人沟通

开发人员和那些从事设计和产品工作的人之间的许多冲突包括误解和不匹配的优先级。用户测试通过提醒每个人最终目标是改善用户体验来帮助团队达成共识。

设计师经常将用户测试的发现转化为交付给开发人员的需求。问题是没有机会观察用户的行为，开发者错过了用户测试数据中强大的情感成分。

如果开发人员偶尔真的有机会看到用户在当前的应用程序中挣扎，或者轻松完成一个新的原型，他们会对设计师或产品请求背后的原因有更好的理解，这将导致团队中更有成效的对话。

### [](#3-you-will-derive-more-meaning-from-your-work)3。你将从工作中获得更多的意义

最后，参与用户测试将有助于提醒开发人员他们为什么要做所有的艰苦工作。例如，当我看到当我们的应用程序显示她的孩子患疾病的风险较低时，一位母亲脸上掠过一丝宽慰，我对我的工作重新燃起了热情。

观察一个活生生的人对一个产品的反应将有助于给开发人员的日常工作赋予更多的目的，记住这个目的是避免精疲力尽和过上更有意义的生活的关键技术之一。

所以，开发人员，如果你准备好培养对用户的同情心，与设计师和产品负责人进行更有成效的对话，并最终在你的工作中找到更多的意义，请考虑时不时地参加一次用户测试！
