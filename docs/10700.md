# 66 天-第一天

> 原文:[https://dev.to/mlescaille/66-days-第一天](https://dev.to/mlescaille/66-days---day-1)

我跟踪了这个计划，并承诺在接下来的 66 天里至少花一个小时编码、阅读和练习算法、设计系统和商业相关的内容。我没有明确的目标，我确实想在工作之外练习，尽可能保持我的技能。

这一切都是从我认识的一位母亲和她的儿子之间的争吵开始的。他争辩说他不可能在学校做所有他必须做的事情。

不知何故，这让我想到了一个想法:事实上，我从未如此努力地学习或练习过。任何事。我没必要这么做，至少我是这么认为的。我停止写散文，然后也停止写诗，因为有人建议如果你不是每天都想写就不要写。

我并不是每天都渴望写诗，当我有灵感的时候，我会写，但主要是当一幅画或一个想法值得写的时候。那是给我的。没什么疯狂的。十几岁的时候，我总是想象自己写书，参加读书会之类的活动。我停下了。
然后大学的时候编程来找我。我不知道那是什么。快进到今天。

但直到昨天，我才意识到我从未承诺尽我所能每天学习并大幅提高我的技能。我在大学的时候确实参加过一些编程竞赛，真的很喜欢，但是在那之后，你只是对项目的细节了如指掌，然后你开始工作，不想在工作之余练习。

这是我试图创造一个新的习惯，并在编码、阅读、学习和写作时享受自己。我绝对喜欢做但并不总是做的事情。

在接下来的 65 天里，我将在这里写下我的旅程，主要是为我自己，写下我一天中的经历。
