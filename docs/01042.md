# SEGA SC-3000 个人电脑的历史默默无闻

> 原文:[https://dev . to/buntine/the-historical-obscity-of-the-sega-sc-3000-personal-computer-1hac](https://dev.to/buntine/the-historical-obscurity-of-the-sega-sc-3000-personal-computer-1hac)

这篇文章是复古 SEGA SC-3000 个人电脑系列的一部分。在下一部分，我将介绍如何在这台机器上编写 BASIC 语言。在最后一部分，我们将制作一个简单的射击游戏，并把它保存到录音带中。

这是澳大利亚墨尔本一个平常炎热的夏天晚上，我听到“咚，咚”的敲门声。

我看见我的邻居站在我面前，手里拿着一个大塑料袋。经过仔细观察，我发现塑料袋里似乎装着一堆旧游戏机和各种外设。我的眼睛亮了一下，但我试图控制住自己的兴奋。

他继续告诉我，当他在远郊的一所废弃的房子里做一些拆除工作时，他偶然发现了一台“旧电脑什么的”。他知道我喜欢那种东西，所以他问我是否想要。

我想要吗？！当然，我做到了。我一直是复古电脑装备的超级粉丝。在废弃的房子里找到这样的东西对我来说就像宝藏一样！

"好吧，让你自己平静下来，安德鲁"，我心想。

我跑进去，拿了一瓶 300 美元左右的唐培里侬葡萄酒，这是我开始上一份工作时得到的。经过一番说服，他拿走了瓶子，我们称之为公平交易。

## [](#better-than-expected)比预期的更好

当我进去倒空休息室地板上的塑料袋时，我发现里面的东西比我最初意识到的要多得多。包里有一台复古的小型机、一些射频电缆、一个交流适配器、五个软件盒、一个操纵杆和一个游戏手柄。所有包装完好无损。它就像一个简单时代的时间胶囊。

## [](#wait-a-sega-computer)等等，一台 SEGA 电脑？

真正让我印象深刻的是每款游戏和电脑本身的标志: **SEGA** 。

SEGA 从什么时候开始制造家用电脑的？

当我还是个孩子的时候，我拥有的第一台游戏机是一台[世嘉主控系统](https://en.wikipedia.org/wiki/Master_System)(或者日本的 Mark III)。这是一个很棒的小系统，有很好的游戏目录。然而，我不知道的是，在[创世纪](https://en.wikipedia.org/wiki/Sega_Genesis)之前，在[土星](https://en.wikipedia.org/wiki/Sega_Saturn)之前，甚至在主系统之前——SEGA 发布了一款名为 [SC-3000](https://segaretro.org/SC-3000) 的家用电脑。

令人惊异的...但也有一些其他突出的东西。这台机器的全名是**“约翰·桑斯·SEGA SC-3000h 个人电脑”**。

对于那些不知道的人来说，John Sands 是一家澳大利亚贺卡和礼品包装供应商。就[看看他们的网站](http://johnsands.com/)——看起来不太像电脑经销商吧？当然，在 80 年代早期，约翰·桑兹确实[进入了羽翼未丰的家用电脑市场](https://www.youtube.com/watch?v=4HbYyCj3gLE)，当时他们帮助 SEGA 销售这种机器和一种叫做 [SG-1000](https://segaretro.org/SG-1000) 的视频游戏。

## 好吧，告诉我具体的细节

你可能从未听说过这款机器的原因是，SEGA SC-3000 只在几个市场发行:日本、澳大利亚、新西兰、法国、意大利和芬兰。

它在欧洲的表现不是特别好，但在日本和澳大利亚/新西兰取得了一些小的成功。事实上，SC-3000 代表了 SEGAs 第一次也是最后一次进军个人电脑市场。虽然 SC-3000 在市场上主要是作为家庭商业和教育用途的工具，但它似乎只用于玩视频游戏。多么令人惊讶啊...

这款机器发布了两种型号:SC-3000(带有橡胶按键)和 SC-3000H，它拥有一个全机械键盘(我必须说，这是一个非常漂亮的键盘！).每个市场似乎都有自己不同的颜色。所有的约翰·桑斯澳大利亚模特都是黑色的。据说白色模特是最稀有的。

在背面有一个电源开关，一个 DC 输入，复合视频端口，打印机端口，音频盒式磁带输入/输出端口(我稍后会讨论这些)和一个射频连接器端口，用于插入您的电视。

在一侧有一个大的空间来放置软件卡带。另一方面，有两个标准的操纵杆端口。

内部有一个 8 位 NEC D780C-1 CPU，运行频率为 4 MHz。我相信它与 80 年代在科学计算器和许多街机上大量使用的更著名的 Z80 CPU 是一样的。

嵌入机器的只有区区 2Kb 的内存。以及为视频显示处理器(德州仪器 TMS9929A)提供的 16Kb VRAM。也许有些人会感到惊讶，但是软件卡带实际上提供了自己的 RAM(从 1Kb 到 32Kb)。当一个游戏运行时，它会通过自己的卡盒上的 RAM 进行读写。所以，不幸的是，这意味着保存你的游戏进度，稍后再回来是完全不可能的！

## [](#and-how-about-the-software)还有软件怎么样？

SEGA SC-3000 没有操作系统。事实上，如果你在没有子弹的情况下打开它，你只会看到一个空无一物的空白屏幕。

正如我前面提到的，分发软件的主要方式是通过盒式磁带。SC-3000 盒式磁带插槽也兼容早期 [SG-1000](https://en.wikipedia.org/wiki/SG-1000%20video%20game%20system) 和 [SEGA【我的卡】](https://en.wikipedia.org/wiki/Sega_Card#My_Card_Mark_III_/_Sega_Card)格式的游戏。

我发现这些墨盒很容易变脏(即使留在盒子里)，所以我花了相当多的时间用棉签清洁它们，然后才能让它们工作。坐在我的客厅里，对着墨盒吹气，然后摇晃着它们，这是一种非常怀旧的感觉。这绝对勾起了我童年的回忆！

## [](#how-do-i-hack-on-this-thing)这个东西我怎么黑？

像当时的许多家用电脑一样，SC-3000 打算通过 [BASIC](https://en.wikipedia.org/wiki/BASIC) 编程语言进行编程。根据我的研究，在我看来，所有型号都配有基本的墨盒。有几个独立的基本墨盒可用，差异在于可用的内存量。最“基本”的版本只有 1Kb(其中 512 字节由基本运行时本身占用)。其他版本分别有 16Kb 和 32Kb。当时 SC-3000 的许多广告都吹嘘有 48Kb 的内存——但这只有在你还买了最贵的基本墨盒的情况下才会发生！

SC-3000 还允许一种相对有趣的 I/O 方式与基本盒式磁带相结合——通过标准盒式磁带。软件是一点一点地写入磁带的，每次记录只需不到 1 毫秒(833.3 微秒)。这导致每千字节数据大约 6.8 秒的音频。因此，从盒式磁带中读取一个 16Kb 的游戏并将其存储到基本盒式磁带的内存中需要将近 2 分钟的时间...

[![SEGA data recorder](../Images/bc5c018de74b377fa9b871ed81560cab.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--4IrwFtsN--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/3eap7c62qvwddf8ew82v.jpg)

一旦您的卡式播放器通过 3.5 毫米音频线插入 SC-3000，它就可以被读入内存并非常简单地执行:

```
LOAD some-game
RUN 
```
