# 追求新的

> 原文：<https://dev.to/cyberomin/in-pursuit-of-new>

作为软件工程师，我们被不同的工具、库、框架和不断发展的最佳实践所淹没。

几乎每周我们都会在黑客新闻上发现下一个最佳框架或库趋势。如今，几乎任何问题都有一个库或框架——从字符串操作、日志管理到声称速度极快的框架，以及自切片面包以来的第二好的东西——伴随着它。

人们很容易被这种不断变化的景观所淹没，这很容易导致疲劳。

举个例子，在 JavaScript 领域，我们已经看到框架来来去去。在我看来，最新最好最棒的框架的寿命是 2 年。

从事物的前端来看，我们已经看到了 EmberJS，Kockout JS，Babel，Grunt 等。这些天来，React 和 AngularJS 是街区的新成员。如果你是做前端的，并且不使用这些工具，这几乎是一个禁忌。

和所有东西一样，我们有了 Prototype 和 Scriptaculous，然后是 jQuery，幸运的是，这个仍然很强大。

### 你是干什么的？

就我而言，我对这些工具抱有相当程度的怀疑。就像所有其他事情一样，我知道他们会失败。你无能为力，这就是生活的事实。只要摩尔定律仍然成立，太阳将从东方升起，事情就会改变。

我一直坚信，牢牢掌握你的工作工具会让你脱颖而出，给你带来急需的满足感。我什么意思？例如，如果你对 JavaScript 本身没有很好的理解，那么追求 JS 框架是没有意义的。学习基础知识。

下次你觉得自己落后时，不要责备自己。不，你不是。你不能也不会知道所有的事情，这很好。我是不是在暗示人们不应该随波逐流？远非如此。跟上他们，但不要忘记基础。

生活中的一切都是短暂的，你的工作工具也不例外。不要过度紧张。继续阅读和学习新的东西，但是当下一件大事出现时，不要太担心。

还记得 CSS 简单明了的时代吗？少，SCSS 和萨斯改变了这一点。

享受这一刻。
