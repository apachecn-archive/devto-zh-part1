# 程序员们，集合！

> 原文:[https://dev.to/captainsafia/programmers-assemble-1nc](https://dev.to/captainsafia/programmers-assemble-1nc)

关于如何为软件开发者和软件用户建立有效的社区，我想了很多。对我个人来说，联系和授权他人是非常重要的，我认为这是一个很好的时机来写下我在构建社区方面学到的一些东西，因为我已经作为组织者/领导者参与技术社区大约三年了。准备好了吗？开始了。

**有意义的指标**

社区，像非营利组织、政府和其他各种组织，需要有有意义的成功衡量标准。你创建社区的主要目标是什么？你想解决哪些问题？如何量化这些问题？如果你不完全确定你想在你的社区中实现或改变什么，那么你可能想花更多的时间去思考它。这些指标将指导你带来什么样的事件、活动、赞助和演讲者，所以尽早弄清楚它们是有好处的。仅仅说你正在为 X 编程语言或 Y 框架的用户建立一个社区是不够的，你需要有一个建立那个社区的具体目标。

**包容性的重要性**

为了繁荣和参与，社区需要多样性。如果你的社区由一群同质的人组成，考虑一下你可能会做些什么来阻止边缘化的个人加入这个社区。行为准则是健康的技术社区不可或缺的一部分，如果你的社区缺乏行为准则，你可能会推开更多你意识到的人。除了有一个行为准则，你还需要警惕地执行一个。努力让不同的演讲者和导师参与进来，可以让被边缘化的人在你的空间里感受到欢迎和重视。

**一致性是关键**

当涉及到社区活动时，最好保持一小组一致的和可重复的活动，而不是引入许多不同的活动。人们从繁忙的日程中抽出时间来参加社区活动，重要的是他们在参加活动之前对自己将从活动中获得什么有一个强烈的理解。这种理解只能来自于定期为社区成员提供类似的活动，让他们参与并确定什么对他们最有效。

**场馆种类**

虽然保持活动类型的一致性很重要，但主办地点应该尽可能多样化。通过在不同的地点举办活动，你鼓励了那些原本无法长途跋涉到镇上“中心”参加活动的人们。我犯了一个错误，那就是坚持在我的家乡(芝加哥)的中心地点举办很多次活动，然后才意识到不同的地点并不会阻止人们参加，它只会鼓励不同的人群参加。

**永远不要说 Meetup**

我不喜欢用 Meetup 这个词。只要有可能，我喜欢称我开发的非数字社区为社区。Meetup 将联系和共同成长的古老概念与一种相当新的技术联系在一起。Meetup 是一个容器，但哪些社区组织起来，它不是社区。

**目的的力量**

最后，但同样重要的是，你必须完全献身于你的社区的目的和目标。无论你是想培养下一代普通的 Lisp 程序员，还是想让书法艺术成为主流，你都必须对这项事业有个人奉献精神。我见过太多的 Meetup 失败了，因为人们想获得 Meetup“创始人”的头衔，在行业中发展自己的关系，或者推动自己的产品/创业/议程。社区不是为你而建，而是为人而建。建立一个成功的社区可能是一个累人和自我牺牲的过程，你必须愿意认识到你在那里是为了帮助别人，而不是你自己。

这些想法仅仅是基于我的经验，但我认为它们非常准确，反映了一些很好的见解。当我收集到更多的成功管理社区的技巧时，我会在这篇博客中补充。
