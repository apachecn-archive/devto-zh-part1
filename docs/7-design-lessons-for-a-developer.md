# 给开发者的 7 堂设计课

> 原文：<https://dev.to/pradeep_io/7-design-lessons-for-a-developer>

我是一名开发人员，过去 6 年我一直在编写代码，构建 android 应用程序和 web 应用程序。我总是面临设计思维的问题，甚至没有尝试去修改我的应用程序的设计。但最近我开始阅读唐·诺曼的《日常用品设计》,并开发了一个应用程序。我想列出我在过去 6 个月中得到的 7 条极其重要的经验。

1.  即使是最简单的事情，用户也会犯错误。你的工作是尽量减少这些错误，让他们对使用你的产品感觉良好，而不是责怪自己。
2.  *制作至少 3 个用户角色*并一直保持在你的眼前。
3.  假设*用户可以在他们的短期记忆*中存储最多 3 件事情。设计这样的流程，它不会耗尽用户的脑力，不管这些事情听起来有多琐碎，就三件事。
4.  阅读关于*心智模型*的内容。
5.  导航和动作应该符合目标用户的心理模型。
6.  *反馈很重要*对于用户执行的每一个动作，用户应该总是得到他们动作的肯定，并立即知道他们动作的结果。良好的反馈消除了认知负荷(例如，我执行的行动发生了什么变化？)，不好的或者没有反馈的效果正好相反。
7.  当有疑问时，*与用户*交谈。
