# 开发人员和 UI 文本

> 原文:[https://dev.to/stipegrbic/developers-and-ui-texts-51a0](https://dev.to/stipegrbic/developers-and-ui-texts-51a0)

这篇文章的灵感来自软件中面向用户的无信息文本。作为一名软件开发人员，我知道有些情况下我们没有意识到我们写的文本对世界上的其他人来说是多么的无意义。这是给所有受此类消息影响的用户的一个借口，也是给其他开发者的一个提醒，他们应该小心如何与他们的用户沟通。

### [](#a-perfect-world)完美的世界

在一个完美的软件开发世界里，有专业的内容作者，所有面向用户的文本都经过他们的批准。这涵盖了可以向用户显示的所有可能的屏幕、信息和错误消息。开发者不仅不需要自己编写字符串，他不需要担心文本的内容甚至语言。当然，在这个世界中，内容作者完全理解软件产品、上下文和他需要呈现给用户的信息。他可以轻松地将我们作为开发人员交流的技术语言翻译成世界上其他人理解的非技术语言。

### [](#the-real-world)现实世界

现实世界中的情况有点不同。即使你足够幸运，有一个为你的软件编写内容的人，你也很可能遇到过不得不自己编写文本的情况。在这种情况下，当你高速编写程序代码时，你不会给予太多的关注，或者不会为了那些小的“细节”而转向非技术性的思考方式，比如面向用户的文本消息。我知道这种情况，但这绝对不是借口。这种情况需要突破技术思维方式，把自己放在用户的位置上。只有这样，你才有机会写出对用户有用的信息性文本。这里有一些如何在 [MSDN](https://msdn.microsoft.com/en-us/library/dn742478.aspx) 上编写用户界面文本的方法。
