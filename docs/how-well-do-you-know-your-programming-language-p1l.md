# 你对你的编程语言了解多少？

> 原文：<https://dev.to/mjswensen/how-well-do-you-know-your-programming-language-p1l>

我正在听[一个由](http://hanselminutes.com/394/demystifying-scala-with-netflixs-dianne-marsh) [Scott Hanselman](http://www.hanselman.com/) 和 [Dianne Marsh](http://diannemarsh.com/) 制作的关于 Scala 的播客，在那里编程语言被简单地与外语进行了比较。在接下来的上班路上，这个想法一直萦绕在我的脑海里。我提出了以下量表，可以用来帮助自我评估一个人对编程语言的熟练程度。

| 水平 | 程序设计语言 | 外语 |
| --- | --- | --- |
| 1(没有经验) | 你不认识语法，要么是因为这种语言使用了一种不熟悉的范式，要么是来自一个与你所熟悉的不同的语系。文件扩展名可能会提供提示，这取决于您的记忆力有多好。 | 你也许能猜出这种语言在哪个大洲使用，但是你不愿意尝试去识别这种语言是否有人在听。 |
| Two | 你知道是什么语言，但是语法感觉挺别扭的(甚至莫名其妙)。 | 你能够根据字母表的外观来识别语言，但是语音、语法和一般结构仍然非常陌生。 |
| three | 您了解该语言所处的上下文，并能够通过从 Stack Overflow 复制和粘贴代码片段来拼凑基本功能。 | 你已经记住了点餐和问洗手间在哪里的一些基本短语。 |
| four | 你可以让事情运转起来，但是代码质量的水平让其他人(以及未来的你)读起来很痛苦。你还不喜欢用新语言写代码。 | 虽然仍然支离破碎，但你对这种语言的结构有了基本的了解，并能在飞行中形成一个基本的句子，尽管语法不健全。 |
| five | 你写代码的水平可以接受。它不再完全是用你更熟悉的语言范式编写的。你开始对新的语言感到舒服了。 | 虽然你仍然只是偶尔用外来语说你的母语，但你能够进行对话。 |
| six | 你越来越多地采用这种新语言的习语。您正在拥抱最佳实践，并避免新生态系统的陷阱和代码味道，这个新生态系统对您来说越来越熟悉。你创造性地用只有新语言才会用的方式解决问题。 | 你开始用你的新语言思考和做梦。你希望有办法用你的母语表达某些只有用外语才能表达的想法。 |
| 7(精通) | 用新语言的惯用方式编写代码是自然而舒适的。阅读他人的优质作品是轻松愉快的。这种新语言已经成为你思考问题的法宝。你甚至为社区做贡献，帮助传播对这门语言的热情。 | 你对这种新语言的掌握程度足以让你理解诗歌的微妙之处。你甚至有能力自己写诗，偶尔会想出一些绝妙的东西，和母语是这种语言的人写的没什么区别。 |

对于你每天使用的语言来说，你属于哪一类呢？
