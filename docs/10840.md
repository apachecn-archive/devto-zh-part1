# 我的八大必听开发者播客

> 原文:[https://dev.to/ben/my-eight-must-listen-podcasts](https://dev.to/ben/my-eight-must-listen-podcasts)

我喜欢听播客。这就是我如何不经意地跟上潮流，了解我甚至不知道我不知道的事情。以下是我经常听的八个播客列表。我可以很容易地在此基础上再列出八个，但我认为八是一个好数字，这是一个相当好的列表。我主要用 JavaScript 或 Ruby 编写 web 应用程序。这影响了我听的内容，但不管你的语言偏好如何，这些节目大多都是很好的听众。这个列表是根据偏好和偏见模糊排序的。

所有这些播客和我的几个最爱可以在我们的新发现工具 [/pod](https://dev.to/pod) 上找到。如果你知道我们应该包括在那里的任何更多的必须听的，让我们知道！

## [软件工程日报](https://dev.to/sedaily)

每周五次精彩的面试。Jeff 为每次面试做了大量的研究，他的努力显示了出来。该展览涵盖了软件开发的每一个角落，但 Jeff 对栈的底层部分最感兴趣。

## [变更日志](https://dev.to/changelog)

Adam 和 Jerod 对开放源代码中的著名人物进行了一次非常干脆的采访。这个节目每周出一次，总是很有价值。该团队现在还有两场演出，分别是[开始时间](https://dev.to/gotime)和[提交请求](https://dev.to/rfc)。我还没有进入 Go Time，但是我真的很喜欢 RFC。

## [代号新手](https://dev.to/codenewbie)

Saron 经营着一个很棒的播客，专注于可以帮助新手实现飞跃的编程部分。即使它是针对新程序员的，但对每个人来说都是有价值和有趣的。

## [店铺脱口秀](https://dev.to/shoptalkshow)

Chris 和 Dave 主持了一个展示，展示了各种格式和围绕前端开发的各种主题。它充满了关于开发堆栈这一部分的实用建议和意见。

## [红宝石盗贼](https://dev.to/rubyrogues)

这是一个由主持人和嘉宾反复出现的专题节目，涵盖了 Ruby 中许多有趣的主题，并且也倾向于进入关于开发的社区方面的大讨论。

## [JavaScript Jabber](https://dev.to/jsjabber)

这是《红宝石流氓》的主持人查尔斯·麦斯武德制作的另一个节目。他在 Devchat.tv 制作了其他几个节目，我也听了。这个节目的形式和 Ruby Rogues 类似，但是它的性质有所不同，因为主持人、嘉宾和 JavaScript 环境与 Ruby 不同。

## [自行车棚](https://dev.to/bikeshed)

本节目深入探讨了 Ruby、Rails 的一些技术细节，并提供了许多关于 Rails 正在进行的开发的内幕消息。德里克和肖恩上演了一场非常有趣的科技和戏谑表演。莱拉·温纳曾参与主持过几次，但她有一个新的播客，[莱拉和布伦达秀](https://dev.to/bikeshed/76-the-one-with-laila--brenda)，遵循类似的格式。

## [汉斯分钟](https://dev.to/hanselminutes)

Scott 带来了一些令人惊讶的客人，并在揭示计算机编程中的利基主题和最有趣的部分方面做了大量工作。这个节目积压了很多。
