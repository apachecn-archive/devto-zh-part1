# 牦牛毛。简短的定义！

> 原文：<https://dev.to/stefanbc/yak-shaving-a-short-definition>

[![Yak shaving. A short definition!](img/6639cde2d403675e1bf1ca5dfea84a0c.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--66KhQ1ZB--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://stefancosma.xyz/conteimg/2015/11/ibaDjk7AeIcvxv.gif)

你是否曾经开始着手一项任务，突然意识到这项任务需要一些其他的小任务提前完成。那是牦牛毛给你的，TL；dr 版本。

你可能已经多次这样做了，你甚至意识到了这一点。

早在 90 年代，麻省理工学院一位名叫[卡林·j·维耶里](http://projects.csail.mit.edu/gsb/old-archive/gsb-archive/gsb2000-02-11.html)的博士创造了这个术语来描述

> 当你在做一些愚蠢的、繁琐的小任务时，你在做什么，这些小任务与你应该做的事情没有明显的关系，但十二个因果关系链将你正在做的事情与最初的元任务联系起来。

让我给你举个例子，你可以理解。

你在做一个老项目。假设您已经准备好进行部署，但是您突然意识到忘记设置数字海洋水滴了。你开始设置它，但是你意识到你需要一个特定版本的 PHP。在您开始下载和设置之前，您会意识到有些安全问题需要通过您的 droplet 来解决。

因此，你发现自己陷入了一个永无止境的小任务的漩涡中，这些小任务分散了你对主要任务的注意力。另一个比较类似于 RPG 类游戏中的支线任务。

我们为什么要这样做？

我认为这与人性有关。我们不是本能做事的物种。我们不知道在任何时间点会发生什么。至少我是这么看事情的。对此还有其他想法吗？

直到下一次，代码长且适当！
