# 新手开发者，不要相信那些教程。

> 原文:[https://dev . to/inidaname/newbie-developers-don-trust-these-tutorials-1ib](https://dev.to/inidaname/newbie-developers-dont-trust-those-tutorials-1ib)

我跟你分享一个故事吧。

讲的是一个穷人养了一只神奇的鹦鹉，神奇到他可以和它进行完整的对话。他决定利用他的鹦鹉的能力赚些钱，所以他把它带到了市场，在那里他遇到了一个非常富有的人，他有兴趣付钱和鹦鹉交谈。几天后，鹦鹉一言不发，带着愤怒和失望，这个可怜的人带着他的鹦鹉回家，当他们到达时，他把沉默的鹦鹉放回笼子里，只听到鹦鹉说“这是紧张的一天。”

是的，这是应用程序的常见情况，在你的本地机器上总是完美的，但一旦投入生产，它就变成了沉默的鹦鹉。

[![](img/6985eac562a5c75a56813d5e40b20821.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--2x8qou66--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/vsyz8wirfkai752pxr7w.gif)

作为开发者，你最害怕的是什么？我知道我的不是`Hello World!`。

当我开始学习编码时，我记得用许多不同的语言和框架构建 ToDo 和天气应用程序，特别是在`Javascript`和`PHP`中，我能够构建像注册、登录/注销、好友请求、帖子状态、评论和喜欢这样的功能。

然后我开始读文章，看教程，去见 ups，他们会提到像**算法**、**数据库设计**、**可扩展应用**和**可重用代码**这样的词。

[![](img/93b0553827ab09c00b500c36a00f9863.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--psfz-j3u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mws6c0g7nnxcqxwcrqbe.gif)

**算法…** 这是另一种语言吗？**数据库设计…** 我需要一支铅笔吗？**可扩展应用…** 那是什么框架？**可重用代码…** 我复制粘贴的还不够多吗？

听到这些话让我觉得自己像是这个领域的鹦鹉，我创建的所有应用程序都只供我使用，代码完全是从我观看的视频教程中克隆出来的，有些甚至是一篇文章的直接复制和粘贴。

这是问题的一部分，另一个事实是，即使我从视频教程中克隆了完整的代码，或者从一篇文章中复制并粘贴了完整的代码，但还是不一样，有些功能总是会失败。

[![](img/ea490079ebd8b815b748acffd9fdfaac.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--sfrwP8yV--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/l3jndeubyybcegtr4rlt.gif)

这在我的脑海中产生了一些大问题，我如何确定自己是一名专业人士？我什么时候从学习者转变为满足者？最重要的是，我如何建立一个开发者的自信？

大多数客户不会要求任何接近我的教程所认为的东西。

[![](img/4627f999e79708d2f7f86b92cf585fec.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--UyJxThov--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/i9efvf3odgbtwuse0ogw.jpg)

一些人要求应用程序可以识别汽车停在哪里，为司机提供早餐、午餐、晚餐的选择，甚至与邻居打架。我想趁他在的时候，我们为什么不给他奶奶打个电话。

我会看到职位空缺，要求初级开发人员有三年的经验，精通普通 Javascript，必须熟悉 DOM，必须理解 RESTful API，必须理解 Everything.js(而不是框架)，必须记住每个同事的名字，能够与 Joker 和 Dark Vader 一起工作，必须拥有**脸书，谷歌，GitHub，Twitter，Medium，Instagram，LinkedIn，雅虎！，Hotmail *(别慌)*** 并且一定要准备好参加健身房。*等等……列出了所有的特征，我们还需要去健身房吗？*

液体错误:内部

网上很少见的一个教程是关于信心的教程，它没有框架或库，没有特定的语言或设计模式。

[![](img/88574a0ce83322dd1c10529b49155a3b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--gXKe437w--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/azrwhdgglihp3gnthvme.png)

学习编码时重要的是理解概念和学习课程的基础知识，自信是你必须建立的东西，即使我们需要不断练习编码来建立它，我们不一定使用编码来建立它，希望我没有让你困惑。

建立自信的最好方法之一是参加科技聚会和科技会议，别忘了吃那些免费的食物，这是建立自信的重要部分。*我觉得*。

另一个重要的方法是**写博客**，你不能写给读者看，先写给自己记住，如果这对你有帮助的话，你就是在间接地帮助别人，因为你是某个人，只是提到几个免费的博客空间，你可以使用【medium.com】、 [dev.to](https://dev.to) 、 [scotch.io](https://scotch.io) 和 [github.io](https://pages.github.com/) 。

另一个方法是我自己在一次[我们代码聚会](https://dev.to/inidaname/we-talked-we-ate-we-learned-and-wecode)上从一位女士那里学到的。就是**开发一些**并发布出去，让公众使用它，让他们给你带来战斗，因为有时防御是最好的攻击。

你要明白这不是战场，让我这么说吧。构建应用程序，发布并等待评论。

[![](img/1b55d9b323d253ca15a758943baa4e2b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--XeKIZ4Rh--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/vrdnsvvjvdu7dsxszl0i.jpg)

真诚地说，这篇文章本来不应该这么长，我只是希望你现在还在读它，这让我想到了另一个问题，不管你在做什么，不管它有多重要，尤其是在有最后期限的时候，简单地休息一下，散步，跑步，与人交谈，只是做一些不包括电子产品的事情。

**加入社区**是一种非常有助于建立自信的方式，你可以与你的同类人互动，**加入在线**论坛，访问[www.stackoverflow.com](https://www.stackoverflow.com)，[脸书团体](https://facebook.com/groups/)和 Twitter。如果你能接受我在 twitter 上的一些推荐，请关注 [@CodeNewBies](https://twitter.com/codenewbies) 、 [@ThePracticalDev](https://twitter.com/thepracticaldev/) 和 [@iNidAName](https://twitter.com/inidaname) 。是啊。你抓到我了，最后一个是我的。

**投入讨论**不要做旁观者，提问，尽你所能回答问题，如果你遗漏了什么，有人会纠正你。*我想这就是我们最初来这里的原因*。只要确保无论你做什么，你的问题或答案都在范围内，请不要冒犯。

还有最重要的**求助**，不要做沉默的鹦鹉，跟有钱人说话，有什么损失。

不要相信那些教程…！说真的，我相信你现在明白了，我只是说我们需要更多样的教程。请遵循那些教程，否则你为什么要寻找勇气。
