# 学习编程的乐趣

> 原文:[https://dev.to/ericnormand/the-joy-of-programming-to-learn](https://dev.to/ericnormand/the-joy-of-programming-to-learn)

你编程的主要动机是什么？

早在 90 年代，我在网上读到一些东西(我不记得在哪里了)，说人们一致认为这是绝对控制机器的快乐。作为一个易受影响的年轻人，我感到被疏远了。这当然不是我的主要动机。

现在，不要误解我，我可以像任何人一样享受对无生命物体的控制。用 C 编程给我那种感觉。但是还有别的东西驱使我去编程。直到大约九年前，我才开始找到能引起我共鸣的其他人，并进一步理解我自己在编程中的乐趣。

我喜欢创意。我喜欢思想史。我喜欢想法是如何被你的环境塑造的。我喜欢探索坏主意的乐趣。我喜欢通过时间和空间将想法传达给另一个人的方式。我喜欢一些想法有力量。

这是我在[我的时事通讯](https://purelyfunctional.tv/newsletter/)和[我的博客](http://lispcast.com)中做的事情之一——我发展想法。我产生了很多错误的想法和一些正确的想法。但是通过发展他们的过程，他们的轮廓和特征才为人所知。这些文章远不是我的信念的表达，而是这个过程和对想法本身的热爱的一种**表达。**

对我来说，编程是一种拓展思路的方式。你用代码建立一个系统，然后你可以凭经验确定它的属性。对我来说，编程的刺激更多地来自于对思想的探索，而不是控制机器的乐趣。我写了神经网络来学习神经科学，写了遗传算法来学习进化，写了其他模拟来学习我正在模拟的东西。

大约十年前，我发现了艾伦·凯，确切地说是计算机革命尚未发生的*和他在 VPRI 的工作。我找到了一个将编程作为思想媒介的人。他感兴趣的是了解如何使计算机变得更容易，以便任何人，包括儿童，都可以开发、探索和交流思想——也许这些思想在其他情况下根本不可能探索。*

 *我读过 Seymour Pappert 和他关于计算机作为一种探索困难想法的工具的观点。真的，去读读 [*Mindstorms*](https://www.amazon.com/Mindstorms-Children-Computers-Powerful-Ideas/dp/0465046746) 。尼古拉斯·尼葛洛庞帝[最好地诠释了帕佩特的愿景](http://longnow.org/seminars/02013/apr/17/beyond-digital/):“编程就像思考思考。调试是对学习的一种近似学习。”当你编程时，你把你的想法转化成可执行的形式。调试你的程序接近于调试你的思想。

现在举着这个火炬的人是布雷特·维克托。他显然对这些想法很深入，并且似乎将他的一生都奉献给了这些想法。更令人惊奇的是他交流思想的能力。把 [*思考不可思议的媒体*](http://worrydream.com/MediaForThinkingTheUnthinkable/) 放在你的必看名单上。如果你不为可能发生的事情感到一丝悲伤，那就观看 [*编程的未来*](http://worrydream.com/dbx/) 。他采纳了艾伦·凯的想法，并让它们变得非常容易理解。

我不怀疑还有更多像他们一样的人。从马歇尔·麦克卢汉的意义上来说，他们把计算机作为一种媒介。虽然计算机越来越多的渗透到我们的环境中，但是它们能够实现新的学习和思考方式的能力还没有被充分开发。

Lisps 和其他高表达能力的语言帮助很大。它们让你更快地表达问题——无需处理控制计算机的许多细节。我一直在寻找一个系统，可以用来快速实现论文中的想法。即使算法已经为你写好了，这也要花很长时间。最好的语言，包括 Clojure，仍然离我们想象的差得很远。

我把它像一盏明灯一样发出去，吸引那些喜欢编程的人，因为它有帮助我们学习的巨大潜力。你是其中之一吗？你为什么编程？在下面评论并告诉我:)
