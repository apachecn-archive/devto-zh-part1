# 成为编程早期采用者的乐趣和好处

> 原文：<https://dev.to/ben/the-joy-and-benefit-of-being-an-early-adopter-in-programming>

昨天我写了关于[晚采用者](../the-zen-of-missing-out-on-the-next-great-programming-tool)的好处，在那里我概述了*不会*太快采用新的编程语言/库/构建工具等的好处。程序员考虑时间的权衡是很重要的。这篇文章呼吁开发者不要让他们的焦虑和对错过的恐惧引导他们做出错误的选择，但是我想看看硬币的另一面。有很多很好的理由让你在技术选择上保持领先。

很多编程都是关于选择的。由于大量开源代码选项和专业软件服务的出现，我们的日常生活比以往任何时候都更需要选择。昨天我提出了让马戏团穿过城镇而不跑去加入的选择；实践纪律，以保持高效，并做出正确的长期选择。但是现在我想先谈谈你什么时候以及为什么会想跳进去。

## 跳进来的好处

1.  当你早期参与一个项目时，你会看到它的实时发展，并对发展过程中出现的问题和解决方案更加熟悉。你会对为什么做出这些选择有一个更大的了解，并且从长远来看会有更深的理解。
2.  早期通常会有一个更简单、功能更少的界面。也许这是因为关键特性丢失了，很快就需要添加，但是需要学习的东西会更少，需要思考的新概念也会更少。当你很晚才加入一个进化的生态系统时，通常会有很多进化的功能和构建链被现任开发人员视为理所当然。
3.  通过建议、编码或文档，有真正的机会来贡献和塑造项目的方向。您可能在决策中有发言权，并确保您的用例得到考虑。
4.  你将有一个很好的机会去了解开发者并建立良好的行业关系。与早期项目相关的较小的聊天室和列表为知识共享和交流提供了一个更加亲密的机会。
5.  您将有机会了解 API 并真正地使用它。如果您想用新的范例延迟编写重要的代码，这给了您在小项目中使用它的机会。尽早动手意味着有更多的时间，即使你没有马上开始使用这个工具，你也会给自己更多的时间来解决你内心深处的问题。
6.  参与新热点的兴奋感。不管生产力和结果如何，如果你对你正在做的事情不感兴趣，你将会有一段糟糕的时间，参与新事物可以让你在所有编程目标上保持动力。
7.  你很早就参与到最新、最伟大的事情中，一旦那些顽固的落伍者加入进来，你就走在了游戏的前面。然而，这也带来了很多不确定性，我将在下面介绍。

## 新技术的彩票

想要参与编程中不可避免的转变当然很好，但预测这种事情并不容易，你不能太快宣布赢家。这是在形势变化如此之大的情况下，对一个想法投入过多精力的最大危险。我们不需要对我们做出的每一个激动人心的选择进行分析，我们确实需要承担风险，在技术上下注，学习很多新东西，但我们不需要愚蠢。

在选择阵营之前，确保这些新想法适合你的大脑。有很多选择，你不需要卷入那些不会让你快乐和高效的程序员的事情。一旦我们确定了个人适合度，我们可以问一些关于技术的问题。如果我们喜欢自己想出的答案，我们可以更自信地投入进去。

## 关键问题

这是对现有选项的明显改进，还是仅仅是另一个选项？很难用没有明显更好的东西来取代现有技术。是的，你应该学习各种各样的想法，但不要跳来跳去做横向移动。注意参与更大更好想法的机会。

这项技术的背后是最初的支持和兴奋，还是主要的既得利益者已经明确表示愿意使用它？React 非常成功，这在很大程度上是因为脸书对此做出了直截了当和透明的承诺。 [Angular](https://angularjs.org/) 和谷歌的其他项目虽然也很受欢迎，并得到了一家大公司的支持，但它们一直在努力展示透明的承诺和既得利益。[著名的](http://famous.org/)，一个在这个领域不太成功的项目，在早期并不是没有得到支持，但未能吸引足够的兴趣而被广泛采用。Ember 仍然非常重要，因为它有一个开放且值得信赖的核心团队，以及做出正确选择的早期历史，但 React 已经成为该领域的基本领跑者。

你个人对这个热门的新事物有需求吗？如果你有一场激烈的斗争，并找到了一个很好的解决方案，你有很多很好的理由去采纳它。这并不意味着你应该有狭隘的眼光，因为可能会有其他更适合你的东西，但是如果你对某个工具有直接的需求，而不是抽象的未来需求，你会过得更好。如果你在一个当前或即将到来的项目中纠结于 [Ruby on Rails](rubyonrails.org) 模式和相对平庸的性能，你应该打开你的搜索，审视一下前景，选择一种新的技术，即使这样你也不一定要在有许多长期存在的性能语言之间权衡你的选择。如果您只是笼统地将性能和可伸缩性作为最终的问题，我认为您在采用新(或旧)技术时不会感到愉快。

## 最后

有很多很好的理由让你尽早涉足某些技术，但是在有大量选择的情况下，要非常注意风险和权衡，这样你才能做出好的选择。技术的早期和晚期采用并不是一个二分法。作为一名开发人员，新技术和传统技术的良好结合将拓展你的思维，增强你的长期潜力。如果你不屈服于闪亮物体综合症，不因为害怕和担心错过而行动，你会做得很好。
