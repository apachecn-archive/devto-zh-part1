# 播客

> 原文:[https://dev.to/funkysi1701/podcasts-14j5](https://dev.to/funkysi1701/podcasts-14j5)

我听播客已经有一段时间了。对于那些不知道播客是一个音频文件，你可以下载并收听，通常是关于你感兴趣的话题。让我感兴趣的话题显然是星际迷航，科技和 IT 之类的东西。我主要是开车的时候在车里听，但有时候在家里做家务的时候也会听。

我想我听的第一个播客是 [Trekmate](http://www.trekmate.org.uk/) 。Trekmate 顾名思义是关于《星际迷航》的，是几个人在谈论《星际迷航》。这和你在酒吧里和朋友的对话非常相似。Trekmate 做了很多播客，几乎每一个版本的《星际迷航》都有一个播客，我并不了解所有的播客，但是我真的很喜欢我听的那些。

我最喜欢的与工作相关的播客是 RunAs Radio。如果你是系统管理员，或者对服务器或 IT 感兴趣，你应该听听这篇文章。主持人 Richard Campbell 就某项特定技术采访 IT 专家，一周前是 Exchange，下一个是安全，然后是 SQL Server。通常我会学到一些我想尝试的东西，有些星期我不会学到一些具体的东西，但会让我对商业 IT 的发展方向有所了解。我想可能是在听了 RunAs Radio 之后，我才开始对我的 it 基础架构有了更长远的规划。

正如我之前在博客中提到的，我正在学习开发，所以很明显我会听[。净岩](http://www.dotnetrocks.com/)。这是由卡尔·富兰克林和理查德·坎布尔(没错，就是经营 RunAs Radio 的同一个人)主持的，他们从更多开发者的角度与开发者谈论代码、学习和 IT。这是一个星期发布 3 次，所以有时有点击中和错过，但大多数星期会有一些与我相关或值得一听的东西。

几天前，我第一次做了一件事，我是一个叫 TenForward 的旅行者的客人。我看了 5 集《星际迷航》。作为一个内向的人，这对我来说是一个飞跃，我不喜欢成为关注的焦点，我当然也不喜欢我的声音被录音。希望当发布时人们会喜欢它，我会在未来做更多这样的事情。一旦它发布了，我会在这里放一个链接，这样你们都可以听听。
