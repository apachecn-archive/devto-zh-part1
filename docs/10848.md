# 所以你希望你的编程语言受欢迎

> 原文:[https://dev . to/acanjiano/so-you-want-your-programming-language-to-be-popular-71i](https://dev.to/acangiano/so-you-want-your-programming-language-to-be-popular-71i)

一些编程语言变得流行起来。其他人没有。这是为什么呢？流行的语言比晦涩的语言好吗？【1】比这还复杂，那我们就拿根长棍戳这个马蜂窝吧。

## 定义流行

首先，对于编程语言的流行程度并没有真正的共识。基于 Google 搜索、StackOverflow 问题、GitHub 存储库等等，有各种各样的努力来对它们进行排名。

根据你考虑的启发和你在排名中的权重，你最终会偏向于支持或反对某些语言。

例如，一种编程语言的受欢迎程度与 StackOverflow 上相关问题的数量之间存在明显的相关性。然而，在所有条件相同的情况下，这种度量标准会对更简单、文档更好的语言不利。

如果你把 GitHub 库作为流行度的指标，你会惩罚那些代码倾向于被托管在其他地方的语言。当然，这同样适用于在开源世界中没有广泛使用的语言。

谢天谢地，我们不需要精确的排名来进行有意义的对话。就本文而言，让我们一致认为“流行”的大致定义是在 [TIOBE 指数](https://www.tiobe.com/tiobe-index/)或 [Redmonk 指数](https://redmonk.com/sogrady/2016/02/19/language-rankings-1-16/)中进入前 20 名。[2]

## 流行的编程语言更好吗？

流行的语言不一定比不太流行的语言好。

我认为 Lisp 家族在很多方面都是最好的语言家族之一。根据我们上面的定义，它最多只是稍微受欢迎。

我无意挑起战争，但我觉得有必要指出 PHP 是世界上最流行的编程语言之一。

仅仅技术上的优势并不能区分赢家和输家。让我们看看是什么。

## 默认语言

列表中几种最流行的语言是我定义的默认语言。

默认语言是给定平台的自然首选语言。

例如:

*   **C** 是 Unix 系统编程的默认语言；
*   **JavaScript** 是网络浏览器的默认语言；
*   **SQL** 是关系数据库的默认查询语言；
*   **Java** 是安卓应用的默认语言；
*   **Swift** (以及在此之前的 Objective-C)是 iOS 和 Mac 应用的默认语言。

默认语言很受欢迎，因为任何对在给定的生态系统中开发感兴趣的人都会默认使用它们。

他们比其他语言有巨大的优势。

使用其他东西需要研究和努力。借用科技界的一个例子，BASH 比 ZSH 和 FISH shells 更受欢迎，原因很简单，因为它是许多*nix 系统的默认设置。

## 科技巨头支持的语言

另一类流行语言是由科技巨头支持的语言。

请注意，仅仅得到大型科技公司的支持是不够的。想想谷歌和 Dart(尤其是在 Flutter 之前)。不，为了让它受欢迎，公司必须支持这种语言。

微软对 C#的推动如此之大，以至于它几乎成了 Windows 编程的默认语言。

## 大型标准库

除了依赖其位置优势的选择默认语言之外，大多数流行语言都有一个大型的标准库。

你需要有一个来吸引最初的开发者种子群体，他们会反过来帮助你发展图书馆的生态系统。

Python 就是一个很好的例子。它的“包含电池”理念使它在很早的阶段就很有用。一颗巨大的图书馆生态系统的种子。

通常，近年来出现并变得流行的新语言利用现有平台作为一种手段，立即提供大型标准库和规模良好的库生态系统。

最常见的例子是 JVM 语言，比如 Scala、Clojure 和 Groovy。

再过几年，Elixir 可能会成为 BEAM(Erlang 虚拟机)上流行语言的一个例子。

## 范式转变

有时编程语言变得流行是因为它们为行业内的特定范式转变提供了答案。

OOP(如 C++，Java)，web(如 JavaScript，Perl，PHP)的出现，移动编程(如 Objective-C，Swift，以及最近某种程度上的 Kotlin)，数据科学(如 R)的出现，想到了。

范式转换并不经常发生，但一旦发生，它们就为新语言打开了大门。

现在，函数式语言有机会被广泛采用。部分是因为它们在一个日益并行和平行的世界中运作得如此之好。[3]

## 黑仔应用

这给我们带来了另一个为什么一些语言在采用方面成功的原因。

在给定的领域中拥有绝对的杀手级应用。

Ruby 是这方面的典型代表。在日本之外，在它开始在其他地方受到重视之前，这种语言曾被忽视了大约十年。

然后 Rails 出现了——这对于 web 开发来说是一个非常有吸引力的提议。[4]人们对此印象深刻，愿意学习这种鲜为人知的叫做 Ruby 的语言，仅仅是因为 Rails。

当他们这样做的时候，他们发现这是一种多么好的小语言。它的技术优势是其被采用的基础(也是 DHH 选择它用于 Rails 的原因)，但光靠技术优势是不够的。[5]

这种情况并不常见，但是在流行领域中为特定问题提供更好答案的语言可以在流行度方面获得巨大收益。

## 还有什么帮助？营销。

营销有帮助。一个不回避好的技术营销的社区无疑会帮助这种语言变得更加流行。

请注意，营销是一个非常广泛的术语。好的文档是好的营销。不用花一个小时安装东西就可以开始，这可以说是你营销方法的一部分。

接受、包容和欢迎新人是好的营销方式。写博客、组织会议、在其他会议上发表演讲、写好书，这些都是以营销为导向的社区所做的事情的好例子。

## 还有什么？熟悉度。

熟悉度很重要。如果你的编程语言与大多数开发人员所习惯的完全不同，它将很难被广泛采用。

所以它必须服从命令。它必须具有足够的创新性，使其值得使用，但又足够熟悉，不会在新开发人员接触它时立即引起负面的本能反应。

这是许多 Lisp 语言不得不面对的一场斗争。大多数开发人员都熟悉 Algol/C 语法，然而，S 表达式对许多人来说有些可怕和陌生。[6]

正因为如此，尽管函数式语言这次有机会，但这不会是一次轻松的销售。

## 还有别的吗？时间。

编程语言不会一夜之间流行起来。不管你怎么切，都需要很多年。

大多数语言至少需要 10 年才能被广泛采用。近年来，采用率越来越快，但这个过程仍然需要相当长的时间(几年)。

最明显的例外是 Swift，它在推出后的短短几年内迅速崛起。Swift 不仅得到了苹果公司的支持，还成为了世界上最流行的开发生态系统之一的默认语言。

因此，如果你是一名编程语言开发人员，或者是一门希望变得更流行的语言的早期使用者，请耐心等待。你将长期参与其中。

## 受欢迎并不容易

让一门编程语言流行起来真的很难。

竞争是激烈的，开发者有这么多有效的选择。

当我们无症状地接近所有可能的编程语言时，创造出比它更好的东西就更加困难了。

开发人员需要令人信服的理由来跳槽，甚至考虑在你的语言和社区上投入时间。

考虑一下为什么程序员想要学习一门对他们来说是新的语言:

*   得到或保持一份工作；
*   变得更有生产力和/或编写更好的软件；
*   开发新的应用类别；
*   学习一种新的范式。

如果你的语言至少能满足这些需求中的一部分，它就有机会了。但这只能让你获得参赛资格。

## 结论

总之，我认为这些是需要考虑的关键因素:

*   成为流行生态系统的默认语言；
*   得到科技巨头的支持和推动；
*   拥有一个大的标准库和/或以一个流行的 VM 为目标(例如 JVM，除非它是一种像 Rust 这样的系统语言)；
*   充分体现了新的范式转变；
*   在特定领域非常有用，就像 Ruby 在 2005 年通过 Rails 做的那样；
*   拥有优秀的文档、新手指南、工具、编辑器集成，并且总体上消除了任何使语言入门困难的工具障碍；
*   培养一个欢迎和包容的社区，鼓励分享、营销和布道；[7]
*   具有某种熟悉的语法，易于阅读、书写和教学；
*   活跃发展了几年；
*   提供技术创新，从而提高生产率和代码的可维护性；
*   运气。

对于大多数语言来说，前两个是不可能的，即使这样，也很少有语言能够满足所有其余的需求。

然而，即使只接受其中的一些，也足以让你的语言有机会成为主流。

如果做不到这一点，你可能仍然会得到一种活跃的、合适的语言，这种语言将在未来的几十年里继续存在并被许多开发人员所欣赏。
