# 你好社区

我是阿德里安，一个编程狂，我从小就服用这种药物(父亲爱好程序员)，我在高中和教职时开始编码，然后作为一份全职工作。我在业余时间发展，而且我足够幸运，可以在白天实践我的爱好并得到报酬。

我必须完成开发。第一次用户体验，我也看到我有一些追随者，虽然我还没有写任何文章，所以我觉得有压力写这个。

除了在度假的时候，我不能停止使用这种药物，我会手指发痒，有大量的想法，并且越来越想知道还有哪些闪亮的蓝色新药被发明出来。

[![drugs](img/594e017f2f4a2cdc79f64d32b6622364.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--F3PEvxV6--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://static.sashe.sk/data/2014/05/11/photos/71554/c/c-N4018648_71554_20140511_69565.jpg)

我是一个网络通才(我认为)，全栈网络开发人员，但我也涉足其他语言和平台(如移动，虚拟现实，小工具和玩具)。我喜欢帮助人们，教授编程(以及相关的技术领域)，指导其他开发人员，并撰写相关文章。

从今年开始[我重新启动了我的博客](https://coder.today/)，我试着写一些通用的帖子(适合任何开发者)和更少的网络开发内容。关于我的更多细节。

我是一个务实的程序员，对“正确的事情”充满热情，我认为我们的工作是一门**手艺**。这是什么意思？嗯，我喜欢写干净的代码，使用最佳实践，我不认为单元测试应该是可选的，但我也已经进入战壕很多年了(初创公司)。

我习惯于每天发布新的版本，并且知道大多数“正确的事情”不能像我们在书中学到的那样实现，至少在“公司之外的世界”是这样。因此，我尝试搜索、开发、[学习和撰写将最佳实践的“现实世界”(明天交付)与“梦想世界”相结合的技术。](https://coder.today/how-to-maintain-a-clean-code-a-3-step-technique-with-example-477e20c882df)

因此，你会注意到我的回答是混杂的:我将攻击所有与 TDD 相关的文章，这些文章将这种技术视为一种信仰(所有时间的救世主，Bob 叔叔的风格)，但是我也建议自动化测试不应该是可选的，并且所有 3 本干净的* Bob 叔叔的书都是必修课(针对中高级开发人员)。

这结束了对我的咆哮，到目前为止，我喜欢这个网站，所以继续努力。
