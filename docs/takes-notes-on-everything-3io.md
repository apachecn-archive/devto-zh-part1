# 对所有事情都做笔记

> 原文：<https://dev.to/maxwell_dev/takes-notes-on-everything-3io>

我最喜欢的一本书是大卫·麦克拉尼的《你并不聪明》。它列出了几十种人类经常犯的常见认知错误，从我们如何扰乱社会关系，曲解我们周围的世界，到发明虚假的动机和记忆。

我为什么会喜欢这本书？这是一个很大的清单，列出了我思想的局限，只有知道这些，我才能绕过它们。其中最重要的一点是我记得我读过的东西的可靠程度。

也就是说，根本不可靠。在编码方面当然不会。

一个例子是我最近买的，劳拉·卡尔巴格的《每个人的可访问性》(或者真的，除了一本书以外的任何令人惊讶的东西)。在深入阅读之前，我先浏览了一下。但是如果我真的想用那本书上的任何东西，比如说在一个新项目中，我实际上不会记得任何有用的东西。我不得不打开书，从头再找一遍——最多我知道从哪里开始找。考虑到大多数开发人员每天处理的信息量，这种情况时有发生。只有我至少看了五遍的信息才会消失。

这就是为什么把我读到的所有东西都记下来是一个重要的习惯，但我很遗憾地放弃了。我自己的偏好是[一个 Github repo](https://github.com/maxx1128/Webdev-Study-Notes) ,里面装满了关于书籍、博客帖子以及任何我想留着以后用的东西的降价文件。重要的语法规则、术语定义、个人技巧或想法、引用或总结——我会写下我从文本中发现的任何鼓舞人心或有用的东西。迄今为止，这些措施包括:

*   可维护 CSS 和前端库的技巧
*   网页排版基础
*   我容易忘记的重要 Git 命令
*   硬汉网页设计的风格指南和 CSS 技巧。
*   有效的用户界面和 UX 不要让我思考。
*   [本质主义的原则](https://www.amazon.com/Essentialism-Disciplined-Pursuit-Greg-McKeown/dp/0804137382/ref=sr_1_1?s=books&ie=UTF8&qid=1509464782&sr=1-1&keywords=essentialism)
*   一本关于不要以单身为耻的书的主要建议和要点(随便你怎么判断，这是一本好书！)

写下这些笔记有很多好处:

*   它帮助我把信号和噪音分开，并保持它们分开。当我不需要浏览一本书或博客文章时，寻找一些东西会更容易，为了获得额外的帮助，我有搜索功能。
*   它迫使我从更大的来源中提取重要的信息。我希望我的笔记越简单，我就需要越快抓住要点。
*   如果你将笔记保存在公共回购中，其他人可以参考你的笔记。这对于开源社区、寻找好资源的初学者来说是一个胜利，或者对于你的职业来说是一个胜利，如果希望招聘的球探或开发者看到你专注于学习更多东西的话。
*   放松和翻阅过去的笔记总是有趣而富有成效的。

我不是说记笔记比写实际代码更重要。但是在我的编码工作流程中，我发现我的笔记是重要的平衡行为的一部分。完成一个项目给我一种巨大的、欣快的成就感，就像嗑药一样——我很焦虑，急于开始下一个项目。记录我的新知识给了我一种逐渐的、更平静的成就感，这种成就感随着时间的推移而增长。它帮助我更好地检查我正在学习的东西，并且不会忽略重要的细节。从长远来看，它使学习更容易，并帮助我记住更多。作为奖励，它甚至可以帮助其他阅读它们的人。

这就是为什么我强烈支持程序员在代码之外写更多东西的部分原因。如果不是给别人看，比给自己提高知识和手艺那要多得多。
