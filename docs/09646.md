# 流畅团队的力量

> 原文:[https://dev.to/teamcoder/the-power-of-fluid-teams](https://dev.to/teamcoder/the-power-of-fluid-teams)

你知道自己做副业项目时这种激动人心的感觉吗？当你致力于它时的难以置信的动力和当你发表它时的这种神奇的感觉！这类项目和你正在工作的项目有什么不同？我认为最相关的区别是，大多数人在工作中做他们被告知要做的事情，而不是他们喜欢做的事情。

想象一个没有人告诉你该做什么的工作环境！如果你能在自己真正相信的事情上努力，那该有多好！如果你能和同事谈论你的想法，每个真正相信它的人都会加入你。团队将是流动的！它们会自发形成并存在，直到工作完成。然后你可以加入另一个说服你的团队。

你觉得这会有多大成效？我认为潜力将是惊人的！功能将会开发得更快，而且充满爱！

问题是没人想做的任务会怎么样？也许，如果没人想做，那就没那么重要了。如果某样东西不能让公司的任何人兴奋，也许它也不能让大多数用户兴奋。

我想尝试一下，但很难说服一家公司放弃所有的控制权，希望一切都会好起来。我看到一家来自美国的软件公司 [Valve](http://www.valvesoftware.com/) 做得非常成功。你有类似实验的经验吗？我很想听听这件事！
