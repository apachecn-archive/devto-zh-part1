# 代码死了！

> 原文：<https://dev.to/stcalica/code-is-dead>

### 1997 年，克拉斯乐队臭名昭著地宣称“朋克已死”。

“朋克已死”这句话是什么意思？这个短语指的是一个地下的局外人场景，比如定义朋克音乐的场景，走向主流，为了高额利润而卖出。到了 20 世纪 90 年代，朋克乐队开始在 MTV 和 VH1 出现，这无疑是事实；商场里挤满了年轻人，他们谈论着热门话题，试图展示最新的朋克潮流(不做评价；我是其中之一！).场景已经转移到一个叛逆的怪人场景，一个被利用来获取巨额利润的场景。

我们可以将死亡的定义，即从地下局外人场景到可开发的高收入利润主流的运动，应用于大多数局外人运动。一口气；我在 Youtube 上看到的会议视频，敏捷软件方法论的创始人之一，迪夫·托马斯宣称敏捷已死。对于那些不知道的人来说，敏捷是一种快速生产灵活软件产品的软件开发方法论。2001 年 2 月在犹他州，17 名软件开发人员聚集在一起，撰写了敏捷宣言。正如 Dave 指出的，敏捷经历了一次病态的转变。敏捷不再那么敏捷了。它变成了一种严格的方法，公司开始冒出来，向其他公司和开发者提供昂贵的培训课程。因此，大卫只能宣布敏捷已死。

我们可以把这个定义应用到敏捷上，并进一步推广。作为一名大学学生，我参加了许多黑客马拉松。每一次黑客马拉松都有很多学生竞争在规定的时间内编写项目代码并赢得现金奖励。所有这些黑客马拉松都有大型赞助商，如微软、LinkedIn、NFL 等。每个人都因为使用了他们的技术而获得了他们自己的奖励。过了一段时间，看到一些项目因为不符合公司希望他们用他们的技术做的事情而被抛弃，令人作呕。创新受到金钱的束缚。另一个趋势是编码训练营；没有什么技能的人去参加一个为期一周的研讨会，学习如何编写非常具体的代码，然后试图被录用。据 SkilledUp 称，代码训练营的费用从 2000 美元到高达 14000 美元不等！现在，人们想编码或找一份编码工作并没有错，但有很多人只对硅谷提供的高薪感兴趣。

让我们把这与旧的地下、局外人的场景进行对比，那里曾经是家酿计算机俱乐部。家酿计算机俱乐部可以追溯到 20 世纪 70 年代；远在每个家庭和人手里都有电脑之前。这些是对电子产品感兴趣的人组成的团体，他们会出于热情一起探索项目。走进任何一个黑客马拉松现场或编码训练营，你仍然会看到对计算机感兴趣和充满热情的人，但我怀疑你是否需要走很远才能看到那些努力赢得大公司大奖的人(在黑客马拉松结束时，让这个项目在 Github 上腐烂掉)或试图给那些大人物招聘人员留下深刻印象。随着大量的人和公司涌入扭曲了过去的边缘场景，我想你可以说我要去哪里。代码已经死了。

但是，仅仅因为计算机编程成为主流并有利润空间，并不意味着这些材料不那么令人兴奋。让我们承认，如果我们想要一份朝九晚五的稳定工作，我们都必须在某种程度上玩公司游戏。让我们也承认这意味着更多的机会被发现，更多的人被引进意味着有更多的机会拥有高质量的人才。诀窍是不要失去激情，努力成为人群中的一员。试着做一个叛逆者。多做兼职不是为了更高的收入，而是因为这是你喜欢做的事情。成为死人中的活人。
