# 展示和讲解

> 原文:[https://dev.to/craigbuchek/show-and-tell-2i1c](https://dev.to/craigbuchek/show-and-tell-2i1c)

我将于 12 月结束目前在 Mercy 的咨询工作，并在 CenturyLink 开始新的工作。我是一名软件开发人员，但这只是我工作的一部分。我真正做的是加入一个团队，帮助他们改进工作方式——包括他们的流程和技术技能。

我认为这是我作为顾问的一个关键优势。大多数顾问(和敏捷教练)进来告诉人们该做什么。我不喜欢告诉人们该做什么。我更愿意与他们并肩工作，更好地了解他们面临的挑战。一旦我对挑战有了更好的理解，我就能更好地集思广益，尝试一些想法。然后，我们可以进行实验，看看什么可行，什么不可行。

我不是告诉人们做什么，而是告诉他们如何做。大多数人通过看比通过听学得更好。如果你解释如何和为什么，而不仅仅是什么，他们也会学得更好。所以告诉他们怎么做比告诉他们更有效。通过展示和行动，你也可以树立一个好榜样。当协作是需要改进的很大一部分时，这一点尤其重要。

我发现这种咨询方式更受每个人的推崇。我通过与开发人员密切合作来与他们建立信任。一旦经理们看到这些方法有多么有效，他们就会喜欢把我留在身边，所以我的工作往往会持续相对较长的时间。

我遇到的最大问题是解释这是如何工作的。我真的不知道该在简历上写些什么。我有时称自己为敏捷实践者，有时称自己为敏捷玩家/教练。但是这些描述并不十分令人满意。我正在考虑把“我帮助团队改进他们的工作方式——包括他们的流程和技术技能”写进简历。但这看起来很尴尬，错过了表演和讲述的部分。我愿意接受任何建议。
