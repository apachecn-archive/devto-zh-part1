# 怠惰

我很懒，我不会否认这一点。早上闹钟响的时候，我会在起床前小睡 20 分钟左右。

在我的工作中，我继续懒惰。远程桌面(或 RDP)可能是我的头号懒惰工具。对于那些不知道 RDP 允许你连接到另一台计算机和访问它，就像你坐在它面前。因此，我可以坐在办公桌前，RDP 到办公室的任何其他计算机，包括任何服务器。然而，这个懒惰的工具有时确实需要一点努力，有时，打开目标计算机的电源，登录本地，确保您正在使用的用户帐户被允许使用 RDP。

写脚本是懒惰工具的另一个例子。我经常被要求做冗长乏味的任务，因为我很懒，所以我会努力学习如何编写脚本来完成这项任务，这样我就可以在几秒钟内运行这个脚本并完成这项冗长的任务。有许多不同类型的脚本，从数据库 SQL 脚本到 PowerShell 脚本，几乎可以在您的服务器上做任何事情。

PowerShell 是微软目前正在大力推动的东西，你甚至可以编写脚本来创建新的用户帐户，这样你就不必再为每个新用户帐户打勾了。由于 PowerShell 是几乎所有 MS 技术的一部分，您可以将 Exchange 链接到 Active Directory，您甚至不必记住正确的语法，因为 PowerShell 有一个内置的帮助命令来告诉您如何运行这个有用的命令。

由于我现在正进入一个更具开发性的角色，我还能继续偷懒吗？我当然可以。代码应该只写一次，尽可能多地重用，这是 OOP(面向对象编程)的特点之一。

它可以像创建一个母版页一样简单，因此您不需要在每个网页上重新创建相同的代码。或者每次你发现自己又要重写同样的代码，你就把它插到一个方法里，这样它就可以被反复调用。

当然，懒惰的终极方式当然是给自己找一些员工，花一整天让他们做所有的事情。如果你幸运的话，他们甚至会尝试采纳这些懒惰的想法。
