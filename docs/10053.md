# 为什么我现在开始学生锈

> 原文:[https://dev.to/andrewlucker/why-i-am-learning-rust-now](https://dev.to/andrewlucker/why-i-am-learning-rust-now)

从一开始，我就一直模糊地意识到这种新语言叫做 Rust。到目前为止，它主要是由系统编程社区开发的，也是为系统编程社区开发的。有人说它会取代 c。

对我来说，我不是强硬派，我会接受旧神继续存在，繁荣，折磨他们的追随者。不过，谢天谢地，我从来没有被绑在这些更大的社区。我和福斯、普洛斯、SML、哈斯克尔以及一些需要逃离围墙花园的 C 一起长大。在学习 Rust 之前，我曾忧虑地看待它:由于缺乏理智的函数式编程。即使在今天，经过几个月的调整，我担心多线程比任何闻起来像旧书的东西更重要。

这就是为什么我对 FP 在 Rust 的未来充满希望。传统上，学术编程和实践编程之间存在裂痕。象牙塔里总是有“好的答案，而其他人只是使用“坏的变通方法”。生锈可以改变这一点。

Rust 中的类型系统，由于对对象生存期的痴迷，已经创造出了一些如此偏执的东西，以至于它几乎类似于构造的演算。这是编程理论从业者可以要求的仅次于 Lambda 的最好的东西。目前 Rust 中的 lambdas/closures 是非常可怕的东西:你可以使用 FnMut 或 closure 并“移动以复制所有的作用域变量”。这种上下文信息形成了 CoC 逻辑的环境。现在所需要的是 Rust 的类型系统的形式化，然后我们将有一种可以被正式操作的语言。

Rust 还结合了一些其他的东西，这些东西在函数式编程的出版期刊中有历史。Rust 异常处理的独特之处在于，它不需要特定的堆栈结构，也不需要长时间跳转到远处的代码路径。所有基本的分类操作都已实现。我已经开始怀疑 Rust 会成为一门宠儿语言，直到我不停地用它编程，尝到了嘴里的恶趣味。

根据我对这种语言的了解。第一，语言社区正在成长。第二，语言维护者对实验持开放态度。第三，越来越多的共同结构将 Rust 与 PLT 群体联系起来。作为一名语言学家，当更多的人开始影响一门语言的发展方向时，我相信它会创造出一个更好的最终产品，或者至少它会磨平粗糙的边缘；而这正是铁锈现在最需要的:好好打磨。

我的 Rust escapades 可以在 Github 上开源获得。

–学生
