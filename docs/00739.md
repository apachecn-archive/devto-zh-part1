在我们日益互联的世界中，API 已经成为新的数字商业模式的基础。它们是应用程序、数据和服务之间的连接器，使您的企业能够在 web 应用程序、移动应用程序、物联网(IoT)传感器和其他接触点中公开数据和服务。

当您开始通过公开 API 来增强您的业务模型时，您会很快发现，为了取得成功，您需要多种能力。

这是因为 API 管理不仅仅是一个安全层或对您提供的 API 的描述。事实上，当启动一个 API 程序时，你在很多方面都是在启动一组产品，这些产品需要像你公司提供的其他产品或服务一样进行管理。因此，如果您希望您的 API 程序成功，您需要将所有的部分(或功能)都准备好。

## [](#about-webmethods-api-cloud)关于 webMethods API 云

Software AG 的 webMethods API Cloud 是一个 API 管理即服务平台，它为您提供了一个用于开发人员协作的 API 门户和一个保护您免受恶意用户攻击的 API 网关。

借助 API Cloud，您可以设计和编写 API，构建和发展您的开发人员和合作伙伴生态系统，安全地公开您的 API，将您的 API 货币化，并分析您的 API 以了解哪里需要改进。

创建一个成功的 API 程序需要哪些能力？让我们来探索它们。

## [](#building-and-managing-your-ecosystem)构建和管理您的生态系统

要创建一个成功的 API 程序，你需要吸引和发展用户，这些用户通常是希望将你的服务或数据添加到他们的应用程序中的合作伙伴。要做到这一点，你需要一个开发者门户，类似于你的 API 产品的市场。

开发者门户是面向公众的网站，致力于帮助您的受众注册和使用您的 API。但是它提供的不仅仅是关于 API 的信息。

**图 1** :使用 webMethods API Cloud 中的 API 门户构建您的 API 生态系统

**API 门户提供以下功能:**

*   **API 发布**:这允许您将 REST、SOAP 和 OData APIs 发布到开发者门户，供您的合作伙伴和其他第三方开发者发现和了解。
*   **用于注册的内置工作流**:开发者门户需要管理第三方访问请求，以便合作伙伴可以注册使用您的 API。API Cloud 具有内置的审批流程工作流，可确保安全可靠的访问管理。这有助于您的合作伙伴快速启动和运行。
*   **文档**:开发者需要知道如何使用你的 API，以及你的 API 会输出什么。在开发人员门户中，您可以提供 API 的描述、如何使用 API 的示例、附加文档的文件附件以及关于在 API 级别实施的策略的信息。基本上，您希望提供您的合作伙伴使用您的 API 所需的一切。
*   **集成 API 测试**:让开发人员能够轻松地在门户中直接测试 API，了解 API 的表现。让他们用不同的输入参数尝试测试调用，这样他们就可以很快看到结果。
*   **社区支持**:记住一个 API 程序要成长，你需要持续的采用。为了建立采用，展示更广泛的社区如何使用你的 API 是有益的。这就是为什么在开发者门户中提供一个社交社区是有帮助的。Software AG 的 webMethods API 门户的独特功能之一是，它允许用户对 API 进行评级，并有助于与其他开发人员进行公开讨论，以及突出显示也在使用您的 API 的第三方应用程序。
*   **内置使用分析**:如前所述，你的 API 就像产品一样。所以理解你的用户是很重要的，这样你就可以确保你正在构建正确的 API。API 门户允许您跟踪 API 使用情况，以获得关于使用模式的宝贵见解，从而帮助您改善客户体验并提高 API 采用率。
*   **品牌化**:因为你的 API 是你业务的延伸，你需要能够定制和品牌化你的开发者门户来匹配你公司的企业形象，从颜色和皮肤到定制图形。

## [](#securing-and-governing-your-apis-at-runtime)在运行时保护和管理您的 API

获得 API 的采用同样重要的是保护它们的需要。这就是 API 管理服务的核心组件是 API 网关的原因。API 网关提供以下功能:

*   **安全性和认证**:API 网关使您能够使用策略驱动的安全性来统一保护和监控使用您的 API 的应用对后端服务的访问。我们的网关支持传输和消息层安全，包括认证、授权、数字加密和数字签名，以确保您的 API 免受恶意或未经授权的攻击。
*   **DMZ 强度安全**:任何时候你向外部用户公开 API，你都在创造潜在的安全威胁。降低风险的一个方法是不要在防火墙上打开任何不必要的漏洞。API cloud 通过在使用您的 API 的消费者应用程序和您的内部服务器之间使用 DMZ 强度的安全性来消除防火墙中的漏洞。您可以配置内部防火墙来拒绝所有入站连接，然后使用从内部服务器打开的永久连接来访问后端服务。

[![](../Images/9bf7593e005b7f9bda26cafc0fb18716.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--24mF2Lzb--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/http://techcommunity.softwareag.com/documents/10157/9398481/fig2-apimgmt.jpg/75792573-56b3-4331-83fe-9cea8c2816af%3Ft%3D1516181504945)

**图 2** :使用 webMethods API Cloud 中可用的 API 网关保护您的 API。

*   **转换和路由**:虽然您想在防火墙后面而不是在网关中完成大部分逻辑，但是您需要一些路由和转换的灵活性。借助 Software AG 对映射和转换的广泛支持，您的 API 消费者可以灵活选择他们想要使用的协议、消息格式和传输方式。
*   API 货币化:因为 API 程序是数字业务的一个关键部分，你需要能够捕捉到你的 API 是如何被消费的，这样你就可以将它们货币化。API gateway 可以利用构成订阅包基础的 API 计划和包对 API 进行货币化，使您可以通过按月订阅或按使用付费的方式对 API 的使用进行收费，以及在部门内和跨部门进行退款。
*   **生命周期模型和版本控制**:为了确保在您的公司创建 API 并在 API 开发生命周期中移动它们时遵循标准和实践，API gateway 提供了生命周期管理和治理功能，以支持版本控制以及 API 随着时间推移的迁移和淘汰。
*   **策略管理**:您需要设计和配置在运行时应用于 API 的策略。策略可以同时应用于多个 API，并且可以通过命名模式、分类或组织来动态确定范围。
