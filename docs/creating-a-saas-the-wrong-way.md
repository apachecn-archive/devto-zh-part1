# 用错误的方式创造 SaaS

> 原文：<https://dev.to/arvesolland/creating-a-saas-the-wrong-way>

所以，我创造了一个产品，却没有真正验证人们是否会购买它。

我知道，我知道——愚蠢的权利？

整个事情开始于去年做一些客户工作的时候，我收到了做类似工作的请求，处理来自多个客户的同一个问题。

我交付的解决方案工作正常——但是很难测试，因为它安装在客户的服务器上，而且交付后我对测试、监控和改进解决方案的了解和访问有限。然后我有了一个绝妙的主意:为什么我不把它变成一个 SaaS 应用程序，让多个客户都可以使用呢？

这听起来像是一个很棒的想法，我的意思是，我已经开发了它的第一个“版本”…我只需要“调整”它，把它变成一个通用工具，我就可以作为产品发布了…简单吧？

[![Yep...I know](img/8e2ad87f873969af770898c6d0fee1e4.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--PqZdMsHE--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/mdyfokvx5my5m7mnqgp0.gif)

哦——几个月后，数百个清晨和周末——我创造了我的解决方案的 SaaS 版本。这不像我想象的那么容易。这比我预期的时间要长，现在是另一部分:我现在该做些什么来销售这个解决方案？？

在开发 SaaS 应用的时候，我消耗了太多太多关于创业和创业的信息。令人恼火的是，他们都重申了同样的事情:在你去建造一些东西之前——确保你与潜在客户交谈并验证你的想法！Â

在我的发展过程中，我一直听到这些话，但我真的不想听到它们，因为我知道我没有这样做。我知道我做错了。

客户对我的新 SaaS 应用程序正在解决的问题仍然很感兴趣，但事实证明，销售未知在线服务的订阅服务与销售我的小时数来创建客户可以在他们的服务器上实施的定制解决方案有很大的不同。

就此而言，我在营销数字产品或非数字产品方面没有直接的经验。我也没有直销这类产品的经验。那么我接下来要做什么呢？

[![What to do now](img/8a0098c17df8836bae57b35cc1a22365.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--EXh7Ehmx--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_66%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nlq3x13bgismn9jjymw6.gif)

无论我走到哪里，都有关于如何发展你的创业公司的建议——但是一旦你开始时没有任何观众——接下来有什么潜在的步骤？大多数建议都假设你已经做了功课，遵守了规则——不像我。

所以我有一个产品——仅仅因为我可能做错了事情，没有真正做好产品/市场适合性验证——我仍然想测试一下这个东西是否能飞起来。我不想因为没有先去找 100 个感兴趣的人就放弃它。

现在，这并不意味着我永远不会放弃我的产品——我知道我对自己创造的这个小东西有感情依恋(大多数开发人员都是这样)——但我会尽最大努力保持现实，因为我尝试了不同的策略来尝试让这个东西启动。

但是我真的很想好好体验一下！

那么像我这样的违规者下一步会做什么呢？我如何在潜在客户面前展示我的解决方案？我营销/推广这个东西的最佳方式是什么？我怎么知道什么时候该放弃？

当我在这段旅程中发现我此时的选择时，我会在每周的博客文章中记录我的发现，以便其他人也可以尝试我发现的一些策略。

我本周的第一步是真正尝试定义我的理想客户，并获得一些关于寻找和瞄准这些客户的想法。

直到下一次…
