# 这是让我的应用程序在国际学生竞赛中获得第三名的三件事

> 原文:[https://dev . to/RS melo 92/heres-3-things-make-my-app-国际学生竞赛第三名-a6n](https://dev.to/rsmelo92/heres-3-things-that-made-my-app-third-place-in-an-international-student-competition-a6n)

我在 IEEEmadC 2017 (移动应用程序开发大赛)上，在世界各地学生开发的 40 多个应用程序中获得了第三名。这是一次奇妙的经历，它让我开阔了思维，以新的方式看待事物，并提高了我作为开发人员的技能。

这里有三点，我相信这三点让我的应用程序从来自全球各地的学生的这么多有趣和非常好的应用程序中被选中，我相信这三点也能让你的应用程序做到这一点。

### [](#1-choose-an-idea-that-is-useful-for-your-own-life-and-for-others-around-you)1。选择一个对你自己的生活和你周围的人有用的想法。

[![LightBuldDrawn](../Images/02efbba9cc5f14044168164fdc847325.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--0ZpBac_W--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/ijm13cblhfvksqugrrro.jpeg)

竞赛的第一阶段是创意阶段。超过 200 个团体提交了惊人的想法，我是其中之一。但是就像大多数人一样，我有很多我认为很棒的想法，我怎么能只选择一个并专注于它呢？

在我的城市，我们有许多美丽的海滩，我们主要是作为巴西的旅游城市而闻名。但是，一年中的大部分时间，我们的海滩都被污染了，因为我们的大多数河流都长期接受污水，而且每条河流都流入海洋，海滩变得很脏。好的一面是海滩在夏天非常适合，但是当下雨的时候，大多数时候在其他季节，每个海滩都不适合下水道的水。

一些人无视这一事实，继续去海滩，冒着因水中大肠杆菌含量高而感染多种疾病的风险。其他人只是避免海滩，失去了我们城市提供的最令人惊叹的东西之一，(我们有非常美丽温暖的海滩)，还有一些人只是不知道这一点，大多数是来自其他国家的游客。这三种人的一个共同点是，他们中的大多数人不知道我们的政府测量我们海滩上的 T2 大肠杆菌水平，并每周在互联网上公布。

当我发现这个想法刚刚出现时:一个应用程序将这些信息返回给公众，使清洁海滩的知识对公众和游客更加开放。

我知道这个想法不仅会帮助我，也会帮助其他人。

当第一阶段结束时，我收到一封电子邮件，告诉我我的想法是 40+中的一个。这是一种令人兴奋的感觉:我即将与来自世界各地的 40 多组人竞争，而我只有一个想法。

### [](#2-choose-your-tools-wisely-and-go-for-what-you-already-know)2。明智地选择你的工具，去做你已经知道的事情。

[![Bullseye](../Images/2ea4b45ddbc01a82ecf05c391b6752ed.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--e2emXk3u--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/kjib8a3gvvq9dbl8ptvb.jpeg)

第二阶段是开发，我的想法必须在三个月内变成一个应用程序，我没有团队，我必须选择如何开发我的应用程序。

我首先想到的是，既然这是一个移动应用程序竞赛，我应该用 Java 为 Android 或 Swift 为 iOS 开发我的应用程序。但是我一个都不知道，我唯一知道的是我在 Java 的一些课程中学到的一些知识，我从来没有真正使用过 swift。我很想学其中一个来参加比赛，但是我的时间真的很短。所以我做出了明智的决定:利用我已经知道的东西。我是一名网络开发实习生，我的主要语言是 javascript，所以我决定使用基于 cordova 的混合平台。而且成功了。在前两个月，我已经用 javascript 从前端和一些后端构建了几乎所有的东西。我能适应这种语言的事实在各个方面都帮助了我。

我知道对于像我们这样的开发人员来说，逃避新事物听起来很可怕，但是我相信这是关于知道发现新事物的正确时间和使用你感到舒适的东西的正确时间。一位老师曾经告诉我，大多数人在某些目标上失败是因为在最关键的时候，他们决定尝试新的东西，而不是他们已经知道的东西。比赛结束时，我很高兴我听到了那些话。

### [](#3-your-app-is-for-humans-not-machines-dont-underestimate-design)3。你的应用是为人类而不是机器设计的:不要低估设计。

[![Chalk](../Images/fcdb7289625b78ea680cedcf43135f7a.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--jrkeswGS--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/lt05rqsteej7g0441b0p.jpeg)

我的应用程序快完成了，我决定问问我的朋友们对它的看法。他们中的大多数人都不喜欢，而我离比赛截止日期只有不到一个月了。

随着我所有的前端和大部分后端的完成，我开始向人们展示我的应用程序，并询问他们对它的看法。令我吃惊的是，他们不喜欢。用他们的话说，它“很丑”，他们不知道如何使用它。我很沮丧:我开发了一个我认为很好的应用程序，但是如果人们不理解如何使用它，也不喜欢它，它就几乎没有任何价值...

[![OldApp](../Images/bcefaf13b752f22424272ebd1edb62c0.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--e5_3pYuR--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/xpejl8eqcluuxtin1l56.png) 
*第一版 app*

所以我决定学习一点设计——我认为这是我一生中不必要和多余的东西——并用我学到的东西来改进我的应用程序，让人们有兴趣使用它。因为我在大学里从来没有上过设计课，所以互联网是我最好的朋友，我开始了我从未学习过的旅程。

设计是它自己的世界。有这么多的领域和理论，你不可能在这么短的时间内学会所有的东西。所以我决定专注两件事，用户体验和色彩理论。经过一些研究和大量的试验和错误，我终于得到了人们开始称赞的东西:它不再难看，每个人都知道如何使用它。

[![FinalApp](../Images/a1bc10eac8902ae6f215580b1b9599df.png)](https://res.cloudinary.com/practicaldev/image/fetch/s--RD0A_uk5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/hgtc6zo87z5svhdrvqb9.png) 
*最终版本*

它让我大吃一惊。

对我来说，这改变了我看待事物的方式。设计真的很重要，应该认真对待，毕竟它是关于用户如何与我们创造的东西互动，用户是每个系统中最重要的部分。没有用户就没有应用。这个比赛永远地改变了我对设计的想法，这也是我参与的每一个项目中我都会带着的东西。

在评审阶段结束时，我又收到了一封邮件，这次告诉我，我的应用程序获得了第三名，即计算机学会特别奖。我真的很高兴，因为我不仅觉得我的努力让我有了一些伟大的成就，还因为我参加比赛时是一个人，比赛结束时我是另一个完全不同的人。

这是我认为让我的应用在 IEEEmadC 2017 上成为第三名的三件事。真正让评委们喜欢我的应用程序的只有他们自己知道，但我相信这三件事产生了影响，我想与人们分享它。这次经历改变了我很多东西，我希望我能分享这种感觉。

我要感谢 IEEE 给我这个机会，让这么多学生聚在一起创造伟大的东西，感谢所有评委在这么多优秀的应用中艰难地选择了三个，感谢我的女朋友和朋友在设计部分给了我这么多帮助。
