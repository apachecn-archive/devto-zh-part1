# 回顾 2014 年

> 原文:[https://dev.to/funkysi1701/looking-back-at-2014-h3g](https://dev.to/funkysi1701/looking-back-at-2014-h3g)

因为今天是除夕，我想这将是一个回顾过去一年的好机会。

[![](img/9fb46abb9a7a262344280a078ec2834b.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--MkSOrat5--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://storageaccountblog9f5d.blob.core.windows.net/blazor/wp-content/uploads/2014/12/tangle.jpg%3Fresize%3D300%252C256%26ssl%3D1)

今年年初，我终于同意安装一条 30MB 的租用线路。这是一个惊人的成就，因为我一直在努力让我们的 ADSL 连接正常工作，而连接的复杂性已经开始引起问题了。

然而，这不是一个顺利的过程，现场调查显示，将需要一笔可观的安装费，然而，委员会即将推出一项代金券计划，将给予我们 3000 英镑的折扣。所以大部分都在等着委员会开始行动。到 4 月份，我们已经安装好了，我对我们选择的 ISP 约克数据服务公司(YDS)非常满意。

今年 4 月，对 Windows XP 和 Office 2003 的支持也将终止。这并没有对我们造成太大的影响，主要是因为我不断地向董事们唠叨，要求更换我们最老的机器。我们现在只有几台运行 XP 的机器，没有运行 Office 2003 的机器。现在大多数机器都运行 Windows 7，我有一个导演运行 windows 8 来做演示，还有一些 surveyor 平板电脑也运行 Windows 8。像许多企业一样，我在等待明年推出 Windows 10，然后再对操作系统进行任何实质性的替换。

五月份，公司决定让我更多地参与开发工作，所以我买了一台新的台式电脑和 Visual Studio。我对我的新机器非常满意(戴尔今天晚些时候会修复一个问题)，到目前为止，我只涉足了开发工作，这主要是因为我需要更多的时间来建立其他人来做我所做的事情。

六月，我取得了另一个巨大的成就。我得到了一个新服务器的协议。这是我们的第一台 Server 2012 服务器，让我们在服务虚拟化方面取得了巨大进步。一旦我们启动并运行了这台服务器，我就能够在明年它停止支持之前淘汰我们的两台 Windows Server 2003 机器。

我是这个新服务器的超级粉丝，我只使用了它的一小部分内存。Hyper V 真的很棒，我希望明年在 HyperV 复制方面取得进展。

同样在 2014 年，我们在云软件方面取得了巨大进展。在过去的几周内，它运行在 Azure 平台上，这不仅使扩展和创建新功能变得更加容易，还将为我们节省成本。随着开发的深入，我期待着学习更多关于 Azure 的知识。

哇，我取得了很多成就，不是吗，除此之外，我还雇佣了一名新的 IT 人员并对他进行了培训(还有更多工作要做)，我还修复了许多打印机、网站、服务器(有一台服务器主板坏了！)，并对我们的内部数据库进行了数百次更改。

我不知道 2015 年会发生什么。我希望有更多的关注在 Visual Studio 中开发，培训我自己和我的部门，并对我们的内部服务器进行最后的一些调整。

我还要感谢在过去的 12 个月里和我一起工作的每一个人(你知道你是谁)，因为你让我刚刚描述的许多事情成为可能。
