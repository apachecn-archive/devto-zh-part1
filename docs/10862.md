# 超级终端或:我如何学会停止担忧，热爱一切的电子化

> 原文:[https://dev . to/seagoj/hyperterm-or-how-I-ve-learn-to-stop-warning-and-love-the-electronic-ization of-everything](https://dev.to/seagoj/hyperterm-or-how-ive-learned-to-stop-worrying-and-love-the-electron-ification-of-everything)

如今，电子应用似乎是孩子们的最爱，但在我尝试了一些最流行的应用之后( [atom](https://atom.io/) 、 [gitkraken](https://www.gitkraken.com/) 等)。)我最终退回到更传统的工具。

直到我尝试了超级终端。我认为这很有可能是因为传统的替代方式 [iTerm2](https://www.iterm2.com/) 从未让我产生像我选择的编辑那样的忠诚。毕竟，终端模拟器只是达到目的的一种手段。只要我能跑 [zsh](http://www.zsh.org/) 、 [neovim](https://neovim.io/) 、 [tmux](https://tmux.github.io/) 不至于让我眼睛出血，谁管得对？

当你开始创建插件时，真正的好处变得显而易见。当修改你日常使用的工具的门槛如此之低，以至于 HTML、CSS 和 JavaScript 可以清除它时，强大的事情就发生了。为你想添加的任何功能编写一个插件，比搜索一个你想做的替代应用程序要快得多。第一天，我为这个平台贡献了几个功能插件，甚至创建了一个插件来移植我非常重要的配色方案。
