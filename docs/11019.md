# 错过下一个伟大编程工具的禅

> 原文:[https://dev . to/Ben/the-Zen-of-missing-out-on-the-next-great-programming-tool](https://dev.to/ben/the-zen-of-missing-out-on-the-next-great-programming-tool)

每当一种热门的新编程语言或框架或方法出现或流行起来，这几乎是每天都会发生的，我看到编程社区中出现了明显的恐惧和偏执。许多程序员担心错过这一浪潮，让他们的技能变得过时。这是编码员作为个人经历的事情，并且会变成一种集体焦虑。没有人想变成编程恐龙。

这种对热门新事物的情绪反应以黑客新闻辩论的形式集体出现，通常令人讨厌，也作为一种对个人技术选择的麻痹而单独出现。在从工具箱中拿出你熟悉的工具或者测试人们赞不绝口的闪亮新品之间，有一种令人沮丧的拉力。

## 旁观的好处

在编程中，不作为的行为可能是一种非常强大的行为。通过等待和成为新技术的后来者，你可以获得很多智慧和未来的生产力。

1.  你让新事物成熟，变得稳定。你将会避免成为每三周改变一次的事物的一部分。
2.  学习资源将变得更加普及。等到一些完整的在线课程已经存在，大量的阅读材料已经建立，可以让你的学习体验更加有效和愉快。你不会浪费时间去寻找稀缺的学习材料。
3.  支持结构将会发展。堆栈溢出问题将变得更加丰富，聊天室将会弹出，有很多人在那里提供帮助。
4.  构建工具和可重用代码的生态系统将会发展。这将有助于你的即时生产力。
5.  你将会发现这个热门的新东西是否会继续流行下去。编程中不受欢迎的工具很难被接受，因为它们失去了支持，周围的生态系统很弱，并且几乎没有支持材料。
6.  耐心学习边缘的点点滴滴，你会学到很多。一旦是时候开始了，你将会有更多的背景和方向。
7.  好的和坏的用例将会出现并被讨论，所以你不必浪费时间用错误的工具构建错误的东西。
8.  受您正在等待的工具的启发，可能会出现更符合您需求的替代方案。
9.  通过使用您已经有经验的工具做更多的工作，您将保持更高的工作效率。
10.  旧的编程语言在所有的播客和讨论板上不再被大量讨论之后，仍然保持相关性，并提供许多就业机会。

## 我的经历

当我第一次接触专业的 web 开发时，我学习了 Ruby on Rails，并且很乐意这样做。我喜欢 Ruby 友好的语法，也和 Rails 相处得很好。与我以前的 Java 经验相比，这是一个很大的个人升级，我的应用程序架构技能也做得很好。但是很快我感到不安，我的 JavaScript 技能需要升级，这样我才能更好地构建和管理我的前端需求。我可以用 JavaScript 写，但是我的前端代码一团糟。我知道我需要学习一个“框架”什么的。我尝试了 Backbone，Angular，并在 React 较新时检查了一下，觉得没有一个真正适合我的想法。我最终所做的是接受这不是我能马上掌握的东西，我继续写混乱的前端代码，承担适合我技能水平的任务。

最终，对我来说，React 显然是正确的选择，当 React 开始真正起飞时，我一路上学到了足够多的东西，并进入了 React 全速状态，我非常感谢我等待了。当我跳进去的时候，环境变得更加成熟和友好。我仍然很早就开始使用 React，但是我很高兴我很晚才开始使用前端框架，并且没有花时间学习 Angular 或者几年前其他最流行的解决方案。这并不是说我不会通过学习 Angular 来丰富我的技能，但对我来说，总是有一个权衡和耐心的回报。

当我第一次进入 React 时，我还是有点早，我觉得在变化中没有明确的架构模式赢家，所以我用自己的风格推出了自己的模式，并工作了一段时间。我知道我最终会需要一些更具伸缩性的东西，并且符合图书馆的发展方向。我等着形成强有力的架构观点，我很感激我这样做了，因为 Redux 作为标准出现了，并且比当时友好得多。

我仍然在大量使用 Ruby on Rails，我对一些新兴的替代品感到兴奋，但我很乐意在担心落后于时代之前花一些时间尝试一下。我的未来可能会涉及很多 Elixir，Go，Clojure，Haskell，Node，或者其他。我要让它发挥出来。

## 最后

耐心是一种美德。我并不是说你不应该过早地接受新事物，而是在感觉合适的时候加入进来，而不是出于对错过的焦虑。对新事物感到兴奋并愿意改变在计算机编程中是至关重要的，但是感觉你需要赶上时代的焦虑是没有帮助的。挑战你自己，但是花时间去做让你快乐和富有成效的事情。
