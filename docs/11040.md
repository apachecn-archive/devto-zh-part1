# 实用开发者调查结果出来了

> 原文:[https://dev . to/Ben/the-practical-developer-survey-results-in](https://dev.to/ben/the-practical-developer-survey-results-are-in)

上周，Practical Developer 发布了一份 58 个问题的调查，围绕程序员的职业和个人生活提出了各种问题。问题包括压力水平、收入、对名人的看法等等。450 名程序员对调查做出了回应，足以收集到一些有趣的结果。这是一个简单的公众调查，而不是一个深入的研究，应该这样对待，但调查结果很有趣，任何人都希望进一步分析或可视化数据。

以下是几种不同格式的结果:

*   谷歌电子表格(可从那里下载)
*   [类型化汇总结果](https://practicaldeveloper.typeform.com/report/RuPhPR/VvTi)
*   [统计数据分析工具](https://www.statwing.com/open/datasets/6ca711a3502787e4493b0e1a9d3bf094b8a3198b#workspaces/124024)

从 Python 程序员对政府缺乏信任到制表符与空格的争论，我们列出了一些我们在调查中发现的有趣的事情。这些发现中的一些可以被认为是绝对有统计学意义的，而一些只是有趣的金块。

当被问及“你的政府有多可信？”从 1 到 10，在小公司工作的程序员最不信任他人，平均信任度为 3.43，而在大公司为 4.71。在更受欢迎的编程语言中，Python 被评为最不值得信任的语言，平均得分仅为 3.27 分(满分 10 分)。总的来说，程序员不信任他们的政府，因为他们的平均得分是 4.22。

女性在工作中受到骚扰的可能性是男性的四倍。大约四分之一(25.2%)的女性在回答“你在编程工作中经历过骚扰吗？”这个问题时回答“是”，而男性的这一比例为 6.2%。这个问题可以有多种解释，并不意味着女性一直受到这样的骚扰，但性别之间的差异非常明显，我们都同意这个数字太高了。

更轻松的是，在制表符和空格之间存在明显的性别偏见。59.6%的男性使用空格来缩进他们的代码，而女性则相反。61%的女性使用 tab。对我来说，制表符和空格之间的统计意义是相当显著的。这看起来很随机，因为对于大多数答案，男人和女人或多或少有非常相似的行为和观点

艾伦·图灵的伟大分布

就“伟大”而言，程序员很少考虑马克·扎克伯格。我不知道会发生什么，但看到阿撒·富兰克林的得分比这两位科技传奇人物都高很有趣。史蒂夫·乔布斯是最两极分化的人物，调查的标准差为 2.56。

值得注意的是，自称“更聪明”的人对乔布斯和扎克伯格的评价更高。这是自我宣称的智力和对这些偶像的评价之间唯一突出的相关性。我很想知道人们还能得出什么样的数据关联。

这些突出显示的结果只是这些数据可以讲述的众多故事中的一小部分。我很有兴趣看看是否有人能发现程序员对即将到来的“科技泡沫”的预期与调查的其他回应之间的相关性。受访者对他们是否认为一个正在接近，但我没有发现其他数据中的任何迹象。顺便说一下，这里有一个链接，是关于“哪个流行语最让你恼火？”这个问题的[答案列表](https://docs.google.com/document/d/1IZ1x37hzM3qowkWOA7Xz55dEp-M4FUnETgEfZN-fF9s/edit?usp=sharing)
