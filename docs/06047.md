# 从 QA 过渡到开发

> 原文:[https://dev . to/donbavand/transitioning-from QA-to-development](https://dev.to/donbavand/transitioning-from-qa-to-development)

## [](#introduction)简介:

去年，我从 [QA 转到了开发](https://danieldonbavand.com/qatodevelopment/)。我做了很多年的 QA，由于很多原因，我觉得我需要改变。

虽然我知道改变需要奉献和努力，但我并没有意识到这有多难。

我在转变过程中犯了一些错误。这篇文章回顾了这些错误，然后看看我是如何纠正它们的。我想强调一下我的缺点，希望能帮助其他想改变角色的人，这样他们就不会犯和我一样的错误。

## [](#my-technical-dev-skills-before-moving-into-development)在进入开发阶段之前，我的技术开发技能。

在进入开发阶段之前，我对自己的技能水平相当有信心。我知道我还有很多东西要学，但我没有再三考虑我是否有能力掌握这些技能。

我有一个 BCA，主修信息系统和电子商务。由于工作繁忙，我没有足够的时间攻读计算机科学学位，所以我决定退而求其次，攻读商业和管理学士学位，在那里我可以通过一些编程和数据库论文来获得一定水平的编码知识。

我在 Xero 大量参与 UI 自动化。我能够设计测试大型应用程序的解决方案。这是我学到大部分编码知识的地方。

我用来提升技能的第一资源是 Pluralsight。对于任何水平的人来说，这都是提高任何编程语言技能的绝佳资源。Pluralsight 提供许多不同编程语言的课程，从初学者到高级，我经常使用 Pluralsight 来提高技能。如果你有兴趣了解更多关于 UI 自动化的知识，我在我的测试博客网站上写了一篇关于开始 UI 测试自动化的博客

虽然我对编码活动有所了解，但主要集中在测试自动化方面。当时我认为这些技能足以轻松过渡到开发角色。

## [](#developer-bootcamp)开发者训练营

搬家时，我参与的第一项培训是由我的团队领导设计和领导的训练营。

就像一个练习训练营一样，这是一个为期 2 个月的紧张集中的训练营，C#语言被分成几个部分，我们每周 4 天，每天花 5 到 6 个小时，围绕那周的 C#部分进行练习、文章、观看视频和讨论。除此之外，我们还会花时间阅读和观看相关视频。

新兵训练营很好地介绍了不同的 C#概念和原则，然而通过这个新兵训练营，我开始意识到我以前的开发技能实际上是多么的少。

## [](#the-struggle)斗争

训练营已经结束，几个月过去了，我开始面对现实。我没有学到成为一名成功的开发人员所需的技能。在我进步的同时，我仍在努力理解一些更基本的概念。

我和我的团队领导讨论过，我是否还想继续发展，或者我更愿意回到 QA 的角色。

我讨厌失败，在这一点上我觉得自己有点失败。Â

我需要后退一步，找出为什么我没有以应有的速度学习。

目标…..或者说缺乏。后退一步，回顾过去我是如何成功的，我意识到我没有为自己设定任何明确的目标。我每天都去上班，希望能有所改善，虽然我有所改善，但并不是以我想要或需要的速度。

## [](#writing-down-your-goals-and-tasks)写下你的目标和任务

我发现并开始使用一个应用程序，它允许我写下我的目标，然后将这些目标分成可执行的任务。Trello 是我们在工作中使用的一个工具，完全符合我的目的。

[![alt](../Images/e3bc2e68c80cdbd77ba26a9de9fa1008.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--6aX8Bs76--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/69k5joudo31vuw2jks7n.png)

我会在我的“待办事项”栏中添加任务，包括我想阅读的开发书籍的章节、附带的项目任务、我想完成的编码练习、我想观看的多视角课程。

当我开始工作并完成每项任务时，我会将它们从左到右移动到“完成”栏

我的 trello board 还包括一个名为 documentation - blog 的专栏，我想开始记录和博客我的开发历程。我会在这个专栏中加入博客文章的想法。

我有一个创意专栏。我发现我会在一天中有这些好想法，这些想法可能会在与同事聊天或阅读 twitter 时出现，如果我在一天结束时等待并写下它们，我经常会努力记住这个想法的特定细节。有一个写下这些的地方，可以让我快速写下这个想法的主要概念和观点，并记住它们。

使用像 Trello 板这样的板还有另一个很大的优势，那就是协作的能力和与导师分享任务和想法的能力。

## [](#finding-a-mentor)寻找导师

当你进入另一个角色，比如发展时，找到并拥有一个导师会非常有帮助。

无论你选择哪种编程语言，它看起来都很庞大和可怕，重要的是你要把它分成易于管理的部分。这对于任何初级开发人员来说都很难独立完成。这就是导师可以帮忙的地方，他们可以和你一起把语言分解成更小的块，他们可以帮助你把重点放在更重要的理解领域上。

在寻找导师时，你需要考虑几个要点

*   你可以信任的人
*   愿意放弃时间帮助你进步的人
*   一个你觉得可以随意提问的人。

你可以信任的人不用说，如果你不能信任这个人，那么他就不应该是你的导师。

你需要确保将成为你导师的人理解时间承诺。做一名导师需要很多时间和努力，如果有人不准备放弃这些时间，那就没用。因此，在一开始就进行这些讨论将会非常有帮助。

我发现有些人比其他人更容易问问题，在寻找导师时，重要的是你有一个你觉得可以问任何问题的人。

## [](#finding-a-side-project)寻找副项目

帮助我提高技能水平的最重要的事情之一是找到一个副业。这个建议来自我的团队领导，虽然我没有立即采纳，但我一采纳就看到我的技能水平迅速提高。

在开始一个主要项目之前，我开始做一些较小的辅助项目，比如创建一个简单的。NET 核心控制台应用程序，与 DynamoDB (AWS)交互与第三方 API 协作和交互。

在做了几个较小的副业项目后，我开始做我的主要副业项目。我目前正在做的兼职项目是一个 Slack 聊天应用程序机器人。我们有一个投注辛迪加，每周有两个人下注，如果赢的钱比他们花的钱多，并且赢了另一个人的结果，那么他们就可以再次下注，否则他们就会走到下注名单的底部。我创建了一个机器人，它会告诉我们，谁在下注，谁在下一次下注，存储并显示一个排行榜和其他一些东西。由于我已经学习了不同的编码技术和设计模式，我能够将这些应用到我的副业项目中，有时很难将这些类型的变化应用到生产中运行的应用程序中，至少对于副业项目来说，如果它失败了，我唯一讨厌的人是我的同事。

## [](#extra-learning-advice)额外学习建议

发展书籍:虽然我通过实践学到了更多，但我仍然阅读发展书籍，以补充我的兼职项目，并给我灵感和想法。我最近读过的几本好书是:

*   代码完成
*   干净的代码
*   C#深度

Pluralsight:对于希望提高任何编程语言技能的任何水平的人来说，这是一个非常棒的资源。Pluralsight 为初学者提供了许多不同编程语言的课程，我经常使用 Pluralsight 来提高技能。Â

编码练习:有很多不同的网站，允许你在线进行编码练习。我目前使用的网站是 Code Wars，给你一个完成一系列失败测试的练习，目标是让所有的测试都通过。我发现解决这些问题有助于训练你的思维，然后解决不同的问题。技能水平从初学者到高级不等，因此每个人都有适合的东西。

## [](#summary)总结:

如果你正在考虑转换角色，那么一个关键的建议是提前做好准备，在实际进行角色转换之前，制定出你可以努力的领域和任务，努力使你的转换在头几个月变得更容易。
