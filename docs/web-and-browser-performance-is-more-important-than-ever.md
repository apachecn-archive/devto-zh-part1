# 网络和浏览器性能比以往任何时候都更加重要

> 原文：<https://dev.to/ben/web-and-browser-performance-is-more-important-than-ever>

我参加了上周在圣克拉拉举行的 [O'Reilly Velocity 大会](http://conferences.oreilly.com/velocity)，这是一个关注 DevOps 和 web 性能的大型会议。会议的 web perf 内容重申了我的信念，即通过浏览器访问的 web 比以往任何时候都更重要，性能是其最重要的特性。随着下几十亿人访问网络，他们将在不平坦的地面上这样做，专注于提供简单的性能将是释放这些新人口并为他们提供他们应得的网络的关键。在美国和其他发达国家，很多关于网络的讨论都是关于它不受约束的本质是如何让它变得破旧和臃肿的。当我们试图建立最好的万维网时，这些主题本质上是联系在一起的。

大会开始的主题演讲，[让糟糕的广告难过。拉德。Opera 首席技术官布鲁斯·劳森(Bruce Lawson)在一次有趣的演讲中谈到了浏览器性能成为如此热门话题的一个重要原因:广告很糟糕，它们正在破坏用户体验。这不是一个全新的话题，但随着用户越来越多地接触广告拦截器，以及原生社交应用越来越多地蚕食互联网，利益相关者终于开始关注这个问题，如果互联网要继续成为一个商业上可行的平台，就需要尽快解决这个问题。网络内容的货币化仍然是一个痛点，绝望已经侵蚀了用户体验。就性能而言，广告的问题本质上主要是技术性的。它们通常由带宽昂贵的渲染阻塞方式提供服务，从而降低了用户体验，尤其是在高延迟环境中。商业利益相关者和性能倡导者需要在这些主题上达成一致。](https://www.oreilly.com/ideas/making-bad-ads-sad-rad)

在特权世界中，如果网络继续成为一个 ad 退化的灾难，那么原生的可能只是蚕食浏览器网络，直到它消失。作为开发者，难道我们不应该滑向冰球要去的地方，忘记网络吗？对你我来说，这似乎并不明显，但随着互联网连接设备方面的技术差距扩大，浏览器的重要性只会越来越大，浏览器是通往互联网的可靠网关。伊利亚·格里戈利克在他的演讲[中讨论了这个话题:构建快速&弹性网络应用。](http://conferences.oreilly.com/velocity/devops-web-performance-ca/public/schedule/detail/50463)全球仍有 60%的人没有接入互联网，即使他们接入了互联网，也不会使用最新的 iPhone 或三星 Galaxy 手机。他们将使用功能不足的客户端和基本的网络浏览器。浏览器仍然是互联网时代的杀手级应用，而渐进式应用就是一切。不太可能会有一系列本机应用程序在所有这些不同的设备上运行。一个可以在所有浏览器上运行的网站，同时逐步利用更先进的 API，是走向世界的最可行策略。

随着越来越多的平台和标准的出现，开发新软件变得越来越复杂。我认为，如果我们要开发可维护的软件，我们将不得不关注格里戈利克和其他人印象深刻的一些基础知识。我们需要降低应用中的表面区域复杂性，这使得我们很难回到以性能为可实现目标的状态。如果我们不能构建稳定的 web 应用程序来逐步满足世界各地不同人群的各种需求，我们将最终构建出庞大复杂的软件，这些软件依赖于对每一点功能的苛刻条件来满足新的 web 采用所带来的越来越多的问题。

在他关于 HTTP/2 的演讲中，Michael Gooding 很好地展示了新协议带来的好处，但也展示了它的缺点和不足。可悲的是，HTTP/2 并没有改善光速或建立新的网络基础设施。新的协议、语言和功能得到了大量的宣传，但它是浏览器网络的基础，将帮助开发人员为每天访问它的 70 亿人提供更好的用户体验。
