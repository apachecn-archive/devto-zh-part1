# 热链接-那是什么？

我们都做过。你在网上看到一张你喜欢的图片，然后把它复制并粘贴到你自己的网站上。现在每次有人访问你的网站，你复制粘贴的图片就会加载进来，让你的网站看起来非常可爱漂亮。但是，如果我告诉你，这种(大部分)无辜的行为会给网站的原所有者带来很多麻烦，在某些情况下，还会给你自己带来很多麻烦，那会怎么样呢？迷茫？请继续阅读。

这种“无害的复制粘贴工作”被称为**热链接**，这里有三个你不应该这么做的理由。

#### [](#1-it-can-cost-the-original-content-owner-a-lot-of-money)**1。这会让原始内容所有者损失很多钱**

每当有人访问您的网站，并且“热链接图像加载”时，该图像使用原始内容所有者的服务器加载。访问你网站的人越多→图片加载的次数越多→你发送到原所有者服务器的流量就越多。对于内容的原始所有者来说，成本可能会增加，最终可能会非常高。(看看这个故事就明白我的意思了，[点击这里！](https://www.theguardian.com/media/2015/oct/28/cartoonist-the-oatmeal-trolls-huffpo-over-images-published-sans-permission))

#### [](#2-the-hotlinked-image-you-use-can-be-replaced-at-anytime-by-the-original-owner)**2。您使用的“热链接图像”可以随时被原所有者替换**

设身处地为内容创作者着想。增加他/她的服务器的流量是在浪费他们的钱。所以为了阻止人们热链接他们的图片，他们使用了一个定制的错误页面。htaccess 文件。所以现在，当你的网站加载而不是你复制粘贴的图像时，会弹出一个错误。或者在其他情况下，用不同的图像替换“热链接的图像”。你能想象一个潜在客户查看你的在线个人作品集，却在上面看到一张不合适的图片吗？不酷。

#### [](#3-lets-be-honest-hotlinking-isnt-very-nice)**3。老实说，热链接不是很好**

从小到大，我妈妈总是告诉我，我希望别人怎样对待我，我就要怎样对待别人。坦率地说，如果有人热链接了你网站上的内容，你会不高兴的。

你基本上是在利用别人的资源为自己谋利。
原始内容所有者已经支付了一定数量的空间来存放他们的文件。所以当你把他们的图片或视频热链接到你自己的网站时，你正在使用他们的服务器 T2 的带宽。你的网站被浏览的次数越多，你从内容的原始所有者那里拿走的资源就越多。这可能会变得很昂贵，特别是对于小企业或博客，如果他们超过了每月支付的带宽和空间的分配量。

所以，下次你想热链接别人的内容时，请三思！
