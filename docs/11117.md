# 你应该使用你的应用程序

> 原文:[https://dev.to/stipegrbic/you-should-use-your-apps-10gf](https://dev.to/stipegrbic/you-should-use-your-apps-10gf)

构建软件会让人精疲力尽。总是试图通过增加功能、修复错误和保持代码质量来改进它。为了改进软件开发过程，软件开发人员在他们的工作流程中加入了许多方法。从原型、线框、软件设计、自动或手动测试开始。尽管如此，有一种方法很少被提及，但它可以极大地帮助你制作更好的应用程序——使用它们。

### [](#how-its-used)它是如何使用的？

澄清一下，我说的是在你空闲时间使用的有意义的应用程序。不是你在日常工作中构建的大型企业软件，甚至可能无法从工作场所之外访问。但是对于你自己的应用程序，那些你在各个方面都拥有的应用程序，这是一件简单而必要的事情。万一这个应用对你来说不那么有用，你应该试着把自己放在用户的位置上。每天都有新的功能或错误修复，浏览应用程序并使用它。从我自己构建软件的经验来看，我可以见证我最引以为豪的应用程序是我自己使用的。原因是让你的应用脱颖而出的细节很难捕捉，尤其是像自动化测试这样的东西。当你使用这个应用程序时，你可以注意到一些你作为开发者或设计师无法注意到的事情。也许这个新特性并不像它在线框中看起来的那样好。也许它拖慢了整个应用程序。

### [](#usability-assurance)可用性保证？

不要误解我的意思，我完全支持自动化测试和专门的 QA，但是对于小的爱好项目，这通常是不可行的。另一方面，使用你的应用程序的成本应该不高。如果它是一个移动应用程序，你可能可以在通勤或喝咖啡时使用它。它的好处有很多，从捕捉自动化测试或测试人员都无法捕捉的错误，到获得如何改进应用程序的新想法。无论是功能、UX、用户界面还是代码质量的改进。
