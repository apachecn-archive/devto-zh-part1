# 选择博客主题

> 原文:[https://dev.to/funkysi1701/choosing-a-blog-theme-2n83](https://dev.to/funkysi1701/choosing-a-blog-theme-2n83)

你可能已经注意到我的博客有了新家。现在可以在它自己的域名 http://www.funkysi1701.com 找到它。这种改变不仅给了我更多的灵活性去做我喜欢的任何改变，而且再次拥有一个网站的感觉也很好。如果你关注了我的博客，请在新的位置继续关注。

我也一直在玩给网站一个新的外观，希望你们都喜欢它。我选择了一个更注重图片的头版。毫无疑问，几个星期后，我会感到无聊，再次改变外观。

我最近一直在阅读关于博客的文章，一个建议是要有一个明确的主题，或者博客是关于什么的想法。我不确定我的博客有明确的主题。

当我开始写博客的时候，比起思考我写的是什么，我对写博客更感兴趣。我做的模糊主题是谈论我做什么，回答这个问题，“你做什么？”

我认为这是一个相当好的主题，但它可能需要一些改进。最好的主题博客是专业的或者关于特定的东西。因为我通常是在学习开发，而不是在做开发，所以我认为这应该是我博客关注的一部分。我的一小部分博客可以继续写我花时间做的其他事情，因为偶尔从博客的主要“内容”中休息一下总是好的。

阅读我的博客的人分为技术人员和非技术人员，所以我想继续用非技术术语解释我所做的事情。这是一个开发人员(或者系统管理员)的基本技能，因为你总是需要解释你在做什么，或者讨论需求。这不是一项简单的技能，就像发展本身一样，我做得越多，我就会做得越好。

因此，请关注这个空间，等待博客的到来。我知道在接下来的几周里我将会和 Azure 一起工作，所以可能会有一些关于这个的博客，可能会有更多的 SQL 博客，或者可能会有一些关于我接下来会学到什么的博客。
