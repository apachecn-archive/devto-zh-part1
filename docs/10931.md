# 我今天让网络变得更好，你也可以。

> 原文:[https://dev . to/pathyatt/I-make-the-web-better-today-you-can-too-17hf](https://dev.to/pathyatt/i-made-the-web-better-today-you-can-too-17hf)

很久以前，大概 5 年前，在从华盛顿贝尔维尤出差回家的路上，我在登机前买了一本书。多亏了该公司(现在换了个名字)关于技术书籍的自由教育政策，它就在他们身上。

我在网上搜索了“开发人员的好书”或类似的关键词后，买了这本书，是这个人写的:

这本书讲述了优秀的设计，它太长了，以至于我想不起它的其他优点(抱歉，史蒂夫！).但在从贝尔维尤回华盛顿的飞机上，我如饥似渴地阅读了这本书。对于一本技术类的书来说，这是迄今为止我读得最快的一本书。这本书的一个部分非常突出，从那以后，我一直很突出，那就是“可访问性和你”(第三版，第二版是“可访问性，层叠样式表和你”)，作者在那里详细介绍了各种实践或属性，以帮助那些有某种残疾的人访问你正在处理的网页。

引用一个五年前的记忆库的话，“可访问性现在不是法律，但它将是”是另一句突出的话，因为这是真的。如果你知道工作的旋风即将来临，你为什么不提前做好准备呢？

在这里吹一下，我今年去了谷歌 I/O，自己掏钱，感谢公司？参加了一个关于无障碍的会议。一些统计数据令人惊讶，使用互联网的人中有五分之一(美国人)有某种残疾，对于那些喜欢统计数据的人来说，这一比例是 20%。我认为，如果你为一家公司工作，你的用户群中会有一部分人有残疾，他们会喜欢使用屏幕阅读器或不斜视来浏览你的象耳食谱网站。

所以，在这个打字的垃圾堆里。帕特，你是如何让互联网变得更好的？

在 Google I/O 上，该公司一直在推动的一个项目是加速移动页面，简称 AMP，这是一种强制执行非常严格的 web 设计准则并添加一些自定义 HTML 元素的技术，底层代码针对交付进行了优化。对于那些使用 Android 手机的人来说，如果你访问你的新闻和天气应用程序，那些旁边有 AMP 的小闪电表示网页正在使用 AMP。事实上，这个博客上的新页面正在使用它，它很快。

我最近的一个帖子是[，我刚刚花了 70 美元买了 pjs](///2016/03/01/i-just-paid-70-for-pjs.html) ，它有一个 Instagram 图像，AMP 需要一个特殊元素用于`<amp-instagram>`。一天晚上，我对可访问性产生了兴趣，试图用 [tota11y](http://khan.github.io/tota11y/) 让这个网站更容易访问。不断出现错误，`img`标签没有`alt`属性。很简单，对吗？只需在`<amp-instagram>`元素上添加一个`alt`属性，然后我们继续。

没有。该死的。AMP 验证开始时不允许使用 alt 标签。(非专业但有文档记录的提示:对于启用 AMP 的页面，在 URL 上添加#development=1，并查看控制台中的错误)

哦该死的。

进入[世纪公关*](https://github.com/ampproject/amphtml/pull/3520) 开源允许你修复东西，并可能在最终项目中得到它，无论项目是大是小。我说过这是谷歌的旗舰产品之一，对吗？比如难以置信的谷歌输入输出？**至少对我来说*

因此，我能够确定一个遗漏的问题，这个问题影响了世界上最大的科技公司之一多达 20%的美国互联网用户。感觉如何？

确实…
