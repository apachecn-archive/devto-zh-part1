# [](#devoxx-belgium-2017-recap)比利时 devo xx 2017-概要

[![](../Images/1f6240a3aa71ce74fbd54bff6d4ab255.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--WZ5amoBK--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/nuu4tc5jjoucaj22himb.JPG)

[Devoxx](https://devoxx.com/) 是欧洲最大的开发者大会之一。它基本上遵循特许经营模式，在英国、美国、法国和其他地方都有版本。

Devoxx 是我最喜欢的开发者大会之一。以前，你必须订购演讲的 DVD，你可能会错过——当时这个会议被称为“Javapolis”。如今，他们实时播放会谈内容，并在 Youtube 上发布所有内容，以方便观看，只需查看他们的[频道](https://www.youtube.com/channel/UCCBVCTuk6uJrN3iFV_3vurg)。

# [](#the-venue)会场

Devoxx 发生在比利时安特卫普的 Kinopolis。讲座在电影院举行，保证顶级座位、顶级音频和顶级视觉质量。由于会议总是座无虚席，房间和会场都挤满了人。真正受欢迎的演讲有时会过多，这意味着有时感兴趣的观众必须坚持录音。

气氛总是有感染力的。黑客、书呆子和开发者的天堂。

# [](#the-conference)发布会

周一和周二是留给实践研讨会和大学会议。实际的会议在周三到周五举行(只有半天)。由于我倾向于跳过研讨会和大学，我将只涵盖会议本身。

对了，3/5 是推荐。

# [](#in-summary)总结

Devoxx。BE 2017 又一次是一场伟大的大会。价格是公平的，内容是非常高的质量。热门主题似乎是无服务器、Kotlin，当然还有 Java 9。我的“学习清单”已经排得满满的了，接下来的几个月我会忙着在 Youtube 上观看错过的演讲。

然而，我想到了两个抱怨。首先，人数惊人。第二，赞助商的数量似乎有点高。Redhat、Pivotal 和 Oracle 人员占据了太多的位置，人们可能会有这样的印象，即社区方面被抛在了后面。请注意，我不想暗示，时间表或任何东西...这只是我个人的印象。我会密切关注发展，因为社区参与一直是 Devoxx 的优势。

如果你想消耗整个 Devoxx。BE 2017 goodness，访问 [Youtube 播放列表](https://www.youtube.com/watch?v=Yacu1yUktjY&list=PLRsbF2sD7JVqZ4RpHYkqSuCNhxumGP5eo)。

除此之外，我只能敦促你尝试购买一张你附近的 Devoxx(或 Voxxed)活动的门票。如果你认为你已经失去了对编程的热爱，来 Devoxx，你会得到治愈:D
