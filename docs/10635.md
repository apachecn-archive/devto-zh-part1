# Kottpd —纯 Kotlin 中的 HTTP 服务器

> 原文:[https://dev.to/gimlet2/kottpd-http-纯服务器-科特林-k7g](https://dev.to/gimlet2/kottpd--http-server-in-pure-kotlin-k7g)

在今天的帖子中，我想谈谈我正在做的一个项目。我把它定义为轻型 **HTTP 框架**并且它是用纯 [**科特林**](http://kotlinlang.org/) 制造的。基于这两个事实，我将其命名为 [**kottpd**](https://github.com/gimlet2/kottpd) 。

这个项目是在长途汽车旅行中诞生的。没有互联网，我决定写一些没有外部依赖的东西。还有什么比实现众所周知的基于文本的协议更好的呢？！

并不是所有的片段都很容易在头脑的宫殿中找到，但是在几个小时内第一个版本已经准备好服务于标准的 HTTP 查询。API 的设计灵感来自于 **Spark-Java** 。所以，看起来挺像的。

我们能在引擎盖下找到什么？首先它是缓存的 ThreadsPool。它有助于为每个 HTTP 请求分配和重用线程。然后，在无限循环中，服务器等待传入的连接，读取它，尝试与配置的路由匹配，如果找到，则传递给相应的处理程序。除了请求处理器之外，你还可以在过滤器之前指定**，在**过滤器之后指定**,以实现某种中间件(例如，登录、事务等)。).**

#### [](#what-for-is-it%C2%A0useful)有什么用途？

嗯，如果你想做一个 web 应用程序，或者一个 REST API，或者仅仅是提供一个静态内容(它也支持这个!),那么它可以被使用。).但是你想用一种轻量级的方式。还有你喜欢的**科特林**！这对你来说是个完美的选择。我提到过它支持 **SSL** 吗？

为了更好地解释它，让我给你看一个在自述文件中很容易找到的例子。但是首先你需要把它作为一个依赖项添加到你的项目中。这个库被发布到 maven central repository 中，所以您应该不会遇到任何问题。
