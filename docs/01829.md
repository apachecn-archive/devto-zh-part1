# 正确的“为什么”很重要

> 原文:[https://dev.to/_patrickgod/the-right-why-matters-a9n](https://dev.to/_patrickgod/the-right-why-matters-a9n)

你认识西蒙·西内克吗？他是一名作家、励志演说家和营销顾问，最出名的可能是他的 TED 演讲“伟大的领导者如何激励行动”。视频的描述说:

> 西蒙·西内克有一个简单而强大的鼓舞人心的领导模式——从一个金色的圆圈和问题“为什么？“他的例子包括苹果公司、马丁·路德·金和莱特兄弟……”

“为什么”最能驱动我们，或者至少**应该最能驱动我们**。不是“如何”或“什么”，而是“为什么”。

好吧，你可能会问，我到底在说什么？西蒙·西内克正在谈论一个公司的领导能力。他希望领导们思考他们为什么要领导公司，为什么他们的公司要做现在正在做的事情。

大多数公司领导——以及员工——都从“什么”开始。他们知道自己在做什么或者想做什么。之后，他们会想出如何去做。大多数时候，它就停在那里。很少有人走得更远，真正地问自己为什么要做这些事情。当然，你可能会说你得起来挣钱，吃点东西。但在这种情况下，我说的不是*而是*“为什么”。这是关于你日常活动的**意义**。

在我看来，和软件开发一模一样。每当我知道我为什么要创建这个特别的功能，以及这个大图有什么意义时，我就没有任何动机问题。我更乐意去实现它。无论你是在用你的软件帮助人们，还是在创造可能成为未来一部分的东西，或者你是在给人们一个他们觉得有趣并让他们开心的软件。或者你正在编写一个小工具，帮助同事更快地完成工作。作为一名软件开发人员，有许多事情会给你的工作赋予意义。发现这一点的感觉真好。

例如，制作游戏给我的感觉是，我会提供一些让其他人玩得开心的东西。对于我参与的电子游戏，游戏玩家可能会有一种成就感和成就感。他们会感觉更好。

在卫生部门工作，我认为我可以对人们的生活产生真正的影响，我可以帮助他们保持健康。

当我在银行工作时，我找不到任何意义，因此，我很快就辞职了。

诚然，找到一份有“正确原因”的工作并不容易。也许你现在的情况是你无法在几个选项中做出选择。也许你得先**找到你的“为什么”**。但是如果你这样做了，它将改变你的工作方式，改变你思考下一个你必须实现的特性或者下一个你必须修复的 bug 的方式。

顺便说一句，这不一定是你的工作。也可以是副业。我猜你的副业不会有这么大的问题去寻找意义。这是你选择的东西。也许这是你长久以来想学的东西？或者，也许你梦想有一款你想在业余时间开发的应用或游戏。

不管怎样，现在是开始思考为什么的好时机。它可能会改变你写下一行代码的方式。
