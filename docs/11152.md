# 我在 Yandex 界面开发学校的冒险经历

> 原文：<https://dev.to/dimaip/my-adventures-at-yandex-interface-development-school-3fm0>

距离我上次写博客已经有一段时间了，有一个很好理由:过去三个月我陷入了一场疯狂的冒险。只是为了好玩，我试着报名参加了 Yandex 界面开发学校。对于那些不知道的人来说，Yandex 是俄罗斯搜索引擎巨头，欧洲最大的 IT 公司，换句话说，这些家伙在前端开发方面有相当多的经验。每年他们都会组织一次免费的强化前端编码课程，具有很强的团队合作和实践性。他们举办这样的编码学校的动机是通过培训更好的开发者来提高俄罗斯互联网生态系统的质量，这确实是一个非常崇高的目标。

我以前听说过这所学校，但并没有认真想过我能通过比赛。一天晚上，我粗略地勾画了三个测试任务的解决方案([修复一些破损的 JS 代码](https://gist.github.com/dimaip/28e478f7c2783405d405)、[用 Web Audio API 编写音频播放器](https://gist.github.com/dimaip/e8acf0b1b87c3083bbe9)和制作一个 [CSS 飞行表](https://gist.github.com/dimaip/c1c4bbb99532fd6c1bce))，一个月后，令我完全惊讶的是，我得到了一个积极的回应。我们总共有 36 个人成功了。

学习过程由两部分组成:由 Yandex 顶级开发者每周 2-3 次的讲座和 3 次黑客马拉松的实际团队工作。第一个讲座是关于 CSS 的。如果我们认为 10 个中有 8 个知道 CSS，请举手。我们大多数人都举起了手，然后我们被告知，事实上我们对 CSS 的了解还远远不够。我们被安排在家观看许多关于 CSS 的视频讲座，后来又得到了更多关于浏览器内部如何工作的讲座和其他深入的东西。讲座非常及时，非常及时:我们有关于 CSS、SVG、WebGL、浏览器、测试、React、Redux(！)，nodejs，git，shell 等等。

但是最大的乐趣是团队合作。我们分成 6 个小组，每组 6 人，每个小组必须创建一个聊天应用程序。我们的团队选择了我们能找到的最时髦的堆栈(React、Redux、webpack、node、express、mongo ),我们开始摇滚。我们用英语记录了我们所有的项目，所以如果你需要一个用 Redux [编写的全功能聊天应用程序的例子，请随意看一看](https://github.com/shri-2015-org/shrimp)。

我很幸运地加入了一个非常熟练的团队，并在项目期间学到了很多新东西。困难的部分是以黑客马拉松的速度建立适当的工作流程和分支策略，但我认为我们做得很好。工作流和部署生命周期可能是将小型自由职业者的工作与 Yandex 中正在进行的工作区分开来的唯一事物:CI 和 CD 在它们的规模上看起来非常可怕和复杂。每个团队可以从 Yandex 开发人员中挑选两名导师。我们选择了[安德烈](https://twitter.com/Andre_487)和[丹尼斯](https://github.com/L0stSoul)，两个在 Yandex 做搜索结果页面开发的熟练开发者。伙计们在我们身上投入了大量的精力，并不是真的干预我们的工作，而是保护我们免于重大错误。

参加这个编码学校绝对给了我很多超出我预期的东西。我从与 Redux 的合作中得到了灵感，我已经用它重写了我的一个工作项目——立竿见影。我也希望现在能更好地为[近地天体 CMS](https://neos.io) 项目做准备。但是最有价值的收获是我遇到的 36 个了不起的男孩和女孩。坦白地说，我以前从未见过来自俄罗斯的熟练开发人员(我主要和来自欧洲的开发人员一起工作)，而在这里我一下子见到了这么多！这让我重新燃起了在俄罗斯看到一个强大的互联网前端社区的希望，我们离那一天不远了。

毕业晚会后的深夜，有人带我们参观了 Yandex 总部。他们仅在莫斯科就提供了 4000 个工作场所，所以逛遍所有楼层需要相当长的时间。这些曲艺让开发者有宾至如归的感觉，事实上，许多人工作到深夜，甚至在节假日去工作……如果不是我的心已经被我目前在圣菲拉雷东正教基督教学院的工作占据，在这样的地方工作对我来说是很有吸引力的。我们在这里对工作的态度非常相似，也许这就是为什么我现在在我的工作场所过国庆假期:)
