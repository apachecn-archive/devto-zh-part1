# 人们都是哑巴(像我一样)...一个人聪明(像我一样)。

> 原文:[https://dev . to/Brian nipper/people-are-dumb-like-me-a-person-is-smart-like-me-3nk](https://dev.to/briannipper/people-are-dumb-like-me-a-person-is-smart-like-me-3cnk)

[![](../Images/16570494993b197c29c006553be21861.png)](http://upload.wikimedia.org/wikipedia/en/f/fb/Men_in_Black_Poster.jpg)1997 年夏季大片《黑衣人》中的两个主要角色，由威尔·史密斯和汤米·李·琼斯扮演，讨论了群体和个人的能力。

威尔·史密斯的角色暗示，告诉整个地球关于外星人的事情是安全的，他说“人们很聪明……”他们能处理好。托米·李的性格回应“一个人是聪明的，人们是愚蠢的……”这意味着，虽然一个人有能力，但作为一个整体的人没有。

总的来说，我倾向于不同意大多数人是哑巴、愚蠢或其他一些负面品质的想法。我坚信，绝大多数人都非常有能力，只要给他们合适的环境，他们就能给人惊喜，让他们高兴，而且通常会超出大多数人的预期。然而，记住“吱吱作响的轮子得到了油”，或者换句话说，最大声的人得到了关注，似乎出于某种原因，最大声的人也往往是“不太”能干的人。

我在日常工作中注意到，在软件开发方面，抱怨/列出问题最多的用户通常最没有能力诊断实际问题，而不是“用户错误”(也就是“这是他们自己的错”)。

在我看来，如果我能简单地让所有用户学会我在解决问题时总是求助的一个原则，那么解决这个问题就很简单了。“这是我的错”(意思是这个问题可能是我自己造成的，而不是其他人/事)。

通常当我解决软件中的问题时，我首先会假设这是我做的事情。不幸的是，取决于你如何看待它，我通常是正确的。我可能会忽略一个条目，产生一个一个的错误，或者其他一些小错误。因为这似乎是开发者经常遇到的情况，他们设计了一个出色的过程来解决这些问题，这被称为测试。这个惊人的、革命性的和突破性的过程有助于消除大多数错误/问题。因此，当(已测试系统的)用户报告错误时，大多数故障排除都是从尝试重现所报告的问题开始的，当这无法实现时，这可能表明报告错误/问题的用户做了一些与预期“不同”的事情。虽然这可能被认为是一种边缘情况(另一篇文章的主题)，但最常见的是基本的用户错误。

如果能教会所有用户如何首先想到“这是我的错”，然后，人们会变得愚蠢。
