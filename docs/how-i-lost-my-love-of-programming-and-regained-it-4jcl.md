# 我是如何失去对编程的热爱又重新获得它的

> 原文：<https://dev.to/andrioid/how-i-lost-my-love-of-programming-and-regained-it-4jcl>

*再版。原写于 2013 年*

当我在冰岛还是个孩子的时候，我总是对所有技术性的东西着迷。无论是我祖母家的电灯开关，还是通常所说的电视机。我一直喜欢这东西，现在也是。

当我八岁的时候，我发现了令人愉快的计算机世界，回想起来，我的许多朋友都是精心挑选的，因为他们有有趣的玩具可以玩。我把所有的时间都花在玩和“摆弄”电脑上了。它从电脑游戏开始，但最终我用完了我们 Amstrad CPC 128k 的游戏，不得不求助于阅读电脑手册，在后面重新键入一些编程示例。我仍然记得我在学校的那个灵光一现的时刻，在我们的英语课上，我明白了“如果”的真正含义。我用 BASIC 语言为我们的第一台家用电脑创建了各种简单的带有像素图形的菜单程序，根本不知道我实际上是在编程。

几年过去了，我想我下一次做我认为是编程的事情是在我十几岁的时候，那时我发现了 IRC 并开始为 mIRC 编写脚本。在我最自豪的创作中，有一个小脚本，我安装在我堂兄弟的电脑上，用可爱的小词如“花”、“亲爱的”和“爱”代替了他所有的咒骂。当他发现我的小程序时，我收到了一条信息，大意是“你对我的宝贝电脑做了什么？!"。

不用说，IRC 主宰了我当时的生活，作为一个在整个高中期间被无情欺负的人，躲在电脑屏幕后面更容易。我从 mIRC 去为 Eggdrop 编写脚本，这是一个全天候坐在 IRC 频道上的机器人。我们当时有一个相当大的频道(IRCnet 上的冰岛)，里面都是不成熟、愤怒和好色的青少年，有时噪音太大，无法手动处理。这促使我创建了“nosense.tcl”，这是一个基于分数的系统，如果你大喊大叫、重复自己和其他令人讨厌的事情，它会给你扣分。随着时间的推移，机器人会“原谅”——所以它并没有真正影响正常的聊天，但破坏了那些可怜的青少年。今天，我仍然会想到一些系统，比如 MMO 运动会，它们可能会从这种噪音控制中受益。

大约在同一时间，我被介绍给 Pascal 和 Delphi，但两者都没有真正引起我的兴趣。练习很简单，我只是从来没有足够的热情用它创造任何东西。在学习了 HTML 之后，学校里的一些人使用 PHP(当时是 v3)在一个当时非常静态的世界里创建了许多动态的东西。大约在 PHP 达到 4.0 的时候(我记得是 1999 年左右)，我被一群朋友雇佣在一家初创公司做 web 开发人员。

这是第一次有人真的愿意付钱让我玩电脑，我很兴奋。然而，我当时的老板并不满意我使用 PHP 作为一种自顶向下的语言，因为我在 v3 中已经学会了，他想用类替换我的函数文件。由于极其固执和骄傲，我只是忽略了他，直到他们最终以没有足够的工作给我为由解雇了我。

在做了一年的网络工程师后，我又做了一份 web 开发人员的工作，并在短时间内为一家小公司创建了 3 个定制解决方案。不幸的是，他们是由一个反社会者经营的，他经营企业的理念是雇佣员工，从不付钱给他们，直到人们放弃并离开为止。人们的薪水总是“在邮寄中”或“在路上”。

当时，我作为一名自由职业者为电影院创建了一个小型 CMS，我们的互动以他向一名客户出售我的系统而告终，并且从未向我付款。由于失业，我请不起律师，所以我决定在 2000 年到 2006 年间专注于此，满足我的其他兴趣；计算机系统和计算机网络，我唯一的编程工作是偶尔用老式的函数式 PHP 编写一个外壳脚本或个人主页。

很久以后，我意识到有人窃取了我的编程作品，这让我很受伤，我得出结论，专业编程不适合我，我只会把它作为一种爱好。

2006 年，我做了一件疯狂的事。我辞去了系统管理员的工作，申请了丹麦的大学预科班。这有助于我认识这里的人，他们是我的第二个家庭，没有他们我不可能成功。奥尔堡大学有一个基于小组的学习方法，所以你的大部分成绩是由一个小组项目决定的，每个学期。在我从事 IT 工作的几年里，我曾多次尝试学习 C 语言，但我从未能在几个小时后保持专注。这些小组项目基本上帮助了我学习 C、C++和 C#。不是因为他们教我们这些语言，而是因为我们的团队已经决定了一个软件项目，并被激励去创造一个伟大的项目。住在宿舍，没有白天的工作，也给了我学习新技术的自由时间。

本科毕业前，我在冰岛一家软件/咨询公司实习。他们给了我一个机会，给了我一份带薪实习，尽管我以前没有做过职业程序员。在这次实习之前，我总是在学习后做什么的问题上在 IT 和软件之间徘徊。他们向我介绍了学习和敏捷的做事方式，虽然这些话变得越来越没有意义，但他们没有向我推行程序或工作流程。相反(不涉及太多的细节)，工作主要是由更多的实践和交流组成，而不是需求、设计和文档。

我每周都会和客户见面，如果在任何时候，客户没有得到他们期望的东西，我们就会交谈。在这次实习中，我感觉到了某种觉醒。我对编程的热情又回来了。而不是像以前那样，作为达到目的的手段。尽可能做出最好的软件的野心是存在的。与“我更喜欢我的方式”相反，设计模式突然变得有趣了，我明白了做软件测试的意义。

在我早年的时候，软件测试只是为犯错误的人准备的。如果我的代码有效，为什么要测试？对吗？对我来说，这个启示是，测试让我有信心改变程序中的基本东西，而不是害怕改变一个大的程序并最终重写整个程序。今天，我刚刚开始了一份 Java 程序员的工作，我希望通过在那里创建一些令人敬畏的解决方案来实现我的抱负。虽然我的关注点改变了很多，但我仍然为了乐趣而编程。我现在也在做，但是规模小多了。我想以感谢几个人来结束我在软件开发方面的自我发展。

*   谢谢你在丹麦照顾我。没有你们，这是不可能的。
*   菲尔曼网的Á·尼:谢谢你解雇了我。很长一段时间我对此很生气，但后来意识到我当时的心不在编程上，我只会继续做最少的工作，永远也学不到任何新东西。
*   Stockvault 的 Bjö ggi:在我学习期间鼓励我做一点自由职业者的工作，即使在我坚持说我不是程序员之后。它帮助我认识到，不仅仅是别人的代码更好——它经常更差。这帮助我为自己设定了一个更现实的期望。
*   为了实习，友谊和灵感。

需要澄清的是，我已经有一段时间没有写博客了，我只是想把它写出来。我非常满意没有人读这本书。
