# 软件开发人员的形

> 原文:[https://dev.to/gimlet2/kata-for-software-developers-1fle](https://dev.to/gimlet2/kata-for-software-developers-1fle)

今天我想谈谈像形这样的东西。最初，形是武术中的一种练习，必须一遍又一遍地重复，以使你的动作干净、快速且几乎是自动的。

因此，同样的方法也适用于软件开发。尤其是在商业开发领域。

怎么做形？首先，你需要选择一个与你想练习的主题相关的话题。比如—测试驱动开发( [**、TDD**](https://en.wikipedia.org/wiki/Test-driven_development) )。对于这个话题，最好的选择是 [**银行武士形**](https://github.com/sandromancuso/Bank-kata) 。即使 TDD 没有被列为这个形的要求，它也是一个很好的额外挑战。也试试 [**赛车形**](https://github.com/emilybache/Racing-Car-Katas) 。这将给你机会实践 [**扎实的**](https://en.wikipedia.org/wiki/SOLID_(object-oriented_design)) 原则。

我是一个编程形世界的初学者，到目前为止，我只训练这两个。但是即使有了这个小小的实践，我也能在我的日常生产代码中看到不同。

只需去 **github** ，结账，看 README，试着实现。然后，当你认为你完成了，一定要恢复到原来的提交，并再次这样做。然后再来一次。每次你的解决方案都会有一点点不同(但可能不只是一点点)。然后试着在一两个月后恢复同样的形。再做一次。将结果与您之前的尝试进行比较。

这是一个两人一组做形的好主意。这不仅仅是知识的交流，也将帮助你更严格地遵循形的要求。此外，它还可以让你练习如何解释你的想法，并就你在开发过程中面临的问题进行辩论。

做形——训练你的技能！
