# 我在谷歌学到的五件事

> 原文：<https://dev.to/jkirchartz/five-things-i-learned-working-at-google-7gm>

2015 年，我在谷歌工作，学到了很多东西，不久后我写了这篇文章；现在我想它终于可以出版了。

# 验尸

失败需要被调查，这样你可以从中吸取教训，避免将来犯同样的错误。任何时候你创建一个程序，最终都会出现人为错误，也许是一系列人为错误。在代码评审和 QA 批准了开发人员的工作后，因为一个 bug 解雇开发人员不会解决任何问题。如果工作流程中有一个错误导致人们互相妨碍或给团队带来压力，那么就需要讨论这个问题，以避免将来重复同样的问题。如果有人犯了一个他们可能都不知道的错误，那么预测软件在现实世界中将要面临的一切是不可能的。从你的错误中吸取教训，不要指手画脚找替罪羊，耸耸肩——解决它。

# 会议

会议需要行程，开始/结束时间，人们需要坚持这些时间。在共享文档中创建会议议程，允许每个人在会议期间提出他们要讨论的问题的简介/链接。会议可以超过 5 分钟，但不应该超过 5 分钟，如果需要更多的讨论，可以安排额外的会议。如果有什么需要进一步讨论的，这个议程甚至可以在会议期间随时编辑。这有助于避免糟糕的会议形式，如“随机故事时间”或“一页页无意义的数字”。议程可以防止会议浪费时间，如果你想浪费时间，可以安排一个“与一个小组同步会议，然后享受”。

# 谷歌不是它的神话

谷歌和这个星球上的其他公司没什么两样，只是人更多，钱也更多。他们制造了一堆整洁的工具(首先是为他们自己——吃他们自己的狗粮),但没有任何超级特别的东西来防止与客户、沟通、工作流程或人员安排有关的问题。经理和团队有机会讨论和修改事情的运作方式，但每个人都有自己的风格，他们被允许使用它。在我的面试中没有恶作剧问题，他们让我做 FizzBuzz 和一些正则表达式。

# 种姓制度

我以前读过一些关于大 G 公司员工等级制度的东西，但我亲眼看到全职员工、实习生，还有 TVC(临时工、供应商、承包商)——每个人都有不同颜色的徽章。实习生需要解决一个特定的问题。全职员工可以喝啤酒、骑滑板车、邀请客人、参加场外活动，甚至可以得到圣诞礼物。出于法律原因，TVC 没有得到这些。我是一名承包商，不是谷歌员工，作为他们入职培训的一部分，他们告诉我，我“为谷歌工作，为谷歌的利益工作，但非常明确地说，不是”为谷歌工作。在整个办公室被邀请参加另一个活动的每一次活动中，TVC 都会问我们是否被允许参加，我们从未被驱逐。

# 这只是生意

作为一名承包商，我非常清楚一年合同何时到期，但我不知道它是否会续签。我错了，因为我的一年合同只计划持续 11 个月，在圣诞节前一个多月结束。我看到承包商离开，一些回来；所以我不确定会发生什么。在我工作的最后两周，我听说明年的资金还没有确定，所以我的合同不会续签。上周，在我们的例会上，我收到了对我工作的掌声和感谢。在我的合同结束后不到两个星期，另一家人事公司联系我申请同样的职位，这一次我的收入比以前的一半多一点——毕竟，这只是生意。
