# 为什么我更喜欢科特林

> 原文：<https://dev.to/grahamcox82/why-i-prefer-kotlin>

我从事企业 Java 开发已经有 10 年多了，使用 Java 也有 16 年多了——那时 Java 1.3 是最新最棒的。这是一段漫长的旅程。那时没有 Lambdas，没有流，没有泛型，没有注释。甚至没有一个`assert`关键字。

如今，Java 在一个非常庞大和强大的生态系统中占有巨大的份额。其中最强大的部分之一是作为同一系统的一部分，内置了对替代语言的支持。JVM 使许多不同的语言成为可能——意味着您输入到编辑器中的实际代码——编译成兼容的字节码，并在同一个应用程序中一起运行。这对软件开发来说是一个巨大的好处，因为你现在可以用最适合的语言编写应用程序的不同部分。

那么 Kotlin 到底是什么，我为什么这么喜欢它？与许多其他 JVM 语言不同，Kotlin 并没有试图变得与众不同。它只是作为一种大大简化的流线型 Java 语言而存在。如果说有什么不同的话，那就是 Java 能做的 Kotlin 做不到的事情很少。不同之处在于，Kotlin 用少得多的代码做了同样的事情，使得阅读和维护变得容易得多。Kotlin 还非常重视 Java 互操作，这意味着将它与现有的 Java 库结合使用是很容易的。如果有可能的话，其他一些 JVM 语言会使这变得更加困难。

所以，总的来说，到目前为止我的经验是，Kotlin 允许我做 Java 做的所有事情，但更简单。

唯一让我纠结的是一些工具——但这真的没有让我那么困扰。这里，我说的是代码覆盖(JVM 字节码包括许多 Kotlin 代码中不存在的代码行，所以覆盖它们不太容易)、FindBugs、PMD、Checkstyle 等等。我还没有探索 Kotlin 的声纳，但如果它工作的话，很可能会填补许多空白。
