# iOS 项目结构

> 原文：<https://dev.to/luisramos1337/ios-project-structure-1obm>

为了更好地组织我的代码，这些年来我一直在改变我的项目结构。我想我终于有了一个，所以是时候为将来记录它了！

创建项目结构时有两个基本原则。首先，保持简单。第二，用对自己有意义的东西。

如果你用谷歌搜索一下这个话题，你可能会找到这个 quora [答案](https://www.quora.com/How-should-I-structure-my-iOS-app)或者这些[指南](https://github.com/futurice/ios-good-practices#project-structure)。他们都提出了类似这样的建议:

*   视图控制器
*   视图
*   查看模型
*   UI(又名故事板)

如果你有小项目，这种结构会工作得很好。但是，当你开始添加更多的文件，它变得混乱。我更愿意用功能性的方式来组织文件，通过应用程序的区域和层次来分离它们。所以它的结局更像这样:

连接

*   基础
*   登机时
*   饲料
*   用户
*   设置

    ## —本地存储

    *   建立工作关系网

班级

*   类别/扩展
*   不适合其他地方的课程
    *   资源

在这些文件夹中，我通常遵循第一个设置，通过视图、视图控制器等分离文件。通过这种方式，你可以在一个文件夹中保存几乎所有与应用程序相关的文件。我所有的图像都存储在 XCAssets 中。字体和其他原始文件在参考资料中(通常每个都有一个专用的文件夹)。

如果你有更棒的设置，或者如果你只是想分享你的设置，请随时给我发推文。
