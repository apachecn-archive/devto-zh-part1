# 程序员:保持敏捷。

> 原文：<https://dev.to/thillerson/programmers-stay-nimble>

专攻还是通才是一种选择，或者说是一种平衡，所有程序员在职业生涯的某个阶段都会考虑这一点。然而，技术的无情发展似乎将淘汰你今天熟悉的任何语言或平台，所以问题不是是否选择一种语言或平台并坚持下去，而是何时开始学习一种新的语言或平台。这不一定是抛弃你选择的语言、平台或框架的问题。更好的思考方式是学会变得敏捷。探索和使用其他技术。在某个时候，你将不再是一名 X 程序员，这里 X 是一种语言、平台或框架，你将成为一名程序员。

从某种程度上来说，这就是实用程序员的七个系列中的七个系列背后的想法:对技术进行分组和分类，在短时间内集中研究它们，然后你不仅会对某项技术有所了解，还会对多组技术有所了解。你还可以将你所学到的知识应用到你更熟悉的技术中。也许通过学习从不同的角度处理问题，或者通过理解你以前没有想过要深入的东西。

这就是为什么我想在七周内写七个手机应用程序的主要原因。对于探索和学习新语言或数据库有效的方法，我相信对于移动框架和平台也是有效的。

我觉得移动和 web 开发社区倾向于虔诚地坚持特定的平台或框架。总的来说，编程社区一直喜欢“圣战”:Windows vs Mac vs Linux，Vim vs Emacs，各种不正确的缩进样式 vs“一个真正的大括号样式”，诸如此类的事情。这是我们文化的一部分，也很有趣。然而，你应该记住这些由来已久的冲突的价值，不要让任何这样的偏见或简单的安慰阻碍学习和保持敏捷。

更具体地说，移动领域现在竞争激烈，精通多个平台是一件非常有价值的事情。作为移动开发人员，您需要解决的大多数问题都是相同的。从 API 加载数据，将其转换为本地数据结构，并显示在屏幕上，这是移动开发人员的主要工作。在多个平台上学习这样做并不困难，对于移动网络来说也是理想的。

无论你选择怎样做，作为一名程序员，要不断成长。当然，在你选择的技术中，但是也通过探索，玩和比较。根据我的经验，无论是否有人付钱给他们，大多数程序员都会编程。在这种情况下，不断地在你的舒适区之外施压不仅能让你保持高就业机会，还能让你保持参与和充实。作为任何技术的开发人员，要不断学习，保持相关性，保持灵活性。
