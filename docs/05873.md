# 编程背后的障碍

> 原文:[https://dev.to/rapidnerd/the-barriers-behind-programming](https://dev.to/rapidnerd/the-barriers-behind-programming)

# 我到底在说什么？

编程对我和很多其他人来说，已经从一个小小的爱好变成了职业。但是就像所有的工作一样，会有一些阻碍你履行职责的障碍。我不认为这有一个合适的技术术语，但我喜欢称之为编码块(像作家块)。

# 亲身经历

目前我的主要语言是 Java，在此之前我有 C++的背景。当我学习 Java 的时候，我经常发现我不知道我在做什么，我经常嫉妒我的朋友，因为他们用 Java 做了这些很酷的东西，而我坐在这里像个土豆一样想我到底在做什么？我永远也到不了我想去的地方。

虽然发生了这种事，但我没有因此而沮丧。我个人认为编程不仅仅是一份工作，而是一种学习经历，每种语言中都有可以学习的东西，一种新的做事方式，一个新的 API/库来帮助你完成日常任务。我一直把这记在心里，为了到达我想去的地方，我不能放弃自己，必须继续前进。

一直推动我前进的是我的朋友们。我目前在一个大型网络中做志愿者，我有员工经理，和一些员工一起工作，使这个网络成为现在这个样子。其中两个人(由于隐私，我不会提及他们的名字)给了我一直在寻找的动力，让我走到了现在。

# 有哪些常见的障碍，我是如何度过的。

就像上面提到的，我看到这类似于作家群，一个作家正在写一本书，他们只是因为某种原因而停滞不前。

*动机*

动机是我们在生活中完成任何工作或任务所需要的。没有它，我们基本上完蛋了。就我个人而言，我很长一段时间都在努力寻找编写任何程序的动力。以上是我找到动力，走到现在的地方。但是如果没有动力，你通常会想这他妈的是什么？我为什么要这么做？重点是什么？哦该死的。

克服了这一点，我发现提醒自己不断提醒自己为什么我成为一名程序员摆在首位的原因。我的爱好就是在 C++中闲逛，看看我能做多少愚蠢的事情。但是有一天我突然想到我可以以此为职业。我有三个兄弟，他们都和技术有些关系，其中一个是一家大公司的程序员。他热爱他的工作，我只是把两者结合起来，事实上我热爱编程，我可以把它作为一份工作来做。

但也许你这么做另有原因。我认识的许多人这样做是因为好玩，其他人这样做是因为这是他们的天性。提醒你自己为什么你现在在这个地方，你为学习到你现在的位置所付出的所有努力都会有所帮助

*嫉妒地*

现在我要坦率地承认这一点，并说有些我的朋友或同事让我嫉妒。为什么？因为我看到这些人创造了一些非常酷的东西，这让我觉得我永远不会像他们一样优秀。但是我错了，哦，我错了。

为了克服这一点，我花了无数的时间研究他们做过的事情和我脑海中的想法，教自己如何去做。我知道学习编程需要时间，但我从来不知道像 Java 这样的语言有无限的可能性。我脑海中的一些想法我不确定它们是否可行，因为我没有做研究。通过这项研究，它教会了我很多东西，教会了我更有效的编写代码的方式和更有效的方法。

另一件要记住的事情是，总会有人比你有更多的技能，或做出更棒的东西，或有更好的知识基础。但是这就让你成为一个糟糕的程序员了吗？不。让一个人成为伟大的程序员的是那些乐于奉献、渴望学习和热爱做他们所做的事情的人。最初，我害怕在代码方面寻求帮助，因为我认为别人会根据我做事的方式来评判我。但是他们问的人(好吧，他们笑了一点点)最终给我的建议比我想象的更有帮助。

*bug*

毫无疑问，对于任何形式的代码，bug 都是其中之一，或者可能是最令人讨厌的。他们比一些托管公司更让人头疼。有时我会被一个 bug 困扰几个小时，这让我坐在这里想“该死的，你为什么不工作？！？！？！？!"。

通常，当有一个我无法解决的问题时，我会离开电脑，做些别的事情来让我的大脑休息一下。有时伴着音乐在树林中散步，我发现这真的很放松。当我回来再次查看代码时，我发现它通常会有拼写错误，使用花括号而不是分号，或者代码通常无法正常工作，因为它做错了
