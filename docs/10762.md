# 在求职过程中要求透明度

> 原文:[https://dev . to/Jodi onis/demand-transparency-in-your-job-search](https://dev.to/jodionis/demanding-transparency-during-your-job-search)

几年前，我在一家大型全球对冲基金担任内部招聘人员。对于一家金融公司来说，我们的招聘策略相当前卫。我们是根据原始能力而不是一系列的经验来雇佣员工的；我们鼓励艺术家在为我们工作的同时追求他们的创作生涯；我们对每一位候选人都进行了严格的多阶段评估，涉及一个由内部招聘人员和高管级招聘经理组成的庞大团队。我们甚至有自己专有的 ATS，由海德拉巴的一个专门团队制造。当一位重要的候选人——即使是资历较浅的候选人——得到聘用通知时，董事总经理会打电话向他们表示祝贺。

然而，有一件事我们没有，那就是让申请人深入了解他们申请的工作和团队。当时，对技术人才的需求很大，但远没有达到今天的狂热程度。最近毕业的学生成群结队地向我们申请，他们被高薪、反华尔街的氛围(没有着装要求、等级制度扁平)以及我们拥有强大的定量分析师骨干的声誉所吸引。我们没有为候选人创建关于他们申请加入的团队的资源，因为我们没有必要这样做。

事情已经变了。我现在为一家科技初创公司工作，专注于帮助开发人员培养他们的职业生涯和技能。开发人员已经成为招聘的圣杯。人才库不能满足当前需求的事实表明，该系统进行变革的时机已经成熟。

这并不是说开发人员可以走进会议室，得到一打理想的工作，并从中挑选最好的。无关/令人沮丧的白板面试、阴险的歧视和设计不良的招聘标准仍然困扰着求职过程。不过，如果你是一个开发者，这是一个卖方市场。因此，在筛选大量招聘新人时，研究应该是你最好的朋友。

你应该试着找出什么？当然，这取决于你的优先考虑。在最近的一次采访中，一名工程师说:“当考虑早期创业时，我脑海中有三个主要问题:我认为团队有能力随着规模的扩大而执行吗？我能和团队里的每个人和平共处吗，他们也能和平共处吗？这个想法是好的还是扯淡的？”

这并不奇怪:候选人希望了解初创公司的生存能力、同事和文化以及核心产品。但是工作列表和职业页面没有这些信息——在数字领域的其他地方也没有。

这种信息真空是 [Devpost Jobs](http://devpost.com/teams) 的种子，这是一个招聘平台，我们建立它来帮助开发人员根据对他们重要的标准评估初创公司的工作机会——即，他们将与谁合作，公司的技术堆栈中有什么以及如何使用，面试过程中会有什么，以及开发过程和日常工作是什么样的。

这是我们的用户告诉我们他们想知道的，我们在设计产品时就考虑到了他们。但我们仍在建设和发展。我们很想听听你的想法，以及你在寻找工作时会寻找什么样的信息。

招募是一个不完美的过程，永远都是。就其核心而言，就像商业中的许多事情一样，它是关于人际关系和互动的——这些东西不能被算法或数据查询完全消化。但我们可以从提供有价值的信息开始——你可以从确保我们以及你考虑为之工作的任何公司给你提供信息开始。
