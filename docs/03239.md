# 编码还是不编码

> 原文:[https://dev.to/brightdevs/to-code-or-not-to-code-84k](https://dev.to/brightdevs/to-code-or-not-to-code-84k)

是否总有编码的好时机？你问过自己这种问题吗？你有没有觉得也许你应该休息一下，而不是写代码，但是你选择了编码？

[![](../Images/3d1180863b8349e2b92f0e601eff22f3.png)T2】](https://res.cloudinary.com/practicaldev/image/fetch/s--FCSVo2jI--/c_limit%2Cf_auto%2Cfl_progressive%2Cq_auto%2Cw_880/https://thepracticaldev.s3.amazonaws.com/i/7hx9niptmhfk1xu0udlc.jpg)

编码不仅仅是写代码。它正在编写代码。通过这样做，你解决了一个问题，通常是你客户的问题。通过编写代码，您同意与语言、平台、架构和整个系统保持一致。您同意遵循优秀编程的所有原则。你同意创造一些其他程序员可读的东西，这些东西将清楚地表明为什么要写它。这就是为什么编写代码需要长时间的集中精力。这就是为什么，如果你感到心烦意乱或疲倦，你应该问自己一个问题，这是否是一个编码的好时机。

有人会问，那么敬业精神呢？我们必须记住，奉献并不意味着也不应该意味着夜以继日地工作。然而，奉献是专业精神不可或缺的一部分，而另一方面，专业精神包括有效地工作，这意味着无休止地工作。所以，如果你筋疲力尽，不要编码。如果你累了，就去睡觉。

精疲力尽是一件可能使你的代码缺乏良好质量的事情。另一件事是你的个人生活或缺乏它。记住，你对家庭的担忧和烦恼会让你分心。即使在潜意识里，它们也会对你的工作造成巨大的伤害。与配偶的争吵、财务问题、孩子的疾病……所有这些以及更多的事情都会让你的代码变得臃肿不堪。试着先解决你的个人问题，然后开始做你的工作。

现在是流动的时候了。是的，我们都知道流动区是不可思议的。我们是代码大师，我们是超级英雄。我们工作得非常快。我不敢说这个流动区总是不好的。但是我敢说，只要想一想，我们的流动区是否总是等于有效功。适度——这是关键。有时候高速工作真的很有用，尤其是在你练习的时候，但是也值得休息一下，和你的队友商量一下你在做什么。记住，进行不可预测的快速编码并不总是让你更有效率。

以这样一种方式工作，这样你就可以永远为你正在做的事情感到自豪。我可以向你保证会有好的结果。
