# 对常用文本使用破折号片段

> 原文:[https://dev . to/Martin _ betz/use-dash-snippets-for-frequency-used-text](https://dev.to/martin_betz/use-dash-snippets-for-frequently-used-text)

如果你像我一样，你会收到很多商业和私人邮件。答案经常被一遍又一遍地重复。当我住在伊斯坦布尔时，我的朋友(甚至他们的朋友)向我寻求旅行建议，其他朋友想开始学习编码，并向我寻求最好的资源。

## TL；速度三角形定位法(dead reckoning)

您可以使用 [*破折号*](https://kapeli.com/dash) (在 OSX 上)快速访问和调整您经常与 [*破折号的片段*](https://kapeli.com/dash_guide#introductionToSnippets) 一起使用的文本模板。

## [](#before-snippets-as-text-files)之前:片段为文本文件

我保存了相当多的文本片段作为简单的`*.txt`文件，我可以通过 *Apple Spotlight* 的全文搜索功能快速找到。因此，以请求编码建议为例，步骤包括:

1.  收到一封关于`coding resources`的邮件
2.  点击回复
3.  用搜索查询`coding resources`点亮*聚光灯* ( `⌘+Space`)
4.  在*文本编辑*中双击打开`reply_coding_resources.txt`
5.  复制内容
6.  贴在回复里
7.  浏览片段并调整与片段不同的称呼和次要细节
8.  点击发送

## 我能简化它吗？

在一遍又一遍地重复这七个步骤后，我的程序员自我思考如何简化这一点。我看了看[text expander](https://smilesoftware.com/textexpander)——我不介意每月支付 3.50 美元购买一个不断发展的产品，但片段的问题集是有限的，并且已经在很好地工作，[at ext](https://www.trankynam.com/atext/)——良好的功能集和价格，但我不太喜欢 UI 和工作流程。

当我插入一些 HTML 代码片段时，我立刻明白了:
我已经有了一个管理代码片段的工具——*`Dash`*。

## [](#after-dash-snippets)后:破折号片段

Dash 是适用于 Mac OS X 编程语言的离线文档浏览器。您可以下载和浏览大量语言的文档，并保存可在任何程序中使用的片段。一次大约 25 美元，但是用这个价格你可以得到一个优秀的文档浏览器和一个代码片段工具。Dash 使用 Mac 的辅助工具，因此您首先需要在 Mac 的设置中启用它。

您可以设置粘贴代码片段的触发器。对于您的非编程文本片段，我建议您使用在您使用的编程语言中没有使用的指示器。

这个我用`§`。在您的代码片段中，您可以使用`__placeholder__`作为占位符，您可以在粘贴代码片段时交互填充该占位符。

这是前面相同的*编码建议*片段的工作流程。我假设 Dash 被设置为在启动时运行，现在正坐在后台等待您的命令。

1.  收到一封关于`coding resources`的邮件
2.  点击回复
3.  类型`§codingresources`
4.  交互式更改称呼
5.  点击发送

这不仅比我以前的解决方案少了两步，而且速度更快，更不容易出错。一开始，我试图想出非常短的指标，比如`§cdrsc`，但是当然，这些很难记住。所以我建议你使用完整的描述。

您不仅可以使用这种技术对编码资源进行快速的个人响应，还可以提供简单的客户支持。它快速、简单，没有很大的学习曲线。
