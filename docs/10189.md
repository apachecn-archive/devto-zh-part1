# 2017 年 MongoDB 赎金

> 原文:[https://dev.to/grahamcox82/mongodb-ransoms-in-2017](https://dev.to/grahamcox82/mongodb-ransoms-in-2017)

你几乎肯定听说了正在进行的 MongoDB 赎金的新闻。已经过了很久了，完全没有理由。

我们很久以前就知道，如果没有某种形式的保护，在互联网上拥有工作站是完全不安全的。本质上，你需要一个防火墙来保护你的台式机或笔记本电脑的安全。

您应该将您的前端和后端系统分离，这也是众所周知的。

最后，众所周知，数据是公司最重要的部分。

由此，我发现数据库 a)没有备份，b)没有安全保护是很可笑的。

备份至关重要。如果数据没有备份，它就可能不存在。此外，备份只有在最后一次成功恢复时才有效。所以，如果你不做备份和测试，你可能一开始就没有这些数据。

这使你能够取回你的数据，但它并不能阻止坏人得到它。为此，您需要首先阻止他们进入数据库。这意味着防火墙和体面的认证。数据库服务器不应该被外部访问——永远不要。如果你需要管理员访问，使用 VPN。就这么简单。

最可笑的是——如果我理解正确的话，MongoDB 支持使用 x.509 证书进行认证。这意味着攻击者不需要用户名和密码——这可能是暴力破解——而是需要正确签名的 SSL 证书。这些都很难破解——如果有人能可靠地做到这一点，MongoDB 是我们最不担心的。
